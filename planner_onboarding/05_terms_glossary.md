# 용어집

이 문서는 개발 용어를 기획 언어로 맞추기 위한 기준이다. 동일한 개념을 화면마다 다르게 부르면 사용자와 운영자가 모두 혼란스러워진다.

## 핵심 콘텐츠

| 용어 | 의미 | 헷갈리기 쉬운 점 |
|---|---|---|
| 이벤트 | 특정 날짜와 장소에 열리는 오프라인 모임 | 클럽에서 만든 정기모임도 이벤트 흐름을 탈 수 있음 |
| 클럽 | 장기적으로 유지되는 관심사 기반 커뮤니티 | 이벤트보다 상위 관계이며 멤버/게시판/기금이 있음 |
| 플랜 | 데이트/모임 코스나 일정 구성안 | 작성물 자체이며, 상품으로 팔리면 마켓 아이템이 됨 |
| 마켓 아이템 | 플랜을 구매 가능한 상품으로 등록한 것 | 플랜과 1:1처럼 보일 수 있으나 판매 상태/가격/리뷰가 붙음 |
| 프라이빗 데이팅 | 본인 인증 기반 1:1 매칭 서비스 | 일반 이벤트와 다른 안전/매칭 흐름을 가짐 |

## 신청, 참석, 체크인

| 용어 | 의미 | 사용자에게 보여줄 표현 |
|---|---|---|
| 신청 | 이벤트 참여 의사를 제출하는 행위 | 신청하기, 신청 완료, 심사중 |
| 승인 | 호스트가 신청자를 받아들이는 행위 | 승인됨, 참석 확정 |
| 거절 | 호스트가 신청자를 받지 않는 행위 | 신청 거절 |
| 참석 | 이벤트 참가자로 확정된 상태 | 참석 예정, 참석 확정 |
| 대기열 | 정원이 찼지만 차례를 기다리는 상태 | 대기중, 대기 순번 |
| 체크인 | 현장에서 실제 참석을 확인하는 행위 | QR 체크인, 체크인 완료 |
| 노쇼 | 참석 확정 후 오지 않은 상태 | 사용자에게 직접 강한 표현을 쓸지는 정책 결정 필요 |

## 돈의 흐름

| 용어 | 의미 | 구분 기준 |
|---|---|---|
| 지갑 | 사용자가 포인트를 보관하고 결제하는 잔액 영역 | 사용자 개인의 돈 |
| 충전 | 외부 결제수단으로 포인트를 늘리는 행위 | 잔액 증가 |
| 결제 | 포인트 또는 결제수단으로 비용을 지불하는 행위 | 잔액 감소 또는 외부 결제 |
| 환불 | 결제된 금액을 되돌리는 행위 | 원거래와 연결됨 |
| 정산금 | 호스트가 받을 수익/지급금 | 플랫폼에서 호스트에게 지급 |
| 모임 정산 | 모임 비용을 참가자끼리 나누는 행위 | 호스트가 참가자에게 요청 |
| 선입금 | 이벤트 전 참가 확정을 위해 미리 받는 금액 | 모임 후 정산과 별도 |
| 기금 | 클럽 단위로 관리되는 공동 자금 | 개인 지갑과 구분 |

## 리뷰, 신고, 신뢰

| 용어 | 의미 | 노출 범위 |
|---|---|---|
| 리뷰 | 이벤트/플랜/사용자 활동에 대한 공개 평가 | 다른 사용자가 볼 수 있음 |
| 취향 평가 | 개인화를 위한 비공개 평가 | 본인과 추천 시스템 중심 |
| 신고 | 부적절한 사용자/콘텐츠를 운영자에게 알리는 행위 | 운영 검토 대상으로 쌓임 |
| 신뢰점수 | 참석, 리뷰, 신고, 노쇼 등 활동을 점수화한 값 | 본인은 상세 이력, 타인은 제한 정보 |
| 취향 프로필 | 좋아하는 카테고리/시간대/태그 등이 누적된 데이터 | 추천과 매칭 품질에 영향 |

## 알림과 설정

| 용어 | 의미 | 기획 포인트 |
|---|---|---|
| 푸시 | OS 알림으로 사용자에게 전달되는 메시지 | OS 권한 필요 |
| 알림함 | 앱 안에서 보는 알림 목록 | 푸시 권한과 별개로 존재 가능 |
| 카테고리 설정 | 이벤트/클럽/결제/데이팅/마케팅 등 유형별 수신 제어 | 중요 알림과 마케팅 알림을 구분 |
| 방해금지 | 특정 시간대 알림을 제한하는 설정 | 예외 알림 정책 필요 |
| 기기 토큰 | 푸시를 보낼 대상 기기를 식별하는 값 | 사용자는 "등록된 기기"로 이해 |

## 위치와 프라이버시

| 용어 | 의미 | 기획 포인트 |
|---|---|---|
| 위치 공유 | 이벤트 참석자가 자신의 현재 위치를 다른 참석자에게 보이게 하는 기능 | 명시적 동의 필요 |
| opt-in | 사용자가 직접 켜는 동의 방식 | 기본값은 꺼짐으로 보는 것이 안전 |
| opt-out | 사용자가 공유를 중지하는 행위 | 쉽게 찾을 수 있어야 함 |
| 프라이버시 대시보드 | 위치 공유 가능한 이벤트와 공유 상태를 한눈에 보는 화면 | 일괄 통제감을 줌 |
| 역지오코딩 | 좌표를 주소로 바꾸는 기능 | 사용자에게는 "주소 자동 표시"로 설명 |

## 상태값 표기 원칙

| 원칙 | 예시 |
|---|---|
| 사용자에게는 한글 상태를 우선 쓴다 | `PENDING` 대신 "심사중" |
| 운영/정책 문서에는 영문 상태를 병기한다 | "심사중(PENDING)" |
| 같은 한국어라도 도메인이 다르면 구분한다 | "정산 완료"와 "체크인 완료" |
| 취소 상태는 내부 영문값 차이를 확인한다 | 신청은 `CANCELED`, 참석/정산은 `CANCELLED`가 쓰일 수 있음 |

## 추천 화면 문구 기준

| 상황 | 권장 표현 | 피할 표현 |
|---|---|---|
| 신청 심사중 | 호스트가 신청을 검토 중입니다 | API 대기중 |
| 대기열 | 현재 대기 3번째입니다 | WAITING 상태입니다 |
| 잔액 부족 | 포인트가 부족합니다. 충전 후 다시 시도해주세요 | 결제 실패 |
| 정산 활성 | 참가자에게 납부 요청이 전송되었습니다 | ACTIVE 전환 |
| 계좌이체 확인 대기 | 호스트 확인 후 납부 완료로 바뀝니다 | confirm 필요 |
| 위치 공유 중 | 이 이벤트 참석자에게 내 위치가 보입니다 | 좌표 업로드 중 |
| 알림 권한 거부 | 기기 설정에서 알림 권한을 켜주세요 | FCM 권한 없음 |
