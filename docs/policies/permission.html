권한 · 역할 매트릭스

정산 이의제기·열람 조건 (2026-06-05)

정산 이의제기는 정산이 신청(활성화)된 이후부터만 가능하다 — 준비 중 정산에는 시스템이 이의 접수를 거부하고 앱도 버튼을 숨긴다. 호스트 본인 분담에 대한 이의는 상태와 무관하게 불가. 모임 정산 열람 자격은 참석 확정자 + 해당 정산에 분담금·송금이 걸린 사람 + 호스트다 — 참석을 취소했어도 내 돈이 걸려 있으면 본인 정산을 볼 수 있다. 준비 중 정산은 참가자에게 총액·상태·내 분담금만 보이는 미리보기 수위로 차등 노출된다.

권한 검토 체크리스트

  • 비로그인 사용자가 이 화면을 볼 수 있는가
  • 로그인했지만 관계 없는 사용자가 할 수 있는 액션은 무엇인가
  • 소유자·작성자·호스트 본인이 대상일 때 금지해야 할 액션이 있는가
  • 관리자급 권한과 소유자 권한을 구분했는가
  • 차단·탈퇴·삭제·비활성 사용자 상태를 고려했는가
  • 한 사용자가 여러 역할을 동시에 가질 때 우선순위가 있는가
  • 화면 진입 후 권한이 바뀌면 어떻게 갱신하는가
  • 권한 없음은 버튼 숨김·비활성·에러 중 무엇으로 표현하는가

수용 기준

  • 주요 액션마다 비로그인·로그인·관계자·소유자/관리자가 각각 어떤 결과를 보는지 적는다 — “이 사람은 이걸 못 한다”까지 포함해서.
  • 권한이 없을 때 화면이 어떻게 보일지(버튼 숨김·비활성·로그인 유도·접근 불가 안내) 한 가지로 정하고 다른 화면에서도 같은 방식을 쓴다.
  • 한 사람이 호스트이면서 참가자, 클럽 관리자이면서 일반 멤버 같은 상황에서 어느 역할이 우선인지 미리 정한다.
