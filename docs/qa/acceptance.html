개발 착수 전 합의하고, 릴리즈 승인 전 다시 확인한다. “기능이 동작한다”에서 한 걸음 더 들어가, 사용자가 결과를 이해하고 다음 행동을 할 수 있는지를 본다.

범주수용 기준
기능 커버리지139개 기능이 모두 PRD·QA 대상에 포함된다.

상태 커버리지생성·진행·완료·취소·만료·삭제·차단을 서로 다른 상태로 다룬다(한 상태로 묶지 않는다).

카드화 우선순위

기능 카드 상세화는 모든 기능을 한 번에 하지 말고, 사용자 피해와 전환 영향이 큰 것부터 진행한다.

우선순위기준해당 예시
P0돈·안전·계정 삭제·위치 공유처럼 사용자 피해가 큰 기능결제·환불, 모임 정산, 데이팅 차단, 위치 opt-in/out, 계정 삭제
P1주요 전환율과 직접 연결되는 기능이벤트 신청, 클럽 가입, 플랜 구매, 온보딩
P2운영·CS 문의가 많이 생길 기능알림 설정, 데이터 내보내기, 리뷰·신고, 대기열
P3조회 중심이거나 실패 영향이 작은 기능목록 조회, 통계, 단순 상세
