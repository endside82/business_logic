릴리즈 준비 체크리스트

릴리즈 직전 한 번 더 보는 17개 항목. 모두 “예”라고 답할 수 있어야 릴리즈 승인. 한 줄이라도 “모르겠다”가 나오면 발견된 시점이 가장 싸다.

1. 기능 범위

  • 이번 릴리즈에 포함되는 기능 ID를 168개 인벤토리 기준으로 표시했다
  • 제외되는 기능과 이유를 명시했다
  • P0/P1/P2 우선순위를 정했다

2. 사용자 흐름

  • 신규 사용자 가입·온보딩 흐름이 깨지지 않는다
  • 홈·검색에서 핵심 콘텐츠로 진입할 수 있다
  • 이벤트 신청·클럽 가입·플랜 구매·데이팅 매칭 중 릴리즈 대상 흐름이 QA됐다

3. 상태와 권한

  • 권한 없는 사용자의 버튼 노출·비활성·에러 정책이 정해졌다
  • 이미 취소·만료·삭제·차단된 대상을 사용자가 클릭했을 때 보일 안내 화면이 정해졌다
  • 멀티 디바이스·중복 클릭 케이스가 QA됐다

4. 돈과 알림

  • 결제 성공·실패·환불 후의 잔액·거래내역이 모두 같은 값으로 맞물려 있다 (한 곳만 다른 값을 보여주면 안 된다)
  • 정산의 작성중·진행중·완료·취소됨 상태가 분리되어 보인다
  • 알림 수신자·딥링크·카테고리 설정·방해금지 정책이 QA됐다

5. 개인정보와 안전

  • 위치 공유의 동의·중지 동선이 명확하다
  • 데이팅 차단·신고가 관련 화면에 즉시 반영된다
  • 데이터 내보내기·계정 삭제·비활성화 문구가 검토됐다

릴리즈 차단 조건

아래 한 줄이라도 해당되면 릴리즈 보류
  • 돈이 빠진 뒤 기능 성공/실패가 사용자에게 불명확함
  • 작성중 정산이 참가자에게 노출됨
  • 공유 중지 후 위치 마커가 남음
  • 차단된 사용자가 채팅·게시판에서 여전히 보임
  • 마지막 로그인 수단 해제가 가능함 (계정 잠김)
  • 비로그인 사용자가 결제 직전 화면까지 진입함
