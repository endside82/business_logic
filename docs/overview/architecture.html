16개 도메인의 책임

제품을 “어디에 어떤 기능이 사는가”로 나눠 둔 지도다. 이 지도가 흔들리면 기능 추가의 책임 소재가 흔들리기 때문에, 새 기능이 도착했을 때 “어느 도메인에 둘 것인가”를 가장 먼저 묻는다.

도메인 간 의존 관계

화살표 방향은 “이 도메인이 동작하려면 저 도메인이 먼저 준비돼야 한다”는 뜻이다.

구조 해석 기준

  • 인증·프로필은 모든 기능의 진입 조건이다. 무엇을 만들든 비로그인 동작과 비교한다.
  • 홈·검색·플랜 마켓은 발견·탐색의 축이다. 추천 알고리즘과 콘텐츠 가시성을 책임진다.
  • 이벤트와 클럽은 오프라인 활동의 핵심 실행 단위다. 무거운 상태 머신이 여기 있다.
  • 결제·지갑·정산은 돈의 상태를 다루는 별도 축이다. 다른 도메인이 직접 잔액을 만지지 않는다.
  • 알림·캘린더·위치는 다른 기능의 “결과”를 사용자에게 다시 연결한다. 단독 기능이 아니다.
  • 리뷰·신고·차단·신뢰점수는 플랫폼 안전의 기반이다. 운영 흐름과 사용자 흐름이 같은 데이터를 본다.

새 기능을 둘 곳을 정하는 법

결정 순서
  1. 이 기능이 풀려는 사용자 문제가 위 5축 중 어디에 있는가
  2. 그 축 안에서 책임을 가지는 도메인은 어디인가
  3. 다른 도메인 데이터를 “쓰기”로 만지는가 — 그렇다면 정책 PRD가 먼저 갱신돼야 한다
  4. 기존 도메인에 들어가면 그 도메인 PRD의 기능 목록에 추가, 들어가지 않으면 새 도메인 후보로 보류
