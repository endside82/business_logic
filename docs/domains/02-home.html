02. 홈 피드

로그인 직후 첫 화면이자, 사용자가 가장 자주 돌아오는 곳이다. 추천 콘텐츠와 다른 도메인으로의 진입점을 함께 보여준다. 기능은 적지만 “부분 실패에도 살아남는다”라는 요구가 가장 강한 도메인.

왜 부분 실패가 중요한가

홈 피드는 여러 섹션(추천 이벤트·인기 클럽·진행중 정산·읽지 않은 알림 등)을 한 화면에서 합쳐 보여준다. 한 섹션의 API가 실패해도 다른 섹션은 보여야 사용자가 그대로 진입할 수 있다. “전체 화면 실패”로 떨어지면 다음 액션이 없어지므로, 기획 초기부터 섹션별 빈 상태·부분 실패를 분리해서 정의한다.

5개 기능

F02-01

홈 메인 조회

섹션별 부분 실패에도 화면이 유지된다. 캐시 정책, 빈 섹션, 비로그인 노출 분리.

8 시나리오
F02-02

새로고침

당겨 새로고침과 자동 새로고침. 네트워크 실패 시 스크롤 위치 복구.

6 시나리오
F02-03

섹션 카드 진입

카드별 라우팅. 잘못된 ID, 백버튼 복귀, 진입 후 상태 변경 처리.

7 시나리오
F02-04

추천 이벤트 더보기

필터·무한스크롤. 필터 초기화, 마지막 페이지, 빈 결과, 추천 배지 노출 규칙.

11 시나리오
F02-05

검색·알림 진입점

상단 검색 아이콘과 알림 진입. 비로그인 가드, 미읽음 배지, 탭 복귀 시 동기화.

6 시나리오

레드 플래그

자주 깨지는 케이스
  • 비로그인과 로그인 결과 차이 — 같은 카드가 게스트에게는 “자세히 보기”, 로그인 사용자에게는 “신청”으로 달라져야 한다. 잘못된 CTA는 가입 전환을 떨어뜨린다.
  • 부분 실패가 전체 화면 실패로 떨어짐 — “정산 알림” API 하나 죽었다고 “추천 이벤트”까지 못 보여주면 안 된다.
  • 캐시로 오래된 CTA 노출 — 이미 신청한 이벤트에 “신청하기” 버튼이 남아 있는 경우. 진입 시 재조회 정책 필요.
  • 비로그인 알림 진입 — 로그인 가드를 빠뜨리면 401 화이트스크린.
