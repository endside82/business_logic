이 도메인의 핵심 난점

이벤트에는 “신청서 심사”와 “참석 확정/대기”가 같이 돌아간다. 두 흐름은 별개다. 유료 + 승인제 이벤트의 경우, 호스트 승인은 결제 권한 부여이지 참석 확정이 아니다. 승인 → 결제 대기 → 결제 성공 → 참석 확정의 4단계를 모두 분리해서 보여줘야 한다. 이 흐름이 한 버튼에 섞이면 환불 분쟁이 시작된다.

핵심 흐름 — 유료 승인제 이벤트의 신청·결제·참석 확정

이 도메인에서 가장 자주 오해받는 흐름. 승인과 결제, 결제와 참석 확정이 모두 다른 단계라는 점이 핵심이다.

flowchart TD A["참가자 신청"] --> B["신청서 심사 대기"] B --> C{"호스트 심사"} C -->|"거절"| D["신청 거절
결제 없음"] C -->|"승인"| E["승인 후 결제 대기
결제 권한 부여"] E --> F{"기한 내 결제?"} F -->|"성공"| G["결제 완료 기록"] G --> H["참석 확정"] F -->|"실패/만료"| I["결제 기한 만료
예약 정원 반환"] H --> J["체크인·위치·리뷰 자격 열림"]
선입금 확장 반영

"승인 후 결제 대기"와 "결제 기한 만료" 상태는 F03-13에서 선입금 흐름과 함께 자세히 다룬다. 승인 즉시 참석 확정으로 처리하지 않고, 결제 또는 입금 확인을 거쳐 참석자가 확정되는 흐름을 기준으로 본다. 자세한 정책은 이벤트 참가 선입금과 결제·정산 정책에서 본다.

일반 신청·참석 상태 흐름

flowchart TD A["참석 신청 버튼"] --> B{"승인 필요?"} B -->|"예"| C["심사 대기"] C -->|"승인"| D["참석 확정"] C -->|"거절"| E["거절됨"] C -->|"사용자 취소"| F["신청 취소"] B -->|"아니오"| G{"정원 여유?"} G -->|"예"| D G -->|"대기열 가능"| H["대기 중"] G -->|"대기열도 가득"| I["신청 실패"] H -->|"자리 발생"| D D -->|"사용자 취소"| J["참석 취소"]
"신청 취소"와 "참석 취소"는 서로 다른 상태다

신청서를 거두는 "신청 취소"와, 참석 확정 후 빠지는 "참석 취소"는 별개의 상태다. 사용자에게는 둘 다 "취소"로 보이지만, 정원 반환·환불·대기열 승급 처리가 서로 달라 운영·검수 단계에서 반드시 구분해야 한다.

QR 체크인의 구체 정책 (F03-08)

항목값
입장 확인 방식QR 코드와, 함께 표시되는 숫자 단축코드로 입장을 확인한다. QR은 발급 후 일정 시간이 지나면 만료된다.
새 코드 발급 시이전 단축코드는 즉시 무효화 — 같은 이벤트에 두 코드가 동시에 살아 있지 않다.
만료 화면호스트 화면에 “새 QR 코드 받기” 버튼이 뜬다. 자동 재발급은 하지 않는다 — 호스트가 직접 눌러 새로 받아야 한다.
참가자 표시참석 확정자에게 QR 코드와 만료 카운트다운이 보인다.

이 도메인의 부수효과

영향여부이유
알림강함승인·거절·대기 승격·취소·정원 변경마다 알림이 발생. 수신자가 호스트·참가자 양쪽이라 룰이 복잡.
결제·정산강함유료 이벤트의 결제·환불, 호스트 수익 정산, 클럽 이벤트의 기금 차감.
캘린더있음참석 확정 시 자동 등록, 일정 변경 시 자동 갱신.
위치조건부위치 공유 이벤트일 때만 위치 도메인이 활성화.
리뷰·신뢰있음참석 확정 + 종료 후 리뷰 자격이 생긴다. 노쇼는 신뢰점수에 반영.

레드 플래그

실제 분쟁 사례 모음
  • 정원·대기열·승인·결제가 한 버튼에 섞이는 경우 — 사용자가 자신이 어느 단계에 있는지 모르게 된다.
  • 호스트가 자기 이벤트에 신청 시도 — 버튼만 막지 말고 “자기 이벤트는 신청할 수 없다”라고 사유까지 같이 보여준다.
  • 정원 마지막 자리 동시 신청 — 둘 다 “정원 여유” 화면을 봤지만 한 명만 성공한다. 실패자에게 “방금 마감됐다”라는 정확한 안내가 필요.
  • 대기열 자동 승격 — 사용자 행동 없이 상태가 바뀐다. 알림·CTA 갱신이 필수.
  • 이벤트 취소 후 유료 참가자 환불 — 환불·알림·정산금이 동시에 움직이는 케이스.
  • 유료 승인제 미승인 사용자가 결제 시도 — 돈을 먼저 받으면 거절 후 환불해야 한다. 승인 후 결제만 허용.
