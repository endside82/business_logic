06. 결제 · 지갑

포인트라는 단일 잔액을 중심에 두고, 충전·결제·환불·구독·수익 정산을 모은다. 다른 도메인의 “돈이 움직이는 액션”은 모두 이 도메인의 거래로 기록된다. 한 줄을 잘못 건드리면 잔액·거래내역·세금계산이 동시에 어긋난다.

지갑 유료/무료 분리 + 인출은 유료분만

지갑 잔액은 유료 / 무료로 분리 관리된다. 무료로 결제된 금액이 수취자에게 어떻게 넘어가는지는 사용처 유형에 따라 다르다 — 구분을 그대로 이어 전달하는 유형, 무료분을 소멸 처리하는 유형, 유료만 받는 유형으로 나뉜다.

구분유입사용인출(현금화)
유료충전, 현금 환불모든 사용처가능
무료프로모션 지급허용된 사용처만(이벤트·마켓·구독·티켓 등)불가

외부 출금·탈퇴 환불은 모두 유료 잔액만 대상이다. 탈퇴 시 무료 잔액은 소멸한다. 사용처 3분류(무료가 수취자까지 전파되는 흐름 / 수취자 없이 소각되는 흐름 / 무료 결제 자체를 막는 유료 전용)와 차등가격(콘텐츠 생산자가 무료 허용 여부와 무료 전용가를 정하는 방식)은 결제·정산 정책에 정리되어 있다.

2026-06-06 종결: 모든 환불 경로(이벤트·구독·마켓·모임 정산·클럽 기부/구독)가 결제했던 유료/무료 구성을 그대로 복원하도록 통일됐다 — "무료 → 유료 둔갑" 구멍이 봉합됐다. 운영자가 직접 처리하는 환불·보상도 별도 규칙으로 우회하지 않고 본 시스템과 같은 규칙을 따른다(운영 보상은 무료 포인트로 지급).

2026-06-06 — 이벤트도 무료가 주최자에게 전달: 그동안 이벤트 참가비만 이 흐름에서 빠져 있었으나, 이제 이벤트 참가비도 무료가 수취자까지 전파되는 흐름에 합류했다. 무료 포인트로 낸 참가비는 주최자의 무료 포인트(인출 불가)로 적립되고, 유료로 낸 몫만 정산·인출 대상이 된다. 무료 결제만 모인 이벤트도 정산이 생겨(수수료·세금 0) 주최자가 그만큼의 무료 포인트를 받는다. 정산이 끝난 뒤의 무료분 환불은 주최자에게서 거두지 않고 플랫폼이 비용으로 흡수한다(정책).

돈 흐름 무결성 — 새는 길·막힌 돈·장부 정합 (2026-06-06)

결제·정산·환불·출금이 각자 다른 장부를 갱신하다 어긋나던 지점들을 한 번에 닫았다. 아래 표는 그 변화에, 같은 맥락의 이벤트 결제 표준화(2026-06-06)를 함께 정리한 것이다.

외부 출금 출금해도 지갑 잔액이 그대로 남아 같은 돈을 쓰며 동시에 인출 가능 신청 즉시 지갑에서 차감(실패 시 복원), 출금 가능액은 정산으로 실제 받은 실적 기준

일일 자동 대사 — 지갑·거래내역·장부 삼중 대조

지갑 잔액·거래 내역·복식 회계 장부는 같은 돈을 세 곳에서 적는데, 그동안 셋이 서로 어긋나도 아무도 몰랐다. 이제 매일 자동으로 8종의 대조를 돌려 셋이 일치하는지 점검하고, 어긋나면 운영자에게 알린다 — "장부는 맞는데 실제 지갑이 다른" 상태를 사람이 손으로 찾지 않아도 드러난다.

남은 것

실제 카드·계좌 결제 대행과의 계약이 끝나야 점검할 수 있는 2건(결제 응답 유실 보정, 외부 출금 대행의 중복 방지 검증)만 출시 전 확인 항목으로 남아 있다 — 그 외 돈 흐름 무결성 작업은 종결됐다.

먼저 충전한 순서대로 쓰는 잔액과 복식 회계

충전·결제·환불은 모두 충전 단위(먼저 충전한 잔액부터 차감되는 환불 가능 단위)와 복식 회계 장부 양쪽에 동시에 기록된다.

  • 중복 결제 차단: 같은 건에 대한 두 번째 결제 시도는 그 자리에서 거절된다 — 잔액은 한 번만 차감된다.
  • 서버 가격 검증: 사용자가 보낸 금액과 시스템에 등록된 모임 가격이 다르면 결제가 거절된다(가격 조작 방지).
  • 반대 분개: 모임 정산을 취소하면 원래 분개를 반대로 한 줄 더 기록해 장부를 되돌린다.

레드 플래그

  • 결제 성공과 기능 성공이 분리되는 경우 — 결제 대행 응답 지연 또는 정원 마감 직후 결제 케이스. 처리중 화면과 재조회 동선 필수.
  • 중복 결제·중복 환불 — 같은 버튼 연타 또는 응답 중복 도착. 같은 결제는 한 번만 처리되고, 같은 건의 두 번째 환불도 차단된다(환불액이 결제액을 넘지 못함).
  • 출금하고도 지갑 잔액이 남는 경우 — 출금 신청 시 즉시 차감되지 않으면 같은 돈을 쓰며 동시에 인출할 수 있다. 출금 가능액은 정산으로 받은 실적 기준.
  • 잔액 부족 vs 결제 대행 실패 구분 미흡 — 사용자는 둘 중 어느 쪽인지 알아야 다음 행동을 결정한다.
  • 자동충전 실패 후 원래 결제까지 실패 — 두 단계의 실패를 분리해서 안내해야 한다.
  • 결제수단 삭제 시 자동충전·구독 영향 미고지 — 다른 곳에서 쓰이던 수단을 그냥 지우면 결제 실패가 연쇄.
