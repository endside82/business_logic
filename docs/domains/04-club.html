04. 클럽

단발 이벤트가 아닌 “관계가 이어지는 커뮤니티”의 단위. 가입·운영·게시판·클럽 이벤트·기금·구독까지 18개 기능이 들어 있어 도메인 중 가장 무겁다. 보상(마일리지)과 제재(경고·징계)도 클럽 단위로 굴러가는 위성 도메인으로 이 줄기에 속한다. 관리자·소유자 권한 분기와 기금이라는 돈 흐름을 동시에 가진다.

구독료 — 클럽 소유자가 플랫폼에 내는 정기 운영비

F04-16의 가장 큰 오해 포인트: 사용자가 클럽을 구독하는 게 아니라 클럽 소유자가 플랫폼에 운영비를 지불한다. UI 스펙(SCR-CF-005)은 사용자 구독처럼 묘사되어 있어 향후 분리 검토가 필요한 부분이다.

기금 인출 — 5% 수수료 + 3.3% 원천징수

F04-15. 클럽 소유자만 가능하며, 외부 계좌가 아니라 소유자 지갑으로 수수료·세금을 뺀 금액이 입금된다.

화면 표시 예

요청 100,000원이라면 사용자에게 보여야 하는 값: “수수료 5,000원, 원천징수 3,300원, 실제 입금 91,700원”. 합계 화면에서 net 금액을 명시하지 않으면 분쟁 발생.

클럽 기금 유료/무료 분리 + 인출 paid-only

기금 잔액도 지갑과 동일하게 유료(paid) / 무료(free)로 분리 관리된다. 기부·가입비는 무료 수취자(기금)가 존재하는 flow-through다.

  • 기부·가입비 split 전파: 멤버가 무료 포인트로 기부·가입비를 내면 그 split이 기금에 free로 적립된다. 무료로 적립된 기금은 클럽 안에서 쓸 수 있으나 인출(현금화) 불가.
  • 인출 가용액 = paid 잔액 − 예비금/미수금. 위 “5% 수수료 + 3.3% 원천징수”는 유료(현금)분에만 부과된다. 폐쇄 인출도 유료만 현금화하고 무료 기금은 소멸한다.

전체 정책(사용처 3분류·차등가격·flow-through)은 결제·정산 정책 참조. (개선 예정: 현재 클럽 기부·가입비·구독 환불은 지갑으로 돌려주는 방식이라 유료/무료 구성이 보존되지 않으며, 원결제수단으로 되돌려 구성을 그대로 보존하는 방식으로 전환할 예정이다.)

레드 플래그

자주 깨지는 케이스
  • 관리자와 소유자 권한 혼동 — “관리자가 클럽 삭제”나 “소유자가 자기 추방”처럼 권한 경계 위반.
  • 차단 시 환불·멤버십 정리 누락 — 차단 직후 게시글에서 사라지지만, 결제된 구독·기부는 남는 경우.
  • 소유권 이전 수락 전 권한 변경 — 이전 수락 전에 새 소유자가 권한 행사 시도.
  • 초대 만료·중복 신청 — 같은 사용자가 두 경로(초대 + 가입 신청)로 들어오는 케이스.
  • 인출 요청 pending 중복 — 소유자가 같은 금액을 두 번 인출 요청.
  • 구독 만료 후 혜택 노출 — 결제 실패로 만료된 사용자에게 멤버 전용 콘텐츠가 캐시되어 보임.

최근 구현 확장 — PRD ID 배정 대기

클럽 활동 운영 묶음

최근 구현에는 마일리지, 경고·이의, 운영자 알림 발송 모니터링, 권한 설정 관리, 클럽 활동 알림 채널 설정, 멤버 신뢰도 점수 위젯이 추가됐다. 아직 PRD 문서의 기능 ID 체계에는 별도 단위가 없으므로 이 도메인에서 구현 반영 대기 묶음으로 추적한다. 기능 ID를 임의로 만들지 않고, 다음 PRD 정리 시 F04 클럽 운영 확장 또는 F11/F12 정책 확장으로 배정해야 한다.

클럽 줄기 — 클럽 안에서 굴러가는 운영 도메인

별도 도메인 번호를 받았지만 실제로는 클럽 단위로 설정·운영되는 기능들이다. 보상도 제재도 클럽 운영진의 도구라는 점에서 클럽 줄기로 묶는다.
