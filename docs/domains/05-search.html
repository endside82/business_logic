05. 검색

이벤트·클럽·플랜을 한 입력창으로 찾는다. 입력 전·입력 중·결과의 세 상태 각각이 별개 화면이며, “결과 없음”과 “필터 너무 강함”을 사용자가 구분할 수 있어야 한다.

검색의 세 상태

flowchart LR A["검색창 진입"] --> B["입력 전
최근/트렌딩"] B --> C["입력 중
자동완성 서제스트"] C --> D["결과 화면
영역별·필터·정렬"] D --> E["빈 결과"] D --> F["저장된 검색으로 등록"]

입력 전·입력 중·결과 화면은 같은 페이지 안의 다른 상태다. 같은 검색창에서 사용자 입력의 길이에 따라 자동으로 전환된다. 디바운스 누락 시 모든 키 입력마다 API가 호출되는 회귀가 자주 발생한다.

레드 플래그

  • 저장된 검색의 알림 연동 — 사용자가 “저장”했으니 새 결과 알림이 자동으로 켜진다고 오해할 수 있다. 별도 토글 노출 필요.
  • GPS 권한이 필요한 필터 — “내 주변” 필터를 GPS 거부 상태에서 적용하면 결과가 0이 되는 케이스.
  • 비로그인 검색 — 비로그인 사용자에게는 “저장된 검색·최근 검색어”가 보이면 안 된다.
