14. 위치 · 길찾기

이벤트 장소·경로 안내·참석자 위치 공유·프라이버시 통제를 모은다. 위치 공유는 가장 민감한 데이터이므로 모든 동작이 opt-in으로 시작하고, opt-out은 즉시 반영되며, 공유 중인 상태가 항상 사용자에게 시각화된다.

위치 공유의 4가지 원칙

flowchart LR A["위치 공유 가능 이벤트"] --> B{"사용자 동의"} B -->|"동의"| C["공유 활성"] B -->|"거부"| D["공유 비활성"] C --> E["위치 업데이트"] C --> F["공유 시간 연장"] C --> G["공유 중지"] G --> D
  • opt-in이 기본 — 동의 없이 위치가 노출되는 동작은 없다.
  • 중지는 즉시 — 옵트아웃 후 1초라도 마커가 남으면 위반.
  • 공유 상태가 시각화 — 어디에 누구에게 보이고 있는지 사용자가 한 화면에서 본다 (F14-04 프라이버시 대시보드).
  • 자동 만료 — 이벤트 종료 시간 + 짧은 버퍼로 자동 만료. 사용자가 잊지 않게 한다.

레드 플래그

  • opt-in 없이 위치가 보임 — 가장 큰 사고.
  • opt-out 후 마커가 남음 — 사용자가 “껐다”라고 생각하지만 다른 사람 화면엔 잠시 보임.
  • 위치 권한 영구 거부 — 앱 내 CTA로는 해결 불가. OS 설정 진입 안내 필수.
  • 위치 공유 자동 만료 후에도 “공유 중”으로 표시 — 사용자가 계속 보인다고 오해.
  • 이벤트 종료 후 위치 데이터 보관 — 보관 기간을 명시하고 자동 삭제.
  • 외부 지도 앱이 깔려 있지 않은 사용자 — 길찾기 버튼을 눌렀을 때 앱 대신 웹 지도로 받아 준다.
