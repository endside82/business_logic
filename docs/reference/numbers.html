F07-09 선입금·환불 (호스트가 모임마다 직접 설정)

  • 입력 단위: hoursBeforeStart(이벤트 시작 N시간 전) × refundPercentage(0~100, 5% 단위 권장)
  • 참가자 결제 방식: POINT 또는 BANK_TRANSFER
  • BANK_TRANSFER는 호스트가 수동 확인. 가상계좌 입금은 PG webhook 자동 처리 (현재 stub 401)
  • 화면 표시 예: “이벤트 3일 전 / 환불률: 50% / 환불금액: 5,000원”
  • 0% 룰일 때는 “환불 불가” 안내 명시

회계·복식부기

결제·정산·인출 흐름 뒤에서 모든 거래를 복식부기로 기록한다

  • AccountingLedgerService — 모든 돈 흐름을 분개로 기록
  • accounting_ledger 테이블 — 충전·결제·환불·기금 인출 등 각 행위가 다수 분개 생성
  • charge_lots — FIFO 환불 가능 단위. 충전 시 신규 생성, 결제 시 오래된 lot부터 차감, 환불 시 lot 단위 무효화
  • 역분개 — 정산 취소 시 AccountingLedgerService.recordMeetingSettlementReversal (F07-03)
  • 관련 거래 타입: CHARGE, PAY, REFUND, WITHDRAW, DONATION 등

읽는 법

  • 이 페이지는 인덱스다. 각 항목의 상세 동작과 분기는 해당 기능 PRD에서 본다.
  • “서버 미구현/Gap”으로 표시된 값은 PRD에는 정의돼 있지만 코드에 아직 들어가지 않은 항목이다. 구현 착수 전 반드시 확인.
  • 법무 문구·UI 카피로 인용할 때는 항상 해당 기능 PRD에서 최종 값을 재확인한다.
