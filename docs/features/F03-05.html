F03-05. 이벤트 신청 · 참석 (참가자)

한 이벤트에 신청해서 정식 참석자가 되거나, 대기열에 들어가거나, 호스트 승인을 받거나, 결제를 거치는 모든 분기를 다룬다. 승인 × 결제 × 정원 × 대기열 네 축의 조합이 본 단위의 모든 시나리오를 만든다. 이벤트 도메인에서 가장 시나리오가 많고 가장 자주 회귀가 일어나는 단위.

분기 축
4
승인·결제·정원·대기
시나리오
13
참석 상태
PENDING/ATTENDING/WAITING/CANCELLED
유료 + 승인제 흐름의 핵심 정책

승인과 결제, 결제와 참석 확정은 모두 다른 단계다. 신청 시 결제 X, 승인 후 결제 권한 부여, 결제 성공 후에만 ATTENDING이 된다. 이 순서가 뒤집히면 환불·정산이 즉시 깨진다.

이 기능 안의 하위 항목

동작 1

직접 신청 (무료 + 승인불요)

액션바 “참석 신청” 즉시 ATTENDING.

동작 2

대기열 등록

정원 가득 시 WAITING + 순번. 자동 승급 알림.

동작 3

승인제 신청서

메시지 첨부, 호스트 승인 후 ATTENDING.

동작 4

유료 결제 흐름

승인 후 결제 권한 부여, 결제 성공 시에만 ATTENDING.

동작 5

자가 취소

ATTENDING/WAITING/PENDING 모두 취소 가능, 대기자 자동 승급.

사용자 시나리오 13개

S1 · Happy Path

무료 + 승인 불요 + 정원 여유

  1. “참석 신청” 탭 → 정원 여유 확인 → ATTENDING + capacity++
  2. 로그·trending INCR
  3. 액션바 “참석 취소”로 교체
S2 · 대기열

정원 초과 대기 등록

  1. 정원 가득 + 대기열 활성
  2. WAITING + 순번 부여
  3. “대기 취소 (N번째)” 표시
S3 · 순번 갱신

다른 사용자 취소로 순번 당김

  1. ATTENDING 1명 취소 → 1번 대기자 자동 승급
  2. 나머지 대기자 순번 재정렬
S4 · 자동 승급

대기 → ATTENDING 알림

  1. 1번 대기자에게 푸시 “참석으로 확정되었습니다!”
  2. 상세 진입 시 1회성 강조 노출
  3. dismiss 후 재진입 시에는 강조 안 보임
S5 · 승인제

신청서 제출 → 호스트 승인

  1. “신청서 작성” → 메시지 입력 → 제출
  2. PENDING + 호스트 fanout
  3. 호스트 승인 시 ATTENDING + APPLICATION_APPROVED 푸시
S6 · 자가 취소

ATTENDING 취소

  1. 환불 정책 안내 시트 → 확인
  2. CANCELLED + 1번 대기자 자동 승급
  3. 유료면 환불 흐름(F06)
S7 · 호스트 시도

호스트 본인 신청 시도

  1. UI 우회 시 400 EVENT_CANNOT_APPLY
S8 · 클럽 가드

클럽 비멤버 신청

  1. “참석 신청” → CLUB_MEMBERSHIP_REQUIRED
  2. 클럽 가입 CTA 표시
S9 · 중복

이미 신청한 사용자가 다시 시도

  1. 409 ALREADY_ATTENDING 또는 APPLICATION_ALREADY_EXISTS
S10 · 가득

정원·대기열 모두 가득

  1. 409 WAITLIST_FULL
S11 · 잔액

유료 + 잔액 부족

  1. 결제 시트에 “잔액 부족 + 충전” CTA
  2. 충전 후 다시 신청 가능
S12 · 마감

모집 마감 후 신청

  1. EVENT_ALREADY_CLOSED
S13 · 유료 + 승인제

승인 → 결제 → 확정

  1. 신청 시 결제 X, PENDING 등록
  2. 호스트 승인 시 “결제 필요” 상태로 전환
  3. 사용자가 결제 → ATTENDING + 거래 기록
  4. 결제 실패/만료 시 예약 정원 반환

현재 갭: 일부 정책은 구현 중. 승인 즉시 ATTENDING이 만들어지는 경로가 남아있어 결제 검증 강화가 필요.

서버 계약

MethodPath설명
POST/api/v1/events/{id}/capacity직접 참석(승인 불요)
POST/api/v1/events/{id}/apply신청서 제출(승인제)
DELETE/api/v1/events/{id}/capacity자가 취소
에러 코드
  • 400 EVENT_CANNOT_APPLY · EVENT_ALREADY_CLOSED
  • 403 CLUB_MEMBERSHIP_REQUIRED
  • 409 ALREADY_ATTENDING · APPLICATION_ALREADY_EXISTS · WAITLIST_FULL
  • 402 결제 부족 (F06)

자주 빠뜨리는 케이스

레드 플래그
  • 승인 즉시 결제 없이 ATTENDING으로 만드는 회귀 — 유료 정산 일관성 깨짐.
  • 거절된 사용자에게 결제 CTA 노출 — 자격 검증 누락.
  • 중복 결제 가능 — 결제 전 신청 상태·기한 검증 필수.
  • 대기자 자동 승급 누락 — 빈 자리가 채워지지 않는 회귀.
  • 승급 알림 미발송 — 사용자가 자리가 났는지 모름.
