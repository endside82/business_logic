F17-06. 고정형 등록·승인·대기열 (FIXED)

FIXED 모임만의 멤버 8 상태 머신. 좌석 점유는 PENDING/APPROVED_PENDING_PAYMENT/ENROLLED 3종. terminal 후 재신청은 동일 row 재사용. FORFEITED는 영구 차단. CLOSED/CANCELED 모임에서는 자가 취소 차단.

멤버 8 상태 머신

flowchart TD A["신청"] --> B{"approvalRequired?"} B -->|"예"| C[PENDING] B -->|"아니오, price>0"| D[APPROVED_PENDING_PAYMENT] B -->|"아니오, price=0"| E[ENROLLED] C -->|"approve"| D C -->|"approve (free)"| E C -->|"reject"| F[REJECTED] D -->|"WALLET 결제"| E D -->|"BANK confirm"| E D -->|"paymentDueAt 만료"| G[PAYMENT_EXPIRED] H["정원 초과"] --> I[WAITING] I -->|"자동 승격"| D I -->|"무료 승격"| E E -->|"자가 취소"| J[CANCELED] E -->|"노쇼 한도+FORFEIT_ON_LIMIT"| K[FORFEITED]

핵심 가드

  • 호스트 자신 enroll 금지 — HOST_CANNOT_ENROLL
  • FORFEITED 재신청 금지 — ENROLLMENT_FORFEITED (영구 차단)
  • non-terminal 재신청 금지 — ALREADY_ENROLLED
  • terminal 후 재신청 가능 — 동일 row 재사용 (UNIQUE (meeting_id, user_id))
  • CLOSED/CANCELED 모임 자가 취소 금지 — 클로백 갭 / 중복 환불 방지

자동 승격

ENROLLED 1명이 취소하면 좌석이 비어, WAITING 중 waitlistOrder가 가장 작은 1명이 자동 승격된다. 가격이 0이면 ENROLLED, 아니면 APPROVED_PENDING_PAYMENT(paymentDueAt = now + 72h)로 전환. materializationService.syncMemberAdded로 미래 세션 ATTENDING도 catch-up된다.
