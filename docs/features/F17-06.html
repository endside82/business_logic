F17-06. 고정형 등록·승인·대기열

고정형 모임은 사용자가 코스 전체로 가입한다 — 회차마다 신청하지 않는다. 호스트가 정한 운영 방식(즉시 가입/승인제)과 가격(유료/무료) 조합에 따라 등록 직후 멤버 상태가 8가지로 갈린다. 자리가 다 차면 대기열로 들어가고, 누군가 빠지면 가장 오래 기다린 사람이 자동으로 승격된다. 한 번 자격박탈된 사람은 같은 모임에 다시 들어올 수 없다.

고정형은 한 사람이 모임에 가입한 순간부터 끝까지 한 행으로 추적된다. 같은 사람이 다시 신청하면 이전 행이 재사용된다(취소·거절·결제만료 후 재신청 가능). 단 자격박탈은 영구 차단 — 노쇼 한도로 박탈된 사람은 같은 모임에 다시 들어올 수 없다.

사용자 시나리오

즉시·무료 가입

승인 없이 자리도 있고 무료

사용자가 "참가 신청" 누름. 곧바로 "참가 확정"으로 들어가고, 모든 회차 자동 참석 명단에 즉시 등록된다.

이 흐름이 남기는 것 — 단순한 모임에서 가입 마찰을 최소화. 사용자가 망설이지 않게.

즉시·유료 가입

승인 없이 자리도 있고 유료

"참가 신청" 누름. "결제 대기" 상태로 들어가고 72시간 카운트다운 시작. 결제 화면으로 자동 진입.

이 흐름이 남기는 것 — 사용자의 의사 표명(신청)과 자리 확보(결제)를 명확히 분리, 결제 기한으로 자리 점유를 보호.

승인제

호스트가 승인할 때까지 기다린다

"참가 신청 + 자기소개 메시지" 누름. "승인 대기"로 들어가고 호스트에게 알림(1차 미배선). 호스트가 승인하면 무료는 즉시 확정, 유료는 결제 대기.

이 흐름이 남기는 것 — 호스트가 신중하게 멤버를 받는 모임 운영 가능.

정원 초과

자리가 다 찼다

"참가 신청" 누름. 정원이 이미 차서 "대기열"로 들어간다. 대기 순번이 카드에 표시. 누군가 빠지면 자동으로 다음 단계로 올라간다.

이 흐름이 남기는 것 — 인기 모임에서 사용자가 "지금은 못 들어가지만 기다리면 자리 날 수도"를 인지하게.

대기열 승격

참가 확정자가 한 명 취소

자리가 빈다. 가장 오래 기다린 대기열 멤버 1명이 자동으로 다음 단계(유료면 결제 대기, 무료면 참가 확정)로 올라간다.

이 흐름이 남기는 것 — 사용자가 운영팀 개입 없이도 자기 차례를 받는다.

본인 취소

참가 확정 후 마음을 바꾸다

모집 중 모임에서 "취소" 누름. 환불 예상액 안내 후 확정. 자동 환불이 발사되고 자리는 다음 대기자에게.

이 흐름이 남기는 것 — 사용자 자가 취소의 자유와 환불 예측 가능성 동시 보장.

박탈 차단

박탈된 모임에 다시 신청

"참가 신청" 누르면 "이전에 자격박탈된 모임이라 다시 가입할 수 없어요" 안내. 차단.

이 흐름이 남기는 것 — 모임 운영자 보호. 한 번 신뢰가 깨진 멤버 복귀 차단.

호스트 자기 모임 가입

"내가 만든 코스에 내가 가입"

호스트가 자기 모임에 신청하려고 하면 "호스트는 자기 모임에 가입할 수 없어요" 차단.

이 흐름이 남기는 것 — 호스트가 자기 자리를 셀프 점유해 다른 사용자 가입을 막는 일을 차단.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
상태 뱃지"승인 대기·결제 대기·참가 확정·대기열" 등 자기 상태가 상단 라벨로 노출.한 사용자가 같은 모임 화면을 여러 번 들르므로, 자기 위치가 화면 맨 위에 보여야 다음 행동이 명확.
대기 순번"대기 5/12번" 표기.사용자가 자기 차례를 가늠해 "더 기다릴지·다른 모임을 볼지" 결정하게.
결제 카운트다운"결제 대기"에서 실시간 카운트다운(67시간 남음).72시간 결제 기한이 핵심 정책이라 긴급성을 시각화.
취소 시 환불 예상액취소 누르면 dialog에 "지금 취소하면 X원 환불됩니다" 안내.사용자가 시점에 따라 환불액이 달라지는 코스 환불 산식을 결정 전에 인지하게.
박탈 안내박탈된 사용자에게는 신청 버튼이 회색·비활성.버튼이 활성이면 사용자가 눌렀다가 차단당해 혼란. 버튼 단계에서 막는다.

운영자가 알아야 할 것

자가 취소·재신청·박탈 규칙
  • 호스트는 자기 모임에 신청 불가 — 자기 자리 셀프 점유 차단.
  • 종료·취소 모임의 자가 취소 차단 — 정산 직전·후 중복 환불을 막기 위한 조치. UI에서도 버튼 자체가 숨김.
  • 자격박탈은 영구 — 한 번 박탈된 멤버는 같은 모임 재신청 불가. 정책으로 사전 안내 필수.
  • 취소·거절·결제만료 후 재신청은 같은 행을 재사용 — 이력이 새 행으로 쌓이지 않고 상태만 갱신.
  • 대기열 승격 알림 1차 미배선 — 멤버가 자기 차례가 됐는지 알림 없이는 모름. 호스트 안내 또는 사용자 새로고침 의존.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 대기열 승격 → 결제 안 함 → 다음 사람 승격 — 자리가 한 명씩 옆으로 흘러가는 동안 멤버는 알림이 없어 못 따라옴.
  • 종료 직전 자가 취소 시도 — 막혀서 안 되지만, 멤버가 환불 의도였다면 항의 가능. 운영자 중재 경로 필요.
  • 박탈 후 재신청 거절 — 사용자 입장에서 "왜 안 되지?" 의문. 박탈 시점 안내가 필수.
