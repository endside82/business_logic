F17-06. 고정형 등록·승인·대기열

고정형 모임은 사용자가 코스 전체로 가입한다 — 회차마다 신청하지 않는다. 호스트가 정한 운영 방식(즉시 가입/승인제) 과 가격(유료/무료) 조합에 따라 등록 후 멤버 상태가 8가지로 갈린다. 자리가 다 차면 대기열로 들어가고, 누군가 빠지면 가장 오래 기다린 사람이 자동으로 승격된다.

먼저 기다린 사람 우선
8가지 상태가 한 멤버를 따라다닌다

고정형은 한 사람이 모임에 가입하는 순간부터 끝까지 한 행으로 추적된다. 같은 사람이 다시 신청하면 이전 행이 재사용된다(취소·거절·결제만료 후 재신청 가능). 단 자격박탈은 영구 차단 — 노쇼 한도로 박탈된 사람은 같은 모임에 다시 들어올 수 없다.

자리가 빌 때마다 가장 오래 기다린 대기열 멤버가 자동으로 다음 단계로 올라간다(승격 후 무료 모임은 즉시 참가 확정, 유료는 72시간 결제 대기).

운영자가 알아야 할 것

자가 취소·재신청·박탈 규칙
  • 호스트는 자기 모임에 신청할 수 없다 — "내가 만든 코스에 내가 가입" 시도는 명시적으로 차단.
  • 종료·취소 모임의 자가 취소 차단 — 정산 직전·후의 중복 환불을 막기 위한 조치. UI 에서도 버튼 자체가 노출 안 됨.
  • 자격박탈은 영구 — 한 번 박탈된 멤버는 같은 모임 재신청 불가. 정책으로 사전 안내 필수.
  • 취소·거절·결제만료 후 재신청은 같은 행을 재사용 — 이력이 새 행으로 쌓이지 않고 상태만 갱신.
  • 대기열 승격 알림이 1차 출시에 미배선 — 멤버가 모름. 호스트가 안내하거나 사용자 새로고침에 의존.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 대기열 승격 → 결제 안 함 → 다음 사람으로 다시 승격 — 자리가 한 명씩 옆으로 흐르는 동안 멤버는 알림이 없어 못 따라옴.
  • 종료 직전 자가 취소 시도 — 막혀서 안 되지만, 멤버가 환불 의도였다면 항의 가능. 운영자 중재 경로 필요.
  • 박탈 후 재신청 거절 — 사용자 입장에서는 "왜 안 되지?" 의문. 박탈 시점 안내가 필수.
