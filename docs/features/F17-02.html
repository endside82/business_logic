F17-02. 정기모임 상세 조회

카드에서 진입하면 도착하는 면. 모임 본체·회차 요약·내 위치(보는 사람이 호스트·멤버·게스트 중 누구인가)를 한 화면에 모은다. 정기모임은 사용자마다 다음 행동이 완전히 다르기 때문에 하단 버튼이 가장 중요한 디자인 결정이다 — "지금 보는 사람이 누구이고, 모임이 어떤 상태인가"의 두 축으로 6가지 버튼으로 갈린다. 비로그인도 들어올 수 있지만, 발행 전 초안은 호스트 본인에게만 보인다.

호스트·멤버·게스트가 같은 화면을 보지만 다음 행동은 완전히 다르다. 호스트는 멤버 관리·발행·종료가 필요하고, 결제대기 멤버는 시간이 흐르고 있고, 게스트는 가입을 결정해야 한다. 화면에 행동 버튼을 여러 개 깔지 않고, 상황별로 가장 의미 있는 버튼 1~3개만 남겨 사용자가 망설이지 않게 한다.

사용자 시나리오

게스트 가입 결정

모집 중 모임을 처음 본다

화면 상단에 운영 방식 뱃지·호스트 닉네임·회차 요약. 하단에 "참가 신청" 버튼 1개. 화면을 스크롤하면 회차 일정을 카드로 본다.

이 흐름이 남기는 것 — 가입 전 사용자가 모임 전체 일정을 확인할 수 있어야 가입 결정의 망설임이 줄어든다.

결제대기 멤버

승인 후 결제 기한이 흐른다

하단 버튼은 "결제하기" + 남은 시간 카운트다운(예: "67시간 남음"). 상단에는 "결제 대기" 라벨이 노란색으로 표기된다.

이 흐름이 남기는 것 — 결제 기한 만료 = 자리 빼앗김 + 대기자 자동 승격. 카운트다운으로 긴급성을 시각화.

참가확정 멤버

이미 가입한 코스를 다시 본다

하단 버튼은 "취소" 1개. 환불 기한 안내가 함께 노출되어 "지금 취소하면 얼마 돌려받는지"를 카드 단계에서 가늠한다.

이 흐름이 남기는 것 — 취소 시점이 환불 금액을 좌우하므로, 결정 화면에서 환불 산정 기준을 미리 보여 준다.

호스트 초안

본인 초안을 다시 연다

하단에 "발행하기" 버튼. 회차가 부족하면 "회차를 N개 더 추가해야 발행할 수 있어요" 안내. 초안은 외부 사용자에게는 "모임을 찾을 수 없습니다"로 가려진다.

이 흐름이 남기는 것 — 호스트가 안전하게 모임을 다듬을 수 있는 보호된 공간 유지.

호스트 운영

모집 중 모임의 운영자 진입

하단 버튼이 "멤버 관리 · 회차 추가 · 출석 확정" 세 개로 갈린다. 결제 대기 멤버 수가 카드 형태로 노출된다.

이 흐름이 남기는 것 — 호스트 일과의 핵심 액션을 한 화면에서 분기 없이 진입.

호스트 종료

코스 종료 후 정산을 본다

하단 버튼이 "정산 보기" 1개. 정산이 자동 생성되기 전이면 "준비중" 라벨 + 새로고침 안내.

이 흐름이 남기는 것 — 종료 직후의 짧은 대기 시간을 사용자가 오류로 오인하지 않게 한다.

취소 모임

취소된 모임을 본다

본문이 회색 처리되고 "취소된 모임" 라벨이 상단에 붙는다. 멤버에게는 환불 안내 라벨이 같이 노출된다.

이 흐름이 남기는 것 — 사용자가 카드 클릭 → 빈 화면 같은 단절을 겪지 않게, 취소된 모임도 상태를 명시한다.

회차 진입

회차 카드 탭

회차 카드를 누르면 일반 이벤트 상세로 이동한다. 단, 화면 상단에 "이건 정기모임의 N회차" 표기가 항상 같이 노출된다.

이 흐름이 남기는 것 — 일반 이벤트 화면을 재사용하면서도 사용자가 컨텍스트를 잃지 않게 한다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
상태 라벨"모집 중·종료·취소·결제 대기" 등 상태가 상단에 색으로 구분.같은 화면이라도 상태에 따라 하단 액션이 달라지므로, 사용자가 자기 위치를 먼저 인지하게 한다.
하단 버튼최대 3개. 상황에서 가장 의미 있는 액션만 남긴다.버튼이 많아지면 사용자가 망설이고, 적은 버튼이 결정 속도를 높인다.
회차 카드 영역회차 시작 시각·장소·정원이 카드로 줄지어 노출.코스 가입 결정에는 "이 코스가 내 일정에 맞는지"가 가장 중요해, 상세 화면에서 일정 가시성을 최대화.
호스트 정보호스트 닉네임·운영 이력 표기.코스는 호스트 신뢰가 가입 결정의 핵심 요소.
결제대기 카운트다운결제 대기 상태에서만 노출. 남은 시간이 실시간으로 줄어든다.72시간 안에 결제 안 하면 자리가 빠지는 정책이 있으므로, 긴급성을 시각화.
회차 컨텍스트 표기회차 상세에서 "정기모임 N회차" 라벨이 상단 고정.일반 이벤트 화면 재사용 시 사용자가 컨텍스트를 잃지 않도록 표기 유지.

운영자가 알아야 할 것

화면 동작
  • 초안 모임은 외부에 "찾을 수 없음"으로 가린다 — 초안의 존재 자체가 노출되지 않게 해, 호스트가 준비 단계에서 노출 부담을 갖지 않게 한다.
  • 정기모임 회차에는 일반 신청 버튼이 노출되지 않는다 — 고정형은 본체 가입이 필수이고, 변동형은 회차에서 신청.
  • 취소된 모임도 본문은 보여 준다 — 환불 안내를 사용자가 확인할 수 있게.
  • 종료 직후 정산은 "준비중"이 잠시 보일 수 있다 — 안전망 재시도 1시간 안에 자동으로 갱신.
  • 비호스트가 초안 URL을 직접 쳐도 노출되지 않는다 — 공유 링크 보호.

레드 플래그

자주 발생하는 케이스
  • 결제 대기 멤버 호스트 관리 화면이 1차 미신설 — 입금 확인은 별도 진입을 통해야 한다. 카운트만 노출되어 호스트가 답답할 수 있음.
  • 회차별 출석 명단 화면 부재 — 호스트가 회차마다 누가 왔는지 한 명씩 보는 UI는 1차에서 빠짐. 일괄 확정만 가능.
  • 비호스트 사용자가 회차 카드 깊이까지 들어가도 정기모임 컨텍스트 유지 — 라벨이 빠지면 사용자가 일반 이벤트로 오인.
