F05-01. 키워드 검색

사용자가 검색창에 한 단어를 입력하면 이벤트·클럽·플랜 세 영역에서 같은 단어로 동시에 결과를 받아 본다. 한 번 입력하면 세 곳의 카탈로그를 같이 둘러보게 되어, "이 앱에 내가 찾는 게 있는지"를 한 자리에서 확인할 수 있다. 이벤트는 무한 스크롤과 다섯 가지 정렬을 지원하고, 클럽·플랜은 첫 묶음을 보여 주어 진입 분기 역할을 한다.

이벤트는 시간 단위 모임이고, 클럽은 지속 모임이며, 플랜은 콘텐츠 상품이다. 사용자가 "등산"이라고 떠올렸을 때 이 셋 중 어느 쪽을 원하는지 미리 정할 필요는 없게 한다. 결과 화면에서 영역 탭으로 갈아타게 두어, 입력 한 번이 세 카탈로그의 둘러보기로 이어진다. 이벤트만 무한 스크롤을 지원하는 이유는 이벤트가 카탈로그의 가장 큰 줄기이기 때문이고, 클럽·플랜은 "검색 결과 안에 있다"를 보여 주는 진입 분기로 충분하다고 판단한다.

이 화면이 하는 일

동시 조회

한 입력으로 이벤트·클럽·플랜을 동시에

사용자가 단어 하나 입력하고 엔터를 누르면 세 영역 호출이 같이 떠난다. 영역 탭에는 각각의 결과 개수가 함께 표시되어, 사용자가 어느 쪽으로 갈지 한눈에 판단할 수 있다.

정렬

이벤트는 다섯 가지 정렬

추천순(기본)·최신순·인기순·가격 낮은순·가격 높은순. "추천순"은 시작 시각이 다가오는 미래 이벤트를 인기·최신 신호와 함께 정렬한다. 사용자가 정렬을 바꾸면 첫 페이지부터 다시 받아 본다.

무한 스크롤

이벤트 카탈로그를 끝까지 둘러본다

이벤트 결과는 끝까지 스크롤하며 둘러볼 수 있다. 클럽·플랜 탭은 첫 20건만 노출되며, 더 둘러보려면 도메인 본 화면으로 진입하게 둔다.

자동 기록

검색하면 최근 검색어로 들어간다

로그인 사용자가 검색을 실행하면 입력 키워드가 자동으로 최근 검색어 목록 맨 앞에 들어간다. 사용자는 검색 후 "저장하라"는 단계 없이도 다음 방문에서 같은 단어를 빠르게 다시 찾을 수 있다 (F05-04).

한 번의 검색이 만드는 흐름

사용자가 단어를 입력하고 엔터를 누른 순간부터 세 영역이 결과 카드로 채워지기까지.

사용자 시나리오

기본 흐름

"등산" 한 단어로 세 카탈로그 둘러봄

주말에 등산 모임을 찾고 싶은 사용자가 검색창에 "등산"을 입력한다. 결과 화면에 "이벤트 24 · 클럽 8 · 플랜 2" 탭이 뜨고, 기본은 이벤트 탭이 활성화되어 있다. 사용자는 이벤트를 둘러본 뒤 "클럽 8" 탭으로 갈아타 같은 단어로 만들어진 클럽 목록을 본다.

이 흐름이 남기는 것 — 입력 한 번으로 세 카탈로그를 함께 본다는 경험. 사용자가 처음부터 "이건 이벤트야, 클럽이야"를 분류해 들어갈 필요가 없다.

깊게 둘러봄

이벤트 결과를 끝까지 스크롤

"독서" 검색으로 이벤트 30건이 나왔다. 사용자가 스크롤하면 자연스럽게 다음 묶음이 이어 붙는다. 끝에 도달하면 더 이상 카드가 나오지 않고 정지한다.

이 흐름이 남기는 것 — 이벤트는 검색의 가장 큰 줄기이므로 끝까지 둘러볼 수 있게 둔다. 클럽·플랜은 진입 분기로 충분하다고 판단해 첫 묶음만 노출한다.

정렬 분기

가격 낮은 순으로 보고 싶다

"요리" 결과를 둘러보다가 정렬 드롭다운에서 "가격 낮은순"을 선택한다. 화면이 첫 페이지부터 다시 받아 와서, 무료 또는 저렴한 이벤트가 맨 위로 올라온다.

이 흐름이 남기는 것 — 정렬 변경은 누적된 묶음에 적용하지 않고 처음부터 다시 받아 본다. "내가 본 위치"가 정렬 기준에서 의미를 잃기 때문에 자연스러운 동작이다.

빈 결과

찾는 단어가 없다

"asdfqwer" 같은 의미 없는 단어로 검색하면 세 영역 모두 0건이 나온다. 화면에는 "검색 결과가 없습니다 — 다른 키워드로 검색해 보세요" 안내가 떠오른다.

이 흐름이 남기는 것 — "결과 없음"과 "필터 너무 강함"이 같은 메시지가 되지 않게, 필터 적용 중 빈 결과(F05-03)와는 다른 안내 흐름을 갖는다.

가입 전 탐색

비로그인 사용자가 둘러봄

로그인 없이도 검색은 열려 있다. 입력하고 엔터를 누르면 세 영역 결과가 정상적으로 보인다. 단, 최근 검색어 영역은 비어 있고, 검색을 해도 기록되지 않는다. 결과 카드를 눌러 상세에 들어간 뒤 신청·가입 같은 행동을 할 때 비로소 로그인 화면이 떠오른다.

이 흐름이 남기는 것 — 가입 전 사용자가 "이 앱에 내가 찾는 게 있는지" 확인할 수 있어야 가입 의사가 형성된다. 검색은 그 확인 수단이라 비로그인에도 열어 둔다.

탭 복귀

결과 카드 들어갔다 돌아옴

이벤트 카드를 눌러 상세에 들어갔다가 뒤로가기로 검색 화면에 돌아온다. 결과 리스트와 스크롤 위치가 그대로 보존되어, 사용자는 둘러보기를 이어 갈 수 있다.

이 흐름이 남기는 것 — 카드 탐색은 보통 여러 장을 비교하는 행위이므로 상태 보존이 핵심.

실패 분기

이벤트 검색이 일시적으로 안 됨

네트워크 문제로 이벤트 응답이 실패하면 이벤트 탭에 "오류가 발생했습니다 — 다시 시도" 표시가 나온다. 같은 입력으로 호출된 클럽·플랜은 정상적으로 보일 수 있다 — 영역별로 독립이다.

이 흐름이 남기는 것 — 한 영역의 실패가 다른 영역의 결과를 가리지 않게 둔다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
검색 입력바 입력창, X(클리어), 필터 아이콘(활성 시 점 표시), 북마크(검색 후만), 취소 버튼. 입력·필터·저장 세 동작이 자주 같이 쓰이므로 한 줄에 모아 둔다. 활성 필터의 점 표시는 "지금 필터가 걸려 있다"를 사용자가 잊지 않게 한다.
활성 필터 칩 적용된 필터가 있으면 검색바 바로 아래에 칩으로 나열. 각 칩은 X로 개별 제거, 맨 앞 칩은 전체 초기화. "왜 결과가 적은가"를 사용자가 한눈에 인지하게 한다. 결과가 빈 상태일 때 필터를 잊고 있던 사용자를 구해 주는 핵심 단서.
영역 탭 "이벤트 N · 클럽 N · 플랜 N" 칩. 활성 탭은 강조 색. 입력 한 번이 세 카탈로그를 동시에 조회한다는 사실을 카운트로 분명하게 보여 준다.
결과 요약 + 정렬 "검색 결과 N건" 좌측 + "추천순/최신순/인기순/가격↑/가격↓" 드롭다운. 이벤트 탭은 사용자가 가장 깊게 둘러보는 영역이므로 정렬 통제권을 분명히 노출한다.
이벤트 카드 썸네일·제목·시작 시각·주소·현재 인원·가격. 사용자 컨텍스트(신청 여부·멤버십)가 있으면 배지로. 이벤트 의사결정에 가장 먼저 필요한 정보(언제·어디서·얼마)를 카드 안에 모아, 상세에 들어가지 않아도 1차 판단이 가능하게 한다.
클럽·플랜 카드 클럽은 카테고리와 멤버 수, 플랜은 가격과 구매 수를 카드 안에 함께 보여 준다. 두 카탈로그는 도메인 본 화면이 따로 있는 분기 진입점이므로, 카드는 진입 결정을 도와주는 최소 정보만 담는다.
빈 결과 안내 검색 아이콘 + "검색 결과가 없습니다" + "다른 키워드로 검색해 보세요". 키워드 자체가 없을 때의 안내. 필터로 좁혀 0건이 된 경우(F05-03)는 다른 메시지로 분리한다.

운영자가 알아야 할 것

검색 결과 화면의 정책 결정점
  • 이벤트는 끝까지 스크롤, 클럽·플랜은 첫 묶음만. 이 비대칭은 검색의 무게 중심을 이벤트에 두는 의도적 선택이다. 클럽·플랜의 더 깊은 둘러보기를 원한다는 데이터가 쌓이면, 도메인 본 화면으로 가는 동선을 강화하거나 검색에서 무한 스크롤을 풀어야 한다.
  • 로그인 사용자가 검색하면 키워드는 자동으로 최근 검색어에 기록된다. 사용자가 별도로 "저장" 동작을 하지 않아도 다음 방문에서 같은 단어가 한 번에 보인다 — 검색의 재방문성을 지탱하는 가장 조용한 동력.
  • 비로그인 사용자에게도 검색은 열려 있다. 가입 결정 전에 "이 앱에 내가 찾는 게 있는지"를 확인할 수 있어야 가입 의사가 형성되기 때문이다. 단, 결과 카드를 누른 후 신청·가입·구매 시점에는 로그인 가드가 동작한다.
  • 거리 필터를 사용하려면 위치 권한이 필요하다. 권한이 없을 때 거리 조건은 결과에 반영되지 않고 조용히 무시된다 — 사용자에게는 칩이 보이지만 결과가 줄지 않는다. 이 사일런트 무효는 F05-03에서 별도 운영 이슈로 다룬다.
