F03-04. 이벤트 수정 · 생명주기 관리 (호스트)

이벤트의 “이후 모든 운영”을 책임진다. DRAFT 수정·발행, OPEN 일정 변경(전원 알림), 취소(자동 환불), 공지 발송(throttle), 삭제(DRAFT만), 종료 후 회고까지. 사용자 환불·알림 같은 다른 단위와 가장 많이 얽혀 있어 상태 전이의 단방향성(DRAFT→OPEN→CLOSED/CANCELED)을 깨면 시스템 전체가 흔들린다.

생명주기 액션
6
시나리오
11
상태
DRAFT/OPEN/CLOSED/CANCELED
상태 전이 일방향

DRAFT → OPEN → (CLOSED 또는 CANCELED). 한 번 떠난 상태로는 절대 돌아오지 않는다. publish 재시도, 종료된 이벤트 재발행 같은 회귀는 모두 상태 전이 가드에서 차단되어야 한다.

상태 전이

stateDiagram-v2 [*] --> DRAFT: 생성 DRAFT --> OPEN: publish DRAFT --> [*]: 삭제 (DRAFT만 가능) OPEN --> OPEN: 일정 변경 (alert fanout) OPEN --> CLOSED: 종료 OPEN --> CANCELED: 취소 (자동 환불 + alert) CLOSED --> [*]: 리뷰·사진첩 진입점 CANCELED --> [*]: 조회만

이 기능 안의 하위 항목

동작 1

수정 (DRAFT)

모든 필드 수정 가능. OPEN 이후에는 일정·정원이 lock.

동작 2

발행

DRAFT → OPEN. publishedAt 기록.

동작 3

일정 변경 (OPEN)

ATTENDING+WAITING 모두에게 EVENT_UPDATED 알림 fanout.

동작 4

취소 (OPEN)

ATTENDING은 자동 환불, 실패는 failed_refund + 호스트 알림.

동작 5

공지 발송

참석자 fanout. 1분당 한도(throttle).

동작 6

삭제

DRAFT만 삭제. OPEN 이후는 취소/종료를 사용.

사용자 시나리오 11개

S1 · Happy Path

DRAFT 수정 후 발행

  1. 수정 화면 진입 → 기존값 prefill
  2. 변경한 필드만 PATCH 전송
  3. 상세 복귀 → “발행” 다이얼로그 → publish
  4. 액션바가 “일정변경/종료/취소”로 교체
S2 · 알림 fanout

OPEN 일정 변경

  1. 일정 변경 다이얼로그에 새 시각 + 사유 입력
  2. 서버: 시간 검증 → UPDATE → ATTENDING/WAITING 전원에게 EVENT_UPDATED 알림
  3. 참석자가 푸시로 새 시각 확인
S3 · 자동 환불

유료 이벤트 취소

  1. 취소 다이얼로그에 환불 금액 안내
  2. 서버: status=CANCELED, ATTENDING 각각 환불 시도
  3. 실패는 failed_refund + 호스트 알림
  4. 참석자/대기자 모두에게 EVENT_CANCELLED 알림
S4 · 공지

참석자 전체 공지

  1. 공지 다이얼로그에 제목·본문
  2. 호스트 제외 참석자 fanout
  3. 발송 알림 카운트 갱신
S5 · throttle

공지 한도 초과

  1. 1분 내 4번째 공지 시도
  2. 429 RATE_LIMIT_EXCEEDED
S6 · 삭제

DRAFT 삭제

  1. “삭제” 확인 다이얼로그 → 204
  2. 마이 이벤트 목록 갱신
S7 · 삭제 차단

발행된 이벤트 삭제 시도

  1. UI에서는 버튼 숨김
  2. 우회 시 400 EVENT_INVALID_STATUS_TRANSITION
  3. “취소/종료를 사용해주세요” 안내
S8 · 권한

비호스트의 수정 시도

  1. 수정 화면 진입 시 권한 가드 → “접근 권한이 없습니다”
  2. 우회 시 403 EVENT_NOT_OWNER
S9 · 검증

일정 변경 시간 검증 실패

  1. 새 시각을 과거로 입력
  2. 클라이언트 picker가 미래만 허용
  3. 우회 시 400
S10 · 종료/취소 수정 차단

CLOSED/CANCELED 수정 시도

  1. 수정 화면 진입 시 “종료되었거나 취소된 이벤트는 수정할 수 없습니다” + 돌아가기
  2. 수정 폼은 마운트되지 않음
S11 · 위치 정책

호스트 위치 공유 안내 화면

  1. 오프라인 이벤트 호스트가 위치 공유 정책을 확인
  2. “참석 확정자에게만 위치 공유” 라벨 표시
  3. 토글 OFF 시 PATCH로 갱신

서버 계약

MethodPath설명
PATCH/api/v1/events/{id}수정(DRAFT 전 필드, OPEN은 일부만)
POST/api/v1/events/{id}/publishDRAFT → OPEN
PATCH/api/v1/events/{id}/reschedule일정 변경 + 참석자 알림
POST/api/v1/events/{id}/cancel취소 + 자동 환불 + 알림
POST/api/v1/events/{id}/announce공지 발송(throttle)
DELETE/api/v1/events/{id}DRAFT만 삭제
에러 코드
  • 400 EVENT_INVALID_STATUS_TRANSITION — DRAFT가 아닌데 삭제, OPEN이 아닌데 일정 변경 등
  • 403 EVENT_NOT_OWNER — 비호스트 운영 액션
  • 429 RATE_LIMIT_EXCEEDED — 공지 throttle 초과
  • 400 INVALID_INPUT — 시각 검증 실패

자주 빠뜨리는 케이스

레드 플래그
  • 일정 변경 후 알림 fanout 누락 — 참석자가 옛 시각으로 이동.
  • 취소 시 환불 실패가 침묵 — failed_refund + 호스트 알림이 필수.
  • 발행된 이벤트 삭제 우회 허용 — 결제·정산 일관성 붕괴.
  • 일정 변경 사유 입력 없이 알림 발송 — 참석자가 무슨 변경인지 모름.
  • 공지 throttle 미적용 — 사용자 푸시 폭주, 차단 위험.
