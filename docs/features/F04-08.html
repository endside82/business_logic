F04-08. 게시판·게시글

멤버가 글로 클럽에서 활동하는 가장 일반적인 면. 클럽을 만들면 공지·자유·가입 인사 세 게시판이 자동으로 같이 생기고, 운영자는 추가 게시판을 만들거나 글을 상단에 고정할 수 있다. 멤버는 글을 쓰고 자기 글을 수정·삭제하며, 관리자는 부적절한 글을 삭제하거나 모임을 알리는 글을 고정한다.

설계 의도 — 글의 종류에 따라 책임이 다르다

공지는 운영자가 클럽 전체에 알리는 면, 자유는 멤버 누구나 일상의 이야기를 나누는 면, 가입 인사는 새 멤버가 자기를 소개하는 면. 세 종류의 글이 한 게시판에 섞이면 사용자는 "어떤 글이 진짜 공지인지" 찾기 어려워지고, 운영자는 "어떤 글을 모니터링해야 하는지" 결정하기 어려워진다. 그래서 게시판이 글의 책임 단위가 된다 — 공지 게시판은 처음부터 운영자만 글을 쓸 수 있게 막혀 있고, 자유·가입 인사는 멤버 누구에게나 열려 있다.

2026-06 기능 추가 — 예약 게시 · 좋아요 · 이미지 안정화
  • 예약 게시(지정 시각 자동 공개) — 글 작성 시 미래 날짜·시각을 지정하면 그 시각 전까지 피드에 노출되지 않는다. 지정 시각이 되면 자동으로 공개 전환되며, 전환에 실패하면 작성자에게 "예약 게시 실패" 알림이 전송된다. 게시판이 삭제될 때 이미 예약 중인 글은 자유게시판으로 옮겨져 보존된다. (참고: 앱에서 예약 상태임을 나타내는 라벨 표시는 추후 보강 예정)
  • 게시글 좋아요 — 게시글에 하트 버튼이 생겼다. 한 번 누르면 좋아요, 다시 누르면 취소. 두 번 눌러도 한 번만 반영되고, 여러 사람이 동시에 눌러도 좋아요 수가 정확히 집계된다. 목록과 상세 화면 모두에서 현재 좋아요 수와 내 좋아요 여부를 확인할 수 있다.
  • 이미지 업로드 안정화 — 이미지 업로드가 끝까지 확정 처리되어야 글에 표시된다. 업로드가 중간에 끊겨 미완료 상태로 남으면 24시간 후 해당 이미지가 자동 소실된다. 또한 서버가 글 등록 시 이미지가 본인이 올린 것인지 재확인해, 다른 사람의 이미지가 도용되는 것을 차단한다.

사용자 시나리오

기본 흐름

모임 후기 글에 사진을 두 장 첨부해서 올린다

멤버가 자유 게시판에 후기 글을 쓴다. 갤러리에서 사진 두 장을 첨부하고 등록하면 사진이 자동으로 사용자 기기에서 클라우드로 올라간 뒤 글이 발행된다. 다른 멤버는 곧장 그 글을 볼 수 있다.

권한 가드

일반 멤버가 공지 게시판에 글을 쓰려 한다

일반 멤버에게는 공지 게시판 진입 시 글쓰기 버튼이 보이지 않는다. 외부에서 강제로 작성을 호출해도 "작성 권한이 없습니다" 안내가 떠오른다. 공지는 운영자만 쓰는 면이라는 사실을 화면과 서버 양쪽에서 보장.

글 고정

관리자가 중요한 모임 안내 글을 상단에 고정

관리자가 글 상세에서 더보기 → "고정"을 누르면 그 글이 게시글 목록 상단에 머문다. 다시 "고정 해제"를 누르면 일반 순서로 돌아온다. 고정 토글은 운영자 단독 권한.

이 흐름이 남기는 것 — 중요한 글이 시간이 지나도 게시판 위에서 보이게 하는 유일한 수단. 공지 도메인과 별개로 게시글 자체를 강조하는 방법.

수정 권한

관리자가 다른 멤버의 글을 수정하려 한다

관리자에게도 다른 멤버의 글 더보기에서 "수정" 항목은 보이지 않는다. 글의 내용은 작성자 본인만 손볼 수 있고, 관리자에게는 삭제와 고정 권한만 있다.

이 흐름이 남기는 것 — 다른 사람의 글 내용을 운영자가 손보는 것은 검열의 영역. 운영자에게는 삭제 권한만 주고 수정은 작성자에게만 남긴다.

조회 카운트

한 글이 여러 멤버에게 읽힌다

멤버가 글 상세에 들어갈 때마다 조회 수가 한 번씩 올라간다. 같은 사람이 다시 들어가도 카운트가 누적된다. 카운트는 글 카드와 상세에 함께 표시되어 어떤 글이 활발히 읽히는지 보여 준다.

사진 한도

사진 열한 장 이상을 첨부하려 한다

화면 자체가 열 장을 넘는 추가 첨부를 막는다. 외부에서 강제로 호출해도 "사진은 최대 10장까지 가능합니다" 안내로 거절된다.

게시판 추가

운영자가 "후기" 게시판을 새로 만든다

게시판 목록 우하단 + 버튼 → 게시판 이름 입력 → 만들기. 기존 게시판들 뒤에 새 게시판이 추가된다. 일반 멤버에게는 + 버튼이 보이지 않으며 새 게시판 만들기는 운영자 단독 권한.

기본 게시판 보호

운영자가 기본 게시판을 삭제하려 한다

공지·자유·가입 인사 세 기본 게시판은 운영자도 삭제할 수 없다. 더보기 메뉴 자체에 "삭제"가 없고, 외부에서 강제로 호출해도 거절된다. 기본 게시판이 사라지면 클럽 활동의 기본 골격이 무너진다.

작성 중 이탈

글을 절반 쓰다가 뒤로 가기를 누른다

제목이나 본문에 내용이 있는 상태에서 화면을 떠나려 하면 "작성 중인 내용이 있습니다" 다이얼로그가 떠오른다. 사용자가 의도적으로 "나가기"를 선택해야 글이 버려진다.

예약 게시

다음 주 월요일 아침에 자동으로 올라가도록 글을 예약한다

글 작성 화면에서 날짜·시각을 지정하고 등록하면 지정한 시각 전까지 피드에 나타나지 않는다. 시각이 되는 순간 자동으로 공개 전환되어 멤버들이 볼 수 있게 된다. 미리 써 두고 원하는 시간에 올리고 싶은 운영자에게 유용하다.

이 흐름이 남기는 것 — 예약 게시글이 들어 있는 게시판을 삭제해도 해당 글은 자유게시판으로 옮겨져 사라지지 않는다.

좋아요

마음에 드는 모임 후기 글에 좋아요를 누른다

게시글 카드와 상세 화면의 하트 버튼을 누르면 좋아요가 반영되고 수가 1 올라간다. 다시 누르면 취소되어 수가 줄어든다. 두 번 연속으로 빠르게 눌러도 한 번만 반영되고, 여러 사람이 동시에 눌러도 좋아요 수가 정확히 집계된다.

운영자가 알아야 할 것

게시판·게시글 운영 정책
  • 게시판이 글의 책임 단위라는 정책은 흐려질 수 없다. 공지 게시판이 일반 멤버에게 열리면 공지의 의미가 사라진다. 새 게시판을 추가할 때도 "이 게시판은 누가 쓸 수 있는가"를 명확히 정해야 한다.
  • 운영자가 글을 수정하지 못한다는 정책은 검열 방지의 핵심이다. 운영자에게 수정 권한을 주면 사용자가 자기 글이 임의로 바뀌었는지 확인할 길이 없다. 운영자는 삭제만 할 수 있고, 삭제된 글은 사용자에게 사실 자체가 드러난다.
  • 기본 게시판 삭제 차단이 풀리면 클럽 골격이 무너진다. 공지·자유·가입 인사 게시판은 클럽 활동의 기본 단위. 한 번 사라지면 같은 이름으로 다시 만들어도 기존 글이 돌아오지 않는다.
  • 사진 첨부 한도가 풀리면 업로드 비용과 사용자 경험이 같이 무너진다. 열 장 이상 사진이 들어간 글은 스크롤이 답답해지고, 사용자 기기의 메모리도 무거워진다.
  • 예약 게시 글은 지정 시각이 될 때까지 다른 멤버에게 노출되지 않는다. 예약 전환에 실패하면 작성자에게 알림이 가므로, 시각이 지났는데도 글이 안 올라왔다는 문의가 오면 알림 내역을 먼저 확인하도록 안내한다.
  • 좋아요는 집계 정확성이 보장되는 기능이다. 같은 사람이 두 번 누르거나 여러 기기에서 동시에 눌러도 중복 반영되지 않는다. 좋아요 수는 글의 인기를 나타내는 신뢰할 수 있는 지표로 활용 가능하다.
  • 이미지 업로드는 업로드 확정 처리까지 완료되어야 글에 표시된다. 네트워크 문제로 업로드가 중간에 끊겨 확정 처리가 이루어지지 않으면 해당 이미지는 24시간 후 자동으로 사라진다. 글 등록 후 이미지가 보이지 않는다는 문의가 들어오면 업로드 완료 여부를 먼저 확인한다.
