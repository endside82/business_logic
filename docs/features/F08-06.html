F08-06. 마켓 아이템 관리

마켓에 노출되는 상품(아이템) 단위를 크리에이터가 직접 관리하는 화면. 플랜과 별개로 상태가 흐른다 — 판매 대기 → 판매 중 → 판매 중지 → 삭제됨. 가격·설명·재고·노출 순서를 손볼 수 있고, 한 번에 한 액션으로 판매를 시작·중지·삭제한다. 플랜은 콘텐츠 자체의 라이프사이클, 아이템은 마켓 진열의 라이프사이클 — 같은 콘텐츠를 두 단계로 통제하는 구조다.

플랜(F08-02)이 콘텐츠의 라이프사이클(작성·발행·숨김)을 다룬다면, 아이템은 마켓 진열의 라이프사이클을 다룬다. 플랜은 그대로 두고 진열만 잠시 내릴 수 있고, 같은 플랜을 다른 가격·설명으로 다시 진열할 수도 있다. 크리에이터가 자기 콘텐츠의 운영을 더 세밀하게 다룰 수 있는 한 단계 위의 통제권이다.

아이템 상태의 흐름

새 아이템은 항상 대기 상태로 만들어진다. 발행을 누르면 판매 중으로 가고, 중지·재발행·삭제가 가능하다. 삭제도 데이터는 보존하는 표시 변경이다.

이 화면이 다루는 네 가지

동작 1

메타 수정

제목·설명·가격·재고·노출 순서를 손본 뒤 "저장"을 누른다. 아이템 종류와 통화는 생성 시 고정이라 본 화면에서는 바뀌지 않는다.

동작 2

발행하기 (대기 또는 중지에서)

대기 상태 또는 판매 중지 상태에서 발행을 누르면 판매 중으로 간다. 그 순간부터 마켓 검색·인기·최신 목록에 노출된다.

동작 3

판매 중지 (판매 중에서)

판매 중인 아이템을 잠시 내려놓는다. 마켓에서 사라지지만 데이터는 그대로 — 다시 발행하면 같은 아이템으로 돌아온다.

동작 4

삭제

아이템 자체를 정리한다. 데이터는 보존되지만 마켓에서 영구히 사라진다. 위험 색상 버튼 + 확인 다이얼로그로 한 번 더 잡는다.

사용자 시나리오

발행

대기 아이템을 처음 마켓에 푼다

발행해 둔 플랜으로 만든 첫 마켓 아이템. 상태 배지가 "판매대기"로 떠 있다. "발행하기"를 누르고 확인하면 상태가 "판매중"으로 바뀌고, 마켓 메인의 최신 목록에 들어간다.

가격 변경

5,000P를 4,500P로 내린다

잘 안 팔리는 아이템의 가격을 낮춘다. 가격 칸을 수정하고 "저장"을 누르면 곧장 반영되고, 마켓 카드에 새 가격이 표시된다. 상태는 그대로 — 가격 변경 자체는 진열 상태에 영향을 주지 않는다.

이 흐름이 남기는 것 — 가격은 자주 손보는 운영 변수라, 별도 상태 전이 없이 한 번에 바꿀 수 있게 둔다.

중지

일정상 판매를 잠시 멈춘다

운영이 어려운 시즌에 판매 중 아이템을 내려놓는다. "판매 중지" 버튼 + 확인 다이얼로그. 상태가 "판매중지"로 바뀌고 마켓에서 사라진다. 시즌이 풀린 뒤 같은 화면에서 "발행하기"를 다시 누르면 같은 아이템이 돌아온다.

삭제

운영을 정리한다

더 이상 운영하지 않을 아이템을 정리한다. "아이템 삭제" + "되돌릴 수 없습니다" 확인 다이얼로그. 데이터는 보존되지만 마켓에는 다시 노출되지 않는다.

권한 가드

다른 사람의 아이템을 손보려 한다

임의 경로로 다른 크리에이터의 아이템 편집에 도달한다. 화면은 열리지만 저장을 누르면 서버가 본인 소유가 아니라는 이유로 거절한다. 사용자에게는 일반 실패 안내가 뜬다.

잘못된 상태

이미 삭제된 아이템을 수정하려 한다

이전에 삭제된 아이템 페이지에 직접 진입해서 저장을 누른다. 서버는 삭제됨 상태에서는 수정을 받지 않는다고 거절한다. 사용자에게는 일반 실패 안내. 정상 흐름에서는 삭제된 아이템 자체가 노출되지 않으므로 드물게 일어난다.

무료 아이템

0원으로 등록한다

가격을 0으로 두고 등록한다. 마켓에서는 "무료" 라벨이 가격 자리에 표시된다. 구매 흐름은 동일하지만 결제는 일어나지 않는다.

운영자가 알아야 할 것

진열 통제와 정산이 같은 화면에서 갈린다
  • 플랜 상태와 아이템 상태는 독립적이다. 플랜이 발행 중이어도 아이템은 대기 상태일 수 있다. 마켓 노출 여부는 아이템 상태가 결정한다.
  • 아이템 종류와 통화는 생성 후 바꿀 수 없다. 잘못 만든 경우 삭제 후 새로 등록해야 한다. 이 규칙이 화면에서 분명하지 않으면 사용자가 같은 폼에서 종류를 바꿔도 저장되지 않는 회색 지대를 만난다.
  • 판매 시작 시점에 수수료·원천세 계산 기준이 정해진다. 10% 수수료 + 3.3% 원천세 — 크리에이터 수익은 판매 단가에서 두 비율을 뺀 값. 가격을 자주 바꾸는 운영은 정산 페이지에서 사용자가 직접 누적 수익을 추적하기 어려워질 수 있다.
  • 월 50건의 구매 한도는 구매자 측 정책이다. 한 아이템이 인기를 끌어도 한 사용자가 무한 구매할 수 없다 — 이는 마켓 운영 안정과 결제 부정 방지 동시 목적.
  • 재고 칸이 비면 무제한. 일반 플랜은 보통 무제한이지만, 인원 한정 이벤트 패키지 같은 종류에는 재고를 설정한다. 사용자가 의도하지 않게 한도가 걸리는 사고를 피하려면 입력 비어 있을 때의 의미가 화면 카피에 분명해야 한다.
