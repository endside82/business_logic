F18-01. 통합 분쟁 케이스 조회

다양한 원천에서 발생한 분쟁·신고·이의를 단일 목록으로 묶어 보여 주는 화면. 대부분의 분쟁은 신고한 사람과 직접 영향을 받은 당사자에게만 보이고, 사용자가 직접 접수한 분쟁만 신고한 쪽과 신고당한 쪽 양쪽 모두에게 보인다. 호스트는 정산 이의(개설자 시점)와 환불 분쟁(판매자 시점) 두 종류의 분쟁을 별도로 확인한다. 원천 도메인마다 상태를 부르는 이름이 다르더라도, 사용자에게는 항상 같은 5단계 중 하나로 통일되어 보인다. 7일 이상 미해결 케이스는 운영팀에 알림이 자동으로 발송된다.

통합되는 분쟁 원천
참가자 8종
호스트는 정산 이의·환불 분쟁 2종
통합 상태
5단계
접수됨·검토 중·상위 이관·처리 완료·종결
처리 기한
7일
초과 시 운영팀 알림
화면이 안내하는 가능 동작
역할별 분기
이의·노트·환불 처리 등
설계 의도 — "내 분쟁이 지금 어떻게 되고 있는지" 한 눈에

이전에는 노쇼 이의는 이벤트 화면, 정산 이의는 정산 화면, 경고 이의는 클럽 화면으로 각자 흩어져 있었다. 이 기능은 그 모든 케이스를 하나의 목록으로 모아 사용자가 현재 진행 중인 분쟁과 종결된 분쟁을 한 화면에서 추적하게 한다. 원천이 다르더라도 사용자가 보는 상태 표시는 항상 동일하다.

상태를 바꾸는 것은 운영팀만 가능

사용자는 앱에서 케이스 상태를 직접 바꿀 수 없다. 직접 접수한 분쟁은 "접수됨"으로 만들어진 뒤 운영팀이 처리할 때까지 상태가 그대로다. 사용자는 현재 상태를 조회하고 이의를 제기하는 것만 가능하다.

사용자 시나리오

기본 흐름 — 참가자

내가 관련된 분쟁 목록 확인

이벤트 노쇼 기록, 클럽 강퇴 이의, 직접 접수한 분쟁이 모두 한 목록에 보인다. "처리 중" 필터를 선택하면 접수됨·검토 중·상위 이관 케이스만 추려 볼 수 있다. 카드를 탭하면 상세 화면으로 진입해 진행 이력과 증빙을 확인한다.

이 흐름이 남기는 것 — 어느 도메인에서 발생했든 "내 분쟁 이력"이 한 화면으로 집약된다.

기본 흐름 — 호스트

내 이벤트·클럽에 걸린 분쟁 확인

호스트가 확인하는 분쟁은 두 종류다. 정산 이의(개설자 기준)와 환불 분쟁(판매자 기준). 두 분쟁은 호스트에게 묶이는 기준이 다르며, 같은 목록·필터 구조를 공유한다. 상세 화면에서는 호스트가 할 수 있는 동작이 별도 패널로 표시된다.

이 흐름이 남기는 것 — 호스트 입장에서 처리해야 할 케이스를 놓치지 않게 한다.

상세 조회

진행 이력과 증빙으로 맥락 파악

케이스 상세에는 진행 이력(누가 언제 어떤 조치를 했는지)과 증빙이 포함된다. 보이는 내용은 역할에 따라 다르다 — 참가자는 양측에 공개된 증빙만 볼 수 있고, 운영팀 전용 항목은 표시되지 않는다.

이 흐름이 남기는 것 — "왜 이 결정이 났는지"를 사용자가 직접 추적할 수 있다.

상태 필터

종결된 케이스만 이력으로 보기

"종결" 필터를 선택하면 처리 완료·종결 케이스만 남는다. 분쟁 종류로도 목록을 좁힐 수 있으나, 일부 분쟁 유형은 유형 필터로 정확히 걸러지지 않을 수 있어 결과를 확인할 때 주의가 필요하다. 무한 스크롤로 이력 전체를 탐색한다.

권한 분기

내가 뭘 할 수 있는지 화면이 알려준다

케이스 상세는 사용자의 역할에 따라 가능한 동작을 다르게 보여 준다. 이의제기 가능 여부, 보완 요청 노트 전송 가능 여부, 환불 분쟁 처리 가능 여부 등에 맞춰 버튼이 표시되거나 숨겨진다. "이의 승인" 버튼은 아직 동작하지 않아 눌러도 처리되지 않는다.

증빙 보존

케이스가 진행 중이면 증빙 보존 안내가 표시된다

증빙 삭제 기능은 현재 제공되지 않는다(조회 전용). 진행 중(접수됨·검토 중·상위 이관) 케이스에서는 "증빙 보존 중" 안내가 표시된다. 케이스가 종결된 뒤에도 삭제 버튼은 아직 제공되지 않는다.

운영자가 알아야 할 것

운영 주의사항
  • 처리 기한 7일 알림은 분쟁 원천마다 따로 점검한다. 한 원천의 점검이 실패해도 나머지 원천의 미해결 케이스 알림은 정상적으로 발송된다. 같은 케이스에 대한 알림은 한 번만 간다.
  • 운영팀 전용 진행 이력은 사용자에게 보이지 않는다. 현재는 운영팀 전용 메모만 가려지고 나머지 항목은 모두 전달된다 — 공개 등급 4종의 세부 구분은 아직 화면에 반영되지 않았다(자세한 내용은 F18-04).
  • 유형 필터가 일부 분쟁 유형을 정확히 거르지 못할 수 있다. 분쟁 종류가 매우 다양해, 특정 유형으로 목록을 좁혔을 때 일부 케이스가 빠지거나 섞일 수 있다. 정밀한 분류가 필요한 운영 작업에서는 결과를 한 번 더 확인하는 것이 안전하다.
  • 호스트의 이의 승인·거절은 아직 앱에서 불가능하다. 화면에 버튼은 있으나 동작하지 않으며, 현재는 운영팀만 처리할 수 있다.

레드 플래그

현재 알려진 미구현·주의 사항
  • 분쟁 알림을 눌러도 해당 화면으로 가지 않는다. 직접 접수한 분쟁이나 클럽 제명 관련 알림을 누르면 홈으로 이동한다. 환불 분쟁 알림만 정상적으로 연결된다.
  • 호스트의 이의 승인 버튼이 동작하지 않는다. 앱 화면에 이의 승인 버튼이 있으나 아직 동작하지 않아, 눌러도 처리되지 않는다.
  • 통합 목록 정확성 검증이 없다. 다양한 분쟁 원천이 목록에 올바른 형태와 상태로, 올바른 공개범위로 들어오는지 자동으로 점검하는 장치가 아직 없다.
