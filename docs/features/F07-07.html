F07-07. 미납자 독촉 · 마감 연장

정산을 활성화한 다음에도 며칠씩 결제하지 않는 사용자가 있다. 호스트가 그 사용자에게 가볍게 한 번 더 알려 주는 동작이 "독촉", 마감일을 뒤로 미루면서 동시에 한 번 알려 주는 동작이 "마감 연장 + 알림"이다. 둘 다 같은 푸시 알림을 사용하지만, 알림이 사용자에게 도배되지 않도록 24시간 안에는 같은 사람에게 두 번 보내지 않는다. 호스트가 활성화 시점에 마감과 자동 독촉 시간을 설정해 두면 미납자에게 자동으로 한 번 알림이 나가는 흐름도 함께 다룬다.

사용자당 알림 간격
24시간
같은 사람에게 도배 방지
마감 연장 폭
최대 7일
1시간 단위로 조정 가능
자동 독촉
1회만
활성화 옵션으로 예약
설계 의도 — 호스트가 직접 카톡으로 닦달하지 않게 한다

"왜 안 보내냐"고 호스트가 사용자에게 개인 채팅으로 묻는 것은 관계를 해친다. 본 단위는 호스트가 한 번의 동작으로 미납자에게 시스템이 대신 알리도록 해서, 사적 관계를 정산 문제로 끌어들이지 않게 한다. 반대로 시스템이 너무 자주 알림을 보내면 사용자가 "독촉 앱"으로 인지하게 된다 — 그래서 같은 사람에게는 24시간 안에 두 번 보내지 않는 약속을 시스템이 지킨다. 호스트가 빠르게 여러 번 누르더라도 사용자에게는 한 번만 도착한다.

네 가지 알림 방식

방식 1 · 수동 독촉

호스트가 의식적으로 보낸다

현황 화면의 "독촉" 버튼을 누르면 미납자 전원에게 한 번 알림이 나간다. 누르기 전 다이얼로그에 "미납자 N명에게 푸시 알림을 발송합니다"가 떠서 호스트가 인지하고 보낸다.

방식 2 · 마감 연장 + 알림 (화면 미제공)

마감을 미루면서 한 번 더 알린다

"이번 주 일요일까지" 같은 마감을 사용자 사정으로 며칠 미뤄야 할 때. 연장과 함께 연장 사실을 미납자에게 한 번 알리는 동작이 시스템에 준비되어 있다. 다만 호스트가 이를 실행할 화면이 아직 없어, 현재는 운영팀을 통해서만 가능하다.

방식 3 · 자동 독촉

마감 후 정해진 시간이 지나면 시스템이 한 번

호스트가 정산을 활성화할 때 "마감 12시간 후 자동 독촉" 같은 옵션을 함께 설정해 두면, 마감이 지나고 정해진 시간이 흐른 뒤 미납자에게 시스템이 자동으로 한 번 알림을 보낸다. 자동 독촉은 한 정산당 최대 1회만 발동.

방식 4 · 멈춰 있는 송금 자동 재알림

혼합결제 계좌이체 부분 또는 환불 되돌리기 실패 건이 며칠 이상 방치되면

이체가 참가자 입금 확인을 기다리는 상태이거나, 환불 되돌리기가 실패해 수동 환불을 기다리는 상태로 3일(기본값) 이상 방치되면, 시스템이 받는 사람·호스트에게 자동 재알림을 보낸다. 호스트의 독촉 없이 시스템이 직접 개입하는 흐름이다. 2회 이상 풀리지 않으면 운영팀 알림이 자동으로 올라간다. 자동 만료는 없다 — 실제 돈이 움직였을 수 있는 상태이기 때문.

도배 방지가 작동하는 방식

호스트가 같은 사용자에게 빠르게 여러 번 알림을 시도해도 사용자에게는 한 번만 도착한다.

사용자 시나리오

기본 흐름 · 수동 독촉

활성화 1일 후 미납자 3명에게 가볍게 한 번

호스트가 정산 활성화 다음 날에 화면을 확인하니 3명이 아직 미납이다. 현황 화면의 "독촉" 버튼을 누르면 "미납자 3명에게 푸시 알림을 발송합니다" 다이얼로그가 뜨고, 확인하면 3명에게 같은 시각 알림이 도착한다. 호스트 화면에 토스트로 "리마인드를 발송했습니다" 안내.

이 흐름이 남기는 것 — 호스트가 개인 채팅으로 닦달하지 않고 시스템이 대신 알리는 경로. 알림 톤이 가볍기 때문에 사용자가 부담을 적게 받는다.

도배 방지

1시간 뒤 같은 사람에게 또 누른다

같은 미납자 3명에게 1시간 뒤에 호스트가 또 독촉을 누른다. 시스템은 마지막 발송으로부터 24시간이 지나지 않은 사용자에게는 알림을 보내지 않는다. 호스트 화면에는 "발송했습니다" 토스트가 뜨지만 실제 전송 건수는 0건이다.

이 흐름이 남기는 것 — 호스트의 의지와 사용자의 평온함을 동시에 지킨다. 향후 응답 메시지에 "발송 N건, 24시간 가드로 건너뛴 M건"을 함께 보여 주면 호스트의 인지가 더 정확해진다.

동시 호출 방지

호스트가 폰과 PC에서 동시에 독촉을 누른다

두 디바이스에서 거의 같은 시점에 독촉이 시작된다. 사용자 단위로 잠금이 걸려 있어 두 요청 중 하나만 처리되고 다른 하나는 즉시 건너뛴다. 결과적으로 사용자에게는 한 번만 알림이 도착한다.

마감 연장 + 알림 (설계된 흐름 · 화면 미제공)

마감을 24시간 미루면서 미납자에게 한 번 알린다

마감이 임박했지만 사용자 사정으로 24시간을 연장하기로 한 호스트. 설계된 흐름은 연장 시간을 고르고 "같이 알림도 보내기"를 켜면, 마감일이 24시간 뒤로 미뤄지면서 미납자에게 "정산 마감이 24시간 연장되었습니다" 알림이 한 번 나가는 것이다. 이 화면은 아직 제공되지 않아 현재는 운영팀을 통해서만 실행할 수 있다.

이 흐름이 남기는 것 — 연장된 마감을 사용자가 모르면 또 미루게 된다 — 연장과 알림이 한 동작으로 묶이는 것이 의도.

잦은 연장 도배 방지

1시간씩 다섯 번 연장하며 매번 알림 켜기

호스트가 사정에 따라 1시간씩 다섯 번 연속으로 연장하면서 매번 "알림 보내기"를 켠다. 첫 번째 연장 때만 알림이 나가고, 24시간이 지나지 않은 두 번째부터 다섯 번째 연장은 알림이 건너뛰어진다. 사용자에게 도배되지 않고 한 번만 도착한다.

자동 독촉

마감 12시간 후 시스템이 알아서 한 번

호스트가 활성화 시점에 "마감 12시간 후 자동 독촉" 옵션을 설정해 둔 정산. 마감일이 지나고 12시간이 흐른 뒤 시스템이 미납자에게 자동으로 한 번 알림을 보낸다. 자동 독촉은 정산당 한 번만 발동하므로 호스트가 추가로 손대지 않아도 미납 알림이 한 번은 무조건 나간다.

이 흐름이 남기는 것 — 호스트가 정산을 만들어 두고 잊어버려도 시스템이 한 번은 챙겨 준다. 호스트의 부담을 줄이고 정산 완료율을 높이는 자동화.

자동 독촉 · 빈 미납

자동 독촉 시점에 이미 모두 결제했다면

자동 독촉이 발동했을 때 모든 사용자가 이미 결제를 마쳤다면 실제 알림은 한 건도 나가지 않는다. 시스템은 이 정산에 대해서는 자동 독촉을 다시 발동시키지 않도록 표시만 남긴다.

차단 · 작성 중·완료 단계

활성화 안 된 정산에서 독촉을 시도

아직 활성화되지 않은 정산이나 이미 완료된 정산에서 독촉을 시도하면 거절된다. 화면에서도 액션 바에 "독촉" 버튼이 진행 중 상태에서만 노출된다.

화면에서 사용자가 인지하는 것

화면 요소호스트가 보는 것그 자리에 둔 이유
"독촉" 버튼 현황 화면 액션 바에 노출. 진행 중 상태에서만 보임. "한 번 누른다"는 행동이 명확해야 호스트가 의식적으로 보낸다는 감각이 만들어진다.
발송 전 확인 다이얼로그 "미납자 N명에게 푸시 알림을 발송합니다" 같은 사람 수가 명시. 알림을 보내기 전 N명이라는 숫자가 보여야 호스트가 "내가 누구에게 무엇을 보내는지" 인지한다.
마감 연장 (화면 예정) 연장 시간 선택 + "같이 알림도 보내기"가 한 동작으로 묶인 설계. 아직 호스트 화면은 제공되지 않는다. 연장과 알림이 한 동작에 묶여 있어야 사용자가 "마감이 바뀐 줄 모른다"는 사고를 막는다.
독촉 이력 누구에게 언제 어떤 종류(수동/자동/연장)로 발송했는지 시간 순. 호스트가 운영 회고에서 "이 정산이 왜 늦게 끝났는가"를 추적할 수 있어야 다음 정산의 정책 결정에 도움이 된다.

운영자가 알아야 할 것

독촉은 사용자 신뢰의 가장 민감한 변수다
  • 같은 사용자에게 24시간 안에 같은 알림을 두 번 보내지 않는다. 호스트가 빠르게 여러 번 누르더라도 시스템이 도배를 막는다 — 이 가드를 완화하면 사용자가 "독촉 앱"으로 인지한다.
  • 발송 후 응답에 "발송 N건 / 건너뛴 M건"을 함께 보여 주는 것이 향후 개선 항목이다. 현재는 호스트가 0건 발송과 N건 발송을 구분하지 못한다 — 의도된 한계지만 사용성 향상의 여지가 있다.
  • 자동 독촉은 한 정산당 한 번이다. 사용자에게 시스템 발 알림이 누적되면 신뢰가 무너진다 — 자동 알림은 한 번이라는 약속을 시스템 차원에서 지킨다.
  • 마감 연장은 사용자에게 알림이 함께 가야 의미가 있다. 연장만 하고 알리지 않으면 사용자는 옛 마감을 기준으로 계속 미루게 된다 — 알림 동반을 기본으로 권장.
  • 마감 연장은 아직 호스트 화면이 없다. 연장 기능 자체는 시스템에 준비되어 있으나 호스트가 직접 실행할 진입 화면이 미구현이다. 현재 마감을 미루려면 운영팀에 요청해야 한다 — 호스트 문의가 들어오는 지점이므로 운영 안내가 필요하다.
  • 동시 호출 방지는 사용자 단위로 잠금이 걸린다. 호스트가 두 디바이스에서 동시에 누르더라도 사용자에게는 한 번만 도착한다 — 이 잠금이 풀리면 24시간 가드가 우회될 수 있다.
  • 작성 중·완료된 정산에서는 독촉이 동작하지 않는다. 완료된 정산에 독촉이 가면 사용자가 "왜 또 알림이"라고 인지한다 — 진행 중 상태에서만 노출되는 것이 원칙.
