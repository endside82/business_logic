F03-15. 이벤트 카풀

이동수단이 CARPOOL인 이벤트에서 운전자가 좌석을 제공하고, 호스트가 참가자를 차량에 배정하는 운영 도구. 참가자는 개별 이동을 선택하거나 카풀 배정을 받을 수 있고, 호스트는 정원·픽업 가능 인원·권한을 기준으로 배정을 조정한다.

설계 의도 — 호스트가 이동 리스크를 운영 가능한 표로 만든다

카풀은 "누가 탈 수 있나"보다 "누가 누구 차에 배정됐나"가 중요하다. 운전자 지원, 호스트 확정, 참가자 배정, 배정 해제, 개별 이동 선택을 모두 분리해야 당일 혼선이 줄어든다.

사용자 시나리오

운전자

참가자가 운전자로 지원

좌석 수와 픽업 가능 인원을 입력해 차량 제공을 신청한다. 호스트가 확정하기 전까지는 배정 대상 차량이 아니다.

호스트

운전자 확정 후 참가자 배정

호스트가 운전자를 확정한 뒤 참가자를 해당 차량에 배정한다. 픽업 가능 인원 초과 배정은 차단된다.

참가자

개별 이동 선택

자차 이동이 허용된 이벤트라면 참가자가 직접 이동을 선택할 수 있다. 이 참가자는 호스트의 카풀 배정 대상에서 빠진다.

권한

참가자는 자기 정보만 본다

일반 참가자가 탑승자 목록을 조회하면 본인 항목만 보인다. 호스트는 전체 탑승자를 보며 운영한다.

방식 가드

카풀이 아닌 이벤트에서 진입

이동수단 방식이 "없음" 또는 "버스"이면 카풀 기능 전체가 닫힌다. 하위 기능은 F03-14의 이동수단 방식을 따른다.

초안 단계 전환

카풀 설정 후 버스로 바꿈

초안에서 이동수단 방식을 버스로 바꾸면 더 이상 유효하지 않은 카풀 데이터가 삭제된다. 공개 전 변경이므로 사용자에게 알림은 필요 없다.

사용자·호스트가 할 수 있는 동작

동작누가무엇이 일어나는가
운전자 지원·수정참가자(운전자)좌석 수와 픽업 가능 인원을 적어 차량 제공을 신청한다.
운전자 확정·거절호스트지원한 운전자를 배정 가능한 차량으로 확정하거나 거절한다.
탑승자 목록 보기·이동 선택참가자·호스트참가자는 카풀과 개별 이동 중 하나를 고르고, 자기 배정 상태를 확인한다.
차량 배정·해제호스트누가 어느 차량을 타는지 정하고, 바꾼 이력이 함께 남는다.

운영자가 알아야 할 것

당일 혼선을 줄이는 제약
  • 확정된 운전자만 배정 대상이다. 지원만 한 운전자의 차량에는 참가자를 배정하지 않는다.
  • 픽업 가능 인원을 넘길 수 없다. 좌석 수와 픽업 가능 인원은 다를 수 있다.
  • 배정 해제도 알림 대상이다. 참가자의 당일 이동 계획이 바뀌므로 배정/해제 모두 알림이 필요하다.
  • 1차 범위는 운영 로직 중심이다. 참가자·호스트 화면은 후속 운영 화면에서 더 구체화될 수 있다.
