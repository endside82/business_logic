F11-06. 취향 평가 · 취향 프로필

본인만 보는 비공개 평가 데이터를 모으고, 그 누적치를 자신의 추천·매칭 입력값으로 통제하는 면이다. 모임에 함께한 사람에 대해 별점·태그·메모를 남기면(공개되지 않음) 긍정·부정 태그 가중치가 자동으로 다시 계산되고, 여기에 사용자가 직접 고른 선호 카테고리·시간대·그룹 크기가 더해져 추천 알고리즘의 입력값이 된다. 공개 리뷰(F11-01)와는 다른 라인의 데이터이며, 누구에게도 그 사람이 받은 평가는 보이지 않는다.

설계 의도 — 누구의 평가를 위한 화면도 아니고, 본인의 추천 품질을 본인이 통제하는 면

공개 리뷰(F11-01)는 다음 사용자의 의사결정을 위한 데이터다 — 평판으로 작동한다. 취향 평가는 정반대 방향이다 — 평가받는 쪽에게는 보이지 않고, 평가한 본인의 다음 추천·매칭 품질을 만든다. 평가가 쌓일수록 긍정 태그 가중치(상대를 어떻게 느꼈는가)와 부정 태그 가중치가 자동으로 누적되고, 여기에 사용자가 직접 고른 선호(카테고리·시간대·그룹 크기)가 더해져 "이 사용자가 좋아할 만한 사람·모임은 어느 쪽인가"의 입력이 만들어진다. 데이터가 적을 때는 차트를 비우는 대신 "더 모일 때까지" 안내를 띄운다 — 빈 차트는 잘못된 추천보다 나쁘다.

두 종류의 데이터가 같은 화면에 모인다

한쪽은 자동으로 누적되고, 한쪽은 사용자가 직접 고른다. 둘 모두 추천 알고리즘의 입력이 된다.

데이터 종류어떻게 만들어지는가화면에서 어디에 보이는가
자동 누적 — 태그 가중치 모임 후 다른 참가자에 대한 비공개 평가에서 사용자가 고른 태그(긍정 10개 / 부정 4개)가 자동 집계. 평가가 등록될 때마다 본인의 누적 가중치가 다시 계산된다. "취향 요약" 레이더 차트, "상위 관심사" 칩, "긍정적 인상" / "개선 포인트" 막대 그래프.
수동 입력 — 선호도 화면의 "선호도 설정" 시트에서 카테고리(8개) · 시간대(4개) · 그룹 크기(3개)를 직접 선택해 저장. 저장된 항목이 화면 아래쪽 영역에 반영. 비어 있어도 자동 누적 데이터가 작동한다.
평점 기록 리스트 본인이 그동안 남긴 비공개 평가 카드 — 누가 받았는지가 아니라 본인이 누구에게 어떻게 줬는지를 본인 기록으로 본다. 화면 아래쪽 평점 기록 영역. 20건씩 페이지로 노출되고 "더 보기"로 누적 로딩.

사용자 시나리오

기본 흐름

평가가 쌓인 사용자가 취향 프로필을 본다

비공개 평가를 5건 이상 남긴 사용자가 마이페이지에서 "취향 프로필"로 진입한다. 화면 상단에 레이더 차트가 그려지고, 상위 관심사 칩 4개, "긍정적 인상" 막대(유쾌함 4.0 / 신뢰감 3.0 / 따뜻함 2.0…), 개선 포인트 막대(지각 1.0) 순서로 보인다. 그 아래에 본인 평점 기록이 깔린다.

이 흐름이 남기는 것 — 사용자가 "내가 어떤 사람을 좋아했는가"를 패턴으로 확인. 자신도 모르게 누적된 호불호를 사용자가 알아차리고, 다음 모임 선택 기준이 분명해진다.

데이터 부족

가입 직후라 평가가 한 건도 없다

새로 가입한 사용자가 화면에 들어온다. 태그 데이터가 3개 미만이라 레이더 차트는 그려지지 않고 "아직 취향 데이터가 부족합니다" 안내만 뜬다. 평점 기록 영역도 "아직 남긴 평점이 없습니다 — 이벤트 참석 후 참가자에게 평점을 남겨보세요" 안내. "선호도 설정"은 그대로 사용 가능.

이 흐름이 남기는 것 — 부정확한 차트로 사용자를 오해시키지 않는다. 동시에 "선호도 설정"을 통한 수동 입력 경로는 열어 두어 신규 사용자가 추천 품질에 즉시 영향을 줄 수 있게 한다.

선호도 설정

선호 카테고리·시간대·그룹 크기 저장

"선호도 설정" 시트가 열린다. 카테고리에서 "음식/맛집", "문화/예술"을 토글하고, 시간대에서 "저녁(18-21)", 그룹 크기에서 "중규모(6-15명)"를 선택한 뒤 "저장"을 누른다. 토스트가 뜨고 시트가 닫히면 추천 도메인이 다음 호출부터 이 값을 입력으로 사용한다.

이 흐름이 남기는 것 — 사용자가 추천 알고리즘에 자기 의사를 직접 전달할 수 있다는 통제감. 자동 누적 데이터에만 의존하지 않는다.

평가 생성

모임 종료 후 함께한 참가자에게 별점·태그·메모

모임이 끝나고 별도 화면에서 함께한 참가자에게 비공개 평가를 남긴다. 별점 4, 태그 "유쾌해요"와 "경청해줘요" 선택, 짧은 메모. 등록되면 본인의 태그 가중치가 자동으로 재계산되고, 본 화면을 다시 열면 변동이 반영되어 있다.

이 흐름이 남기는 것 — 평가 한 건이 자동으로 추천 입력에 반영. 사용자가 따로 "프로필 갱신" 같은 행동을 하지 않아도 시스템이 알아서 다음 추천을 다듬는다.

자기 평가 차단

실수로 자기 자신을 평가하려 함

잘못된 호출 또는 외부 링크로 자기 자신에 대한 평가가 시도된다. 자기 평가는 받지 않는다는 안내가 떠오른다.

이 흐름이 남기는 것 — 본인이 자기 추천 입력을 직접 조작하는 우회 경로를 막는다.

중복 차단

같은 사람을 같은 모임에서 두 번 평가

같은 모임에서 만난 같은 사람에 대해 두 번째 평가가 시도된다. 한 모임당 같은 상대에게 한 번이라는 규칙으로 거절된다. 다음 모임에서 같은 사람을 다시 만나면 그때는 새 평가를 남길 수 있다.

이 흐름이 남기는 것 — 평가 한 번이 그 모임에서 본 인상의 무게를 가진다. 같은 모임에서 마음이 바뀌어도 평가는 한 번으로 잡힌다.

긍정·부정 분류

한 평가에 긍정·부정 태그가 섞임

한 사람에 대해 "시간 잘 지켜요"와 "지각했어요" 같이 모순된 태그가 들어왔다. 시스템은 부정 태그만 따로 분류해 "개선 포인트" 막대에 누적한다.

이 흐름이 남기는 것 — 한 사람을 단일 인상으로 가두지 않고, 긍정·부정이 같은 평가 안에 공존할 수 있게 한다. 사용자의 진짜 인상을 더 풍부하게 모은다.

운영자가 알아야 할 것

취향 데이터가 다른 도메인에 닿는 자리
  • 이 데이터는 평가받은 사람에게 노출되지 않는다. 공개 리뷰(F11-01)는 평판이고, 본 단위는 본인의 추천 품질이다 — 두 데이터가 한 화면에 섞이면 사용자가 "내가 받은 평가"를 다른 사용자가 본다고 오해한다. 본인 외 노출 가드는 어떤 회귀에서도 풀려서는 안 된다.
  • 평가 한 건이 등록될 때마다 본인의 태그 가중치가 다시 계산되고, 협업 필터링 이웃도 함께 재구축된다. 본 단위는 트리거만 책임지고 알고리즘은 추천 도메인이 처리하지만, 본 단위에서 평가 생성이 막히면 추천 품질도 동시에 정지한다.
  • 레이더 차트는 태그 3개 미만이면 비표시한다. 데이터가 적을 때 부정확한 차트로 사용자를 오해시키지 않으려는 결정 — "더 모일 때까지" 안내가 빈 차트보다 정직하다.
  • 선호도(카테고리·시간대·그룹 크기)는 빈 리스트로 저장할 수 없다. 시트에서 아무 것도 고르지 않고 "저장"을 누르면 그 항목은 갱신되지 않는다 — 사용자가 의도적으로 "초기화"하고 싶다면 별도 경로가 필요하다(현재는 미지원).
  • 본인 평점 기록은 페이지 단위로만 받는다(20건씩). 누적이 많은 사용자는 "더 보기"로 추가 로딩 — 한 번에 모든 평가를 보여 주면 화면 부하가 즉시 커진다.
  • "활동 패턴 히트맵", "월별 차트" 같은 화면 항목은 현재 미구현이다. 서버가 해당 데이터를 내려주지 않고 화면도 그리지 않는다. 운영·기획에서 필요해지면 데이터 갈래와 노출 위치를 동시에 결정해야 한다.
