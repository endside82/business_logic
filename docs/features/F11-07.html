F11-07. 호스트 리뷰 모더레이션 (답변 · 임시 숨김)

이벤트 호스트가 본인 이벤트에 달린 리뷰에 직접 대응할 수 있는 두 가지 도구다. 답변은 리뷰 1건에 공개 댓글을 붙이고(리뷰당 1개, 작성 후 24시간 이내만 수정 가능), 임시 숨김은 6가지 사유 중 하나를 선택해 리뷰를 일시적으로 가린다. 허위 리뷰 사유로 숨기면 운영자 검토를 위한 신고가 자동으로 접수된다(F11-04 연동). 모더레이션 권한은 해당 이벤트의 호스트에게만 있다.

설계 의도 — 호스트에게 발언권을 주되, 삭제 권한은 주지 않는다

모임 후기는 다른 사용자의 판단 근거다. 사실과 다른 후기에 호스트가 침묵해야 한다면 커뮤니티 신뢰는 유지되지 않는다 — 답변 기능이 그 발언권이다. 동시에 호스트가 부정적인 후기를 마음대로 지워버릴 수 있다면 리뷰 자체가 무력화된다. 그래서 삭제는 없고 임시 숨김만 있다. 숨김도 사유를 남기고, 운영자가 검토할 수 있는 신고 경로와 연동된다. "호스트가 불쾌하다고 숨겼느냐, 진짜 문제 있는 후기를 정리했느냐"를 운영 기록이 추적한다.

답변 — 발언권의 구조

호스트의 답변은 리뷰 카드 아래에 녹색 카드로 붙는다. 다른 사용자가 리뷰를 볼 때 호스트의 시각도 같이 보인다.

알려진 갭 — 답변 수정 경로 미연결

현재 앱에서 이미 답변이 달린 리뷰의 더보기를 누르면 수정이 아니라 새 답변 작성으로 처리된다. 이미 답변을 단 리뷰에는 다시 답변할 수 없으므로, 호스트가 24시간 이내 답변을 수정하려 할 때 오류 안내가 뜬다. 답변이 이미 있는 경우 수정 경로로 보내도록 화면을 손봐야 한다.

임시 숨김 — 사유를 남기고 가린다

숨김은 리뷰를 지우는 게 아니다. 사유를 남기고 가리는 것이다. 운영자가 나중에 그 흔적을 볼 수 있다.

알려진 갭 — 자동 신고 기본값이 양쪽에서 다르다

앱에서는 어떤 사유로 숨기든 "자동 신고"가 기본으로 켜진 상태이며, 호스트가 체크박스를 직접 꺼야만 해제된다. 허위 리뷰를 골라도 자동 신고가 켜지지만 끄는 것은 여전히 가능하다. 반면 서버 쪽 기본값은 꺼짐이다. 그 결과 호스트가 체크박스를 의식하지 못하면 허위 리뷰가 아닌 일반 숨김까지도 운영자 신고가 자동으로 접수된다. 이것이 의도한 운영 정책이라면 그대로 두되 양쪽 기본값을 "켜짐"으로 맞추고, 의도하지 않았다면 앱 기본값을 "꺼짐"으로 바꿔야 한다.

숨겨진 리뷰를 사용자가 보면

숨긴다는 것은 없애는 게 아니다. 내용이 가려지고 왜 가려졌는지가 대신 보인다.

숨김 상태 리뷰 카드의 모습
  • 리뷰 작성자와 별점은 그대로 보인다. 누가 쓴 리뷰인지, 몇 점을 줬는지는 가려지지 않는다 — 신뢰점수에도 계속 반영된다.
  • 본문 자리에 "임시 숨김" 안내가 들어온다. 호스트가 적은 사유가 있으면 그 내용이, 없으면 선택한 숨김 사유의 의미가 안내문으로 보인다.
  • 숨김 배지가 붙는다. 허위 리뷰 사유면 "허위 리뷰 검토 중", 그 외이면 "임시 숨김" 라벨이 카드 상단에 붙는다.
  • 호스트 답변이 있으면 숨김 상태에서도 함께 보인다.

사용자 시나리오

기본 흐름 — 답변

억울한 후기에 입장을 남긴다

호스트가 본인 이벤트 리뷰 목록을 보다 사실과 다른 내용이 적힌 후기를 발견한다. 카드 더보기를 눌러 "답변하기"를 선택하고, 공식 입장을 적는다. 확인하면 녹색 카드가 리뷰 아래에 붙는다. 다른 사용자가 그 리뷰를 볼 때 호스트의 입장도 같이 보인다.

이 흐름이 남기는 것 — 호스트가 침묵하지 않아도 된다는 장치. 리뷰 한 줄이 전부인 상황에서 호스트 시각이 추가되어 조회자가 더 균형 잡힌 판단을 할 수 있다.

수정 윈도우

잘못 입력한 답변을 24시간 안에 수정

답변을 달았는데 오탈자가 심각하다. 24시간 이내라면 더보기 → "답변 수정"으로 고칠 수 있다. 24시간이 지나면 수정 메뉴가 비활성화된다 — 공개 답변은 함부로 고치지 못한다는 정책이다.

이 흐름이 남기는 것 — 답변을 언제든 수정할 수 있으면 호스트가 후기에 따라 입장을 바꾸는 식으로 악용될 수 있다. 24시간은 교정 시간이지 번복 시간이 아니다.

임시 숨김

광고성 스팸 후기를 잠시 가린다

이벤트와 전혀 무관한 광고 문구가 적힌 후기가 달렸다. 더보기 → "임시 숨김" → "광고·스팸" 선택. 확인하면 후기 본문 자리에 "임시 숨김" 안내가 들어온다. 다른 사람은 그 후기의 내용 대신 "임시 숨김 처리된 리뷰"라는 안내를 본다.

이 흐름이 남기는 것 — 리뷰가 사라지지 않는다는 것을 사용자가 인지한다. 호스트가 부정적인 후기를 모두 없앨 수 있다는 오해를 막는다.

허위 리뷰 신고 연동

"이건 허위다" — 숨김과 신고가 한 번에

참석하지 않은 사용자가 리뷰를 남긴 것이 의심된다. 더보기 → "임시 숨김" → "허위 리뷰" 선택. 확인하면 리뷰가 가려지는 동시에 운영자 신고 목록에 우선 검토 표시가 붙은 신고가 자동으로 접수된다.

이 흐름이 남기는 것 — 호스트가 "이건 신고감이다"고 판단할 때 두 번 행동(숨김 + 신고)하지 않아도 된다. 허위 리뷰를 발견한 순간 한 번의 선택으로 운영 검토까지 이어진다.

숨김 해제

오판이었다 — 되돌린다

숨겼는데 돌아보니 과한 판단이었다. 더보기 → "숨김 해제"를 누르면 리뷰가 다시 보이기 시작한다. 별점과 본문이 원래대로 복구된다.

이 흐름이 남기는 것 — 호스트의 실수를 되돌릴 수 있다. 단, 숨김 중에 자동 생성된 신고는 이미 접수된 상태라 함께 취소되지 않는다 — 운영자 검토는 별도로 진행된다.

권한 경계

다른 이벤트의 호스트가 내 리뷰에 접근 시도

B 이벤트 호스트가 A 이벤트 리뷰에 답변을 달려 한다. 서버에서 "해당 이벤트의 호스트가 아닙니다" 거절이 반환된다. 앱에서는 본인이 아닌 이벤트의 리뷰에는 더보기 모더레이션 메뉴 자체가 보이지 않는다.

이 흐름이 남기는 것 — 답변과 숨김 권한이 이벤트 단위로 격리된다. 호스트 A가 B의 이벤트에 개입할 수 없다.

화면에서 사용자가 인지하는 것

호스트와 일반 사용자가 같은 리뷰를 볼 때 각각 다른 것이 보인다.

화면 요소일반 사용자호스트(본인 이벤트)
리뷰 카드 더보기 신고 아이콘만 신고 + "답변하기" + "임시 숨김" / "숨김 해제" (상태에 따라)
숨김 상태 카드 본문 자리에 "임시 숨김" 안내 + 사유 배지 동일. 단, 더보기에 "숨김 해제" 메뉴가 붙는다.
호스트 답변 카드 리뷰 아래 녹색 카드 — 호스트 닉네임 + 답변 내용 동일. 작성 후 24시간 이내이면 더보기에 "답변 수정" 메뉴 노출. (현재 미연결 — Gap)
답변 수정 여부 보이지 않음 작성 후 24시간 이내이면 수정 메뉴 노출, 지나면 비활성.

운영자가 알아야 할 것

모더레이션 흔적은 남는다
  • 임시 숨김을 해제해도 그 사실이 별도 이력으로 남지 않는다. 숨김 해제 시 숨김 사유·실행자 등의 기록이 비워질 뿐, 누가 언제 해제했는지를 따로 남기는 이력은 없다. 다만 자동 신고가 켜진 채 숨겼다면 그때 접수된 신고는 삭제되지 않고 그대로 남으므로, 숨김과 해제를 반복하는 패턴은 신고 목록을 통해 운영에서 감지할 수 있다.
  • 허위 리뷰 숨김은 자동으로 신고가 접수된다(우선 검토 표시 포함). 운영자 목록 상단에 우선 노출된다. 숨김을 해제해도 신고 기록은 남는다. 운영 처리(승인/기각)는 별도로 진행되어야 한다.
  • 별점은 숨겨도 신뢰점수에서 제외되지 않는다. 현재 정책은 임시 숨김 중에도 별점을 신뢰점수 계산에서 빼지 않는 것이다. 이 정책이 바뀌면 신뢰점수 재계산이 필요하다.
  • 답변 삭제는 호스트도 운영자도 일반 화면으로는 할 수 없다. 현재 답변을 지우는 경로 자체가 화면에 열려 있지 않다. 꼭 지워야 한다면 운영자용 별도 도구를 거쳐야 한다.

알려진 갭 요약

지금 바로 처리해야 하는 항목
  • 답변 수정 경로 미연결 — 이미 답변이 있는 리뷰에서 더보기를 누르면 수정이 아니라 새 답변 작성으로 처리돼 오류가 난다. 리뷰 목록 화면에서 답변이 이미 있는 경우 수정 경로로 보내도록 분기를 추가해야 한다.
  • 자동 신고 기본값 운영 정책 결정 — 앱은 자동 신고가 기본으로 켜진 채이고 호스트가 체크박스로 끌 수 있는데, 서버 기본값은 꺼짐이다. 체크박스가 잘 보이는지와 기본 방향이 의도한 정책인지 확인하고, 필요하면 안내를 강조하거나 양쪽 기본값을 맞춰야 한다.
  • 답변 알림 미구현 — 리뷰 작성자가 호스트 답변이 달렸다는 사실을 알 방법이 없다. 답변 알림 추가를 검토해야 한다.
