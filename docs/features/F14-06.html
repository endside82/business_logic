F14-06. 좌표 → 주소 변환 (역지오코딩)

위·경도 좌표를 한국식 주소 문자열("서울특별시 강남구 역삼동 ...") 로 바꾸는 보조 인프라. 사용자에게 좌표 숫자는 의미가 없으므로 지도에서 위치를 고른 직후나 현재 위치를 표기할 때 본 변환이 일어난다. 본 단위는 전용 화면이 없고, 주소 등록·이벤트 장소 등록·길찾기 같은 다른 화면이 호출하는 lookup.

설계 의도 — 사용자는 좌표가 아니라 주소로 위치를 이해한다

지도에서 핀을 옮기거나 현재 위치를 잡으면 결과로 받는 것은 위·경도 숫자다. 그러나 사용자가 "여기가 어디인가"를 인지하려면 "강남구 역삼동" 같은 한국식 주소가 필요하다. 본 단위는 그 변환을 빠르게 처리해 다른 화면이 사용자에게 의미 있는 라벨을 보여줄 수 있게 한다. 같은 좌표가 반복 호출되어도 외부 변환 서비스 비용을 늘리지 않도록 변환 결과는 24시간 동안 캐시된다.

본 변환이 호출되는 자리

본 단위 자체에는 화면이 없다. 다른 화면이 사용자에게 위치를 보여줄 때 본 변환이 보조로 호출된다.

주소 등록

지도에서 위치 찍기

사용자가 "집 주소 등록" 같은 화면에서 지도 핀을 옮기거나 길게 눌러 좌표를 잡으면 그 좌표가 곧장 한국식 주소로 변환되어 입력 칸에 채워진다. 사용자는 변환된 주소를 그대로 쓰거나 한 줄 수정해 등록한다.

이벤트 생성

모임 장소 선택

호스트가 이벤트를 만들면서 지도에서 모임 장소를 고르면 동일하게 좌표 → 주소 변환이 일어나 모임 장소 라벨이 자동 채워진다.

현재 위치 라벨

"내 현재 위치"를 사람이 읽을 수 있는 형태로

현재 위치를 잡은 직후 사용자에게 "지금 어디에 있는지"를 주소로 한 번 보여주고 싶을 때 호출된다.

운영자가 알아야 할 것

변환 정책과 외부 의존
  • 변환은 외부 지도 서비스(카카오 로컬)를 호출한다. 외부 서비스가 일시적으로 응답하지 않으면 본 변환도 같이 멈춘다. 다른 화면은 이 경우 "주소를 찾을 수 없습니다" 같은 안내로 좌표 숫자를 그대로 노출하거나 사용자에게 직접 입력을 유도해야 한다.
  • 같은 좌표는 24시간 동안 캐시된다. 같은 위치를 여러 사용자가 반복 조회해도 외부 서비스 호출은 한 번만 일어난다. 자주 보이는 주소(많이 등록되는 모임 장소 등)는 변환 비용이 거의 들지 않는다.
  • 바다·산속·미등록 지역의 좌표는 변환되지 않는다. "주소를 찾을 수 없습니다" 안내가 호출 화면에서 노출되어 사용자가 다른 위치를 선택하도록 유도한다.
  • 외부 지도 서비스 인증 키 설정이 끊기면 본 변환 전체가 멈춘다. 주소 등록·이벤트 장소 등록 같은 사용자 흐름이 동시에 막힌다 — 운영 환경 키 만료 시점을 모니터링 대상에 포함해야 한다.
