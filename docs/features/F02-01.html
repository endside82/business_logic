F02-01. 홈 피드 메인 조회

앱 진입의 첫 화면. 4개 섹션(추천 이벤트·예정/인근·인기 클럽·최신 플랜)을 한 번에 그리되, 한 섹션이 실패해도 다른 섹션은 살아남아야 한다. 비로그인 게스트도 같은 화면을 본다(가입 유도 카드 포함). 5분 캐시로 재진입을 빠르게 만들고, 위치 권한은 첫 진입에 한 번만 묻는다.

각 섹션은 자기 상태(로딩/데이터/빈/에러)를 독립적으로 그린다. “전부 에러”일 때만 전체 화면 에러로 떨어진다. 빈 상태와 에러 상태는 시각적으로 명확히 구분되어 사용자가 “데이터가 없어서 비어있다” vs “네트워크 문제로 못 받았다”를 알 수 있어야 한다.

이 기능 안의 하위 항목

섹션 1

추천 이벤트

관심사·위치 기반 맞춤 추천. “더보기”로 F02-04 진입.

섹션 2

예정/인근 이벤트

시작 임박 또는 인근 이벤트 10건. 가로 스크롤.

섹션 3

인기 클럽

멤버 수 내림차순 5건. 빈 상태에서도 헤더는 유지.

섹션 4

최신 플랜

신규 마켓 플랜 5건. 세로 리스트.

섹션 상태 머신

한 섹션의 4가지 상태. 다른 섹션과 독립적으로 전이한다.

stateDiagram-v2 [*] --> 로딩 로딩 --> 데이터: 200 + 비어있지 않음 로딩 --> 빈상태: 200 + 빈 배열 로딩 --> 에러: 4xx/5xx/네트워크 데이터 --> 로딩: 새로고침 빈상태 --> 로딩: 새로고침 에러 --> 로딩: "다시 시도"

사용자 시나리오 8개

S1 · Happy Path

로그인 사용자 첫 진입

  1. 홈 진입 → 4개 섹션 동시 로딩 → 스켈레톤 표시
  2. 위치 권한 다이얼로그(첫 진입 1회만)
  3. 4개 모두 200 → 정상 렌더
  4. 권한 거부 시 상단에 위치 권한 배너 노출

종료 — 캐시 시각 기록, 5분 유효.

S2 · 캐시 hit

30초 만에 재진입

  1. 상세 화면에서 백버튼 → 홈 재진입
  2. 캐시 유효 → 네트워크 호출 0건
  3. 스켈레톤 없이 즉시 렌더
S3 · 부분 실패

인기 클럽만 500

  1. 4개 중 인기 클럽만 500
  2. 나머지 3개 섹션은 정상
  3. 인기 클럽 섹션만 “다시 시도” 표시
  4. 사용자가 재시도 → 4개 모두 재호출 → 정상화

종료 — 사용자 흐름이 막히지 않음.

S4 · 전체 실패

비행기 모드

  1. 4개 모두 네트워크 오류
  2. 전체 화면 “피드를 불러올 수 없습니다” + 재시도
  3. 네트워크 복구 후 사용자가 명시적으로 재시도

주의: 자동 재시도는 하지 않는다.

S5 · 게스트

비로그인 사용자

  1. 토큰 없이 4개 호출
  2. 서버가 사용자 컨텍스트 없이 응답 (myMembershipStatus 등 null)
  3. “내 예정”, “위시리스트” 같은 개인화 섹션은 미노출
  4. 4개 메인 섹션만 렌더 → 가입 퍼널 진입
S6 · 파라미터 방어

잘못된 카테고리

  1. 외부 도구가 잘못된 카테고리 값으로 호출
  2. 서버: 400 CLUB_INVALID_CATEGORY

주의: 현재 UI는 카테고리 파라미터를 보내지 않음. 향후 칩 추가 시 클라이언트 화이트리스트 필요.

S7 · 빈 상태

인기 클럽 빈 상태 노출

  1. 인기 클럽 응답이 빈 배열
  2. 섹션 헤더 “인기 클럽” + 본문 “인기 클럽이 없습니다” 동시 노출
  3. 섹션이 사라지지 않음(에러와 시각적으로 구분)

핵심: 빈 배열은 에러가 아니다. 헤더 유지가 “이 섹션이 존재함”을 알린다.

S8 · 위치 권한 분기

권한 거부 후 배너 노출

  1. 위치 권한 다이얼로그에서 거부
  2. 상단에 권한 안내 배너 표시
  3. 위치 기반 추천은 비활성, 다른 섹션은 정상

수용 기준 (AC)

ACGivenWhenThen
AC-01로그인 + 5분 캐시 없음홈 진입4섹션 병렬 호출 + 스켈레톤
AC-025분 이내 재진입홈 빌드네트워크 호출 0, 즉시 렌더
AC-03한 섹션만 5xx응답 수신해당 섹션만 재시도 UI
AC-044섹션 모두 실패응답전체 화면 에러
AC-05비로그인홈 진입개인화 섹션 미노출, 메인 4섹션 정상
AC-06한 섹션 빈 배열렌더헤더 유지 + 빈 카피
AC-07위치 권한 거부홈 진입배너 노출, 거리 기반 정렬 비활성
AC-08한 섹션 “다시 시도”탭4섹션 전체 재호출

자주 빠뜨리는 케이스

레드 플래그
  • 한 섹션 실패가 전체 화면 실패로 떨어짐 — 가장 흔한 회귀.
  • 빈 배열을 에러로 표시 — 사용자가 “문제 있나?”로 오해. 빈 카피로 분리.
  • 캐시로 오래된 상태 표시 — 이미 신청한 이벤트에 “신청하기”가 남음. 진입 시점 정책 필요.
  • 위치 권한을 매 진입마다 묻기 — 한 번만 묻고 결과를 기억.
  • 비로그인 사용자에게 개인화 섹션이 빈 상태로 노출 — 섹션 자체를 미노출하는 것이 옳다.
