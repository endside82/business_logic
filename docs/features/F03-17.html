F03-17. 차량 레이아웃 카탈로그

버스대절 기능에서 사용할 차량 좌석 구조를 조회 전용으로 제공하는 카탈로그. 호스트는 활성 레이아웃을 고르고, 시스템은 그 레이아웃의 좌석 배치를 기준으로 이벤트 버스 좌석표를 만든다.

초기 시드
4종
28 · 45 · 20 · 8인승
출시 범위
조회
관리자 편집은 후속
사용처
F03-16
버스 좌석 배치 생성 원천
설계 의도 — 좌석표를 이벤트마다 새로 그리지 않는다

차량 좌석 구조는 반복해서 재사용되는 운영 자산이다. 카탈로그를 별도로 두면 28인승, 45인승 같은 표준 차량을 여러 이벤트에서 같은 방식으로 쓸 수 있고, 좌석 선택 가능 여부도 레이아웃 단위로 통제할 수 있다.

사용자 시나리오

기본 조회

활성 차량 4종 노출

시드가 적용된 DB에서 호스트가 버스 추가 화면을 열면 28인승 A타입, 45인승, 20인승, 8인승이 보인다.

좌석 그림

레이아웃 좌석 배치 조회

레이아웃을 고르면 좌석의 행·열 위치, 좌석 종류, 선택 가능 여부를 받아 좌석표를 그린다.

비활성

운영자가 비활성화한 레이아웃

비활성으로 표시된 레이아웃은 활성 목록에 보이지 않는다. 기존 이벤트가 이미 참조한 좌석표는 그 이벤트의 좌석으로 그대로 남는다.

권한

비인증 조회 차단

차량 카탈로그는 이벤트 운영 도구의 일부이므로 로그인이 필요하다. 로그인하지 않은 채 조회하면 거절된다.

이 화면이 제공하는 것

동작누가무엇이 일어나는가
활성 차량 목록 보기호스트지금 쓸 수 있는 차량 레이아웃 목록을 받는다.
좌석 배치 보기호스트고른 레이아웃의 좌석 배치를 받아 좌석표를 그린다.

운영자가 알아야 할 것

1차 범위는 조회 전용이다
  • 관리자 편집 기능은 후속이다. 1차 운영은 초기 등록 데이터로 레이아웃을 관리한다.
  • 레이아웃은 원본이고 이벤트 좌석은 복사본이다. 레이아웃이 바뀌어도 이미 생성된 이벤트 좌석표를 소급 변경하지 않는다.
  • 좌석 선택 가능 여부가 중요하다. 운전석, 통로, 비선택 좌석을 구분해야 좌석표가 안전하게 동작한다.
