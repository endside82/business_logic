F03-09. 이벤트 사진첩

참석 확정 사용자만 사진을 업로드·조회할 수 있다. 첫 업로드 시 앨범이 자동 생성되고, 종료 후 일정 기간이 지나면 업로드가 차단되며 일정 시점에 일괄 정리된다. 외부 스토리지 업로드 + DB 등록의 두 단계 사이가 끊기면 orphan 파일이 생길 수 있어 운영 정리가 필요하다.

하위 동작
4
조회·업로드·삭제·만료 안내
시나리오
9
한도
100장/앨범

이 기능 안의 하위 항목

동작 1

사진 업로드

외부 스토리지 PUT → DB 등록 + 앨범 자동 생성.

동작 2

그리드 조회 + 풀스크린 뷰어

동작 3

본인 사진 삭제

외부 스토리지 best-effort 삭제 + DB DELETE.

동작 4

만료 안내

만료 임박 배너 + 만료 후 FAB 숨김.

사용자 시나리오 9개

S1

첫 업로드(앨범 자동 생성)

  1. FAB → 갤러리 → 캡션 입력
  2. 외부 스토리지 PUT → 서버 등록
  3. 앨범 lazy 생성 + 1장 그리드 표시
S2

추가 업로드(앨범 reuse)

  1. 기존 앨범에 사진 추가 → photoCount++
S3 · 만료 임박

2일 후 만료 배너

  1. 상단에 “2일 후 사진첩이 만료됩니다” 배너
  2. 업로드는 여전히 가능
S4 · 만료 후

업로드 차단

  1. 조회만 가능, FAB 숨김
  2. 우회 시 400 EVENT_PHOTO_ALBUM_EXPIRED
S5 · 한도

100장 초과 시도

  1. 외부 스토리지 PUT은 성공해도 DB 등록 단계에서 400
  2. 부작용: orphan 파일 발생 가능 → 운영 정리 필요
S6 · 삭제

본인 사진 삭제

  1. 풀스크린 뷰어 → 메뉴 → 확인 다이얼로그
  2. 외부 스토리지 + DB 삭제
  3. photoCount-- + 토스트
S7 · 호스트

호스트의 타인 사진 삭제

  1. UI에 진입점 없음, 직접 API 호출 시 호스트는 통과
S8 · 비참석자

비참석자 진입 시도

  1. 라우터 redirect로 사전 차단
  2. 우회 시 403 EVENT_PHOTO_NOT_ATTENDEE
S9 · 업로드 실패

외부 스토리지 PUT 중 네트워크 오류

  1. 토스트 노출, DB 등록 단계 미도달
  2. fileKey 무효, 재시도 시 새 사전 서명 URL 발급

서버 계약

MethodPath설명
GET/api/v1/events/{id}/photos앨범 + 사진 목록(빈 앨범은 404)
POST/api/v1/events/{id}/photos사진 등록(앨범 lazy 생성)
DELETE/api/v1/events/{id}/photos/{photoId}삭제

자주 빠뜨리는 케이스

레드 플래그
  • 첫 진입 404를 에러 화면으로 노출 — 빈 상태 UI로 자연스럽게 처리하는 게 옳다.
  • 외부 스토리지 PUT 성공 후 DB 등록 실패의 orphan — 한도 초과·동시성으로 자주 발생.
  • 만료 후에도 FAB 노출 — 사용자가 업로드 시도 후 실패.
  • 비참석자 가드 누락 — 사진 노출 위험.
