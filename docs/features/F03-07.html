F03-07. 정원 · 대기열 관리 (호스트)

정원·대기열·승인제·모집 마감 토글을 한 폼에서 묶어 관리한다. 정원을 늘리면 대기자가 자동 승급하고, 한 명을 수동 승격하거나 강제 제거할 수도 있다. 모집을 잠시 닫았다가 다시 여는 라이프사이클도 본 단위가 책임진다.

관리 항목
4
정원·대기열·승인제·모집 마감
시나리오
7

이 기능 안의 하위 항목

동작 1

정원 설정 폼

baseCapacity·waitlistEnabled·maxWait·approvalRequired 한 폼.

동작 2

수동 승격

대기 중 사용자를 호스트가 지목해 ATTENDING으로.

동작 3

강제 제거

ATTENDING 사용자 제거 → 다음 대기자 자동 승급.

동작 4

모집 마감/재개

isClosed 토글. 신규 신청 자동 차단/허용.

정원 변경 시 자동 승급

flowchart LR A["baseCapacity 10 → 15"] --> B["availableSlots = 5"] B --> C["대기 1~5번 ATTENDING 승격"] C --> D["currentCapacity = 15"] C --> E["5명 WAITLIST_PROMOTED 푸시"] C --> F["대기 6,7,8 → 1,2,3 재정렬"]

사용자 시나리오 7개

S1 · 수동 승격

대기 2번을 우선 승격

  1. 대기열 화면에서 카드의 “승격” 탭
  2. WAITING → ATTENDING + manuallyPromoted
  3. 나머지 순번 재정렬 + 푸시
S2 · 자동 승급

정원 확장으로 5명 일괄 승급

  1. baseCapacity 10 → 15
  2. 대기 1~5번 ATTENDING + 푸시
  3. 나머지 순번 재정렬
S3 · 강제 제거

노쇼 사용자 제거

  1. 참석자 목록에서 “제거” 확인
  2. CANCELLED + 체크인 정리
  3. 다음 대기자 자동 승급
S4 · 마감/재개

모집 일시 마감 후 재개

  1. “모집 마감” → isClosed=true → 신규 신청 차단
  2. 검토 후 “모집 재개” → isClosed=false → 액션바 다시 활성
S5 · 자동 승급 알림

대기자가 푸시 수신

  1. 다른 사용자 취소 → 1번 대기자 자동 승급
  2. 푸시 “대기열에서 참석으로 승급되었습니다”
  3. 액션바 자동 “참석 취소”로 갱신
S6 · 잘못된 승격

이미 참석 중인 사용자 승격 시도

  1. 409 PROMOTION_FAILED
S7 · 권한

일반 사용자가 설정 진입 시도

  1. 라우터 redirect
  2. 우회 시 403 EVENT_NOT_OWNER

자주 빠뜨리는 케이스

레드 플래그
  • 정원 확장 시 자동 승급 누락 — 자리가 났는데 채워지지 않음.
  • 강제 제거 후 다음 대기자 승급 누락 — 빈 자리.
  • 마감 후 재개 시점에 신청자 알림 누락 — 사용자가 자리가 다시 열린 걸 모름(향후 알림 검토).
  • 비관적 잠금 없는 정원 변경 — 동시 신청과 충돌.
