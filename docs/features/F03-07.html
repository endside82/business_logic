F03-07. 정원·대기열 관리 (호스트)

호스트가 자기 이벤트의 정원·대기열·모집 마감을 운영하는 면. 정원 확장으로 자동 승급을 일으키거나, 노쇼 우려 사용자를 직접 제거하거나, 모집을 잠시 닫는 동작이 본 단위에서 일어난다. 사용자 측 대기열은 F03-05에서 처리되고, 본 단위는 호스트 시점의 운영 도구.

정원 설정 항목
4개
기본 정원·대기열·최대 대기·승인제
자동 승급 트리거
2가지
정원 확장 · 다른 사용자 취소
설계 의도 — 정원 변경 한 번이 여러 사용자에게 동시 영향

호스트가 정원을 10에서 15로 늘리면 대기열 1~5번까지가 한 번에 자동 승급되고, 각각의 사용자에게 푸시 알림이 도달한다. 한 번의 동작 안에서 5명의 상태가 바뀌고 5건의 알림이 발송된다. 그래서 본 단위의 동작은 정원 수치 변경에서 끝나지 않고, 대기열 재정렬·자동 승급·알림 발송까지가 같이 일어나야 한다. 호스트가 "정원을 늘렸다"고만 인지해도 사용자 측에서는 "대기열에서 참석으로 바뀌었다"는 결과가 나와야 한다.

사용자 시나리오

자동 승급

정원을 10 → 15로 확장

호스트가 정원 설정에서 기본 정원을 변경. 대기열 1~5번까지가 자동으로 참석 확정으로 전환되고 5명에게 푸시 알림이 도달한다. 대기열에 남은 6·7·8번은 1·2·3번으로 순번이 재정렬.

이 흐름이 남기는 것 — 정원 변경 한 번이 다수 사용자에게 동시 영향을 준다. 모든 사용자에게 알림이 도달해야 정원 운영이 정상.

수동 승급

대기열의 단골을 우선 승격

호스트가 대기열 화면에서 특정 사용자의 "승격" 버튼을 누른다. 그 사용자는 참석 확정으로 전환되고, 정원이 +1 늘어난다(기본 정원보다 커질 수 있음). 다른 대기자들의 순번이 재정렬.

이 흐름이 남기는 것 — 호스트의 운영 재량. 정원 상한을 일시적으로 넘는 것을 호스트 결정으로 허용. 단골 운영의 핵심 도구.

강제 제거

노쇼 우려 사용자를 미리 제거

이전 두 번 무단 결석한 사용자가 또 신청한 상태. 참석자 목록에서 "제거"를 누르면 확인 후 그 사용자는 취소 상태로 전환되고, 1번째 대기자가 자동 승급된다. 체크인 기록이 있었다면 함께 정리.

이 흐름이 남기는 것 — 호스트의 안전 운영 도구. 잠재 노쇼를 사전에 제거해 모임의 실질 운영 인원을 보장.

모집 마감

갑자기 신청이 폭주해서 잠시 닫음

인플루언서 공유로 신청이 폭주하자 호스트가 "모집 마감"을 누른다. 신규 신청은 차단되고 상세 화면 액션바도 "마감"으로 바뀐다. 호스트가 검토 후 "모집 재개"를 누르면 다시 신청 가능.

이 흐름이 남기는 것 — 호스트의 임시 안전장치. 모집을 완전히 끝내지 않고 잠시만 닫는 동선이 본 단위의 특별한 가치.

대기열 알림 수신

참가자 시점 — 자동 승급 알림

본인이 대기열 1번. 다른 사용자가 취소 → 자동 승급 → 푸시 "대기열에서 참석으로 승급되었습니다". 알림 탭 → 상세 진입 → 액션바가 "참석 취소"로 바뀌어 있음.

이 흐름이 남기는 것 — 사용자가 자기 상태 변화를 인지하지 못하면 노쇼로 이어진다. 알림 도달은 본 단위의 핵심 신호.

잘못된 승격 시도

이미 참석 중인 사용자를 승격 시도

호스트가 오타로 이미 참석 중인 사용자에 "승격"을 누른다. "참석자 승격에 실패했습니다" 안내. 상태 변경이 일어나지 않는다.

권한

일반 사용자가 정원 설정 URL 진입

호스트와 공동호스트가 아닌 사용자가 URL로 진입해도 이벤트 상세로 리다이렉트된다. 우회 호출도 서버에서 차단.

화면에서 사용자가 인지하는 것

화면사용자가 보는 것그 자리에 둔 이유
참석자 목록 현재 참석자 N / 정원 M명 헤더, 본인 상태 카드(본인이 참석/대기 중일 때), 닉네임 검색, 카드별 "승격"·"제거" 버튼(호스트). 호스트가 화면 하나에서 누가 와 있고 누구를 빼고 누구를 우선으로 올릴지 결정하게.
대기열 상단 "대기열 (N명)" 배너, 순번 표시 카드. 호스트는 카드마다 "승격" 버튼, 본인이 대기 중이면 상단에 "내 대기 순번 N번" 카드. 대기열은 순번이 핵심 정보. 순번이 명확하지 않으면 사용자가 자기 위치를 가늠하지 못한다.
정원 설정 폼 기본 정원(0 = 제한 없음), 대기열 활성 토글, 최대 대기 인원, 승인제 토글, "저장" 버튼. 네 가지 옵션을 한 화면에 모아 호스트가 모임 운영 방식을 한 자리에서 결정.
제거 확인 다이얼로그 "닉네임님을 참석자에서 제거하시겠습니까?" 빨강 강조 + 확인 버튼. 제거는 사용자에게 큰 영향을 주는 동작이므로 한 단계 더 강조해서 확인을 받는다.

운영자가 알아야 할 것

정원·대기열 운영의 결정점
  • 정원 변경 시 자동 승급·알림이 한 동작 안에서 일어나야 한다. 정원만 늘리고 자동 승급이 누락되면 "정원은 늘었는데 대기열에 사람이 그대로"라는 사고가 발생.
  • 수동 승급은 정원 상한을 일시적으로 초과할 수 있다. 호스트의 운영 재량. "기본 정원 + 1, + 2"의 상태를 시각화해 호스트가 인지하게 한다.
  • 강제 제거 시 체크인 기록도 같이 정리된다. 잘못 제거하면 체크인 기록까지 사라지므로, 제거는 한 번 더 확인 단계를 거친다.
  • 모집 마감은 완전 종료가 아니다. 호스트가 다시 재개할 수 있다. 일시적 안전장치로 사용하도록 안내가 필요.
  • 대기열 자동 승급의 알림 도달이 노쇼율의 직접 신호. 사용자가 알림을 못 받으면 자기 상태 변화를 모른 채 모임에 나타나지 않는다.
