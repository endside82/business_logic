F11-05. 신뢰점수 · 변동 이력

사용자가 그동안 한 활동(참석·주최·인증·확정된 신고)을 0~100점 신뢰점수와 다섯 등급(DIAMOND·PLATINUM·GOLD·SILVER·BRONZE)으로 환산해 보여 주는 면이다. 본인 화면에서는 변동 이력까지 펼쳐 볼 수 있고, 다른 사용자 화면에서는 점수와 등급, 구성 비율만 보인다. 이 점수는 추천·매칭의 노출 가중치로도 쓰이기 때문에 한 번이라도 식이 어긋나면 다른 도메인이 동시에 잘못된 결정을 한다.

설계 의도 — 점수는 한 줄 식이고, 영향은 사방으로 흩어진다

신뢰점수는 본인에게는 "내 활동의 결과"이고, 다른 사용자에게는 "이 사람이 어떤 사람인지"의 첫 단서다. 추천·매칭 도메인은 이 점수를 노출 순서의 가중치로 직접 가져간다 — 같은 모임을 검색해도 신뢰점수가 다른 사용자가 만든 모임은 다른 순위에 뜬다. 그래서 산정식은 단순해야 한다("활동 절반, 인증 1/4, 패널티 -1/4"). 동시에 사용자가 "내 점수가 왜 이렇게 됐는지" 추적할 수 있어야 한다 — 본인 한정 변동 이력이 그 자리다. 남에게는 변동 과정을 노출하지 않는다 — 점수의 결과만 보이게 한다.

이 단위가 다른 도메인의 결정을 움직인다

신뢰점수가 바뀌면 그 사용자에 관한 여러 면이 같이 흔들린다
  • 추천·매칭의 노출 가중치. 검색·홈 추천·데이팅 후보 카드 노출에 신뢰점수가 가중치로 반영된다 — 점수가 한 번이라도 잘못 잡히면 사용자가 보는 풍경 전체가 어긋난다.
  • 호스트의 다음 모임 신청 결정. 다른 사용자가 모임을 신청할 때 호스트의 점수·등급이 신뢰의 첫 단서. 평균 별점과는 다른 차원의 누적 평가다.
  • 데이팅 도메인의 카드 노출. 인증·활동·패널티가 모두 반영된 점수가 매칭 후보 순서에 영향을 준다 — 인증이 약한 사용자는 노출에서 자연스럽게 내려간다.
  • 운영 측의 다음 결정. 패널티가 빠르게 누적되는 사용자는 운영 모니터링 대상에 올라간다 — 신고 한 건 한 건이 아니라 점수의 변동 패턴이 관찰 신호.

점수가 만들어지는 세 갈래

점수는 활동·인증·패널티 세 갈래의 가중합이고, 0~100 범위로 잘린다. 식 자체는 한 줄이지만 각 갈래에 들어가는 입력 신호는 여러 도메인에서 모인다.

갈래가중치들어가는 신호어디서 오는가
활동 +50% 참석한 모임 수, 주최한 모임 수 (클럽·연속 활동은 현재 기본값 고정). 이벤트·참석 도메인. 모임에 정상 참석하면 자동으로 누적된다.
인증 +25% 이메일(30) · 전화(30) · 프로필 사진(10) · 자기소개(5) · 실명(5) · 소셜 연동(10~20). 인증·프로필 도메인. 인증 한 단계가 더해질 때마다 점수가 즉시 오른다.
패널티 −25% 운영자가 승인·이관 처리한 신고 1건당 20점, 최대 100. 신고 도메인(F11-04). 접수만으로는 안 깎이고, 운영자 결론이 난 뒤에 잡힌다.

최종 점수는 활동·인증·패널티를 가중합한 뒤 0~100으로 잘라 소수 첫째까지 반올림한다. 신규 사용자는 활동이 0이라 인증·기본값으로 결정되며, 보통 50점 근처에서 시작한다.

등급 임계 — 점수가 어디서 갈리는가

점수는 다섯 등급으로 나뉘어 카드·프로필에 배지로 노출된다. 임계는 서버 정책이 기준이며, 클라이언트 화면이 다음 등급까지 남은 점수를 계산할 때 같은 임계를 써야 정확하다.

등급도달 점수의미
DIAMOND90점 이상활동·인증이 모두 누적된 최상위 — 호스트 추천 노출의 최상단.
PLATINUM75점 이상꾸준한 활동과 인증을 모두 갖춘 사용자.
GOLD60점 이상활동이 어느 정도 누적된 단계 — 매칭·추천에서 일반 사용자보다 노출 우위.
SILVER40점 이상인증을 마치고 첫 활동을 시작한 단계.

이전에는 화면이 "다음 등급까지 N점"을 표시할 때 PLATINUM 80점 · DIAMOND 미노출로 하드코딩되어 있었다. 커밋 b0dc370(2026-06-04)에서 서버가 제공하는 TrustScoreVo.nextGradeScore 값을 직접 소비하도록 전환했다. 이제 서버 임계(BRONZE <40 / SILVER 40+ / GOLD 60+ / PLATINUM 75+ / DIAMOND 90+)가 화면 표기에도 그대로 반영된다.

변동 이력은 본인에게만, 미세 변동은 이력에서 제외

사용자가 본인 점수를 가장 자주 보는 자리. "왜 내 점수가 어제와 다른가"를 추적할 수 있어야 한다.

사용자 시나리오

본인 첫 조회

마이페이지에서 "신뢰점수"로 진입

가입 두어 달 된 사용자가 본인 신뢰점수를 본다. 화면 중앙에 원형 게이지로 점수가 그려지고, 등급 배지("BRONZE")와 "다음 등급까지 N점" 안내가 따라 붙는다. 아래에 활동·인증·패널티 세 갈래의 구성이 카드로 깔리고, 그 아래에 7일 기준 변동 이력 리스트가 뜬다.

이 흐름이 남기는 것 — 사용자에게 "내 활동이 어떻게 점수로 환산됐는지"가 한 화면에서 보인다. 추가 인증이나 활동의 동기 부여로 이어진다.

기간 전환

7일 → 1개월 → 3개월

변동 이력 위의 기간 셀렉터에서 "1개월"을 누르면 30일치 스냅샷이 다시 들어와 채워진다. 한 번 더 "3개월"을 누르면 90일치까지 펼쳐진다.

이 흐름이 남기는 것 — 사용자가 자신의 활동 누적 추세를 본인이 정한 기간으로 본다. 추세선을 따라가다 보면 "최근 활동이 줄었구나"를 스스로 인지하는 동기 회복의 자리.

타인 조회

호스트의 신뢰점수를 가늠

모임 신청 전에 호스트의 점수가 궁금한 사용자가 프로필에서 "신뢰점수"로 들어온다. 점수·등급·구성 비율은 보이지만 변동 이력 섹션은 노출되지 않는다.

이 흐름이 남기는 것 — 다른 사용자에게는 결과만 보이고, "어떻게 그 점수에 도달했는지"의 과정은 본인 외에는 접근할 수 없다 — 점수 변화에서 일상이 추론되는 일을 막는다.

간접 변동

방금 새 리뷰를 받은 호스트가 점수 화면을 열었을 때

다른 사용자가 후기를 남기면 호스트의 점수가 다시 계산되고, 변동이 1점 이상이면 그날의 변동 이력에 한 줄이 추가된다. 호스트가 점수 화면을 열어 보면 새 스냅샷이 리스트 끝에 보인다.

이 흐름이 남기는 것 — 외부 도메인의 행동(리뷰)이 본 단위의 화면에 즉시 반영된다는 일관성. 점수가 사용자 활동과 함께 살아 있다는 인지.

미세 변동

작은 활동이라 이력에는 잡히지 않음

신청 취소 같은 작은 변동이 일어나 점수가 0.4점만 움직였다. 점수 자체는 갱신되지만 변동 이력 리스트는 그대로다.

이 흐름이 남기는 것 — 잦은 미세 변동이 이력 그래프를 어지럽혀 "큰 변화"가 묻히는 것을 막는다. 1점이라는 임계가 노이즈와 신호를 가른다.

패널티 누적

승인된 신고가 한 건 추가됐을 때

다른 사용자의 신고가 운영자에 의해 승인 처리됐다. 그 시점에 대상자의 패널티가 20점 누적되고, 본인의 다음 화면 진입에서 점수가 떨어진 모습이 보인다. 변동 이력에 한 줄이 추가된다.

이 흐름이 남기는 것 — 접수 시점이 아닌 운영자 결론 시점에 점수가 움직인다는 정책이 사용자에게 화면으로 일관되게 보인다.

빈 상태

가입 직후라 변동 이력이 비어 있다

신규 사용자가 본인 점수를 본다. 점수는 50점 근처(인증·기본값) 등급은 BRONZE, 변동 이력은 "변동 이력이 없습니다" 안내만.

이 흐름이 남기는 것 — 신규 사용자에게 "이 자리가 앞으로 채워질 곳"이라는 인지를 만든다. 빈 상태가 오류로 인지되지 않게.

화면에서 사용자가 인지하는 것

실제로 화면을 열었을 때 보이는 정보와, 그 정보가 그 자리에 있는 이유.

화면 요소사용자가 보는 것그 자리에 둔 이유
원형 게이지 화면 상단 중앙에 큰 원형 그래프. 중앙에 숫자(소수 첫째), 그 아래 "/100" 보조 글자. 점수의 "절대 위치"보다 "100점 중에서 어디쯤"이 사용자의 직관에 가깝다 — 원형이 좌우의 여유를 시각화한다.
등급 배지 게이지 아래에 캡슐 모양 배지. 등급별 다른 색상과 아이콘(다이아·왕관·메달). 점수 숫자만으로 의미를 짐작하기 어렵다 — 등급 라벨이 의미를 한 단어로 잡아 준다.
"다음 등급까지 N점" 배지 아래 한 줄 안내. DIAMOND(최고 등급)에서는 노출되지 않는다. 다음 단계로 갈 동기를 만든다 — "조금만 더 활동하면" 신호. 2026-06-04 기준 서버가 제공하는 nextGradeScore 값을 직접 사용하여 임계 불일치가 해소됐다.
점수 구성 카드 리스트 활동·인증·패널티 갈래의 점수와 가중치가 카드 형태로 깔린다. 양수는 파란 계열, 음수는 빨간 계열. "내 점수가 어디서 만들어졌는가"를 사용자가 추적할 수 있게 한다. 패널티가 빨간색으로 표시되는 자리에 사용자의 시선이 가장 먼저 간다.
기간 셀렉터(본인 모드) "7일 / 1개월 / 3개월" 세 탭. 현재 선택된 탭만 강조. 사용자가 자기 활동 호흡에 맞춰 기간을 선택. 3개월보다 긴 기간은 화면에서 직접 노출하지 않는다 — 사용자의 일상에서 의미 있는 호흡 범위.
변동 이력 리스트(본인 모드) 날짜(MM-DD) + 변동 사유 + 그날의 총점. 점수가 어느 날, 왜 움직였는지를 한 줄로 보여 줘 사용자가 자기 활동을 추적할 수 있게 한다.

측정해야 하는 것

본 단위가 정상이라고 부르려면 이 다섯 가지가 추적되어야 한다
  • 등급별 사용자 분포. 한 등급에 사용자가 쏠리면 등급의 변별력이 사라진다. 임계 조정 여부의 첫 신호.
  • 인증 단계별 점수 기여도. 어떤 인증이 점수에 가장 크게 기여하는지 추적해 사용자에게 어떤 인증부터 권하는 것이 효율적인지 결정.
  • 패널티 평균과 최댓값. 누적 패널티가 한 사용자에게 빠르게 쌓이는 패턴이 관찰되면 그 사용자의 다음 활동(이벤트 호스팅·매칭 노출)을 운영 측이 미리 통제할 결정 시점.
  • 변동 이력 적재 비율. 호출당 이력 INSERT 비율이 너무 낮으면 1점 임계가 너무 높다는 신호, 너무 높으면 임계를 더 올릴 수 있다.
  • 본인 화면 진입과 추가 활동 사이의 상관. 점수 화면이 사용자의 추가 인증·참석을 유도하는지 측정 — 화면이 정말 동기 부여로 작동하는지의 핵심 신호.

운영자가 알아야 할 것 — 비즈니스 리스크

본 단위에서 무너지면 도메인 전체가 같이 흔들린다
  • 점수 산정식 한 줄이 추천·매칭·검색의 노출 가중치를 동시에 결정한다. 식의 어느 한 갈래가 잘못 계산되면 사용자가 보는 모든 노출 결과가 어긋난다 — 본 단위의 모니터링이 가장 우선.
  • 등급 임계는 서버 단일출처로 전환됐다(2026-06-04 해소). 화면이 nextGradeScore 서버 값을 직접 소비하므로 임계 불일치가 해소됐다. 서버 임계(40·60·75·90)와 화면 표기가 이제 일치한다.
  • 본인 한정 변동 이력 정책이 흔들리면 사생활이 노출된다. 점수 변동 사유에는 운영자 결론(신고 승인) 같은 민감 정보가 섞일 수 있어 다른 사용자가 보면 안 된다. 본인 외 노출 가드는 어떤 회귀에서도 풀려서는 안 된다.
  • "승인된 신고만 패널티에 반영" 정책이 무너지면 평판이 무기가 된다. 접수 시점에 점수가 깎이는 회귀가 들어가면 사용자 간 보복 신고가 발생, 무고한 사용자가 즉시 노출에서 밀린다.
  • 노쇼 카운트는 점수 안에 자리는 있지만 현재 패널티 식에 들어가지 않는다(보강 후보). "노쇼한 사용자가 평판에서 손해 본다"는 운영 기대와 어긋난다 — 노쇼 카운트를 패널티에 포함시킬지 정책 결정이 필요.
  • 1점 미만 변동을 이력에서 제외하는 정책은 노이즈를 막지만, 누적된 미세 변동이 이력에서 사라진다. 사용자가 "내 점수가 어느 날 슬그머니 변했다"고 느끼는 시점이 오면 임계를 0.5 같은 더 작은 값으로 조정할지 검토.
