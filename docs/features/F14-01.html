F14-01. 이벤트 참석자 위치 공유

참석 확정한 모임에서 일행끼리 지도 위에서 서로 어디쯤 오고 있는지를 보는 화면. 사용자가 직접 토글을 켜야만 본인 위치가 다른 사람 화면에 보이고, 토글을 켠 뒤에는 30초마다 좌표가 갱신된다. 위치 공유는 이 도메인에서 가장 민감한 동작이므로, 사용자 동의 · 모임 시간대 · 호스트의 이벤트 단위 허용 세 가지가 모두 맞아떨어진 경우에만 다른 참석자에게 좌표가 노출된다. "지금 어디쯤?"이라는 질문을 채팅으로 다시 묻지 않게 하는 것이 본 단위의 목표다.

위치는 본 서비스에서 가장 민감한 정보다. 그래서 본 단위의 모든 동작은 "본인이 직접 켜지 않으면 좌표는 한 번도 노출되지 않는다"를 보장한다. 동의를 받았다 해도 모임이 아닌 시간에 보이지 않도록 시간대 게이트가 있고, 호스트가 모임 단위로 위치 공유 자체를 꺼두면 사용자 토글이 켜 있어도 좌표가 더 이상 흘러가지 않는다. 이 세 겹 중 한 겹이라도 풀리는 회귀가 발생하면 "내가 켜지도 않았는데 위치가 보였다"라는, 본 서비스에서 가장 치명적인 신뢰 사고가 즉시 발생한다.

이 단위가 한 번에 처리하는 세 가지

사용자에게는 토글 하나처럼 보이지만, 본 단위는 동의·갱신·조회 세 가지를 함께 책임진다.

동의

위치 공유 켜기

사용자가 토글을 켜면 운영체제 위치 권한을 한 번 요청하고, 좌표를 처음 한 번 보낸 다음 자동 갱신 루프를 시작한다. 권한을 거부하면 토글은 꺼진 상태로 유지되며, 영구 거부면 운영체제 설정 화면으로 안내한다.

유지

30초마다 좌표 갱신

토글이 켜져 있는 동안 30초마다 본인 좌표가 서버로 보내지고, 다른 사람 화면에서도 마커가 새 위치로 움직인다. 사용자가 "지금 바로 갱신"을 원하면 화면 우하단의 작은 버튼을 눌러 즉시 한 번 더 보낼 수 있다.

조회

일행 위치 보기

지도 위에 모임 장소(빨강), 본인(파랑), 다른 참석자(초록) 마커가 함께 표시되고, 하단 시트에는 각 참석자의 이름과 모임 장소까지의 거리·예상 소요 시간이 리스트로 보인다. 본인이 공유를 켜지 않으면 다른 사람 좌표도 보이지 않는다 — 일방적 관찰을 막는다.

호스트 우회

호스트는 조회만 우회

모임 호스트는 본인이 토글을 켜지 않아도 일행 위치를 볼 수 있다 — 모임을 안내해야 할 책임이 있기 때문. 단, 호스트도 자기 위치를 보내려면 일반 참석자와 똑같이 토글을 켜야 하며 시간대 안에 있어야 한다.

위치가 다른 사람에게 보이기까지의 세 겹 통과

사용자가 켰다는 것만으로는 부족하다. 시간대, 호스트 허용, 그리고 본인 동의 — 셋 다 통과해야 좌표가 응답에 들어간다.

위치 공유의 상태 흐름

사용자 한 명의 공유 상태는 네 가지를 오간다. 자동 종료는 마지막 좌표 갱신 시점에서 2시간 뒤에 일어나고, 갱신할 때마다 종료 시각이 다시 2시간 뒤로 밀린다.

사용자 시나리오

기본 흐름

모임 30분 전, 위치를 켠다

러닝 모임 참석을 확정한 사용자가 "참석자 위치" 화면에 들어온다. "위치 공유" 토글을 켜면 운영체제 권한을 한 번 묻고, 허용을 누르는 순간 본인 좌표가 지도에 파란 마커로 찍힌다. 하단 시트에는 일행의 이름·모임 장소까지의 거리가 한 줄씩 보이고, 30초마다 자동으로 갱신된다. "위치 공유를 시작합니다" 토스트가 한 번 떠 사용자가 자기 의사로 켰음을 확인시켜 준다.

이 흐름이 남기는 것 — 모임 직전의 가장 큰 불확실성("일행이 어디쯤 오고 있나")이 채팅 없이 해소된다. 이 한 화면을 위해 위치 도메인 전체가 존재한다.

자동 갱신

이동하는 중에 마커가 따라온다

토글을 켠 채 5분간 화면을 보면 매 30초마다 본인 마커가 새 위치로 이동하고, 일행 마커도 자기 자리에서 갱신된다. 사용자가 "지금 한 번 더" 갱신하고 싶으면 우하단의 작은 동그란 버튼을 한 번 더 누른다 — 30초를 기다리지 않고 곧장 새 좌표가 반영된다.

이 흐름이 남기는 것 — 사용자가 "내 위치가 진짜로 보내지고 있는가"를 직접 확인할 수 있는 통제 장치.

호스트 우회

호스트는 위치를 켜지 않아도 일행을 본다

모임 호스트가 같은 화면을 열면 본인 토글이 꺼져 있어도 일행의 마커가 즉시 보인다 — 모임을 안내할 책임이 있기 때문에 조회만 따로 허용된다. 다만 본인 위치를 보내려면 호스트도 똑같이 토글을 켜고 시간대 안에 들어와 있어야 한다.

이 흐름이 남기는 것 — 책임 비대칭을 정책에 그대로 반영. 호스트가 일행을 못 보면 "도착했냐"를 채팅으로 다시 물어야 한다.

시간대 분기

모임 30시간 전에 미리 들어와 봤을 때

호기심에 너무 일찍 위치 화면을 연 일반 참석자는 좌표가 한 점도 보이지 않는다. "공유 가능 시간이 아닙니다" 안내만 노출되고, 모임 시작 24시간 전 시점이 되면 그제서야 일행이 보이기 시작한다.

이 흐름이 남기는 것 — 모임과 무관한 시간대에 위치가 노출되지 않게 한다. 사용자가 의식하지 못한 시점의 위치 노출을 막는 안전장치.

권한 분기

운영체제 위치 권한을 영구 거부한 사용자

과거에 "다시 묻지 않기"로 위치 권한을 거부한 사용자가 토글을 켜면 권한 다이얼로그가 더 이상 뜨지 않는다. 앱이 "위치 권한이 영구 거부되었습니다. 설정에서 변경해주세요" 토스트를 띄우고, 그대로 운영체제 설정 화면으로 보낸다. 토글은 꺼진 상태로 남고, 서버에 동의 기록이 만들어지지 않는다.

이 흐름이 남기는 것 — 앱 안에서는 풀 수 없는 영구 거부를 사용자가 어디로 가서 해결해야 하는지 한 번에 안내한다.

호스트 차단

호스트가 모임 단위로 위치 공유를 꺼두었다

호스트가 모임 설정에서 "이 모임은 위치 공유 안 함"을 선택한 경우, 일반 참석자가 토글을 켜도 좌표 갱신 자체가 막힌다. 사용자 입장에서는 토글이 켜져 있는데 본인 마커가 다음 갱신에서 사라진다 — 현재 구현은 별도 안내 토스트가 없어 사용자가 "왜 안 보이지?"라고 의아해할 수 있는 분기.

운영 관찰 포인트 — 호스트의 모임 단위 비활성화가 발동되었을 때 일반 참석자에게 명시적 안내가 가도록 보강하는 것이 향후 과제. 현 상태는 "조용한 차단"에 가깝다.

중도 비공개

같이 가던 일행이 도중에 위치를 꺼버렸을 때

모임 중 한 사람이 토글을 끄면 다음 30초 갱신에서 그 사람 마커가 다른 사람 화면에서 사라지고, 하단 시트의 거리 리스트에서도 빠진다. 그 사람의 이름이 "비공개"로 남는 게 아니라 목록에서 즉시 사라지는 동작이다 — 사용자가 끄는 순간 그 효과가 단호히 즉시 반영된다.

이 흐름이 남기는 것 — "껐다"는 사용자 의사 표시가 다른 사람의 다음 30초 안에 반드시 반영된다는 보장. 이 보장이 깨지면 위치 공유 중지 기능 자체의 신뢰가 무너진다.

화면에서 사용자가 보는 것

위치 화면을 열었을 때 보이는 정보와, 그 정보가 그 자리에 있는 이유.

화면 요소사용자가 보는 것그 자리에 둔 이유
풀스크린 지도 모임 장소(빨강 마커), 본인(파랑), 다른 참석자(초록). 지도 상단에 마커 색의 의미를 알려주는 작은 캡슐 범례. 색만으로 "나/장소/일행"을 즉시 구분할 수 있어야 시선이 흩어지지 않는다. 범례는 처음 본 사용자가 색의 의미를 한 번에 이해하도록 둔다.
하단 시트 토글 행("위치 공유 중/위치 비공개") + 본인 공유 카운트다운("약 N분 후 종료" + "+30분 연장" 버튼) + 참석자 카운트 + 일행 리스트(이름·거리·예상 시간·마지막 갱신). 지도와 함께 한 화면에 정보가 있어야 사용자가 시선을 바꾸지 않는다. 시트는 위·아래로 끌어 비율 조절이 가능해 지도 비중과 리스트 비중을 사용자가 정한다.
위치 공유 토글 시트 맨 위 큰 스위치. 꺼진 상태에서는 라벨이 회색, 켜진 상태에서는 색이 들어간다. 사용자가 자신의 공유 상태를 항상 시각적으로 인지하게 만든다 — 위치 공유의 4가지 원칙 중 "공유 상태 시각화"의 실현.
내 위치 갱신 버튼 화면 우하단의 작은 동그란 버튼(현재 위치 아이콘). 30초를 기다리지 않고 즉시 갱신할 수 있는 사용자 통제 장치. "내가 직접 보냈다"는 통제감을 자동 폴링과 분리해 둔다.
상단 톱니바퀴 오른쪽 위 아이콘 — 누르면 위치 프라이버시 대시보드(F14-04)로 이동. "내 위치가 어디에 보이고 있는가"를 사용자가 언제든 한 번에 확인하러 갈 수 있는 출구. 위치 공유의 4가지 원칙 중 "공유 상태 시각화"가 이 진입로로 완성된다.
일행 카드 한 줄 닉네임·프로필 사진(호스트가 보고 있을 때만) + 모임 장소까지의 거리·예상 시간 + "방금 전 / N분 전" 같은 마지막 갱신 시각. "이 사람이 지금 어디쯤?"이라는 질문을 한 줄로 답한다. 마지막 갱신 시각이 같이 보여 "데이터가 신선한지"를 사용자가 직접 가늠한다.

운영자가 알아야 할 것

본 화면이 정상이라고 부르려면 다음이 보장되어야 한다
  • 본인이 위치 공유를 켜지 않으면 다른 참석자의 좌표도 보이지 않는다. "남 위치는 보면서 내 건 안 보낸다"는 비대칭이 만들어지면 위치 공유의 신뢰 구조가 무너진다. 일방적 관찰 차단이 본 단위의 첫 번째 안전선.
  • 호스트가 모임 단위로 위치 공유를 끄면 일반 참석자에게 안내가 가야 한다. 현재는 토글이 켜져 있어도 좌표가 안 가는 "조용한 차단"에 가깝다 — 사용자가 "내가 진짜 보내고 있는가"를 모른다. 향후 명시적 토스트 안내 보강 필요.
  • 좌표 노출은 매번 접근 이력으로 기록된다. 누가 누구의 위치를 봤는지의 이력이 30일간 남아, 프라이버시 대시보드(F14-04)에서 사용자가 확인할 수 있는 원천이 된다. 이 기록이 빠지면 "내 위치를 누가 봤는가"를 추적할 수 없다.
  • 이벤트가 끝나고 2시간이 지나면 본 화면은 더 이상 열리지 않는다. 위치 데이터를 모임 시간대에 갇혀 있게 하는 정책. 모임 끝난 뒤에 위치를 보거나 보내는 일이 없도록 차단되며, 보관된 데이터는 별도 청소 작업이 30일 안에 제거한다.
  • 운영체제 위치 권한 영구 거부는 앱 안에서 해결 불가능하다. "다시 묻기" CTA를 만들지 않고 운영체제 설정 화면으로 보내는 동작을 유지해야 한다. 앱 안에서 거부를 풀 수 있다고 사용자가 오해하면 권한 다이얼로그를 무한 반복으로 보게 된다.
  • 30초 자동 갱신은 화면을 벗어나면 멈춰야 한다. 화면을 나갔는데 갱신 루프가 계속 돌면 배터리·데이터가 새고, 사용자가 모르는 사이에 위치가 계속 서버로 흘러간다. 화면 dispose 와 앱 백그라운드 진입 시 둘 다 중지가 보장되어야 한다.

측정해야 하는 것

위치 공유가 사용자에게 가치를 주고 있는지를 보여주는 지표
  • 모임 시작 24시간 전 진입 → 토글 켜기까지의 비율 — 화면에 들어왔지만 켜지 않는 비율이 높다면 사용자가 가치를 못 느끼거나 안내가 부족한 신호.
  • 토글 켠 채로 모임 종료까지 유지된 비율 — 중간에 꺼버리는 사용자가 많다면 갱신 주기·배터리 부담·노출 범위 어딘가에 문제가 있다는 신호.
  • 운영체제 권한 영구 거부 비율 — 너무 높다면 앱 외부에서 해결해야 하는 사용자가 너무 많다는 뜻 — 첫 권한 요청 시점의 설명이 부족한지 점검.
  • 호스트 차단(모임 단위 비활성화) 발동 후 사용자 행동 — 명시적 안내가 부족한 현 상태에서 사용자가 어떻게 반응하는지를 추적하면 보강 우선순위가 결정된다.
