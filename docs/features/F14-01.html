F14-01. 이벤트 참석자 위치 공유

모임 시작 24시간 전부터 종료 2시간 후까지, 참석 확정자가 본인 의지로 opt-in한 경우에만 일행끼리 위치를 공유한다. opt-in × 시간 게이트 × 호스트 설정 세 축 모두 통과해야 잔여 좌표가 응답에 들어가고, opt-out한 순간 다른 사람 화면에서 본인 마커가 즉시 사라진다.

  1. opt-in 명시 동의 — 토글 ON + OS 권한 “허용”
  2. 시간 게이트 — 이벤트 시작 −24h ~ 종료 +2h. 호스트는 조회만 우회 가능.
  3. 호스트 글로벌 토글 — Event.locationShareEnabled. 호스트도 update는 우회 못 함.

이 기능 안의 하위 항목

동작 1

opt-in

토글 ON → OS 권한 요청 → 좌표 획득 → opt-in API → 즉시 update → 30초 폴링 시작. 권한 영구 거부 분기에서는 OS 설정 화면으로 안내.

동작 2

30초 위치 폴링 + 수동 새로고침

Timer.periodic 30초로 GET. FAB(내 위치) 탭은 즉시 update 후 GET. 마커 수가 변할 때만 fitBounds 재계산.

동작 3

일행 위치 지도/시트 표시

지도 마커 + 하단 시트 리스트. opt-out한 참석자는 응답에서 제외되어 자동 사라짐. opt-in 한 사람만 access_log에 기록.

동작 4

2시간 만료 자동 종료

마지막 update 시점 + 2h. update마다 슬라이딩 갱신. 만료 시 좌표 응답 제외. 연장은 F14-03.

동작 5

호스트 토글 비활성화 응답

이벤트 단위로 호스트가 위치 공유를 꺼두면 update 자체가 LOCATION_SHARE_DISABLED. 사용자 토글이 ON으로 보여도 실제 갱신은 막힘.

사용자 시나리오 7개

S1 · Happy Path

모임 30분 전 opt-in

  1. EventLocationScreen 진입 → GET → 본인 opt-in 안 됨 → 403
  2. OS 권한 다이얼로그 → 허용 → 토글 ON
  3. opt-in API + update API 연속 호출 → location_share 생성(expiresAt=now+2h)
  4. 30초 폴링 시작, “위치 공유를 시작합니다” 토스트
  5. 지도/시트에 본인 + 일행 마커
S2 · 폴링

5분간 화면 유지하며 이동

  1. 30초마다 refresh() → 좌표 응답
  2. 마커 수 동일이면 fitBounds 유지, 변할 때만 재계산
  3. FAB(내 위치) 수동 탭 → 즉시 update + refresh
S3 · 호스트

호스트는 opt-in 없이 조회

  1. 호스트 진입 → GET → isHost 분기로 opt-in/시간 게이트 우회
  2. 응답에는 여전히 opt-in 한 참석자 좌표만 포함(opt-out은 제외)
  3. 단, host 본인의 update는 시간 게이트 + locationShareEnabled 검사 받음
S4 · 시간 차단

24h보다 일찍 진입

  1. 참석자가 30시간 전 진입 → 403 LOCATION_ACCESS_DENIED
  2. “공유 가능 시간이 아닙니다” 안내, 사용자는 윈도우 진입 후 재시도
S5 · 권한 영구 거부

OS 설정 분기

  1. 토글 ON → checkPermission → deniedForever
  2. “위치 권한이 영구 거부되었습니다” 토스트 + openAppSettings()
  3. 토글 OFF 유지, opt-in API 미호출
S6 · 호스트 차단

locationShareEnabled=false

  1. 토글 ON → opt-in API 성공
  2. update 시도 → 403 LOCATION_SHARE_DISABLED
  3. 본인 좌표가 응답에 들어가지 않아 다음 폴링에서 본인 마커 미노출 → 토글이 자동 OFF로 추론

UX 보강 후보: 현재는 silent fail에 가깝다. 사용자에게 명시적 안내 토스트 필요.

S7 · 비공개 마킹

opt-out한 참석자는 응답에서 사라진다

  1. A는 opt-in, B는 opt-in 후 opt-out
  2. A의 다음 폴링 응답에서 B의 좌표 제외
  3. A 화면의 마커/리스트에서 B 사라짐

설계 결정: UI 스펙은 “비공개” 라벨로만 노출하라고 했지만 현 백엔드는 좌표 자체를 제외. 사용자에게는 “목록에서 사라짐”으로 보임.

자주 빠뜨리는 케이스

레드 플래그
  • 호스트 우회를 update에도 적용 — 호스트도 시간 게이트와 토글에 묶여야 한다.
  • opt-out 사용자 좌표가 응답에 남음 — 비공개 보장 위배.
  • 30초 폴링 정지 누락 — 화면을 나가도 Timer가 살아있으면 배터리·트래픽 낭비.
  • fitBounds를 매 폴링 재계산 — 지도가 떨려서 사용자 경험 저하.
  • LOCATION_SHARE_DISABLED 안내 누락 — 사용자는 토글이 ON인데 본인 마커가 안 보이는 이유를 모름.
  • access_log를 GET 시점이 아니라 update 시점에 기록 — “누가 누구의 위치를 봤는가” 추적이 깨진다.
  • OS 권한 영구 거부 시 “허용” 다이얼로그 반복 — openAppSettings로 명시 안내해야 한다.
