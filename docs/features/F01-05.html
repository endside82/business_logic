F01-05. 세션 유지·로그아웃

한 번 로그인한 사용자가 일정 기간 동안 다시 로그인하지 않아도 되도록 세션을 유지하고, 사용자가 명시적으로 로그아웃을 누르면 그 자리에서 세션을 끝낸다. 세션 유지의 갱신은 사용자 모르게 백그라운드에서 일어나며, 사용자 입장에서는 그저 앱이 끊김 없이 동작한다.

세션 유지
30일
앱을 한 달 안 켜도 자동 로그인
동시 로그인
1대
새 기기 로그인 시 이전 기기 자동 로그아웃
설계 의도 — 세션 갱신은 사용자가 모르게, 종료는 사용자가 분명하게

세션 유지는 인프라성 동작이지만 사용자 인지에는 직접 영향을 준다. 갱신 과정이 사용자에게 보이는 순간 신뢰가 흔들리기 때문에 ("왜 갑자기 로딩이 길지"), 본 단위는 갱신을 백그라운드에서 끝낸다. 반면 로그아웃은 사용자의 명시적 의사 표시이기 때문에 한 번 누르면 즉시 끝나야 하고, 다시 로그인 화면이 나오면서 "로그아웃되었습니다" 안내가 떠야 한다.

사용자 시나리오

기본 흐름

한 시간 이상 켜둔 채 다시 동작

사용자가 앱을 띄워두고 잠시 다른 일을 했다가 돌아온다. 화면이 정상적으로 다시 그려진다 — 사용자에게는 약간의 지연만 보이고 내부적으로는 세션 갱신이 한 번 일어났다. 같은 자리에서 같은 데이터를 본다.

이 흐름이 남기는 것 — 사용자가 매시간 다시 로그인할 필요 없는 자연스러운 사용 경험. 가장 자주 일어나는 동작이라서 갱신이 안 보이게 끝나는 것이 정책의 핵심.

장기 미사용

한 달 넘게 앱을 안 켜다가 다시 켬

30일이 지나면 세션이 자동으로 종료된다. 앱을 다시 켜면 로그인 화면이 뜨고 사용자는 다시 로그인해야 한다. "세션이 만료되었습니다" 안내가 별도로 뜨지는 않는다 — 그저 로그인 화면으로 돌아간다.

사용자 액션

프로필에서 로그아웃

사용자가 프로필 화면의 "로그아웃" 항목을 탭한다. 확인 다이얼로그가 뜨고 한 번 더 "로그아웃"을 누르면 로그인 화면이 나타난다. 로그인 화면 상단에는 "로그아웃되었습니다" 토스트가 한 번 떠오른 뒤 사라진다 — 의도된 동작이었음을 확인시켜 준다.

이 흐름이 남기는 것 — 사용자가 본인 의사로 끝낸 세션이라는 사실이 분명히 인지된다. 다음 진입자(다른 사용자일 수도 있음)의 정보가 화면에 남지 않게 한다.

기기 정책

다른 기기에서 새 로그인

사용자가 폰 외에 태블릿에서도 같은 계정으로 로그인한다. 폰의 세션은 그 즉시 끊어지고, 폰에서 다음 동작을 시도하면 자동으로 로그인 화면이 떠오른다 — 본 서비스는 한 계정에 한 기기만 활성 세션을 유지한다.

이 흐름이 남기는 것 — 데이팅·결제 같은 민감 도메인에서 계정 탈취 영향이 두 기기로 동시에 확장되지 않게 한다.

운영자가 알아야 할 것

세션 정책과 결정점
  • "왜 갑자기 로그인 화면이 떴냐"는 CS 문의의 대부분은 두 가지 원인이다. (1) 30일 미사용으로 세션 자동 만료, (2) 본인이 다른 기기에서 새로 로그인한 결과. 매뉴얼에 이 두 케이스를 우선 포함.
  • 로그아웃을 누른 즉시 다른 기기에서도 같은 계정으로는 다음 동작이 차단된다. 단일 기기 정책의 연장선 — 운영 측이 사용자 차단·강제 로그아웃이 필요한 경우 본 동선을 활용할 수 있다.
  • 로그아웃 API 통신이 실패해도 클라이언트에서는 세션이 끝난 것으로 처리된다. 사용자 의사가 분명하기 때문에 네트워크 오류로 화면이 멈추지 않게 하는 정책 — 다만 서버 측 세션은 다음 호출 시점이나 유효 기간 종료까지 살아있을 수 있다.
  • 비밀번호 변경(F01-04 결과)으로는 다른 기기 세션을 끊지 않는다. "비밀번호를 바꿨는데 다른 기기에서 계속 로그인된다"는 사용자 인지가 발생할 수 있다 — 보안 강화 후보로 운영 측이 인지.
