F04-11. 사진첩

멤버가 모임 사진을 앨범 단위로 모아 올리는 면. 게시판의 글에 사진을 덧붙이는 것과 다르게, 사진 자체가 중심이 되어 그리드와 뷰어로 본다. 앨범은 멤버 누구나 만들 수 있고, 한 앨범에 최대 50장까지 사진이 쌓인다. 자기 사진은 본인이 일괄 삭제하고, 부적절한 앨범은 운영자가 통째로 정리한다.

설계 의도 — 사진은 글과 다른 흐름이 필요하다

게시글에 사진 두세 장을 덧붙이는 흐름과, 모임에서 찍은 사진 30장을 한 자리에 모으는 흐름은 다르다. 후자를 게시판에 강제로 묶으면 글 본문이 의미를 잃고 사진은 묻힌다. 그래서 사진첩은 앨범 단위로 사진을 모으는 별도 면으로 두고, 글 작성·검색·고정 같은 기능은 빠져 있다. 앨범 자체는 멤버라면 누구나 만들 수 있게 열어 두어, 사진을 올리고 싶은 사용자가 곧장 시작할 수 있다.

2026-06 개선 — 이미지 업로드 확정 처리 안정화

사진 업로드가 끝까지 확정 처리되어야 앨범에 정상 등록된다. 업로드가 중간에 끊겨 확정 처리가 이루어지지 않으면 해당 사진은 24시간 후 자동으로 사라진다. 앱은 업로드 완료 후 확정 처리까지 자동으로 수행하도록 개선되어, 이전보다 사진이 나중에 누락되는 사례가 크게 줄었다.

사용자 시나리오

기본 흐름

모임이 끝난 멤버가 새 앨범을 만들고 사진을 올린다

사진첩 화면에서 + 버튼 → "새 앨범 만들기" → 제목 "2월 한강 러닝" → 만들기. 앨범이 생기면 그 자리에서 갤러리에서 5장을 골라 업로드. 사진들이 사용자 기기에서 클라우드로 차례로 올라가고 그리드에 다섯 장이 들어찬다. 첫 사진은 자동으로 앨범 커버가 된다.

한도

이미 50장 찬 앨범에 한 장 더 올리려 한다

업로드 시도 시 "앨범당 최대 50장까지 업로드 가능합니다" 안내가 뜬다. 새 앨범을 만들어서 거기에 올리거나, 기존 앨범의 사진을 정리해야 한다.

일괄 삭제

본인이 올린 흐릿한 사진 세 장을 한꺼번에 지운다

앨범 상세에서 "선택" 모드로 들어가 본인 사진 세 장을 체크하고 일괄 삭제. 그 자리에서 사진들이 사라지고 앨범의 사진 카운트가 셋만큼 줄어든다.

권한 가드

다른 멤버의 사진을 포함해 일괄 삭제하려 한다

일반 멤버가 본인 사진 두 장 + 다른 멤버 사진 한 장을 함께 선택해 삭제하려 하면 한 장이라도 본인 사진이 아닌 게 섞이면 동작 전체가 거절된다. 다른 사람의 사진은 그대로 남고 본인 사진도 지워지지 않는다.

이 흐름이 남기는 것 — 부분 성공이 일어나면 사용자에게 "지운 줄 알았는데 일부 남았다"는 혼란이 생긴다. 모두 통과해야 동작이 일어나는 정책.

운영자 정리

관리자가 부적절한 앨범을 통째로 지운다

관리자가 앨범 더보기 → "앨범 삭제" → 확인. 앨범 안의 모든 사진이 같이 사라진다. 저장된 사진 파일도 함께 삭제된다.

업로드 실패

5장 업로드 중 3장째에서 네트워크 단절

2장은 이미 올라가 있고 3장째에서 실패하면 해당 사진이 빨간색으로 표시되고 "재시도" 버튼이 떠오른다. 사용자가 신호를 회복한 뒤 재시도를 누르면 실패한 사진만 다시 시도된다.

앨범 수정

앨범 작성자가 제목을 손본다

본인이 만든 앨범의 더보기에서 "앨범 수정" → 제목·설명만 변경. 커버 사진은 이 흐름에서는 바꿀 수 없다.

진입 가드

비멤버가 사진첩에 진입하려 한다

비멤버에게는 클럽 커뮤니티의 "사진첩" 탭 자체가 보이지 않는다. 외부 링크로 사진첩 화면에 들어와도 서버가 거절한다.

운영자가 알아야 할 것

사진첩 운영 정책
  • 한 번에 10장, 앨범당 50장 한도는 화면 사용성과 운영 비용의 균형점이다. 풀면 한 사용자가 한 앨범에 수백 장을 쌓아 다른 앨범이 묻히는 흐름이 만들어진다.
  • 일괄 삭제의 권한 검사는 한 장이라도 다른 사람 것이 섞이면 전체가 거절되도록 작동한다. 부분 성공은 사용자 혼란의 원인. 운영자가 강제로 정리하려면 앨범 통째 삭제 권한을 사용한다.
  • 업로드 확정 처리가 완료되지 않은 사진은 24시간 후 자동으로 사라진다. 앱이 업로드 완료 후 확정 처리를 자동으로 수행하도록 개선되어 이 사례가 크게 줄었다. 그러나 네트워크가 확정 처리 직전에 끊기면 여전히 미완료 상태가 될 수 있다. 업로드 직후 사진이 보이지 않는다는 문의가 들어오면, 사용자에게 해당 사진을 다시 올리도록 안내한다.
  • 비멤버 진입 가드는 라우터·화면·서버 세 곳에서 작동한다. 사진은 멤버 사이의 사적 콘텐츠이므로 비멤버에게 노출되면 안 된다.
