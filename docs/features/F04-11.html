F04-11. 클럽 사진첩

앨범 단위로 사진을 모은다. 새 앨범 생성, 사진 일괄 업로드/삭제, ADMIN의 앨범 통째 삭제까지 책임진다. 앨범당 50장 한도, 본인 사진은 본인이, ADMIN은 누구든 삭제 가능.

하위 동작
4
시나리오
6
한도
50장/앨범

이 기능 안의 하위 항목

동작 1

앨범 생성

제목·설명. 첫 사진의 fileKey가 자동 커버.

동작 2

사진 업로드

외부 스토리지 PUT → DB 등록. 한도 50장.

동작 3

일괄 삭제

본인 사진만 또는 ADMIN/OWNER.

동작 4

앨범 삭제(ADMIN)

앨범 + 사진 메타 + 외부 객체 정리.

사용자 시나리오 6개

S1

앨범 생성 + 첫 5장

  1. 앨범 INSERT → 사진 5장 PUT/등록
  2. 첫 사진이 커버
S2

50장 초과 시도

  1. 외부 PUT 성공 후 등록에서 PHOTO_ALBUM_LIMIT_EXCEEDED
  2. orphan 가능 — 사전 검사 권장
S3

본인 사진 일괄 삭제

  1. 다중 선택 → batch-delete
  2. 외부 + DB 일괄 삭제 + photoCount-=N
S4

비ADMIN의 타인 사진 포함 삭제

  1. 전체 트랜잭션 롤백 → 변경 없음
S5

ADMIN의 앨범 통째 삭제

  1. 외부 객체 best-effort + DB DELETE
S6

비멤버 접근

  1. 탭 자체가 보이지 않음, 우회 시 401/403

서버 계약

MethodPath

POST/api/v1/clubs/{id}/photo-albums/{albumId}/photos
POST/api/v1/clubs/{id}/photos/batch-delete
DELETE/api/v1/clubs/{id}/photo-albums/{albumId}
