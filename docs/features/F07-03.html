F07-03. 정산 활성화 · 취소

DRAFT 정산을 ACTIVE로 전환하면 참가자 푸시 + transfer 자동 생성이 시작된다. 활성화 후에도 호스트는 정산 전체 취소가 가능(미완료 transfer 환불).

이 기능 안의 하위 항목

동작 1

활성화

DRAFT → ACTIVE + transfer 자동 생성 + 푸시 fanout.

동작 2

취소

완료된 transfer 환불, status=CANCELLED.

S2
