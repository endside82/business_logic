F17-01. 정기모임 발견·탐색

사용자가 발행된 정기모임을 찾는 두 면 — 전체 목록과 내 모임. 전체 목록은 지금 모집 중인 모임만 노출해 사용자가 "지금 들어갈 수 있는 코스가 있는지"를 빠르게 가늠하게 한다. 내 모임은 호스팅 중인 모임과 등록한 모임을 한 번에 모아 종료된 모임까지 보여 줘 이력 확인에 쓴다. 일반 이벤트 카드와 시각적으로 구분되는 뱃지가 붙어, 사용자가 "이건 코스로 가입하는 모임"임을 카드 단계에서 인지한다.

설계 의도 — 코스 가입은 일반 신청과 다르다, 카드 단계에서 알려야 한다

정기모임은 한 번 가입하면 여러 회차에 자동으로 묶이고, 결제·환불이 코스 단위로 일어난다. 일반 이벤트와 똑같이 보이면 사용자가 "회차 하나 신청한다" 정도로 가볍게 가입했다가 환불·결제 시점에서 충돌이 생긴다. 카드 단계에서 고정형/변동형 뱃지로 운영 방식을 미리 알리고, 카테고리·지역 필터로 자기 성향에 맞는 모임만 좁힐 수 있게 한다.

사용자 시나리오

기본 흐름

"러닝"으로 주말 코스 찾기

사용자가 키워드를 입력하고 카테고리·운영방식 칩으로 결과를 좁힌다. 카드를 보고 "이건 8주 코스구나"를 뱃지로 인지한 뒤 상세로 진입해 회차 일정을 본다.

이 흐름이 남기는 것 — 정기모임은 일반 이벤트와 결제·환불 구조가 다르므로, 가입 결정 전에 운영 방식이 노출되어야 분쟁이 줄어든다.

가입 전

비로그인 사용자가 둘러봄

카드와 상세는 모두 보인다. 단, "참가 신청"을 누르는 순간 로그인 화면이 뜬다. 비로그인에서는 내 모임 탭은 숨겨진다.

이 흐름이 남기는 것 — "가입하고 둘러봐" 대신 "둘러보고 마음 들면 가입" 패턴 유지. 정기모임 카탈로그가 가입 전환 동선의 일부.

내 모임

호스팅 + 멤버 합산

내가 호스트인 모임과 멤버로 들어가 있는 모임이 한 화면에 모인다. 종료·취소된 모임도 같이 보여 "지난달 코스 정산은 어떻게 끝났더라" 같은 이력 확인이 한 진입으로 가능하다.

이 흐름이 남기는 것 — 사용자가 자기 코스 이력을 별도 페이지를 찾아 헤매지 않게 한다.

초안 보호

호스트만 자기 초안을 본다

발행 전 초안 모임은 일반 검색에 노출되지 않는다. 호스트 본인은 내 모임에서 자기 초안을 계속 볼 수 있다.

이 흐름이 남기는 것 — 호스트가 충분히 준비된 모임만 외부에 노출하도록 보호.

무한 스크롤

20장 보고 더 둘러봄

리스트 끝에 다다르면 다음 20장이 자연스럽게 이어진다. 끝까지 가면 추가 로딩 표시가 사라진다.

결과 없음

지금은 모집 중인 코스가 없음

필터에 맞는 모임이 0건이면 "조건에 맞는 정기모임이 없습니다" 안내. 카테고리 칩을 풀거나 키워드를 비우면 일반 목록으로 복귀한다.

이 흐름이 남기는 것 — 정기모임 자체가 적은 시기에도 "오류"로 인지되지 않게 한다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
운영 방식 뱃지카드 상단에 "고정형" 또는 "변동형" 뱃지가 색깔로 구분되어 붙는다.가입 결정 전에 "코스 일괄"인지 "회차별 신청"인지 알려야 환불·결제 시점에서 사용자 기대와 어긋나지 않는다.
호스트 닉네임카드에 호스트 닉네임이 같이 노출된다.정기모임은 호스트 신뢰가 가입 결정의 큰 요인이라, 카드 단계에서 누가 운영하는지 보이게 한다.
회차 수·기본 정원고정형은 "8회·정원 10명" 같은 규모 표기.코스 가입은 시간 약속이 길기 때문에, 사용자가 카드 단계에서 코스 규모를 가늠해 망설임을 줄인다.
가격고정형은 코스 전체 가격, 변동형은 "회차별 가격 변동"으로 표기.변동형에 코스 가격을 표기하면 사용자가 "총액"으로 오인하기 쉬워, 표기 자체를 다르게 한다.
내 모임 탭비로그인 시 숨김. 로그인 시 호스팅·등록을 합산한 카운트 뱃지.로그인 사용자에게만 의미 있는 면이라 비로그인에서는 진입점 자체를 노출하지 않는다.

운영자가 알아야 할 것

노출 정책
  • 전체 목록은 모집 중만 — 초안·종료·취소는 노출 대상 아님. 사용자가 클릭해도 "지금 들어갈 수 없는" 모임은 카드 단계에서 제외해 헛걸음을 줄인다.
  • 내 모임은 전 상태 포함 — 정산·환불 이력 확인용이라 종료·취소도 같이 보여 준다.
  • 일반 이벤트 검색에서도 정기모임 회차가 노출된다 — 정기모임 본체는 전용 검색이지만, 회차는 일반 이벤트 검색·홈·트렌딩에도 포함되어 검색 표면이 분리되어 있지 않다.
  • 호스트 닉네임 노출이 카드 성능을 좌우한다 — 닉네임 표기 누락 시 카드가 비어 보여 가입 결정이 떨어진다.

레드 플래그

자주 발생하는 케이스
  • 일반 이벤트 카드 영역에 뱃지 미통합 — 정기모임 회차가 일반 검색·홈·트렌딩에 일반 이벤트 카드 형태로 노출될 때 뱃지가 빠지면 사용자가 코스인 줄 모르고 신청한다.
  • 지역 요약이 너무 길면 카드 깨짐 — 주소 원문이 그대로 들어올 경우 잘림 처리 필요.
  • "가까운 순" 정렬은 1차 미지원 — 정기모임은 위치보다 일정 적합도가 더 중요하다는 판단으로 1차에서 빠짐.
