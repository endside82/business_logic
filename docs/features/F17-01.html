F17-01. 정기모임 발견·탐색

사용자가 발행된 정기모임을 찾는 두 화면 — 전체 목록과 내 모임. 전체 목록은 현재 모집 중인 모임만 노출하고, 카테고리·지역·운영방식(고정형/변동형)으로 필터한다. 내 모임은 호스팅 중인 모임과 내가 등록한 모임을 한 번에 모아 종료된 모임까지 보여준다.

설계 의도 — 일반 이벤트와 정기모임을 시각으로 구분

정기모임은 한 번 등록하면 여러 회차에 묶이기 때문에 일반 이벤트와 같은 카드처럼 보이면 사용자가 헷갈린다. 카드에 고정형/변동형 뱃지를 붙여 "이건 코스로 가입하는 것"임을 미리 알리고, 카테고리·운영방식 필터로 자기 성향에 맞는 모임만 좁힐 수 있게 한다.

두 화면이 보여주는 것

전체 목록

모집 중인 모임만

발행되어 모집을 받는 모임만 카드로 노출. 키워드·카테고리·운영방식·지역으로 좁힐 수 있다. 비로그인 상태로도 열람 가능. 발행 시각 최신순이 기본.

내 모임

호스팅 + 멤버 합산

내가 호스트인 모임과 멤버로 들어가 있는 모임을 한 화면에 모은다. 종료·취소된 모임도 포함해 이력을 놓치지 않게 한다.

뱃지

고정형·변동형 구분

카드 상단에 운영방식 뱃지가 붙는다. 사용자가 카드만 봐도 "코스 일괄 등록"인지 "회차별 신청"인지 즉시 알 수 있다.

초안 보호

호스트만 자기 초안을 본다

발행하지 않은 초안 모임은 일반 검색에 노출되지 않는다. 호스트 본인은 내 모임에서 자기 초안을 계속 볼 수 있다.

서버 계약

MethodPath사용자 의미
GET/api/v1/regular-meetings전체 목록 — 모집 중인 모임만, 키워드·카테고리·운영방식·지역 필터
GET/api/v1/regular-meetings/my내 모임 — 호스팅 + 멤버 합산(상태 무관)

응답에는 호스트 닉네임이 한 번에 담겨 카드 표기에 추가 호출이 필요 없다. 페이지 단위로 잘려 내려오며, 무한 스크롤로 다음 페이지를 요청한다.

운영자가 알아야 할 것

노출 규칙
  • 전체 목록은 모집 중인 모임만 — 초안·종료·취소는 노출 대상 아님.
  • 내 모임은 종료·취소 포함 — 정산·환불 이력 확인용이라 전 상태 포함.
  • 일반 이벤트 검색에서 정기모임 회차도 자동 분류되어 노출 정책이 따로 적용된다 — 도메인 17에서 통합 관리.
  • 호스트 닉네임은 한 번에 묶어서 가져온다(N+1 회피) — 페이지가 커도 카드 표기 성능 안정.

레드 플래그

자주 발생하는 케이스
  • 홈·트렌딩·관심인 알림에 정기모임 카드 미통합 — 일반 이벤트 카드 호출처가 십수 개라 점진적으로만 뱃지가 적용된다(잔여).
  • 지역 요약 텍스트가 너무 길면 카드 깨짐 — 주소 원문이 그대로 들어올 경우 잘림 처리 필요.
