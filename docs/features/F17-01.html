F17-01. 정기모임 발견·탐색

호스트가 발행(OPEN)한 정기모임을 사용자가 찾는 진입점. 검색·내 모임 두 화면. OPEN 상태 모임만 노출되고, 카테고리·지역·타입(FIXED/VARIABLE)으로 필터한다. 일반 이벤트 카드와 시각적으로 구분되는 FIXED/VARIABLE 뱃지가 붙는다.

전체 흐름

flowchart TD A["목록 진입"] --> B["GET /regular-meetings?keyword=&category=&meetingType="] B --> C["서버 search()"] C --> D["RegularMeetingQueryRepository
QueryDSL: status=OPEN 강제"] D --> E["호스트 닉네임 일괄 조회"] E --> F["Page<RegularMeetingSimpleVo>"] F --> G["Flutter 카드 + RegularMeetingBadge"] G -->|"카드 탭"| H["F17-02 상세"] A2["내 모임 탭"] --> I["GET /regular-meetings/my"] I --> J["호스팅 + 멤버 합산"]

서버 계약

  • GET /api/v1/regular-meetings — optional auth, RegularMeetingSearchParam extends PagingParam (keyword, category, meetingType, region, page, size) → Page<RegularMeetingSimpleVo>. status=OPEN 강제, publishedAt DESC 기본.
  • GET /api/v1/regular-meetings/my — 인증 필수, PagingParam → Page<RegularMeetingSimpleVo>. 호스트(hostUserId=userId) + 멤버(seat-occupying) 합산.
