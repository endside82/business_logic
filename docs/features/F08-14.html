F08-14. 플랜 마켓 환불

마켓 아이템·번들·플랜 직접 구매분에 대한 환불 흐름. 구매자가 신청하고 판매자가 승인/거절하며, 판매자 무응답에는 자동 처리가 붙는다. 사용 흔적이 있는 구매는 자동 환불을 막고 운영자 검토로 넘기며, 거절에 동의 못 하는 구매자는 분쟁을 열어 운영자 중재로 갈 수 있다.

3
구매자·판매자·운영자
자동 처리
7일
무응답 + 사용 흔적 없음 → 자동 승인
분쟁 기한
7일
거절 후 1회 분쟁
설계 의도 — 판매자 무응답이 구매자를 인질로 잡지 않는다

플랜 마켓은 1인 크리에이터 판매자가 많아 무응답·지연 응답이 잦다. 그래서 판매자 결정만 기다리지 않고 7일 자동 처리를 깐다 — 사용 흔적이 없으면 자동 승인(환불), 사용 흔적이 있으면 운영자 검토로 올라간다. 판매자가 추가 자료를 요청해 타이머를 한 번 멈출 수 있는 안전장치도 있다.

세 결정자가 만드는 흐름

구매자

보유함에서 신청

사유 분류(품질·설명 불일치·실수·하자·기타) + 메모. 신청 즉시 카드는 잠긴다(사용·다운로드·이벤트 연결 불가). 거절 후 24시간 지나면 1회 추가 신청 가능.

판매자

대시보드에서 결정

구매자 정보·사유·결제 금액·사용 흔적(다운로드·이벤트 연결 등)을 보고 승인/거절/추가 자료 요청 중 선택. 추가 자료 요청은 1회만 가능하며 7일 타이머를 멈춘다.

운영자

분쟁 중재 + 자동 에스컬레이션 검토

구매자가 거절에 분쟁을 걸면 운영자 큐로 진입. 거절 유지·강제 환불·추가 자료 요청 중 선택. 사용 흔적이 있는 7일 무응답 건도 운영자 검토로 자동 올라간다.

자동 처리

판매자 7일 무응답

사용 흔적 없으면 자동 환불 승인 → 구매자에게 즉시 환불. 사용 흔적이 있으면 운영자 검토 큐로 이관.

운영자가 알아야 할 것

사용 흔적·자동 처리·재신청
  • "사용 흔적"의 정의 — 다운로드 횟수, 이벤트 연결, 활성화 시각. 이 중 하나라도 있으면 자동 환불 불가, 운영자 검토.
  • 거절 후 24시간 1회 재신청 — 같은 구매에 무한 신청을 막기 위함.
  • 분쟁은 거절로부터 7일 이내 1회 — 단발성. 분쟁이 끝나면 결과 확정.
  • 운영자 강제 환불은 별도 라벨 — 정산·노출 패널티 누적에 영향(별도 정책).
  • 구매 잠금 동안 사용 불가 — 환불 신청 즉시 카드는 잠기고, 결정이 날 때까지 사용·다운로드 막힘.

레드 플래그

자주 발생하는 케이스
  • 지갑 환불의 유료/무료 분리 미보존 — 현재는 환불 시 전액이 유료 포인트로 복원된다(원결제의 유료/무료 비율 보존 후속). 이게 회계 분개와 충돌하지는 않지만, 후속 일정에 잡혀 있다.
  • 매출 귀속(F08-15)과 의존 — 환불 회계는 크리에이터 매출 원장을 잠그거나 역전시키므로 F08-15가 정확해야 환불도 정확하다.
  • 추가 자료 요청 후 기한 만료 — 사용자가 응답 안 하면 환불 신청이 만료된다(거절 아님). 사용자에게 명확히 안내 필요.
  • 판매자가 잠수타면 7일 동안 카드 잠금 — 사용자 입장에서 답답할 수 있어, 신청 시 "최대 7일 소요" 안내 필요.
