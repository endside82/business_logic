F08-14. 플랜 마켓 환불 (Purchase Refund)

마켓 아이템·번들·플랜 직접 구매분에 대해 구매자가 환불을 신청하고, 판매자(크리에이터)가 승인 또는 거절한다. 판매자 무응답에 대비한 자동 처리, 사용 흔적 기반 차단, 거절에 대한 운영자 중재까지 본 PRD가 다룬다. API는 이미 구현됨 (PurchaseRefundController / PurchaseRefundService).

followup — 지갑 환불 split 미보존

현재 환불 회계 분개에는 paid/free split이 들어가지만, 지갑 환불 자체는 walletService.refundToWallet로 전액 paid 복원된다. 원결제의 유료/무료 split 보존을 위해서는 refundByTransaction 경로로 전환해야 한다. EVENT 환불 followup과 함께 처리.

환불 흐름

  • 구매자 환불 신청 → PurchaseRefundService 진입
  • 판매자 승인/거절
  • 판매자 무응답 시 자동 처리 (스케줄러)
  • 사용 흔적 기반 환불 차단 (이미 사용한 플랜은 환불 거절)
  • 판매자 거절에 대해 구매자가 분쟁 진입 → 운영자 중재
  • 회계 분개: paid/free split 기록 (PointTransaction + AccountingLedger)
