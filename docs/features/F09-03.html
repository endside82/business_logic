F09-03. 후보 카드 스와이프 · 매칭 액션

데이팅 도메인에서 가장 중요한 화면. 사용자에게 한 사람씩 카드를 보여 주고, ♥(호감) 또는 ✕(통과)를 누르면 상대가 이미 표시한 의사와 비교해 매칭 여부를 즉시 결정한다. 양쪽 모두 호감이면 곧장 채팅이 열리고, 한쪽만 호감이면 7일 동안 상대의 응답을 기다린다. 안전 정책(차단·인증), 매칭 알고리즘, 그리고 첫 관계 형성이 모두 한 동작 안에서 결정되는 단위다.

♥ 한 번이 누르는 동안 서비스는 (1) 본인인증 가드, (2) 양방향 차단 검사, (3) 같은 사람과의 기존 매칭 확인, (4) 양쪽 의사 합치, (5) 매칭 성사 시 채팅방까지를 하나의 분리 불가능한 동작으로 처리한다. 이 다섯 중 어느 하나라도 따로 처리되거나 누락되면, 사용자에게는 "매칭됐다는데 채팅이 없네", "차단했는데 호감이 통과되네" 같은 안전·신뢰의 균열로 즉시 나타난다. 데이팅은 다른 어떤 도메인보다 이 균열이 치명적인 영역이다.

이 단위가 도메인 잔류·이탈을 결정한다

가입·인증·프로필을 모두 통과한 사용자가 머무를지 떠날지는 이 화면의 경험으로 결정된다
  • 첫 매칭까지의 시간 — 가입 후 첫 매칭이 만들어지기까지의 시간이 길수록 이탈이 늘어난다. 그래서 본 단위는 카드 노출 → 호감 → 매칭 성사 → 채팅 진입을 한 동작 안에서 끝낼 수 있게 설계되어 있다.
  • 안전 가드의 즉시 반영 — 차단·인증 같은 안전 장치는 본 화면 안에서 바로 동작해야 한다. "차단했는데 같은 사람이 또 보인다", "이미 매칭됐는데 호감이 통과된다" 같은 오작동이 한 번이라도 발생하면 사용자가 서비스를 신뢰하지 못한다.
  • 카드 노출의 공정성 — 한 사람이 계속 상위에 노출되거나 반대로 거의 노출되지 않는 편향이 누적되지 않도록, 사용자별 노출 횟수를 점수에 반영한다. 이 보정이 빠지면 매력적인 사용자에게만 신청이 몰리고 다른 사용자는 빠르게 이탈한다.

한 번의 ♥ 탭이 통과하는 다섯 관문

차단·중복 매칭·인증 같은 안전 검사가 가장 먼저 작동하고, 매칭 결정과 채팅 개설은 가장 마지막에 한 동작으로 이루어진다.

관계의 라이프사이클

한 쌍의 사람들 사이에 만들어지는 "매칭"은 다섯 상태를 거친다. 한 쌍당 매칭은 한 개만 존재한다.

매칭 성사의 결정적 순간

상대가 이미 나에게 호감을 표시해 둔 상태에서 내가 호감을 누르는 순간. 본 서비스의 가장 가치 있는 1초가 어떻게 한 동작으로 처리되는지.

sequenceDiagram actor 나 participant 카드 as 카드 화면 participant 서비스 as 매칭 서비스 나->>카드: ♥ 호감 탭 카드->>서비스: 매칭 의사 전달 서비스->>서비스: 본인인증 / 차단 / 중복 매칭 확인 서비스->>서비스: 기존 보류 매칭 발견(상대가 먼저 호감) 서비스->>서비스: 내 의사 호감으로 갱신 → 양쪽 호감 충족 서비스->>서비스: 매칭 성사 + 채팅방 동시 개설(분리 불가) 서비스-->>카드: 매칭 성공, 채팅방 식별자 함께 전달 카드-->>나: "매칭됐어요" 안내 + "채팅 시작" 버튼 나->>카드: 채팅 시작 탭 카드-->>나: 새로 열린 채팅방으로 진입

사용자 시나리오

결정적 순간

호감 → 상대도 호감 → 즉시 채팅 시작

활성 프로필을 가진 사용자가 카드를 보고 호감을 누른다. 상대가 이미 나에게 호감을 표시해 둔 상태였기 때문에 그 자리에서 매칭이 성사되고 "매칭됐어요" 안내가 떠오른다. "채팅 시작" 버튼을 누르면 방금 열린 채팅방으로 곧장 들어간다.

이 흐름이 남기는 것 — 매칭 성사부터 채팅 첫 진입까지의 시간이 짧을수록 사용자 만족과 재방문이 늘어난다. 본 화면은 그 시간을 가능한 한 짧게 만드는 것이 목표다.

기다리는 시간

호감을 보냈지만 상대가 아직 답이 없다

처음 마주친 사람에게 호감을 누른다. 매칭은 보류 상태로 7일간 살아 있고, 사용자에게는 다음 카드가 곧장 보인다. 매칭 목록의 "전체" 탭에 보류 카드와 남은 시간 안내가 표시된다.

이 흐름이 남기는 것 — "기다리는 동안에도 다른 카드를 계속 보게 한다"는 게 본 단위의 핵심 정책. 한 사람의 응답을 기다리느라 흐름이 멈추지 않게 한다.

관계 정리

양쪽 모두 통과를 선택

상대가 이미 통과한 카드에 나도 통과를 누른다. 매칭은 종결로 정리되고 같은 쌍이 다시 카드로 노출되지 않는다.

이 흐름이 남기는 것 — "관심 없음"의 의사 표시가 명확히 기록되어, 사용자가 같은 카드를 반복해서 보는 피로를 막는다.

안전 가드

차단한 사람의 카드가 어쩌다 보였다

호감을 누르는 순간 양방향 차단 검사가 작동한다. 매칭은 만들어지지 않고 "차단 관계가 있는 사용자입니다" 같은 안내만 사용자에게 표시된다.

이 흐름이 남기는 것 — 한 번의 차단이 그 사용자와의 모든 후속 매칭 시도를 막는다는 정책. 이 가드가 한 번이라도 풀리면 사용자가 차단 기능 자체를 신뢰하지 않게 된다.

안전 가드

이미 매칭된 사람에게 또 호감을 누름

이미 매칭이 성사된 상대에게 호감을 다시 누르면 거절된다. 사용자에게는 "이미 매칭된 사용자입니다" 안내가 뜨고, 자연스럽게 채팅방으로 안내된다.

이 흐름이 남기는 것 — 매칭 통계가 부풀어 오르는 것을 막고, 채팅방이 중복으로 열리는 사고를 차단.

진입 가드

본인인증/프로필 없이 카드를 보려 함

본인인증을 마치지 않은 사용자가 카드 화면에 진입하면 카드 대신 인증 안내가 뜬다. 프로필이 없는 사용자에게는 "프로필 만들기" 카드가 보인다.

이 흐름이 남기는 것 — 안전과 진정성을 매칭 액션이 시작되기 전에 강제하는 정책의 첫 발현 지점.

즉시 이탈

카드에서 곧장 차단

카드 우상단 더보기 → 차단을 누르면 사유 입력 시트가 떠오른다. 차단을 확정하면 그 사용자와의 매칭은 즉시 차단 상태로 바뀌고, 만약 이미 매칭된 사이였다면 채팅방도 함께 닫힌다. 해당 사용자는 카드 풀에서 영구적으로 제외된다(차단 해제 전까지).

이 흐름이 남기는 것 — 데이팅의 가장 중요한 안전 경로. "뭐가 이상하다 → 한 동작으로 즉시 끝낼 수 있다"는 통제감이 사용자 신뢰의 핵심.

간접 안전

신고는 본 화면에서 직접 받지 않는다

본 화면에는 신고 버튼이 없다. 사용자는 차단으로 즉각 단절하고, 운영팀 신고는 별도 화면(11 리뷰·신고 도메인)에서 수행한다.

이 흐름이 남기는 것 — 카드 한 장 안에 가드가 너무 많으면 결정의 속도가 깨진다. 차단을 1차 즉시 안전장치로, 신고를 2차 정식 경로로 분리한 정책.

관찰 포인트

같은 카드에 더블탭으로 호감 두 번

흥분한 사용자가 같은 카드에 호감을 두 번 빠르게 누르는 경우. 첫 번째 호감이 매칭을 만들고, 두 번째 호감은 이미 매칭된 사람에 대한 시도가 되어 거절된다. 다만 클라이언트가 첫 응답 전에 이미 다음 카드로 인덱스를 옮겨 둔 상태라면 두 번째 응답이 다음 카드 사용자에 영향을 줄 위험이 있다.

운영 관찰 포인트 — 클라이언트 측 더블탭 보호(첫 응답이 올 때까지 추가 탭 차단)가 필요한지 검증.

화면에서 사용자가 인지하는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
한 장 카드 화면 가득한 사진 1장, 하단에 표시명·나이·지역·자기소개. 우상단에 더보기 메뉴. 한 번에 한 사람만 보여 줘 사용자가 그 사람에 대해 판단하게 한다. 여러 장을 동시에 보여 주면 빠르게 넘기는 모드로 들어가 매칭 품질이 떨어진다.
두 개의 큰 버튼 하단에 ✕(통과), ♥(호감) 두 버튼. 엄지로 한 번에 누를 수 있는 크기. 화면 가득한 카드 아래 두 개의 버튼만 두어, 사용자가 "지금 이 사람에 대해" 결정한다는 사실을 분명히 한다.
매칭 성공 안내 "매칭됐어요" 큰 글씨 + "{이름}님과 매칭됐어요" + "계속 둘러보기" / "채팅 시작" 두 버튼. 매칭 직후 사용자의 다음 행동을 사용자가 선택하게 한다 — 강제로 채팅에 밀어 넣지 않는다. 흐름의 통제권은 사용자에게.
차단 사유 입력 바텀시트에 자유 입력 칸. 입력을 취소하면 차단도 취소. 사유 없는 차단은 운영에서 패턴을 학습하기 어렵다. 동시에 사용자가 "그냥 차단"하고 싶을 때를 막지도 않는다 — 취소가 가능한 형태.
모든 카드 소진 안내 "오늘은 새 카드가 없어요" 같은 안내 + 새로고침 동작 안내. 카드가 다 떨어진 것을 "오류"로 인지하지 않게. 사용자에게는 자연스러운 경험의 끝맺음.

측정해야 하는 것

본 단위가 정상이라고 부르려면 이 다섯 가지 지표가 추적되어야 한다
  • 호감 → 매칭 전환율 — 호감 한 번이 매칭으로 이어지는 비율. 너무 낮으면 사용자가 호감 행동 자체에 의미를 잃는다.
  • 가입 → 첫 매칭까지의 시간 분포 — 신규 사용자의 이탈을 가장 잘 예측하는 신호.
  • 매칭 → 채팅 첫 메시지까지의 시간 — 매칭 성사 다이얼로그 → 실제 채팅 진입 → 첫 메시지의 시간 차. 이 시간이 길면 매칭의 가치가 다음 단계로 이어지지 않고 사라진다.
  • 한쪽 호감의 7일 만료 비율 — 너무 높으면 카드 노출 알고리즘이 사용자 관심과 어긋난다는 신호.
  • 차단 발동 빈도와 사유 분포 — 사용자 안전 환경의 핵심 지표. 특정 사유가 증가하면 카드 제출 단계의 검수 강화가 필요한 신호.

운영자가 알아야 할 것 — 비즈니스 리스크

본 단위에서 무너지면 데이팅 도메인 전체가 무너진다
  • 매칭 성사와 채팅방 개설은 반드시 같은 동작으로 처리되어야 한다. "매칭됐는데 채팅이 없다"는 사고가 한 번이라도 발생하면 해당 사용자는 곧장 서비스를 이탈한다.
  • 양방향 차단 검사 누락 — 내가 차단한 쪽뿐 아니라 나를 차단한 쪽까지 양쪽을 모두 검사해야 한다. 한 방향만 검사하면 차단 자체의 의미가 무너진다.
  • 이미 매칭된 사람에게 호감이 통과되는 회귀 — 매칭 통계 부풀림, 채팅방 중복 생성, "왜 한 사람과 두 번 매칭되지" 라는 혼란. 이 가드는 어떤 경우에도 풀려서는 안 된다.
  • 한쪽 호감의 7일 만료가 동작하지 않으면 매칭 목록의 "전체" 탭에 보류 카드가 영원히 누적된다. 사용자가 한 번 "기다림"의 경험을 잃으면 다음 호감을 누르지 않게 된다.
  • 노출 카운터가 부정확하면 같은 사용자가 영원히 1위로 노출되거나, 영원히 매칭 후보에서 빠지는 편향이 누적된다. 양쪽 끝에서 동시에 이탈이 발생.
  • 매칭 성공 안내에서 채팅 진입 식별자가 누락되는 회귀 — 매칭은 됐는데 어느 채팅방으로 가야 할지 클라이언트가 모르는 상태. 이 안내 흐름의 끝까지 채팅방 정보가 일관되게 전달되어야 한다.
  • 더블탭 보호가 없으면 한 사용자가 빠르게 두 번 누른 호감의 두 번째가 다음 카드 사용자에게 닿는 사고가 일어날 수 있다. 첫 응답이 올 때까지 추가 동작은 차단하는 것이 바람직.
