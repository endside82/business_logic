F09-02. 데이팅 프로필 관리

본인 인증을 마친 사용자가 매칭 대상이 되기 위해 작성하는 자기 표현 단위. 표시 이름·생년·성별·자기소개·키·직업·지역·선호 조건과 사진 여러 장을 모아 한 사람의 데이팅 프로필을 만든다. 작성된 프로필은 매칭 풀에 노출되어 다른 사용자의 후보 카드에 등장하고, 본인이 원할 때 노출 토글 하나로 매칭을 잠시 멈출 수도 있다. 여기서 사용자가 자신을 어떻게 보여주는지가 매칭 품질의 절반을 결정한다.

필수 사진
3장 이상
한 장 이하 프로필은 매칭 대상이 되지 않는다
사진 최대
6장
6장을 넘기는 자기 노출은 막는다
자기소개
300자
짧게 쓰되 안 비도록
공개 범위
3단계

설계 의도 — "처음 진입한 사용자가 곧장 사용 가능한 프로필을 만들 수 있어야 한다"

데이팅에서 프로필이 비어 있으면 매칭이 일어나지 않는다. 반대로 프로필을 만들기까지 너무 많은 단계를 요구하면 인증을 통과한 사용자도 곧장 이탈한다. 그래서 본 단위는 표시 이름·생년·성별·자기소개와 사진 3장 — 한 사람을 식별할 수 있는 최소한의 정보 — 만 필수로 두고, 나머지(키·직업·지역·선호 조건)는 비워 둔 채로도 프로필이 활성화되도록 설계되어 있다. "활성/비활성 토글"을 따로 둔 것은 출장·휴식·관계 형성기처럼 사용자가 잠시 매칭을 멈추고 싶을 때 프로필을 삭제하지 않고도 노출만 끌 수 있게 하기 위한 장치다.

이 단위가 만드는 세 가지 결과

한 사람이 프로필을 작성하면 데이팅 도메인 안에서 이 세 가지가 동시에 자리 잡는다.

매칭 풀 진입

다른 사용자의 후보 카드에 등장

프로필이 활성화되어 있는 동안 본인은 다른 사용자의 후보 카드에 후보로 들어간다. 사진·자기소개·기본 정보가 그대로 카드에 보여 매칭 결정의 단서가 된다. 비활성으로 돌리면 노출이 즉시 멈춘다 — 데이터는 그대로 살아 있다.

자기 표현 통제

공개 범위로 노출 깊이 조절

프로필을 어디까지 보여 줄지 사용자가 선택한다 — 모두에게 / 매칭된 상대에게만 / 아무에게도. "비공개"를 선택하면 후보 카드에서 빠진다. "매칭된 상대에게만"은 사진 일부만 미리 보여주고 매칭이 성사된 후에 전체를 풀어 주는 정책의 자리.

잠시 멈춤

출장·휴식 시 노출 토글

한 번의 스위치로 매칭 노출을 끄거나 켤 수 있다. 비활성 상태에서는 다른 사용자에게 본인이 더 이상 카드로 뜨지 않고, 매칭 목록에는 "비활성 중" 안내가 자연스럽게 노출된다. 데이터를 지우지 않고도 흐름을 잠시 멈출 수 있는 안전 경로.

프로필이 활성화되기까지의 길

인증을 마친 사용자가 매칭 대상이 되는 데까지 통과해야 하는 자리.

사용자 시나리오

기본 흐름

인증 직후 프로필 만들기

본인 인증을 막 마친 사용자가 데이팅 탭으로 돌아오면 "프로필을 만들어 주세요" 흐름으로 안내된다. 표시 이름·생년·성별·자기소개·사진 3장을 채우면 곧장 프로필이 만들어지고, 매칭 풀에 노출이 시작된다. 나머지 정보(키·직업·지역·선호 조건)는 그 자리에서 비워 두고 나중에 채워도 된다.

이 흐름이 남기는 것 — 인증부터 첫 매칭 노출까지의 시간을 가장 짧게 한다. 이 구간이 길어지면 인증을 통과한 사용자가 매칭을 보기 전에 이탈한다.

잠시 멈춤

출장 가는 동안 매칭 멈추기

일주일 출장으로 데이팅을 잠깐 멈추고 싶은 사용자가 프로필 화면의 활성 토글을 끈다. "프로필이 비활성화되었습니다" 토스트가 뜨고, 그 순간부터 본인이 다른 사용자의 카드에 더 이상 등장하지 않는다. 매칭 목록 화면에는 "프로필이 비활성화 상태입니다…" 라는 띠가 한 줄 자리한다. 출장에서 돌아와 다시 토글을 켜면 노출이 즉시 재개된다.

이 흐름이 남기는 것 — "쉬고 싶을 때 삭제 말고 토글" — 한 번 작성한 프로필을 사용자가 보존하면서 매칭 흐름만 통제할 수 있는 경로.

편집과 변심

편집하다가 마음이 바뀌어 뒤로가기

자기소개 일부를 고치다가 마음이 바뀐 사용자가 뒤로가기를 누른다. 앱은 "변경 사항을 저장하지 않고 나가시겠습니까?" 다이얼로그로 한 번 더 확인을 받는다. "나가기"를 누르면 변경 내용은 버려지고 화면이 닫힌다. "취소"를 누르면 편집 화면에 그대로 남는다.

이 흐름이 남기는 것 — 사용자의 의도하지 않은 변경 손실을 막는다. 자기 표현 항목은 사용자가 신중하게 정한 카피이기 때문에, 실수로 변경되거나 사라지면 다시 작성하기 어렵다.

사진 상한

6장을 채운 사용자가 7장째를 시도

이미 사진 6장을 등록한 사용자가 한 장을 더 올리려고 한다. 앱은 "사진은 최대 6장까지 등록 가능합니다" 안내로 추가를 막는다. 기존 사진 중 하나를 지워야 새 사진을 올릴 수 있다.

이 흐름이 남기는 것 — 너무 많은 사진은 결정 피로만 키운다. 6장이라는 상한이 자기 표현의 폭을 보장하면서 의사결정을 가볍게 유지하는 균형점.

공개 범위 선택

"매칭된 상대만"으로 바꾸기

모든 사용자에게 노출되는 게 부담스러운 사용자가 공개 범위를 "매칭된 상대만"으로 바꾼다. 저장 직후부터 후보 카드에서는 일부 정보만 보이고, 매칭이 성사된 후에 비로소 프로필 전체가 풀린다. "비공개"를 선택하면 아예 후보 카드에서 빠진다 — 활성/비활성 토글과 비슷한 효과지만 의미가 다르다(공개 범위는 의도, 토글은 시간).

운영 관찰 포인트 — "비공개"가 활성 토글과 사실상 같은 효과를 내므로, 두 옵션의 차이를 사용자가 인지하기 어려울 수 있다. 사용자 혼란 신호가 들어오면 정리할 여지.

가드 작동

인증을 안 한 사용자가 프로필 생성 시도

어떤 경로로든 인증을 통과하지 않은 사용자가 프로필 생성을 시도한다. 서버가 즉시 거절하고, 클라이언트는 본인 인증 화면으로 안내한다. 본 단위는 인증 가드를 자체적으로 한 번 더 확인한다 — 화면 진입 가드만 믿지 않는다.

이 흐름이 남기는 것 — 인증·프로필·매칭의 가드가 한 자리에서만 작동하지 않고 여러 단계에서 중복으로 검사된다. 한 곳이 풀려도 다음 단계가 잡는다.

화면에서 사용자가 인지하는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
활성 토글 카드 상단에 가장 먼저 보이는 큰 스위치. 활성일 때는 초록 카드 + "매칭 추천에 노출됩니다", 비활성일 때는 노란 카드 + "매칭을 받으려면 활성화하세요". 매칭 노출 여부는 사용자가 가장 빠르게 통제하고 싶은 항목이다. 프로필 본문보다 위에 둬서 한 번에 인지·전환 가능하게.
사진 그리드 2×3 정사각 슬롯 "사진 (n/6)" 표기. 빈 칸에는 + 아이콘. 최대 장수와 현재 장수가 한눈에 보인다. 6장이라는 상한과 본인 자기 표현의 여유를 동시에 인지하게.
기본 정보 카드 나이("{생년}년생"), 성별("남성"/"여성"), 키(cm), 직업 — 4가지를 깔끔히 묶어 표시. 매칭 의사결정의 가장 큰 단서가 한 카드 안에 모인다. 사용자가 자기 프로필을 검토할 때도 "내가 어떻게 보일지"를 그대로 본다.
자기소개와 카운터 자기소개 본문 + 오른쪽 끝에 "{현재}/300" 회색 카운터. 너무 짧거나 너무 길지 않게 사용자가 스스로 가늠하게. 카운터는 강요가 아니라 안내 — 미달이어도 저장은 가능하다.
공개 범위 세 옵션 "전체 공개 / 매칭된 상대만 / 비공개" 세 개의 라디오 카드. 각 옵션에 설명문 한 줄. 옵션의 의미를 사용자가 텍스트로 확인하고 선택하게. "이걸 누르면 무슨 일이 벌어지는지"를 카드 안에서 모두 보여 준다.
차단 목록 진입 본문 하단에 한 줄짜리 "차단 목록" ListTile. 차단 해제는 자주 쓰는 동작이 아니지만 필요할 때 어디서 찾아야 할지 명확해야 한다. 프로필 화면 안에서 본인의 데이팅 관련 설정을 한 자리에 모은다.
편집 시 저장 버튼 변경이 있을 때만 분홍색으로 활성화. 변경이 없으면 회색으로 비활성. "내가 바꾼 게 있다"는 신호를 사용자에게 시각적으로 돌려준다. 의도 없이 누르는 저장을 막는다.

측정해야 하는 것

본 단위의 건강 신호
  • 인증 완료 → 프로필 작성 완료 전환율 — 인증을 마친 사용자가 매칭 풀에 진입하는 비율. 이 숫자가 떨어지면 본 단위가 인증과 매칭 사이의 끊김 지점이 되고 있다는 신호.
  • 프로필 작성 소요 시간 — 입력 시작부터 첫 저장까지의 시간. 너무 길면 폼이 무겁다는 의미, 너무 짧으면 사용자가 성의 없이 채우고 있다는 의미일 수 있음.
  • 활성 토글 사용 빈도와 비활성 지속 기간 — 사용자가 토글을 어떻게 쓰는지. 비활성 기간이 길어지면 그 사용자는 사실상 휴면이다.
  • 공개 범위 옵션 사용 비율 — "전체 / 매칭된 상대만 / 비공개"의 분포. 비공개 사용자가 많아지면 후보 카드 풀이 줄어든다는 신호.
  • 사진 평균 장수 — 3장에 머무는지 6장까지 채우는지. 평균이 낮으면 매칭 의사결정의 단서가 부족하다는 의미.

운영자가 알아야 할 것

본 단위가 어긋나면 매칭 품질이 직접 영향을 받는다
  • 사진 3장 미만은 매칭 풀에 들이지 않는다. 한두 장 사진으로는 다른 사용자의 의사결정 단서가 부족해 매칭 품질이 떨어진다. 상한 6장은 결정 피로를 막는 동시에 자기 표현의 여유를 보장하는 균형점.
  • 비활성 토글과 "비공개" 공개 범위는 의미가 다르다. 비활성은 시간(잠시 멈춤), 비공개는 의도(보여 주지 않음). 사용자에게 두 옵션의 의미가 명확히 구분되지 않으면 운영 데이터가 섞일 수 있다.
  • 사진 권한 위반은 서버에서 한 번 더 검사한다. 본인 프로필이 아닌 사진을 삭제하려는 시도가 들어와도 서버 가드가 잡는다. 이 가드가 풀리면 다른 사용자의 데이팅 사진이 임의 삭제될 수 있는 사고로 이어진다.
  • 인증 가드는 프로필 생성 단계에서도 한 번 더 검사된다. 데이팅 탭 진입 가드가 풀려도 본 단위가 마지막에 다시 잡는다 — 가드는 한 줄에 한 번이 아니라 여러 줄에서 중복으로 작동한다.
  • 한 번 작성한 프로필은 데이터로 보존된다. 비활성 토글·공개 범위 비공개·재인증 만료 어느 시점에도 본문은 지워지지 않는다. 사용자가 돌아왔을 때 빈 화면 대신 자신의 자기 표현을 그대로 본다.
