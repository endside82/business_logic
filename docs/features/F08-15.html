F08-15. 크리에이터 매출 귀속 보정 (Creator Earning Coverage)

플랜 마켓의 세 가지 구매 소스(MARKET_ITEM, MARKET_BUNDLE, PLAN_DIRECT)는 모두 크리에이터 매출 원장(CreatorEarning)에 잡힌다. 수수료·원천징수는 유료분에만, 무료분은 무수수료 free_credit으로 적립된다. F08-14 환불 PRD의 하드 선행이다.

소스 타입
3
MARKET_ITEM · MARKET_BUNDLE · PLAN_DIRECT
상태
구현됨
마켓/플랜 split 적용
followup
EVENT split
legacy 미이관

매출 귀속 진실

  • MarketPurchaseService가 MARKET_ITEM, MARKET_BUNDLE을 grossPaid/gross_free split으로 CreatorEarning 생성
  • PlanPurchaseService가 PLAN_DIRECT를 동일 split으로 생성
  • MarketplaceSettlementService가 세 source type 모두 정산 대상으로 집계 (과거 MARKET_ITEM만 잡히던 결손 해소)
  • 수수료·원천징수는 유료분(grossPaid)에만 적용. 무료분은 free_credit으로 적립(수수료 없음)
  • EVENT 정산의 earning split만 legacy 경로로 미이관 (followup)
