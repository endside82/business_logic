F03-12. 내 이벤트 관리 · 참석 로그

내 신청·주최·지난 이벤트를 한 화면에서 본다. 호스트는 추가로 이벤트의 참석 로그(누가 언제 어떤 단계로 갔는지)를 시간 순서로 확인 가능. 종료된 이벤트도 disabled 상태로 노출되어 회고·리뷰 진입점으로 연결된다.

이 기능 안의 하위 항목

탭 1

참석 예정

본인 신청 중 APPROVED만 필터.

탭 2

주최 중

본인이 host인 이벤트(DRAFT/OPEN).

탭 3

지난 이벤트

CLOSED/CANCELED 이벤트(리뷰 진입점).

호스트 전용

참석 로그 타임라인

이벤트별 신청/취소/체크인 등 단계 순서.

사용자 시나리오 9개

S1

참석 예정 확인

  1. APPROVED 카드만 노출
  2. 임박 카드 탭 → 이벤트 상세
S2

주최 중 임시저장 발견

  1. DRAFT 카드 회색 “임시저장” 뱃지
  2. 탭 → 상세 → “발행”

알려진 이슈: 컨트롤러가 status 쿼리 무시. CLOSED/CANCELED 섞일 가능성, 추가 필터 필요.

S3

지난 이벤트 회고

  1. disabled 카드 + 상태 라벨
  2. 탭 → 리뷰 작성 진입점(F11)
S4 · 호스트

참석 로그 타임라인

  1. 상세 → 참석자 목록 → “로그 >”
  2. 최신순 타임라인 + 날짜 구분선
  3. “참석 확정/대기열 등록/참석 취소/호스트 거절” 라벨
S5

로그 무한 스크롤

  1. page=0→1→2 누적
  2. last=true 시 추가 호출 차단
S6

참석 예정 중 호스트 취소

  1. Application=APPROVED지만 event=CANCELED
  2. disabled + “취소됨” 라벨
  3. 탭 → 환불 안내(F03-04/F06)
S7

빈 상태

  1. “참석 예정인 이벤트가 없습니다” + “이벤트 탐색”
S8

풀투리프레시

  1. 리스트 reset → 0페이지 재호출
S9 · 권한

비호스트가 로그 URL 직접 접근

  1. 라우터 redirect
  2. 우회 시 403 EVENT_NOT_OWNER

서버 계약

MethodPath설명
GET/api/v1/events/users/me/applications내 신청 목록
GET/api/v1/events/my?page=&size=내가 호스트인 이벤트
GET/api/v1/events/{id}/capacity/logs참석 로그(호스트 전용)

자주 빠뜨리는 케이스

레드 플래그
  • 주최 중 탭에 종료된 이벤트 섞임 — 컨트롤러 status 필터 누락.
  • 참석 예정에 취소된 이벤트가 활성 카드로 노출 — 상태 라벨 반영 누락.
  • 호스트 로그 권한 검증 누락 — 우회 시 다른 사용자 행적 노출 위험.
  • 풀투리프레시 후 페이지 인덱스 reset 누락 — 중복 호출.
