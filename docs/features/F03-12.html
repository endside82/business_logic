F03-12. 내 이벤트 · 참석 로그

본인이 신청·주최한 모든 이벤트를 한 곳에서 보는 면. 참석 예정, 주최 중, 지난 이벤트의 세 탭으로 나뉘어 사용자가 자기 모임 생활을 한 자리에서 관리한다. 같은 단위에 호스트가 자기 이벤트의 변경 이력을 검토하는 "참석 로그"도 함께 자리한다 — 누가 언제 어떤 상태로 바뀌었는지의 타임라인.

분류 탭
3개
참석 예정·주최 중·지난 이벤트
로그 분류
9종
신청·취소·승급·제거·정원 변경 등
설계 의도 — 사용자의 모임 생활을 한 자리에 모으되, 역할별로 분기

한 사용자가 어떤 이벤트는 참가자로, 어떤 이벤트는 호스트로 관여한다. 본 단위는 그 둘을 같은 화면 안의 다른 탭으로 분리해, 사용자가 자기 역할을 헷갈리지 않게 한다. 종료된 모임은 별도 탭으로 묶어 "지금 이번 주에 해야 할 일"과 "지나간 모임의 회고"를 분리. 호스트가 자기 이벤트의 모든 상태 변경을 시간순으로 보고 싶을 때는 별도 "참석 로그" 화면이 열린다.

사용자 시나리오

참석자 시점

다음 주 일정 확인

참가자가 "참석 예정" 탭에서 승인된 신청만 본다. 시작 시간이 가까운 카드를 누르면 이벤트 상세로 이동.

이 흐름이 남기는 것 — 사용자가 자기 모임 일정을 한 자리에서 확인하지 못하면 "어떤 모임에 신청했더라" 혼란이 누적. 본 단위의 핵심 가치 자리.

호스트 시점

"주최 중"에서 임시저장 이벤트 발견

호스트가 어제 임시저장한 이벤트가 회색 "임시저장" 배지로 표시된다. 카드를 눌러 상세로 가서 "발행"을 눌러 공개.

호스트 회고

"지난 이벤트"에서 종료된 모임 회고

호스트가 종료된 이벤트의 회고나 다음 모임을 위한 템플릿 복제 진입. 카드는 흐릿한 오버레이 + 상태 라벨로 표시되어 "이미 끝난 모임"임을 명확히 한다.

호스트 운영

참석 로그로 누가 노쇼였는지 확인

모임 끝난 뒤 호스트가 이벤트 상세 → 참석자 목록 → "로그" 진입. 시간순 타임라인에서 신청·취소·승급·제거·체크인 유무를 모두 확인. 노쇼 사용자를 식별해 신뢰점수 반영 같은 후속 처리로 안내.

이 흐름이 남기는 것 — 호스트의 운영 회고. 누가 약속을 지키지 않았는지를 데이터로 확인할 수 있어야 신뢰점수 정책이 동작.

취소 표시

참석 예정이 호스트 취소로 무효화됨

본인은 승인된 상태였지만 호스트가 이벤트를 취소. "참석 예정" 탭의 카드가 흐릿한 오버레이 + "취소됨" 상태 라벨로 표시. 환불 안내는 별도 알림으로 도달했고, 본 단위는 표시만 책임진다.

빈 상태

신규 사용자가 처음 진입

신청한 모임이 0건이면 "참석 예정인 이벤트가 없습니다" + "이벤트 탐색" 안내. 사용자가 탐색 화면으로 이동하도록 자연스럽게 다음 동선 제시.

권한

일반 사용자가 참석 로그 URL 진입

호스트와 공동호스트가 아닌 사용자가 참석 로그 URL로 진입해도 차단된다. 우회 호출도 서버에서 막힘.

무한 스크롤

로그 50건의 다음 묶음

참석 로그는 20건 단위로 이어 붙는다. 마지막 페이지에 도달하면 추가 로딩이 멈춘다. 사용자가 위로 당겨 새로고침하면 첫 페이지부터 다시 받아 온다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
"참석 예정" 탭 본인이 승인된 신청만 표시. 카드에 일정·주소·승인 배지. 종료/취소된 이벤트는 흐릿한 오버레이 + 상태 라벨. "이번 주에 갈 모임"의 자리. 심사 중·거절은 다른 탭에서 다루어 시야를 단순하게.
"주최 중" 탭 본인이 호스트인 모임. 임시저장은 회색 배지. 정원·현재 인원·가격 표시. 호스트의 운영 일감을 한 곳에 모은다. 임시저장이 시각적으로 분리되어 "공개 안 된 초안"을 호스트가 잊지 않게.
"지난 이벤트" 탭 본인이 호스트였던 종료/취소 모임. 항상 흐릿한 오버레이 + 상태 라벨. 회고와 템플릿 복제의 자리. 현재는 호스트 시점만 노출 — 참가자의 "지난 참석" 모음은 추후 도입 검토.
참석 로그 타임라인 시간순 세로 타임라인. 각 항목에 행동(신청·취소·승급·제거 등)과 사용자 닉네임. 날짜가 바뀌면 구분선. 호스트가 누가 어느 시점에 무엇을 했는지를 시간순으로 추적할 수 있게. 노쇼 식별·신뢰점수 정책의 데이터 소스.

운영자가 알아야 할 것

내 이벤트·로그의 결정점
  • "주최 중" 탭에 임시저장과 모집을 함께 표시한다. 호스트가 임시저장 이벤트를 잊지 않게 같은 자리에 둔다. 사용자에게 보이는 모집과 시각적으로 구분(임시저장 배지).
  • 참가자의 "지난 참석" 탭은 현재 없다. 참가자가 자기가 참석했던 모임 목록을 보려면 위시리스트·알림 등 우회 경로 필요. 운영 분석에서 회고 요청이 잦으면 우선 도입 가능 영역.
  • 주최 중 탭의 호스트 필터링은 클라이언트 측 분류로 동작한다. 실제로는 호스트의 모든 상태 이벤트가 한 번에 내려오므로, 종료된 이벤트가 "주최 중"에 잠시 섞여 보일 수 있는 잠재 영역.
  • 참석 로그는 호스트·공동호스트만 본다. 일반 참가자는 본인 관련만 보이는 별도 화면이 필요한데 현재는 미구현. 호스트의 운영 도구로만 자리.
  • 참석 로그는 노쇼·신뢰점수 정책의 데이터 소스. 로그 누락이 일어나면 신뢰점수 정책이 부정확해진다 — 본 단위는 시각화 자리이고, 로그 자체의 생성은 신청·승급·취소 단위에서 일어나야 한다.
