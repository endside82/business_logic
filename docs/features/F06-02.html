F06-02. 포인트 충전 (PG 결제)

사용자가 PG 결제로 포인트를 충전한다. 서버가 PENDING 거래를 만들고 PG WebView를 통해 결제 후 webhook 또는 client-confirm으로 멱등하게 완료된다. 잔액 부족 흐름에서는 부족액을 prefill로 전달한다.

시나리오
10+
멱등
webhook + client-confirm

이 기능 안의 하위 항목

동작 1

충전 요청 + PENDING 생성

동작 2

PG WebView

동작 3

완료 처리(멱등)

webhook과 client-confirm 둘 중 먼저 도달한 것이 처리.

동작 4

잔액 부족 prefill

사용자 시나리오 10개

S1

일반 충전 Happy

  1. 프리셋 선택 → POST → PG → client-confirm → 잔액 +amount
S2

잔액 부족 prefill

  1. 이전 결제 실패의 shortfall로 입력 자동 채움
S3

결제수단 미등록

  1. CTA가 결제수단 등록 안내
S4

PG WebView 취소

  1. PENDING 거래 취소 마킹
S5

webhook 먼저 도착

  1. client-confirm은 멱등 응답
S6

PG 실패

  1. 거래 FAILED, 사유 안내
S7

금액 범위 위반

  1. 최소·최대 검증 + 안내
S8

충전 중 앱 종료

  1. 다음 진입 시 webhook 결과 반영
S9

중복 client-confirm

  1. 멱등 응답
S10

회계 분개

  1. 충전 lot INSERT + ledger 분개
