F01-06. 온보딩

첫 로그인 사용자를 “추천이 가능한 상태”까지 끌어올리는 4단계 흐름. 모든 단계는 건너뛰기 가능하지만, 단계별 입력이 비어 있을수록 추천 품질이 떨어진다. 사진·태그·위치 어디서 실패해도 사용자가 메인 셸까지 도달할 수 있게 흐름을 끊지 않는 것이 본 단위의 핵심이다.

모든 화면은 “더 나은 추천을 위한 정보 수집”이지만, 어느 입력도 메인 셸 진입을 가로막지 않는다. 사용자가 가장 빠른 경로로 메인까지 가도 추천 시스템은 빈 입력에 대비해 동작해야 한다.

이 기능 안의 하위 항목

단계 1

웰컴 (4페이지 캐러셀)

“새로운 만남을…” 문구 시리즈와 함께 서비스 소개 4페이지를 스와이프. 마지막 페이지에서 “시작하기”를 누르거나 어디서든 “건너뛰기” 가능. 완료 상태를 기록해 두 번째 진입 시 표시되지 않게 한다.

스킵 가능
단계 2

프로필 (1/3)

프로필 사진(갤러리 선택 → 사전 서명 URL → 외부 스토리지 PUT) · 닉네임(가입 시 입력 유지) · 자기소개. 사진 업로드가 실패해도 닉네임·자기소개만으로 다음으로 진행 가능.

사진 실패 허용
단계 3

관심사 (2/3)

“3개 이상 선택” 안내 + 카테고리 선택. 선택 후 “다음 (N개 선택)” 버튼 활성화. 선택된 태그는 각각 별도 요청으로 등록되며, 중간 실패 시 부분 저장이 될 수 있어 클라이언트가 멱등 처리(중복 409 무시)를 해줘야 한다.

서버 한도 20개멱등 필요
단계 4

위치 (3/3)

GPS로 현재 위치 자동 감지 후 역지오코딩으로 “시·구·동” 표시. 권한 거부·외부 API 실패·서버 오류 어디서 막혀도 “건너뛰기”로 메인 셸 진입 가능.

권한 의존스킵 가능

화면 전이

선형 4단계지만 각 단계마다 건너뛰기 분기가 있다. 마지막 단계 통과 시 메인 셸 진입.

온보딩 진행과 재개

각 단계 완료 시점에 “여기까지 했음” 플래그가 저장되어, 앱을 닫았다 다시 열어도 같은 자리에서 다시 시작한다.

사용자 시나리오 11개

S1 · Happy Path

4단계 모두 완료

  1. 웰컴 캐러셀 스와이프 → “시작하기”
  2. 프로필 사진 업로드(사전 서명 URL → 외부 PUT) → 자기소개 입력 → “다음”
  3. 관심사 3개 이상 선택 → “다음 (N개 선택)” → 각 태그 순차 등록
  4. “현재 위치 자동 감지” → 권한 허용 → “시·구·동” 표시 → “이 위치로 설정”
  5. 온보딩 완료 플래그 = 참 → 홈 진입

종료 — 프로필·태그·주소 모두 설정, 메인 진입.

S2 · 전부 건너뛰기

모든 단계 건너뛰기로 가장 빠른 메인 진입

  1. 웰컴 → “건너뛰기”
  2. 프로필 → “건너뛰기” (서버 호출 없음)
  3. 관심사 → “건너뛰기”
  4. 위치 → “건너뛰기” → 온보딩 완료

종료 — 사용자 정보는 비어 있고 추천 품질은 떨어지지만, 메인 셸 진입은 보장된다.

S3 · 외부 스토리지 실패

프로필 사진 업로드 실패

  1. 갤러리에서 이미지 선택
  2. 사전 서명 URL 발급 성공 → 외부 스토리지 PUT 실패
  3. “프로필 사진 업로드에 실패했습니다” 토스트, 미리보기는 유지
  4. 사용자가 “다음” → 사진 URL 없이 닉네임·자기소개만 저장

종료 — 사진 미업로드, 다음 단계 진행 가능. 흐름 단절 없음.

S4 · 부분 저장

관심사 등록 중 네트워크 끊김

  1. 3개 태그 선택 → “다음”
  2. 첫 태그 등록 성공 → 두 번째에서 네트워크 오류
  3. “저장 중 오류가 발생했습니다” 토스트, 다음 단계로 이동 안 함
  4. 재시도 시 이미 등록된 첫 태그는 409 중복으로 응답될 수 있음

종료 — 부분 저장 상태. 클라이언트 측 멱등 처리 필요(중복 응답을 “이미 등록됨”으로 간주하고 다음 태그로 진행).

S5 · 권한 거부

위치 권한 거부 후 건너뛰기로 진행

  1. “현재 위치 자동 감지” 탭 → OS 권한 다이얼로그 거부
  2. “위치 권한이 필요합니다” 토스트
  3. 사용자가 “건너뛰기” → 위치 미설정 상태로 메인 진입

종료 — 주소 미설정. 메인 진입 가능. 위치 기반 추천은 동작하지 않음.

S6 · 중간 재개

앱 재시작 시 마지막 완료 지점부터 다시 시작

  1. 프로필 단계까지 완료 후 앱 종료
  2. 다시 앱 실행 → 토큰 유효 확인
  3. 사용자가 온보딩 미완료 상태 → 저장된 “마지막 완료 단계”로 라우터 분기
  4. 관심사 단계부터 다시 시작

종료 — 사용자가 처음부터 다시 하지 않아도 된다.

S7 · 서버 오류

위치 등록 실패

  1. “이 위치로 설정” 탭 → 서버 500
  2. 토스트로 알림 + 위치 완료 플래그 미설정
  3. 사용자가 재시도 또는 건너뛰기

종료 — 사용자가 흐름에 갇히지 않음.

S8 · 중복

닉네임 중복으로 프로필 갱신 실패

  1. 가입 시 자동 생성된 닉네임을 수정하려는 소셜 가입자
  2. “다음” → 서버 409 NICKNAME_ALREADY_EXISTS
  3. 토스트 안내 + 사용자가 다른 닉네임으로 재시도

종료 — 변경 실패, 흐름 유지.

S9 · 한도 초과

태그 21개 시도

  1. 현재 UI는 선택 개수 상한을 강제하지 않음
  2. 21번째 태그 등록 시 서버 400 MAX_PREFERENCE_TAGS_EXCEEDED
  3. 토스트 노출, 진행 중단

종료 — 서버 한도가 마지막 방어선. UI 보강 후보: 클라이언트에서도 사전 차단.

E2E-1 · 가입~온보딩 전부 건너뛰기

가입 → 첫 로그인 → 4단계 모두 건너뛰기 → 홈

  1. 가입 폼에서 매번 고유한 신규 이메일로 가입
  2. 로그인 화면 복귀 후 동일 자격증명으로 로그인
  3. 첫 로그인이므로 웰컴 화면 진입 — “새로운 만남을…” 텍스트 가시
  4. “건너뛰기” 4회 연속 — 프로필 “1/3”, 관심사 “3개 이상 선택”, 위치 “위치를 설정해주세요” 가시 확인
  5. 최종적으로 홈 화면 진입

종료 — 가입부터 홈까지 한 세션에서 모든 단계 스킵 가능.

E2E-2 · 회귀 방지

기존 사용자 로그인 시 온보딩으로 돌아가지 않음

  1. 이미 온보딩을 완료한 시드 사용자로 로그인
  2. 홈 화면이 단일 노출되어야 함
  3. 웰컴 화면이 다시 보이면 회귀

종료 — 메인 셸 진입, 웰컴 화면 미노출.

자주 빠뜨리는 케이스

레드 플래그
  • 중간 실패가 흐름을 끊는 회귀 — 어느 단계도 실패가 메인 셸 진입을 막아서는 안 된다.
  • 건너뛰기가 단계 완료 플래그를 안 남기는 회귀 — 다음 진입 시 같은 단계가 반복 노출된다.
  • 태그 부분 저장 후 재시도 시 409를 토스트로 노출 — 사용자가 “이미 등록된 것”을 모르고 화내는 케이스. 멱등으로 무시.
  • 이미 온보딩 완료한 사용자가 다시 웰컴으로 회귀 — 가장 분명한 회귀. 메인 셸 단독 노출이 보장돼야 한다.
  • 클라이언트가 태그 상한을 강제하지 않아 사용자가 21번째 클릭에서야 막힘 — UI 보강 후보.
  • 권한 거부 후 “건너뛰기”가 안 보이는 화면 상태 — 사용자가 갇히지 않게 항상 건너뛰기 노출.
