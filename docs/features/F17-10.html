F17-10. 호스트 정산 (flow-through)

FIXED 모임 종료 시 코스당 1건의 Settlement(regular_meeting_id=...) 생성. flow-through 모델: WALLET 결제의 retained 유료 포인트만 호스트 수익(gross), retained 무료 포인트는 freePointSubsidy(플랫폼 보조, payout 비대상)로 분리한다. BANK_TRANSFER는 off-ledger(payout 대상 아님).

platform fee
10%
withholding tax
3.3%
멱등
3중
uk_settlement_rm · existsByRegularMeetingId · existsBy ref SETTLEMENT
설계 핵심 — 정산 후 환불 위험 0

close가 통과하려면 모든 정션이 FINALIZED여야 한다. 정산은 close 커밋 후 afterCommit REQUIRES_NEW로 만들어진다. cancelEnrollment는 CLOSED/CANCELED를 거절하고, cancel도 findByIdForUpdate로 잠궈 정산 스냅샷과 환불 경합을 차단한다. FORFEIT 환불은 finalize 시점(=close 전)에 끝나서 settlement gross의 retained paid에 자연 반영된다.

flow-through 산식

// WALLET, !isHostDirect, pointTxId != null, status ∈ {PAID, REFUNDED} 결제 순회
retainedPaid = Σ (origTx.paidAmount − refundTx.paidAmount)
retainedFree = Σ (origTx.freeAmount − refundTx.freeAmount)

if retainedPaid ≤ 0 → skip (settlement-skipped 잔여 후속)

gross = retainedPaid
freePointSubsidy = retainedFree  // payout 비대상, 플랫폼 보조 기록
platformFee = gross × 0.10
withholdingTax = gross × 0.033
netAmount = gross − platformFee − withholdingTax

Settlement / failed_refund 확장

  • Settlement.event_id long → Long nullable
  • Settlement.regular_meeting_id Long 신규
  • Settlement.reserved_refund BigDecimal 신규 (미정리 환불 게이트)
  • uk_settlement_rm(regular_meeting_id) UNIQUE — 코스당 정산 1건
  • CHECK ((event_id IS NULL) <> (regular_meeting_id IS NULL)) — 양립 불가
  • failed_refund.event_id nullable + failed_refund.regular_meeting_id 신규

fail-closed 안전성

payment의 pointTxId 또는 refundPointTxId가 가리키는 PointTransaction이 없으면 TRANSACTION_NOT_FOUND를 던지고 트랜잭션 롤백. failsafe 스케줄러가 데이터 보정 후 재시도. 과소/과대 지급을 절대 만들지 않는다(silent skip 금지).

서버 계약

GET /api/v1/regular-meetings/{id}/settlement — host만. 정산 미생성이면 204 No Content. 생성됐으면 RegularMeetingSettlementVo: settlementId, regularMeetingId, status, grossAmount, platformFee, withholdingTax, netAmount, freePointSubsidy, reservedRefund, periodStart, periodEnd, settledAt, createdAt.
