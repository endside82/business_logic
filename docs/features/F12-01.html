F12-01. 알림 목록 · 읽음 관리

신청 승인·결제 완료·정산 실패·클럽 공지처럼 서비스 안에서 일어난 변화를 사용자에게 모아 보여 주는 인박스 화면. 시간순으로 쌓인 알림을 살펴보고, 한 건 또는 전체를 읽음으로 정리하고, 더 이상 필요 없는 알림은 좌측 스와이프로 지운다. 알림 한 건을 탭하면 그 알림이 가리키는 도착 화면(이벤트 상세·클럽 상세·정산 화면 등)으로 곧장 빠진다.

우선순위 시각 강조
9종
결제·정산 실패류는 노란 바 + "주의" 칩
설계 의도 — 알림은 도착지로 가는 출입구다

알림 화면 자체가 목적지는 아니다. 사용자가 알림을 누르는 이유는 거의 항상 "그 일이 일어난 곳"으로 가기 위해서다. 그래서 본 단위는 한 번의 탭으로 "읽음 처리 + 해당 콘텐츠로 이동"을 같이 끝내는 것을 가장 중요한 동작으로 둔다. 결제·정산 실패처럼 사용자가 놓치면 손해를 보는 알림은 좌측 노란 바와 "주의" 칩으로 한 번 더 눈에 띄게 한다. 반대로 청소 동작(읽음·삭제)은 사용자가 자유롭게 할 수 있도록 풀어 둔다 — 알림 인박스는 "쌓이지 않게 사용자가 통제할 수 있다"는 감각이 더 중요하다.

사용자가 하는 일

탐색

최신부터 시간순으로 본다

오늘 / 어제 / 이번 주 / 이전 네 단계 날짜 헤더로 묶여 나온다. 한 묶음(20건)이 끝까지 스크롤되면 다음 묶음이 자동으로 이어 붙는다.

한 건 처리

알림을 탭해 해당 화면으로 간다

탭하면 그 알림은 즉시 읽음으로 바뀌고, 알림 종류에 맞는 화면이 열린다. 예: 신청 승인 알림 → 이벤트 상세, 결제 실패 알림 → 결제 화면.

일괄 처리

모두 읽음으로 비운다

상단 "모두 읽음" 한 번으로 모든 미읽음을 정리한다. 알림이 누적된 사용자를 위한 빠른 청소 동작.

개별 삭제

좌측 스와이프로 한 건을 지운다

지운 알림은 인박스에서 영구히 사라진다(되돌리기 없음). "이 알림은 더 안 봐도 된다"는 명확한 정리 의도를 반영한다.

알림 한 건이 닫히기까지

탭 한 번에 읽음 표시와 도착 화면 이동이 같이 일어난다. 도착지를 찾을 수 없을 때만 별도 안내 시트로 빠진다.

사용자 시나리오

기본 흐름

며칠 만에 쌓인 알림을 한 번에 정리

3일 만에 앱을 연 사용자가 알림 탭을 누른다. 신청 승인·결제 완료·클럽 공지 12건이 쌓여 있다. "모두 읽음"으로 일괄 정리한 뒤, 가장 궁금했던 "주말 러닝 모임 신청 승인" 한 건을 다시 탭해 이벤트 상세로 들어간다.

이 흐름이 남기는 것 — 사용자가 인박스를 "처리해야 할 작업"이 아니라 "지나간 일의 기록"으로 받아들이게 한다.

우선순위 인지

결제 실패 알림이 눈에 띈다

일반 알림들 사이에 좌측 노란 바 + "주의" 칩이 붙은 결제 실패 알림 하나가 끼어 있다. 사용자는 그 한 줄을 먼저 발견하고 곧장 탭해 결제 화면으로 빠진다.

이 흐름이 남기는 것 — 결제·정산 실패처럼 놓치면 손해가 발생하는 알림이 일반 알림에 묻히지 않게 한다. 매출과 신뢰 양쪽에 직접 영향이 가는 시각 강조 정책.

도착지 분실

대상 콘텐츠가 사라졌다

리뷰 작성을 권유하는 알림을 탭했는데 해당 모임이 운영자에 의해 삭제된 상태다. 빈 화면으로 떨어뜨리지 않고, "알림을 열 수 없습니다 / 해당 콘텐츠가 삭제되었거나 접근 권한이 없을 수 있습니다" 안내 시트로 받아 준다.

이 흐름이 남기는 것 — 알림과 콘텐츠의 수명 차이를 사용자에게 자연스럽게 설명한다. 읽음 처리는 정상적으로 끝나 알림함이 깨끗해진다.

즉시 정리

관심 없는 알림을 좌측으로 지운다

스와이프로 한 건이 사라지고 "알림이 삭제되었습니다" 토스트가 떠오른다. 되돌리기는 없다.

이 흐름이 남기는 것 — 인박스 청소는 사용자 통제 영역. 한 번의 정리 동작에 확인 다이얼로그까지 끼우면 청소 자체를 안 하게 된다 — 알림을 켜 두는 비용을 사용자가 감당 가능한 수준으로 유지하는 정책.

실시간 갱신

다른 화면에서 새 알림이 도착했다

홈을 보고 있는 사용자에게 새 채팅 메시지가 도착한다. 하단 알림 탭의 배지 숫자가 자동으로 +1이 되어 새 알림이 있다는 사실을 사용자가 인지한다.

이 흐름이 남기는 것 — 푸시를 못 받는 사용자(권한 거부·방해금지)도 인박스 배지로는 변화를 알 수 있어야 한다 — 알림 한 줄이 "기록되지 않은 변화"가 되지 않게 하는 보장.

현재 한계

알림 화면에 머무는 동안에는 새 알림이 자동으로 끼어들지 않는다

알림 탭을 보고 있는 동안 새 알림이 도착하면 하단 배지는 갱신되지만, 화면의 리스트 자체는 사용자가 당겨서 새로고침를 하기 전까지 그대로다.

운영 관찰 포인트 — 한 화면을 오래 띄워 두는 사용자가 많다면 자동 갱신을 검토할 여지. 지금은 사용자 의도(당겨서 새로고침) 시점에만 갱신해 데이터·배터리 비용을 낮추는 트레이드오프.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
상단 액션 3종 그룹/리스트 토글, "모두 읽음" 텍스트 버튼, 설정 진입 아이콘. 인박스에서 사용자가 가장 자주 하는 세 가지 행동(보기 방식 전환·일괄 청소·환경설정 진입)을 한 줄에 모은다.
알림 권한 안내 띠 OS 권한이 거부된 사용자에게만 노출되는 분홍빛 인라인 배너. 중요 알림(결제·정산)을 놓치고 있는 사용자에게 회복 경로를 한 줄 안내. F12-06이 담당.
날짜 그룹 헤더 오늘 / 어제 / 이번 주 / 이전 네 단계. 알림은 "언제 일어난 일"이라는 시간 인지가 이해의 핵심. 분 단위는 알림 카드 안의 상대 시각으로 보조.
알림 카드 한 줄 타입 아이콘, 제목, 본문 2줄, 상대 시각, 미읽음일 때 우측 점. 결제·정산 실패류는 좌측 노란 바 + "주의" 칩이 추가로 붙는다. 알림 한 줄로 "무엇이 일어났는지"와 "지금 봐야 하는 일인지"를 동시에 판단할 수 있게 한다.
좌측 스와이프 빨간 배경 + 휴지통 아이콘이 슬쩍 보였다가 끝까지 밀면 사라진다. 표준 모바일 제스처를 그대로 따른다. 별도 더보기 메뉴 없이 정리를 빠르게.
빈 상태 알림 아이콘 + "알림이 없습니다" 한 줄. 신규 가입 직후나 모두 정리한 직후의 깨끗한 인박스. 사용자에게는 도달 상태로 인식된다.

운영자가 알아야 할 것

인박스 정책 결정점
  • 알림은 인박스(앱 내)와 푸시(단말)에 각각 도달한다. 사용자가 푸시 권한을 거부했거나 방해금지 시간대여도 인박스에는 정상 적재된다. "알림함 자체가 비어 있다"고 사용자가 인지하면 그건 사고다 — 발송 트리거 자체가 빠졌는지 점검 대상.
  • "주의" 우선순위 9종은 정책 결정이다. 결제 실패·정산 실패·환불 실패·정산 만료·취소·인출 거부·정산 정정 보류 등 사용자가 놓치면 손해가 발생하는 알림에 한해 시각 강조. 남발하면 강조가 평탄해진다 — 이 묶음을 늘릴 때는 보수적으로.
  • 도착지가 없는 알림이 늘어나면 알림 카피·발송 정책의 신호다. "이 모임은 종료됐어요" 안내 시트가 자주 뜨면, 모임 종료 후에도 리뷰 알림이 발송되는 등 알림 수명과 콘텐츠 수명이 어긋난다는 신호.
  • 삭제는 영구 동작이다. 되돌리기를 두지 않은 것은 의도된 결정 — 인박스 청소의 마찰을 최소화한다. 사용자가 실수로 중요한 알림을 지웠다고 주장하면, 알림 본체가 가리키는 도메인(이벤트·결제 등) 화면에서 동일 정보를 다시 찾을 수 있어야 한다.
  • 알림 인박스에 머무는 동안의 자동 갱신은 현재 없다. 하단 탭 배지는 갱신되지만 리스트 자체는 당겨서 새로고침 시점에만 새로워진다. 이 정책 변경(자동 갱신)을 검토할 때는 데이터·배터리 비용 증가와 트레이드오프를 같이 본다.
