F12-01. 알림 목록 · 읽음 관리

사용자의 알림 인박스. 시간순 그룹 + 미읽음 카운트 + 개별/일괄 읽음 + 딥 링크 진입. 모든 액션은 낙관적 UI로 즉시 반영되고, 서버 실패 시 자연 롤백된다. 포그라운드 푸시는 인박스가 자동으로 인지하지 못해 풀투리프레시가 필요한 알려진 한계가 있다.

설계 의도

알림은 “정리”와 “진입” 두 행동이 빠르게 이뤄져야 한다. 모두 읽음은 화면 갱신 후 서버 호출하고, 개별 탭은 읽음 처리와 딥 링크 라우팅을 같은 동작에 묶는다. 알림 자체에 referenceId가 누락되면 “열 수 없습니다” 폴백을 모달로 안내해 사용자가 갇히지 않게 한다.

이 기능 안의 하위 항목

동작 1

목록 + 미읽음 카운트 병렬 호출

화면 진입 시 GET /notifications + GET /unread-count 동시 호출. 페이지 20건씩 무한 스크롤.

동작 2

개별 읽음 (탭과 동시에)

알림 카드 탭 → 낙관적 isRead=true + 배지 -1 → PATCH /{id}/read → 같은 동작에서 딥 링크 라우팅.

동작 3

모두 읽음

앱바 “모두 읽음” → 모든 카드 즉시 흰 배경 + 배지 0 → PATCH /read-all.

동작 4

좌측 스와이프 삭제

Dismissible로 영구 삭제. 현재 Undo 미구현.

복구 불가
동작 5

딥 링크 라우팅

NotificationRouter가 type별로 적절한 화면 경로 결정. referenceId 누락 시 폴백 모달.

동작 6

FCM 포그라운드 처리

포그라운드 수신 시 unreadCount만 자동 invalidate. 목록 자체는 풀투리프레시 필요(알려진 한계).

알림 탭 → 라우팅 시퀀스

사용자 시나리오 8개

S1 · Happy Path

일괄 정리 후 딥 링크 진입

  1. 알림 탭 진입 → 두 호출 병렬 (목록 + unreadCount)
  2. “모두 읽음” → 12장 즉시 흰 배경 + 배지 0 → PATCH /read-all
  3. 특정 알림 탭 → 이미 읽음이므로 read 호출 없이 NotificationRouter.navigate
  4. 대상 화면(예: 이벤트 상세) push
S2 · 포그라운드

FCM 수신 직후

  1. onForegroundMessage → unreadCount만 invalidate + 인앱 배너
  2. 목록 자체는 갱신 안 됨(알려진 한계)
  3. 사용자가 풀투리프레시 → 새 알림 포함
  4. 탭 → optimistic 읽음 + 라우팅
S3 · 삭제

좌측 스와이프 삭제

  1. Dismissible 끝까지 스와이프 → DELETE /{id}
  2. 토스트 “알림이 삭제되었습니다”
  3. Undo 액션 없음 — 영구 삭제

UI 보강 후보: Undo 액션 추가.

S4 · 권한

다른 사용자 알림 변조 시도

  1. 외부자가 다른 사용자 notificationId로 PATCH
  2. 서버: notification.userId 검증 실패 → 403 NOTIFICATION_NOT_OWNER
  3. 클라이언트 optimistic 롤백
S5 · 무한 스크롤

100개 누적 사용자

  1. page=0 → 20건
  2. 스크롤 끝 → loadMore() → page=1 → 누적
  3. 마지막 페이지(last=true) 도달 → hasMore=false, 이후 호출 없음
S6 · 폴백

referenceId 누락 알림

  1. 탭 → optimistic 읽음
  2. NotificationRouter.resolve() = null
  3. “알림을 열 수 없습니다” 폴백 모달
  4. 사용자가 닫고 다른 알림으로 이동
S7 · 그룹 헤더

시간 그룹 표시

  1. 오늘/어제/이번 주/과거 헤더
  2. 각 그룹 안에서 createdAt DESC 정렬
S8 · 빈 인박스

알림이 하나도 없는 사용자

  1. 0건 응답 → 빈 상태 일러스트 + “아직 알림이 없습니다”
  2. “모두 읽음” 비활성

자주 빠뜨리는 케이스

레드 플래그
  • optimistic 롤백 누락 — 서버 실패 후에도 읽음 상태가 그대로면 인박스와 실제 상태가 어긋남.
  • “모두 읽음” 후 새 알림이 즉시 안 보임 — 포그라운드 push 시 목록 invalidate를 빠뜨림.
  • Undo 없는 삭제 — 사용자 실수 복구 불가. UX 보강 후보.
  • 딥 링크 폴백 누락 — referenceId 없는 알림이 무한 로딩.
  • 본인 검증 누락 — 외부자가 임의로 다른 사람 알림 PATCH.
  • 그룹 헤더 가짜 정렬 — 그룹 안의 시간순이 깨지면 사용자가 “어제 본 알림이 오늘 위에” 같은 혼란.
