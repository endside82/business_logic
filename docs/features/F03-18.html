F03-18. 이벤트 구성인원 인구통계

이벤트 상세에서 호스트·공동호스트·참석 확정자의 성별·나이대·MBTI I·E 비율을 가로 비율 바로 보여준다. 데이터는 본인인증으로 검증된 값(성별·생년월일)과 프로필에서 직접 입력한 MBTI를 사용한다. 소수 셀(개인)이 드러나지 않도록 프라이버시 3단계 게이트를 거친다.

3

프라이버시

중복 없이 집계
설계 의도 — 가입 전 구성 가늠 + 개인 식별 방지

참가 전 사용자가 모임의 구성을 가늠할 수 있게 하면서, 소수 인원에서 한 사람의 속성이 그대로 노출되는 일을 막는다. 노출되는 모든 버킷 카운트가 최소 5명 이상일 때만 그 차원을 보여주고, 응답에는 비율만 내려보낸다.

조회 흐름

flowchart LR A["이벤트 상세 진입"] --> B["구성 통계 조회"] B --> C{"접근권한"} C -->|"공개+진행/종료"| D["인증 사용자 OK"] C -->|"비공개·초안 등"| E["호스트·공동호스트·참석 확정자만"] D --> F["대상 참가자 수집"] E --> F F --> G["회원 정보 결합(성별·생년월일·MBTI)"] G --> H["3단계 게이트"] H --> I["비율 바 렌더 or 안내"]

사용자 시나리오

정상 표시

참가자 10명, 성별 5:5, MBTI 입력 10명

모든 차원이 게이트를 통과 — 성별(50%/50%), 나이대 분포, I·E 비율이 가로 비율 바로 표시된다.

전체 게이트

참가자 4명

전체 인원이 5명 미만이라 분포 자체를 보여주지 않고 “구성원이 적어 통계를 표시하지 않습니다” 안내만 노출.

차원 게이트

여성 4명·남성 6명

여성 버킷이 5명 미만이라 성별 차원만 숨김. 나이대·I·E는 자체 게이트로 따로 평가.

접근권한

비공개 이벤트의 비참가자

호스트·공동호스트·참석 확정자가 아니면 접근이 막힌다. 공개+진행/종료 이벤트만 로그인 사용자 전체에게 허용.

이 화면이 제공하는 것

동작누가무엇이 일어나는가
구성 통계 보기접근 권한이 있는 사용자이벤트 구성인원의 성별·나이대·MBTI I·E 분포를 비율 바로 확인한다.

화면에 내려오는 값은 사람 수가 아니라 비율과 전체 인원, MBTI 입력률뿐이다. 게이트를 통과해 노출되는 버킷은 항상 5명 이상이라 한 사람이 식별되지 않는다.

운영자가 알아야 할 것

데이터 원천과 책임
  • 성별·생년월일은 본인인증 검증값이다. 인증 성공 시 Member로 동기화된다(F09-01).
  • MBTI는 사용자가 프로필에서 직접 입력한다(F13-02). 미입력자는 I·E 분포 분모에서 제외.
  • 나이는 만나이이며 기준일은 이벤트의 timezone(없으면 Asia/Seoul) 기준.
  • 버킷 최소셀 게이트 때문에 소규모 모임에서는 차원이 자주 숨겨질 수 있다 — 정상 동작.
  • Phase 0 이전 인증자는 본인인증 backfill 전까지 가입 시 입력값으로 동기화 — 분포 정확도 보강은 backfill 실행 시 향상.
