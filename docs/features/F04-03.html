F04-03. 클럽 생성 · 수정 · 삭제 · 소유권 이전

클럽의 일생을 책임진다. 생성 시 기본 게시판 3개·기금 0원·OWNER가 한 트랜잭션으로 만들어지고, 삭제 시에는 진행 이벤트 취소·기금 환불·구독 일할 환불까지 일괄 정리된다. 소유권 이전은 “마지막 책임자가 사라지지 않도록” OWNER 탈퇴를 사전에 차단한다.

이 기능 안의 하위 항목

동작 1

클럽 생성

이름·카테고리·가입방식·커버. 트랜잭션 안에서 게시판·기금·OWNER 동시 생성.

동작 2

설정 수정

OWNER만 가능. MapStruct null-skip 패치.

동작 3

클럽 삭제

구독 환불 + 이벤트 취소 + 기금 반환 + 멤버 정리. INACTIVE로 마킹.

동작 4

소유권 이전

현 OWNER → 멤버 한 명. role swap + 알림.

사용자 시나리오 8개

S1

신규 생성

  1. 커버 업로드 → POST
  2. OWNER + 기본 게시판 3 + 기금 0 자동 생성
S2

이름 중복/한도

  1. 중복: INVALID_INPUT
  2. 5개 초과: CLUB_LIMIT_EXCEEDED
S3

설정 수정

  1. 기존값 prefill
  2. 변경 필드만 PATCH
S4

비OWNER 수정 시도

  1. 403 CLUB_NOT_OWNER
S5

클럽 삭제(전체 정리)

  1. 구독 일할 환불 + 이벤트 CANCELED + 가입비 환불 + 기금 반환
  2. INACTIVE 마킹 + 17명에게 CLUB_CLOSED 알림
S6

소유권 이전

  1. role swap + ownerId 갱신
  2. 새 OWNER에게 CLUB_OWNER_CHANGED 알림
S7

자기 이전/비멤버 이전

  1. 각각 INVALID_INPUT/CLUB_NOT_MEMBER
S8

비활성 클럽 이전

  1. CLUB_INACTIVE

서버 계약

POST/api/v1/clubs/{id}/transfer-ownership소유권 이전

자주 빠뜨리는 케이스

레드 플래그
  • 삭제 트랜잭션 일부 실패 — 구독 환불 + 이벤트 취소 + 기금 반환은 모두 같은 트랜잭션.
  • OWNER 이전 후 본인 ADMIN 강등 누락 — role swap 필수.
  • 비활성 클럽에서 작업 허용 — INACTIVE/SUSPENDED 가드 필수.
