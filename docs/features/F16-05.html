F16-05. 적립규칙·등급·배지·교환 프리셋 관리

운영진(정책 담당)이 마일리지의 뼈대를 만드는 CRUD. 자동 적립 규칙, 등급, 배지 정의, 차감 프리셋, 시즌을 정의한다. 등급에는 개수·이름·임계치 제약이 있고, 차감 프리셋은 삭제 대신 비활성화만 된다.

설계 의도 — 마일리지가 "돈처럼 보이지 않게"

등급 이름에 "원·캐시·포인트" 같은 금전성 단어를 막는 것은 의도된 안전장치다. 마일리지가 현금처럼 보이면 법적·세무적 문제와 사용자 오해를 부른다. 차감 프리셋을 삭제하지 않고 비활성화만 하는 것도 과거 거래의 사유 기록을 보존하기 위해서다.

관리 대상과 규칙

대상핵심 규칙
적립 규칙체크인·후기·사진 승인 트리거별로 점수·활성 여부 설정. 같은 트리거 재저장은 덮어쓰기(upsert)
등급클럽당 최대 10개. 임계치 단조·중복 금지. 이름에 금전성 단어 금지
배지 정의클럽당 최대 50개. 활성/비활성 토글. 멤버 화면엔 활성만 노출
차감 프리셋차감 사유 템플릿. 삭제 불가 — 비활성화만
시즌기간 중복 금지. 종료(close)하면 다음 예정 시즌 자동 시작

운영자가 알아야 할 것

에러 메시지가 사유를 구분하지 못한다
  • 등급 생성 실패가 한 가지 에러로 뭉뚱그려진다. 개수 초과, 금전성 단어, 임계치 중복이 모두 같은 에러로 나와 사용자가 "왜 안 되는지"를 알기 어렵다. 사유별 메시지 분리가 필요하다.
  • 적립 규칙 "생성"은 사실 덮어쓰기다. 같은 트리거를 또 만들면 기존 규칙이 갱신된다. UI에서 "수정"으로 보이게 해야 한다.
  • 종료된 시즌 수정 거부가 "없음" 에러를 재사용한다. 실제로는 있지만 수정 불가 상태인데 "찾을 수 없음"으로 표시된다.
