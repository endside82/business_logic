F04-07. 내 클럽 · 멤버 통계

사용자가 자기가 속한 클럽들을 한눈에 보고, 운영자에게는 클럽 활동의 흐름을 숫자로 보여 주는 두 화면을 묶는 단위. 내 클럽 화면은 운영하는 클럽과 가입한 클럽을 분리해 보여 주고, 통계 화면은 운영자에게 멤버 활동의 농도를 한 화면에 정리해 준다.

설계 의도 — 두 화면은 서로 다른 질문에 답한다

내 클럽 화면은 "내가 지금 어디에 속해 있는가"라는 사용자의 질문에 답하고, 통계 화면은 "내 클럽이 살아 있는가"라는 운영자의 질문에 답한다. 두 질문의 답이 같은 화면에 섞이면 사용자는 자기 위치를 찾지 못하고 운영자는 결정 신호를 놓친다.

사용자 시나리오

내 클럽 한눈에

운영 1개·관리 1개·가입 3개를 보유한 사용자

마이페이지에서 "내 클럽"으로 들어오면 다섯 개 클럽이 두 묶음으로 정렬되어 보인다 — 운영하는 클럽 두 개, 가입한 클럽 세 개. 각 카드에는 역할 배지와 다음 이벤트 미리보기가 함께 있어, 어디로 들어갈지 한 화면에서 결정한다.

빈 상태

아직 어떤 클럽에도 속하지 않은 사용자

"아직 가입한 클럽이 없어요" 안내와 "클럽 둘러보기" 버튼이 떠 있다. 버튼을 누르면 클럽 목록(F04-01)으로 이동해 가입 흐름이 시작된다.

통계

소유자가 자기 클럽의 활동 농도를 점검한다

전체 멤버 수, 최근 한 달 활동한 멤버 수, 멤버별 참석·기부 기록, 활동 상위 다섯 명을 한 화면에서 본다. 이번 달 신규 멤버 카운트와 역할 분포는 멤버 목록 데이터를 합쳐 화면에서 계산된다.

이 흐름이 남기는 것 — 운영자에게 클럽의 건강 상태를 한 화면에서 보여 준다. 활동 멤버 수가 줄어드는 시점이 운영 개입의 신호.

진입 가드

일반 멤버가 통계 화면에 직접 들어오려 한다

일반 멤버에게는 통계 메뉴 자체가 보이지 않는다. 외부 링크로 통계 화면에 들어오면 클럽 상세로 되돌아간다. 통계는 운영자가 클럽의 건강 상태를 보는 면이지, 멤버의 둘러보기 면이 아니다.

운영자가 알아야 할 것

통계 정책 결정점
  • 큰 클럽일수록 통계 페이지 이동이 느려진다. 멤버별 활동 데이터가 페이지 단위로 다시 계산되므로 멤버 100명 이상 클럽에서는 페이지 이동이 눈에 띄게 느릴 수 있다. 운영 모니터링에서 응답 시간 추적 필요.
  • "이번 달 신규" 같은 가공 지표는 화면에서 계산된다. 서버는 멤버별 원본 데이터를 내려 보내고, 화면이 가입일 기준 신규 카운트나 역할 분포를 만든다. 지표 정의를 바꾸려면 화면 쪽 가공 로직을 함께 손봐야 한다.
  • 운영자 진입 가드는 라우터·메뉴·서버 세 곳에서 작동해야 한다. 일반 멤버가 통계 화면을 보면 다른 멤버의 활동량이 노출되는 사고가 일어난다.
