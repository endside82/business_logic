F14-02. 위치 공유 중지

사용자가 토글을 내리는 순간 일행 화면에서 본인 위치가 즉시 사라지게 한다. "끄겠다"는 의사 표시 한 번이 다른 사람의 다음 갱신 안에 반드시 반영되어야 하므로, 본 단위는 위치 공유의 4가지 원칙 중 "중지는 즉시"를 단호히 지키는 자리다.

중지 반영 시간
다음 갱신 안
최대 30초 안에 다른 사람 화면에서 사라진다
확인 단계
1단계
위치 화면에서는 다이얼로그로 한 번 더 확인
자동 중지
모임 종료 2시간 후
사용자가 끄지 않아도 자동으로 멈춘다
설계 의도 — "끄면 즉시 사라진다"가 위치 공유 전체의 신뢰 기반

사용자가 위치 공유를 다시 켤 수 있는 이유는 "꺼지면 진짜 꺼진다"라는 보장이 있기 때문이다. 중지 후 1초라도 마커가 남는다면, 그 사용자는 더 이상 토글을 켜지 않는다 — 그리고 그 의심은 위치 도메인 전체로 번진다. 본 단위는 "사용자 의사 표시 한 번이 다음 30초 안에 반드시 반영된다"를 단호히 지키는 자리.

중지의 세 가지 진입 경로

사용자가 "끄겠다"라고 표시할 수 있는 자리는 두 곳이고, 사용자 손이 없어도 자동으로 멈추는 경로 한 가지가 더 있다.

진입 1 — 위치 화면

이벤트 위치 화면 하단 시트의 토글

지도를 보고 있는 도중에 토글을 내리면 "위치 공유를 중지하시겠습니까? 다른 참석자에게 내 위치가 표시되지 않습니다." 다이얼로그가 한 번 뜬다. "중지"를 누르면 곧장 처리되고, "취소"를 누르면 토글이 시각적으로 그대로 켜진 상태에 머문다. 한 번 더 확인하는 이유는, 지도를 보면서 토글을 잘못 건드리는 사용자가 많기 때문.

진입 2 — 프라이버시 대시보드

위치 프라이버시 화면에서 끄기

위치 화면 우상단 톱니바퀴를 통해 들어간 프라이버시 대시보드에서는 확인 다이얼로그 없이 즉시 처리된다. 이 화면은 사용자가 "내 위치가 어디에 보이고 있는가"를 점검하러 일부러 들어온 자리이므로, 한 번 더 묻는 단계가 오히려 흐름을 끊는다고 본 설계.

진입 3 — 자동

모임 종료 2시간 후 자동 중지

사용자가 끄는 것을 잊어도 모임이 끝나고 2시간이 지나면 더 이상 좌표가 다른 사람 화면에 보이지 않는다. 위치 공유의 4가지 원칙 중 "자동 만료"의 실현 — 사용자가 끄는 것을 잊었다는 이유로 위치가 계속 노출되지 않게 한다.

중지 흐름

사용자가 토글을 내리는 순간부터 다른 사람의 다음 갱신 응답에서 본인이 사라지기까지.

사용자 시나리오

기본 흐름

약속 장소에 도착해서 끈다

모임 장소에 도착한 사용자가 시트의 위치 공유 토글을 내린다. 다이얼로그가 한 번 뜨고, "중지"를 누르면 "위치 공유를 중지했습니다" 토스트와 함께 시트 라벨이 회색으로 바뀐다. 다른 일행의 다음 갱신 응답에서 본인 마커가 사라진다.

이 흐름이 남기는 것 — 도착 후 위치를 끄는 자연스러운 마무리. 사용자가 "끝났다"를 직접 표시하게 한다.

취소

잘못 건드린 토글

지도를 보다가 손가락이 토글에 닿아 다이얼로그가 뜬다. 사용자가 "취소"를 누르면 어떤 호출도 일어나지 않고 토글은 시각적으로도 켜진 상태 그대로다. 자동 갱신은 끊김 없이 계속된다.

이 흐름이 남기는 것 — 위치 공유처럼 민감한 동작은 한 번 더 확인하는 단계가 사용자 신뢰의 기반.

다른 진입점

프라이버시 대시보드에서 즉시 끄기

모임이 끝나서 "내가 어디에 위치를 공유하고 있는지" 점검하러 들어온 사용자가 카드의 스위치를 내린다. 여기서는 다이얼로그 없이 즉시 처리되고, "위치 공유를 중단했습니다" 토스트가 뜬 뒤 카드 라벨이 "위치 비공개"로 바뀐다.

이 흐름이 남기는 것 — 사용자가 의도하고 들어온 화면에서는 한 번 더 묻지 않는 것이 자연스럽다.

자동 만료

모임이 끝나고 화면을 켜둔 채로 둔다

사용자가 직접 끄지 않아도 모임 종료 2시간이 지나면 화면이 더 이상 새 좌표를 받지 못한다. 마지막으로 받은 좌표가 잠시 그대로 표시될 수 있지만, 보관된 위치 데이터는 별도 청소 작업이 30일 안에 제거한다.

이 흐름이 남기는 것 — 사용자가 끄는 것을 잊는 인간적 실수를 정책이 덮어 준다.

재개

껐다가 다시 켠다

잠시 끈 사용자가 토글을 다시 켜면, F14-01의 켜기 흐름이 그대로 다시 작동한다 — 권한 확인은 이미 받았으니 곧장 좌표가 보내지고 30초 갱신이 다시 시작된다.

이 흐름이 남기는 것 — 한 번 꺼졌다고 영구히 막히지 않는 자유. 사용자가 자기 의사로 재개할 수 있어야 중지가 가벼운 동작이 된다.

호스트 차단의 간접 효과

호스트가 모임 단위로 위치 공유를 끄면

일반 참석자의 토글은 시각적으로는 켜진 상태로 유지되지만 좌표 갱신이 막힌다. 사용자 본인이 명시적으로 "끄겠다"는 의사를 표시한 것은 아니므로 본 단위의 흐름은 아니지만, 실질적으로 그 모임에 한해 위치가 더 이상 흘러가지 않는 결과가 된다.

이 흐름이 남기는 것 — 호스트의 모임 단위 결정이 모든 참석자의 위치를 사실상 동시에 멈춘다. "호스트는 위치 공유를 끌 책임이 있다"라는 모임 운영 신호.

화면에서 사용자가 보는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
위치 화면 토글 시트 맨 위의 스위치. 켜져 있을 때는 "위치 공유 중" 라벨, 꺼지면 회색 "위치 비공개". 지도와 같은 화면 안에 있어 "내 위치가 보이고 있는가"를 사용자가 항상 인지할 수 있어야 한다.
확인 다이얼로그 "위치 공유 중지" 제목 + "다른 참석자에게 내 위치가 표시되지 않습니다" 본문 + "취소" / "중지"(빨강) 두 버튼. 지도를 보면서 토글을 우연히 건드린 경우와 의식적으로 끄는 경우를 분리한다. 빨간색 "중지"는 되돌릴 수 없는 동작이라는 점을 사용자가 색만으로 인지하게 한다.
완료 토스트 "위치 공유를 중지했습니다" (위치 화면) / "위치 공유를 중단했습니다" (프라이버시 대시보드). 처리가 끝났다는 신호 — 사용자가 "진짜 꺼졌나"를 재확인하지 않게 한다.
꺼진 후의 시트 토글 라벨이 회색으로 바뀌고, 만료 카운트다운과 연장 버튼이 사라진다. "공유 상태가 시각화"되어야 하는 위치 도메인 원칙의 실현. 한 번에 "내가 지금 공유하고 있지 않다"를 알 수 있다.

운영자가 알아야 할 것

위치 공유 중지가 정상적으로 동작한다고 부르려면
  • "끄겠다"는 사용자 의사 표시는 다음 30초 안에 다른 사람 화면에서 반영되어야 한다. 이 보장이 깨지면 사용자가 다시는 위치 공유를 켜지 않는다. 본 단위에서 가장 강하게 지켜야 하는 정책.
  • 위치 화면에서는 확인 다이얼로그, 프라이버시 대시보드에서는 즉시 처리. 두 진입점의 확인 단계가 다른 이유는 사용자가 의도하고 들어온 화면이 다르기 때문 — 한쪽에 통일하려면 이 의도 차이를 먼저 다시 검토해야 한다.
  • 중지 후에도 보관된 좌표는 즉시 삭제되지 않는다. 사용자 동의 상태는 즉시 꺼지지만 마지막 좌표 기록은 만료 시각이 지나거나 30일 청소 작업이 돌 때 삭제된다. 운영 관점에서는 "꺼졌는데도 데이터가 잠시 남아 있다"가 사실 — 다만 중지한 사용자의 좌표는 다른 사람 응답에서 자동으로 제외되므로 사용자 화면에는 보이지 않는다.
  • 모임 종료 2시간 후 자동 중지는 운영 모니터링 대상. 이 자동 중지가 동작하지 않으면 사용자가 끄지 않은 좌표가 모임이 끝난 뒤에도 흘러간다 — 위치 공유의 "자동 만료" 원칙이 무너지는 회귀.
  • 중지 실패(자격 미충족, 네트워크 오류)는 현재 별도 안내 없이 조용히 처리된다. 사용자가 토글을 내렸는데 실제로는 꺼지지 않은 상태로 머무는 분기다. 향후 명시적 토스트 보강 필요.
