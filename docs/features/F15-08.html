F15-08. 제재 집행

누적된 경고에 대해 운영진이 실제 조치를 내리는 면. 공지/주의, 이벤트 신청 제한, 역할 제한, 강제 탈퇴, 수동 잠금까지 강도가 다른 6가지 도구가 있다. 가장 강한 강제 탈퇴는 소유자만 쓸 수 있고, 제재는 사유와 함께 기록되며 나중에 철회할 수 있다.

제재 유형
6종
공지~강제 탈퇴
강제 탈퇴
소유자 단독
서비스에서 재검증
신청 차단 연동
17개 진입점
클럽+플랫폼 전역 2축, 신청 시 실시간 검사
2026-06-04 기준 — 강제 탈퇴가 실제 멤버십 제거로 연결됨

강제 탈퇴 제재가 확정되면, 그 처리가 완전히 끝난 뒤에만 클럽 멤버십 제거·유료 가입비 환불·강퇴 알림이 자동으로 이어진다 — 제재 기록이 확정되지 않았는데 멤버십만 빠지거나 환불만 나가는 일은 없다. 뒤따르는 처리가 실패하더라도 제재 기록 자체는 유지되고 운영자 알림이 발송된다. 기한부 이벤트 신청 제한도 만료 시 자동 해제된다(매일 새벽 04:00 자동 종료 + 목록 조회 시 즉시 만료 처리). 이벤트 신청 차단은 클럽 제재뿐 아니라 플랫폼 전역 제재(고객지원 관리자가 플랫폼 차원에서 부과한 제재)도 함께 검사한다.

사용자 시나리오

신청 제한

반복 노쇼 멤버의 이벤트 신청을 막는다

이벤트 신청 제한 제재를 걸면, 그 멤버가 이벤트를 신청할 때 이벤트 도메인이 실시간으로 검사해 막는다.

강제 탈퇴

심각 등급 멤버를 강제 탈퇴시킨다

소유자만 가능. 사유를 적어 확정하면 제재가 기록되고, 멤버십 제거·유료 가입비 환불·강퇴 알림이 자동으로 따라온다.

권한 가드

관리자가 강제 탈퇴를 시도한다

소유자가 아니면 “소유자만 가능합니다”로 막힌다. 외부에서 강제 호출해도 서버가 재검증한다.

수동 잠금

특정 멤버의 자동 등급 재평가를 잠근다

수동 잠금을 걸면 점수가 바뀌어도 자동으로 등급이 재평가되지 않는다. 해제하면 다시 평가된다.

철회

오해가 풀려 제재를 해제한다

활성 제재만 철회 가능. 강제 탈퇴 철회도 소유자만 할 수 있다.

운영자가 알아야 할 것 — 비즈니스 리스크

운영자가 알아야 할 잔여 사항 (2026-06-04 기준)
  • 강제 탈퇴 후 유료 클럽 구독 정리 연동 여부는 아직 미확인이다. 가입비 환불은 처리되지만, 구독(월정액)이 강퇴 시 자동으로 해지되는지는 별도 검증이 필요하다.
  • 플랫폼 전역 이벤트 호스팅 제한과 데이팅 차단의 실효는 아직 미배선이다. 제재를 부과해도 실제 이벤트 생성이나 데이팅 프로필에 차단이 걸리지 않는다.
  • 공지·역할 제한의 실효도 알림/권한 파이프라인에 위임돼 있다. 그 파이프라인이 안 돌면 “제재가 기록만 되고 효력은 없는” 상태가 된다.
  • 만료 자동 종료와 즉시 만료 필터는 정상 동작한다. 매일 새벽 4시 자동 종료 스케줄러 + 목록 조회 시 즉시 만료 처리가 모두 활성화돼 있다.
