F04-01. 클럽 발견·탐색

앱 하단 "클럽" 탭에 처음 들어오면 보이는 면. 카테고리 칩과 검색바로 클럽을 골라 보고, 카드 한 장에 클럽 이름·카테고리·멤버 수와 함께 "내가 이미 가입했는지"가 함께 보인다. 비로그인 사용자도 둘러볼 수 있게 열려 있어, 가입을 결정하기 전 사용자가 클럽 도메인의 분위기를 확인하는 진입로다.

한 번에 받는 카드
20개
스크롤하면 다음 묶음
가입 전 진입
허용
가입 신청 시점에만 로그인 요구
카드의 본인 상태
3가지
멤버 · 신청 중 · 차단됨
설계 의도 — 카드 한 장에 "내가 어디 서 있는지"가 함께 보여야 한다

클럽 목록은 단순히 "어떤 클럽이 있는가"를 보여 주는 면이 아니라, "내가 그 클럽과 어떤 관계인가"도 같이 보여 줘야 한다. 이미 가입한 클럽인지, 신청 중인지, 차단된 곳인지가 카드 단계에서 드러나야 사용자가 상세 화면에 들어가지 않고도 다음 행동을 결정할 수 있다. 그래서 카드 한 장에는 클럽 정보와 본인 상태 배지가 같이 들어가 있고, 로그인한 사용자에게는 클럽 정보와 함께 자신의 멤버십 상태가 한 번의 호출로 채워져 내려온다.

사용자 시나리오

가입 전 둘러보기

회원가입 전 사용자가 카테고리로 훑어본다

아직 회원가입하지 않은 사용자가 "스포츠" 칩을 누른다. 스포츠 카테고리 클럽 20개가 카드 그리드로 보이고, 어느 카드를 눌러도 클럽 상세까지는 들어간다. 가입을 시도하는 순간에만 로그인 화면이 떠오른다.

이 흐름이 남기는 것 — "가입하고 둘러봐" 대신 "둘러보고 마음 들면 가입" 패턴. 가입 전환의 첫 진입 경로.

내 클럽 찾기

이미 가입한 클럽을 검색해서 빠르게 찾는다

로그인한 사용자가 검색바에 "러닝"을 입력한다. 결과 카드 중 자기 클럽에는 "멤버" 배지가 함께 보이고, 카드를 눌러 상세에 들어가면 가입 버튼 대신 "클럽 홈" 버튼이 떠 있다.

이 흐름이 남기는 것 — 카드에서 본인 상태를 보여 주지 않으면 매번 상세에 들어갔다 나와야 자기 클럽인지 알 수 있다.

스크롤

끝까지 스크롤하면서 둘러본다

화면 하단까지 스크롤하면 다음 20장이 자연스럽게 이어 붙는다. 마지막 묶음에 도달하면 더 이상 호출하지 않고 그리드 끝에서 멈춘다. 위로 당겨 새로고침하면 첫 묶음부터 다시 받는다.

잘못된 진입

외부 링크에서 잘못된 카테고리로 들어옴

딥링크에 카테고리 값이 잘못 인코딩된 채로 진입하면 에러 화면이 뜨고 "다시 시도" 버튼이 보인다. 사용자가 카테고리를 "전체"로 바꾸면 정상 목록으로 복귀한다.

네트워크 단절

지하철 진입 직전 진입

네트워크가 끊긴 채 진입하면 그리드 전체가 비고 "다시 시도" 안내가 뜬다. 자동 재시도는 하지 않고, 사용자가 신호를 회복한 뒤 직접 다시 시도한다.

새 클럽 만들기

+ 버튼으로 클럽 생성 진입

로그인 사용자에게만 화면 우하단 + 버튼이 보인다. 누르면 클럽 만들기 화면으로 이동한다. 비로그인 사용자에게는 + 버튼 자체가 보이지 않는다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
카테고리 칩 띠 전체/스포츠/독서/음악/여행 등 가로 스크롤 칩. "무엇을 둘러볼지" 사용자가 떠올린 단어 한 개로 1차 좁힘을 끝내게 한다.
검색바 한 줄 검색 입력. 클럽 이름·소개 부분일치. 카테고리만으로 좁힐 수 없는 특정 단어("러닝"·"보드게임")가 있는 사용자에게 별도 출구.
2열 그리드 카드 커버 이미지, 클럽명, 카테고리, "멤버 N명", 그리고 본인 상태 배지(멤버/신청 중/차단). 카드 한 장에서 "이 클럽이 어떤 곳인가"와 "내가 거기와 어떤 관계인가"를 동시에 인지하게 한다.
+ 버튼 로그인 사용자에게만 보이는 우하단 + 버튼. 가입자만 클럽을 만들 수 있다는 정책을 화면에서 분명히 드러낸다. 비로그인은 둘러볼 권리만.
빈 결과 안내 필터 적용 시 "검색 결과가 없습니다", 데이터 자체가 없으면 "아직 클럽이 없어요" + 만들기 유도. 두 빈 상태의 의미가 달라 안내 문구도 다르다 — 첫째는 검색 조건 조정, 둘째는 데이터 생성 유도.

운영자가 알아야 할 것

탐색 정책 결정점
  • 비로그인 진입 허용은 클럽 도메인 전환의 핵심 입구다. 잠재 사용자가 가입 전 "이 앱에 내가 들어갈 만한 클럽이 있는지"를 확인하지 못하면 가입 의사 자체가 형성되지 않는다.
  • 카드의 본인 상태 배지는 로그인 사용자에게만 의미가 있다. 비로그인 카드에는 배지 자리가 비어 있어야 자연스럽다. 한 화면에 두 가지 사용자 컨텍스트가 공존하므로 카피 검수 필요.
  • 카테고리 목록은 서버 정의를 따른다. 새 카테고리는 서버 쪽에서 먼저 추가되어야 화면 칩이 활성화된다. 운영 측이 새 카테고리를 켜기 전에 클럽 데이터를 채워 두지 않으면 빈 결과가 사용자에게 노출된다.
