F17-07. 고정형 결제 (지갑·계좌이체)

결제 대기 상태가 된 멤버가 코스 전체 금액을 한 번에 납부한다. 두 가지 방식 — 지갑(포인트) 결제는 즉시 차감되고 회계에 잡혀 자동으로 참가 확정. 계좌이체는 호스트가 직접 받는 외부 송금이라 회계에 잡히지 않고, 호스트가 입금을 확인해야 참가 확정으로 마감된다. 멤버 한 명에게는 동시에 살아 있는 결제가 최대 1건뿐이다.

두 방식의 회계상 차이

지갑 결제는 플랫폼이 중개하므로 매출·수수료·정산이 자동으로 잡힌다. 계좌이체는 호스트가 직접 받는 외부 송금이라 회계에 안 잡히고 정산 대상도 아니다(호스트 직거래). 그래서 환불도 지갑 결제는 자동, 계좌이체는 호스트가 직접 송금하고 "환불 완료"를 눌러야 마감된다.

두 가지 결제 방식

지갑 결제

즉시 차감 → 참가 확정

플랫폼 포인트에서 코스 전체 금액이 즉시 빠지고, 정산이 잡히는 거래 이력이 남는다. 잔액이 부족하면 충전 화면으로 자동 안내. 환불 시에는 같은 경로로 자동 복원된다.

계좌이체

호스트 직접 입금 확인

호스트가 사용자에게 별도로 알려준 계좌로 송금한다(플랫폼 외부). 호스트가 입금을 확인하면 참가 확정. 72시간 안에 입금 안 되면 자동으로 결제 만료되고 자리가 다음 대기자에게 넘어간다.

호스트 거절

잘못된 송금자 등

호스트가 "입금 확인 안 됨" 등 사유로 거절할 수 있다. 멤버 상태는 결제 대기로 유지되어 재결제가 가능하다.

결제 만료

72시간 안에 결제 안 함

자동으로 결제가 만료되고 자리가 비워진다. 가장 오래 기다린 대기자가 자동 승격된다.

한 멤버에게 동시에 살아 있는(진행중) 결제 행은 최대 1건뿐이다. 시스템이 중복 결제를 거절하므로 결제 화면이 두 번 열려도 두 번 결제되지 않는다. 거절·취소·환불이 끝난 결제는 비활성이라 같은 멤버가 새 결제를 다시 만들 수 있다.

운영자가 알아야 할 것

결제·환불 책임 분리
  • 지갑 결제는 정산 대상 — 호스트 수익으로 잡혀 추후 정산에 포함.
  • 계좌이체는 정산 대상 아님 — 호스트가 직접 받고 직접 환불하는 외부 거래.
  • 잔액 부족 → 충전 화면 — 사용자가 즉시 충전 후 재시도할 수 있게 안내.
  • 결제 만료 시 대기자 자동 승격 — 자리는 항상 다음 대기자에게 흘러간다.
  • 변동형은 본 화면 사용 안 함 — 회차마다 일반 이벤트 결제 흐름을 그대로 쓴다.

레드 플래그

자주 발생하는 케이스
  • 계좌이체 입금 확인 무대응 — 호스트가 확인을 안 하면 자리가 예약된 채로 72시간이 흘러간다. 다른 대기자가 못 들어옴.
  • 계좌이체 환불 마감 무대응 — 호스트가 송금만 하고 "환불 완료"를 안 누르면 멤버 입장에서 영구 "환불 진행중".
  • 지갑 잔액 부족 — 충전 화면 진입 후 다시 결제 화면으로 복귀하는 동선이 길어 이탈 위험.
