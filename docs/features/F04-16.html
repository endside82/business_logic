F04-16. 클럽 구독

클럽 소유자가 플랫폼에 운영비를 정기적으로 내는 단위. 사용자가 클럽을 구독하는 게 아니라 소유자가 플랫폼에 운영비를 지불하는 모델이며, 무료 클럽은 월 10,000 / 연 100,000원, 유료 클럽은 월 30,000 / 연 300,000원이다. 결제는 클럽 기금을 먼저 쓰고 부족분만 소유자 지갑에서 빠진다. 자동 갱신 결제가 세 번 실패하면 일시 정지 후 7일 유예를 거쳐 만료되며, 유료 클럽이 만료되면 공지 작성과 이벤트 게시가 차단된다.

설계 의도 — 운영비는 클럽 활동의 결과로 모인 돈에서 먼저 빠진다

클럽 운영비를 소유자 개인 지갑에서만 빼면, 활발한 클럽일수록 소유자가 돈을 더 많이 내야 하는 역설이 생긴다. 그래서 결제는 클럽 기금(멤버 기부로 모인 돈)을 우선적으로 사용하고, 기금이 부족할 때만 소유자 지갑에서 차감한다. 이 우선순위가 의미하는 것은 — 클럽이 멤버들에게 사랑받아 기부가 쌓이면 소유자의 부담이 줄어든다는 신호. 반대로 활동이 약해 기부가 멈추면 소유자가 운영비를 모두 부담해야 하므로, 클럽을 살리거나 닫는 결정을 자연스럽게 마주하게 된다.

사용자 시나리오

기본 흐름

새 클럽 소유자가 월 구독을 시작한다

방금 만든 클럽 상세 → "구독" → 플랜 카드(월간 10,000원·연간 100,000원) 두 개가 보임. 월간을 누르면 결제 수단 시트가 떠오른다 — 클럽 기금에 잔액이 없으므로 부족분 10,000원이 소유자 지갑에서 빠지고 그 자리에서 구독이 시작된다. 만료일은 한 달 뒤로 설정된다.

자동 갱신

만료 하루 전 자동으로 다음 달 결제

스케줄러가 만료 임박 구독을 찾아 갱신 결제를 시도한다. 성공하면 만료일이 한 달 더 늘어나고 별도 알림은 가지 않는다. 실패 시에만 사용자에게 알림이 도달한다.

이 흐름이 남기는 것 — 정상 갱신은 사용자에게 보이지 않는 흐름이어야 한다. 매달 "결제됐어요" 알림이 도달하면 사용자가 알림을 끄게 된다.

갱신 실패

자동 갱신이 잔액 부족으로 1회 실패

스케줄러가 결제를 시도했는데 기금과 지갑 합쳐도 부족하면 1회 실패로 기록된다. 소유자 폰에 "구독 자동 갱신 실패. 24시간 안에 재시도됩니다" 알림이 도달한다. 구독 상태는 그대로 이용 중.

일시 정지

결제가 세 번 연속 실패하면 일시 정지

세 번째 실패가 누적되면 구독이 일시 정지 상태로 바뀌고 7일 유예가 시작된다. "구독 일시 정지 / 7일 안에 결제 수단을 확인해 주세요" 알림이 도달한다.

재활성

소유자가 지갑을 충전한 뒤 재활성

구독 관리 화면에서 "재활성"을 누른다. 그 자리에서 같은 플랜으로 다시 결제가 시도되고, 성공하면 구독이 정상 이용 중으로 복귀한다. 만료일이 한 달 뒤로 갱신된다.

유예 종료

7일 유예가 지나도 결제가 안 됐다

스케줄러가 유예 종료 구독을 만료 상태로 바꾼다. 알림이 소유자 폰에 도달한다. 유료 클럽이라면 그 시점부터 공지 작성과 이벤트 게시가 차단되어, 멤버에게 운영 능력이 멈춘 사실이 자연스럽게 드러난다.

이 흐름이 남기는 것 — 운영비를 내지 않은 클럽이 멤버 알림을 계속 발송하는 모순을 막는다.

해지 예약

소유자가 자동 갱신을 취소

구독 관리 → "구독 해지" → 다이얼로그. 해지를 누르면 자동 갱신만 꺼지고 만료일까지는 정상 이용이 유지된다. 화면에는 "해지 예정 (만료일까지)" 배지가 떠오른다. 만료일이 도래하면 스케줄러가 만료 상태로 정리.

이 흐름이 남기는 것 — 이미 결제한 한 달은 끝까지 사용할 권리가 있다는 정책. 해지가 곧 환불을 의미하지 않는다.

플랜 변경

월간에서 연간으로 플랜을 바꾼다

이용 중인 월간 구독에서 "연간으로 변경" → 풀가격(100,000원) 결제 → 만료일이 1년 뒤로 갱신된다. 현재는 남아 있던 월간 잔여 기간에 대한 환불은 일어나지 않는다 — 사용자에게 분명히 안내되어야 하는 한계.

권한 가드

소유자가 아닌 사용자가 구독 관리에 접근

소유자가 아닌 사용자에게는 구독 관리 진입로 자체가 보이지 않는다. 플랜 조회 화면은 누구나 볼 수 있게 열려 있어, 가입 전 사용자도 "이 클럽이 어떤 등급에서 운영되는지" 확인할 수 있다.

플랜과 가격

플랜무료 클럽유료 클럽
월간10,000원 / 월30,000원 / 월
연간100,000원 / 년300,000원 / 년

연간 결제는 월간의 10개월 가격으로 책정되어 있어 2개월 분의 할인 효과가 있다. 유료 클럽은 가입비를 받는 운영 모델이므로 운영비가 더 높다.

운영자가 알아야 할 것 — 비즈니스 리스크

구독은 클럽 운영 모델의 골격
  • 기금 → 지갑 우선순위가 풀리면 활성 클럽일수록 손해라는 역설이 생긴다. 항상 기금을 먼저 쓰고 부족분만 지갑에서. 멤버 기부가 운영비를 줄이는 보상으로 작동해야 한다.
  • 유료 클럽 만료 시 공지·이벤트 차단은 모델의 일관성이다. 운영비를 내지 않는 클럽이 모든 기능을 계속 사용하면 누구도 구독을 갱신할 동기가 없어진다. 만료 시점에 자연스럽게 운영 능력이 막혀야 사용자가 갱신을 결정한다.
  • 자동 갱신 실패 알림은 단계별로 분리되어야 한다. 1회 실패 / 2회 실패 / 3회 실패(일시 정지) 알림 메시지가 같으면 사용자는 위기감을 인지하지 못한다. 단계가 올라갈수록 안내 강도도 올라가야 한다.
  • 해지가 즉시 종료가 아니라는 사실은 화면에 분명히 표시되어야 한다. "해지 = 환불"로 인지한 사용자가 만료일을 기다리지 않고 분쟁을 제기하는 케이스가 발생한다. "해지 예정 (만료일까지)" 배지가 핵심 가드.
  • 플랜 변경 시 잔여 환불 미구현은 현재의 한계로 명시되어야 한다. 월간에서 연간으로 바꿀 때 남은 월간 분이 풀가격에서 차감되지 않는 사실을 사용자에게 결제 직전에 안내하지 않으면 분쟁이 발생한다.
  • 외부 카드·간편결제 미연동. 결제 수단은 현재 기금과 소유자 지갑 두 곳뿐이다. 외부 결제 옵션이 화면에 표시되어 있다면 안내 카피 정비 필요.
