F04-02. 클럽 상세 · 가입 · 탈퇴

클럽 한 곳의 정보·CTA를 한 화면에 모은다. 가입 유형(자유/승인)과 클럽 유형(무료/유료)의 조합으로 가입 흐름이 분기하고, OWNER에게는 운영 진입점이, 비멤버에게는 가입 CTA가 노출된다. 탈퇴는 시점에 따라 가입비 환불(7일 내) 분기까지 책임진다.

가입 유형
2
자유·승인
시나리오
7
상태
ACTIVE/SUSPENDED

이 기능 안의 하위 항목

동작 1

상세 조회 + 역할별 CTA

비로그인/비멤버/멤버/관리자/OWNER로 분기.

동작 2

자유 가입

즉시 MEMBER, BUSINESS는 가입비 결제.

동작 3

승인 가입 신청

메시지 입력 → PENDING → 관리자 알림.

동작 4

탈퇴

OWNER 차단, 7일 내 가입비 환불 분기.

가입 분기

flowchart TD A["가입하기"] --> B{"가입 유형"} B -->|"자유 + 무료"| C["즉시 MEMBER"] B -->|"자유 + 유료"| D["MEMBER + 가입비 결제"] B -->|"승인"| E["PENDING + 관리자 알림"] B -->|"BAN 상태"| F["CLUB_BANNED 차단"]

사용자 시나리오 7개

S1

자유가입 즉시 가입

  1. “가입하기” → 즉시 MEMBER
  2. CTA가 “클럽 홈”으로 교체
S2

승인가입 신청

  1. 메시지 입력 → PENDING
  2. 관리자에게 CLUB_JOIN_REQUEST 푸시
S3 · 중복

이미 가입/신청/차단

  1. 각각 CLUB_ALREADY_MEMBER/APPLIED/BANNED 토스트
S4

탈퇴(30일 경과)

  1. 환불 미해당, DELETE + memberCount--
  2. OWNER에게 CLUB_MEMBER_LEFT 알림
S5

OWNER 탈퇴 시도

  1. UI에서 항목 제거
  2. 우회 시 CLUB_CANNOT_LEAVE_AS_OWNER
S6

BUSINESS 가입(가입비 결제)

  1. 가입 + 지갑 차감 + ClubFund 적립 + 분개
S7

SUSPENDED 클럽

  1. 전체 컨텐츠 차단 + 안내

서버 계약

MethodPath설명
GET/api/v1/clubs/{id}상세 + viewer 컨텍스트
POST/api/v1/clubs/{id}/join가입(자유/승인 분기)
DELETE/api/v1/clubs/{id}/leave탈퇴

자주 빠뜨리는 케이스

레드 플래그
  • OWNER 탈퇴 차단 누락 — 소유권 이전 없이 사라지면 클럽 좀비화.
  • 가입비 결제와 멤버 등록 트랜잭션 분리 — 한쪽만 성공 시 불일치.
  • 승인 신청 시 관리자 알림 누락 — 처리 지연.
