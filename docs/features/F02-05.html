F02-05. 검색 · 알림 진입점

홈 AppBar 우측의 두 아이콘이 검색·알림 도메인으로 보낸다. 본 단위는 라우팅과 트래킹 이벤트까지 담당하고, 나머지 화면 동작은 검색(F05)과 알림(F12) 단위가 책임진다. 비로그인 게스트가 알림 아이콘을 누르면 로그인 화면으로 안내된다.

이 단위가 책임지는 것

아이콘 탭 시점의 라우팅과 트래킹 이벤트 기록까지. 미읽음 배지·검색 입력 UI 같은 후속 동작은 각 도메인 단위가 처리한다.

이 기능 안의 하위 항목

아이콘 1

검색 진입

AppBar 검색 아이콘 탭 → 트래킹 이벤트 + 검색 화면 라우팅.

아이콘 2

알림 진입

AppBar 알림 아이콘 탭 → 트래킹 + 알림 목록 화면 라우팅. 비로그인은 로그인 화면으로 안내.

탭 흐름

사용자 시나리오 6개

S1

검색 진입

  1. 검색 아이콘 탭 → 트래킹 1건 기록 → 검색 화면 진입
S2

알림 진입

  1. 알림 아이콘 탭 → 트래킹 + 알림 목록 진입
S3 · 복귀

홈 탭으로 돌아오기

  1. 알림 화면에서 홈 탭 탭 또는 백버튼
  2. indexedStack에서 홈이 복원 → 스크롤 위치·데이터 유지
  3. 추가 API 호출 없음
S4 · 게스트

비로그인 알림 탭

  1. 게스트가 알림 아이콘 탭
  2. publicRoute에 알림 미포함 → 라우터가 로그인 화면으로 redirect
  3. 로그인 후 다시 알림 화면 진입
S5 · 미확인 영역

미읽음 배지(현재 미구현)

  1. 서버상 미읽음 5건이라도 현재 홈 AppBar에는 배지 위젯이 보이지 않음
  2. 탭하면 알림 화면에 진입해 배지/카운트 확인 가능

후속 검토: F12에서 배지 위젯 추가 여부.

S6 · 트래킹

전환율 측정 데이터 흐름

  1. 검색/알림 탭 시 각각 1건의 트래킹 이벤트 기록
  2. BI에서 홈 → 검색 / 홈 → 알림 전환율 측정

수용 기준 (AC)

ACGivenWhenThen
AC-01로그인 사용자검색 아이콘 탭트래킹 + /search 진입
AC-02로그인 사용자알림 아이콘 탭트래킹 + /notifications 진입
AC-03다른 탭에 있음홈 탭 탭indexedStack 복원, 호출 없음
AC-04게스트알림 아이콘 탭로그인 화면으로 redirect
AC-05각 탭 탭BI 수신각각 1건 이벤트 기록

자주 빠뜨리는 케이스

레드 플래그
  • 게스트 알림 탭이 401 화이트스크린 — 로그인 가드를 반드시 통과.
  • 탭 전환 시 홈 데이터 reset — indexedStack 보존이 깨지면 매번 4섹션 재호출 발생.
  • 트래킹 이벤트 누락 — BI 측정 데이터 손실.
