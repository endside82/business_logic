F02-05. 검색·알림 진입점

홈 화면 상단에 늘 자리하는 두 아이콘 — 검색과 알림 — 의 의미와 도착지를 정의한다. 본 단위는 두 진입점에 한정되며, 실제 검색 동작과 알림 목록은 각 도메인이 담당한다. 홈에서 어디까지 스크롤했든 두 아이콘은 항상 화면 상단 같은 자리에 있어, 검색과 알림으로 한 번에 빠져나갈 수 있다.

설계 의도 — 둘러보기와 명시적 탐색은 다른 동선이어야 한다

홈은 "무엇이든 한 번 둘러보자"의 면이고, 검색은 "찾을 게 있다"의 면이다. 이 둘이 같은 동선이 되면 사용자는 "둘러봐도 되고, 찾아도 된다"는 자유를 잃는다. 그래서 검색은 별도 화면으로 분리되고, 홈의 상단 아이콘은 그 화면으로 빠져나가는 일정한 출구로 자리한다. 알림도 마찬가지다 — "지금 무슨 일이 있었는지"는 둘러보기와 다른 호흡이다.

사용자 시나리오

검색 진입

특정 단어를 떠올렸을 때

둘러보다가 "와인" 같은 단어가 떠오른 사용자가 상단 검색 아이콘을 누른다. 검색 화면이 열리고, 홈 화면 상태는 보존되어 검색을 마치고 돌아오면 그대로다.

알림 진입

새 알림이 왔는지 확인

알림 아이콘을 눌러 알림 목록으로 빠진다. 새로운 댓글·신청 결과·정산 안내 같은 일들이 한 곳에 모여 있다.

가입 전

비로그인 사용자가 알림 아이콘을 누름

알림은 회원 전용 정보이므로, 비로그인 상태로 누르면 로그인 화면으로 안내된다. 검색은 로그인 없이도 사용 가능하다 — 가입 결정 전에도 "이 앱에 내가 찾는 게 있는지" 확인할 수 있어야 가입 의사가 형성된다.

운영자가 알아야 할 것

두 진입점 정책
  • 알림 아이콘에는 새 알림 표시(읽지 않은 개수)가 들어갈 자리가 있다. 현재는 아이콘 본체만 노출되고 미읽음 표시는 알림 도메인 측 작업 항목으로 남아 있다. 운영 측이 "새 알림이 와도 사용자가 바로 인지하지 못한다"는 신호를 분석에서 발견하면 이 항목의 우선순위가 올라간다.
  • 홈 → 검색, 홈 → 알림 전환은 측정한다. 어디서 사용자가 둘러보기를 멈추고 명시적 탐색·확인으로 넘어가는지가 추천·콘텐츠 전략의 핵심 신호.
  • 탭 전환·뒤로가기로 홈에 돌아왔을 때는 추가 호출 없이 즉시 표시된다. 스크롤 위치·로딩 상태를 잃지 않는 것이 본 진입점들이 유지되어야 하는 이유 중 하나.
