F17-03. 정기모임 생성 (호스트)

호스트가 정기모임 본체를 만드는 면. 무조건 초안 상태로 먼저 저장한 뒤, 회차를 다 꾸리고 가격·정원·정책을 확정한 다음 별도로 발행한다. 고정형은 코스 전체 회차 수·정원·가격이 필수이고, 변동형은 회차마다 따로 정의되므로 본체에서는 비워둘 수 있다. 한 번 정한 호스트는 끝까지 바꿀 수 없다 — 정산 수익자와 환불 책임자를 흔들지 않기 위한 결정이다.

항상 초안
발행은 별도 단계
가격 단위
100원
100원 미만 자동 절삭

설계 의도 — 만든 사람이 끝까지 책임진다, 그 사이 자유롭게 다듬는다

정기모임은 코스 단위 결제·정산·환불을 동반하므로 호스트가 한 번 정해지면 못 바꾼다. 정산 수익자와 환불 책임자가 흔들리는 순간 분쟁이 일어나기 때문이다. 반면 초안 단계는 자유롭다 — 발행 전까지 회차·가격·정책을 자유롭게 손볼 수 있고 운영중 사용자에게 영향이 없다. 모임을 "확정"하는 순간(발행)을 별도 단계로 분리해 호스트가 충분히 준비된 다음에 사용자에게 노출되도록 한다.

사용자 시나리오

고정형 코스 생성

"8주 러닝 코스" 만들기

호스트가 운영방식을 "고정형"으로 선택. 총 회차 8, 기본 정원 10, 코스 전체 가격 80,000원, 승인제 입력. 저장 누르면 초안으로 들어가고 회차 추가 화면으로 이어진다.

이 흐름이 남기는 것 — 호스트가 회차 셋업 전에 본체 정책(가격·정원·승인)을 먼저 결정하게 해, 회차 추가가 일관성을 잃지 않는다.

변동형 모임 생성

"매달 다른 주제 독서 모임"

운영방식을 "변동형"으로 선택. 총 회차·정원·가격 필드는 비워둔다(회차마다 따로 정의). 승인제·노쇼 정책 정도만 본체에 잡고 저장.

이 흐름이 남기는 것 — 회차마다 운영을 다르게 가져가야 하는 모임에 필요한 자유도. 본체는 "공통 정책 컨테이너" 역할만 한다.

가격 자동 절삭

12,345원 입력 → 12,300원 저장

가격 입력 후 저장. 100원 미만 단위는 자동 절삭되어 12,300원으로 저장된다. 상세 화면에서 저장된 가격을 다시 보여 줘 사용자가 인지하게 한다.

이 흐름이 남기는 것 — 원 단위 미세 금액을 막아 환불 산식이 깨끗하게 떨어지게 한다(일반 이벤트와 동일).

필수값 누락

고정형인데 회차 수가 비어 있음

고정형 + 총 회차 수가 비어 있으면 저장이 거절되고 "고정형은 총 회차 수가 필요해요" 안내.

이 흐름이 남기는 것 — 고정형의 핵심(코스 전체 규모) 확정이 누락된 채 발행으로 흘러가지 않게 한다.

온라인 모임

장소를 온라인으로 선택 + URL 입력

장소 종류가 온라인이면 URL 입력칸이 활성화. URL 형식이 아니면 "올바른 URL 형식이 아닙니다" 안내.

이 흐름이 남기는 것 — 회차마다 URL을 다시 입력하지 않게, 본체에서 한 번 정해 회차 추가 시 자동 상속 가능.

노쇼 정책 기본

아무 선택 안 해도 저장됨

노쇼 정책을 지정하지 않으면 "없음"으로 저장된다. 호스트가 명시적으로 "경고" 또는 "자격박탈"을 선택해야 한도 추적이 켜진다.

이 흐름이 남기는 것 — 노쇼 정책이 강한 모임은 의도적 선택이어야 한다(자동 활성화 시 분쟁 가능).

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
운영방식 토글"고정형/변동형" 라디오. 선택에 따라 하단 필드가 달라진다.본체 폼이 운영방식에 따라 달라지므로 가장 먼저 결정하게 한다.
총 회차·정원고정형에서만 활성. 변동형에서는 비활성·회색.변동형 본체에 입력해도 의미 없는 값이라 폼 단에서 비활성화하는 게 사용자 혼동을 줄인다.
가격입력 후 100원 단위 안내. 저장 직후 절삭된 값으로 표기."내가 입력한 값과 저장된 값이 다르네?" 혼동을 막기 위해 자동 절삭을 명시 안내.
승인제·선결제토글 두 개. 본체에서 한 번 결정.모임 전체의 운영 방침이라 회차마다 바꿀 수 없게 본체로 묶는다.
노쇼 정책"없음·경고·자격박탈" + 한도 카운트 방식 선택.현금성 박탈 정책이라 명시적 선택을 요구해 분쟁 가능성을 낮춘다.
저장 후 안내저장 직후 회차 추가 화면으로 자연스럽게 이어진다."본체만 만들어 두고 발행을 잊는" 케이스를 줄이기 위해 다음 단계로 흐름 연결.

운영자가 알아야 할 것

초안 단계의 자유도와 호스트 제약
  • 호스트당 동시 초안 수 제한 없음 — 1차 출시는 quota 없이 자유롭게. 운영상 청소 정책은 후속.
  • 호스트 교체 불가 — 발행 후뿐 아니라 초안 단계에서도 불가. 모임을 다른 사람에게 넘기려면 새 모임을 만들어야 한다.
  • 변동형 본체에 가격·정원 입력은 무시된다 — 회차마다 따로 정의되므로 본체 값은 의미 없음. 폼에서 비활성 처리 권장.
  • 환불 정책 기본은 "전액" — 호스트가 부분 환불 정책을 명시적으로 선택하면 적용.

레드 플래그

자주 발생하는 케이스
  • 가격 절삭 혼동 — 입력값과 저장값이 다르므로 저장 직후 상세에서 절삭된 가격이 노출되어야 한다. 누락 시 호스트가 "왜 가격이 줄었지?" 혼동.
  • 변동형인데 본체에 가격을 적은 모임 — 사용자가 카드 단계에서 본체 가격을 "코스 총액"으로 오인할 수 있다. 변동형에서는 입력 자체가 막혀야 함.
  • 초안 많이 만들어 놓고 발행 안 함 — 호스트당 초안 quota 없어 청소가 안 됨. 운영 모니터링 필요.
