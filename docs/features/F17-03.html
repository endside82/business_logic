F17-03. 정기모임 생성 (호스트)

항상 DRAFT로 시작하고 별도 publish 호출(F17-04)에서 발행한다. FIXED는 totalSessionCount>0, baseCapacity>0 불변식이 강제되고, 가격은 100원 단위로 자동 절삭(RoundingMode.DOWN)된다. VARIABLE은 세션별로 재정의되므로 본체 필드 0/null 허용.

초기 상태
DRAFT
가격 정규화
100원 floor
Event.price와 동일 정책
호스트 교체
불가

FIXED 불변식

  • totalSessionCount > 0 — 위반 시 INVALID_INPUT
  • baseCapacity > 0 — 위반 시 INVALID_INPUT
  • publish 시점: 활성 세션 수 == totalSessionCount (F17-04 §3.1)

서버 계약

POST /api/v1/regular-meetings — 인증 필수. Body RegularMeetingAddParam: title, description, category, meetingType, locationType, address, addressDetail, latitude, longitude, onlineUrl, startDate, endDate, totalSessionCount, price, prepaymentRequired, refundPolicy, refundDeadlineHours, baseCapacity, approvalRequired, noShowPolicy, noShowLimit, noShowCountMode, thumbnailUrl. 응답 201 RegularMeetingVo (status=DRAFT).

도메인 builder가 status 기본값 DRAFT, refundPolicy 기본값 FULL, noShowPolicy 기본값 NONE을 강제한다. 외부에서 status 강제 불가.
