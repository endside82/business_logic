F17-03. 정기모임 생성 (호스트)

호스트가 모임 본체를 만든다. 무조건 초안 상태로 저장돼서, 회차를 다 꾸리고 가격·정원을 확정한 다음 별도로 발행한다. 고정형은 코스 전체 회차 수·정원·가격이 필수이고, 변동형은 회차마다 따로 정의되므로 본체에서는 비워둘 수 있다.

설계 의도 — 모임을 만든 사람이 끝까지 책임진다

정기모임은 코스 단위 결제·정산·환불을 동반하므로 호스트가 한 번 정해지면 못 바꾼다. 정산 수익자와 환불 책임자가 흔들리는 순간 분쟁이 일어나기 때문이다. 또 모임은 항상 초안으로 시작하므로, 발행 전까지는 회차·가격·정책을 자유롭게 손볼 수 있고 운영중 사용자에게 영향이 없다.

운영자가 알아야 할 것

초안 단계의 자유도
  • 호스트는 초안을 여러 개 동시에 가질 수 있다 — 1차 출시에는 호스트당 모임 수 제한이 없다.
  • 온라인 모임 URL은 형식 검증을 거친다 — 잘못된 형식이면 가입 단계에서 안내.
  • 노쇼 정책 기본은 "없음" — 호스트가 명시적으로 선택해야 한도·강제박탈 정책이 켜진다.
  • 환불 정책 기본은 "전액" — 호스트가 명시적으로 부분 환불 정책을 선택할 수 있다.

레드 플래그

자주 발생하는 케이스
  • 변동형 본체에 가격·정원 입력 — 의미 없는 값이라 무시되지만 사용자는 입력 가능. 폼에서 비활성 처리 권장.
  • 가격 절삭 혼동 — 입력값과 저장값이 다를 수 있으므로 저장 직후 상세에서 절삭된 가격을 노출해야 함.
  • 호스트당 초안 quota 없음 — 동시에 다수 초안을 만들 수 있어 운영상 청소가 필요할 수 있음.
