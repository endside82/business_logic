F03-11. 위시리스트 (찜)

하트 탭 한 번으로 추가·해제되는 가장 가벼운 사용자 액션. 낙관적 토글로 즉시 시각 피드백을 주고, 실패하면 자연스럽게 롤백된다. 이벤트 상태(종료·취소·PRIVATE)와 멤버십 가드가 서버에서 추가로 검증된다.

이 기능 안의 하위 항목

동작 1

낙관적 토글

탭 즉시 UI 변경 + 서버 호출. 실패 시 캐시 재조회로 자연 롤백.

동작 2

위시리스트 목록

페이징 + 풀투리프레시. 낙관적 제거.

동작 3

상세 캐시 동기화

토글 시 detail provider invalidate로 두 화면 일관성 유지.

사용자 시나리오 9개

S1

검색 결과에서 첫 찜

  1. 회색 하트 탭 → 즉시 빨강
  2. 서버 INSERT → 토스트 “찜했어요”
S2

위시리스트 화면에서 해제

  1. 하트 탭 → 즉시 리스트에서 제거
  2. 서버 DELETE → 토스트 “찜 해제했어요”
S3

종료된 이벤트 찜 시도

  1. 낙관 patch 후 서버 400 EVENT_ALREADY_CLOSED
  2. detail 재조회로 롤백 + 토스트
S4

PRIVATE 이벤트 비멤버 시도

  1. 서버 403 → 낙관 롤백 + 토스트
S5

PRIVATE 멤버 통과

  1. 멤버십 통과 → INSERT 성공
S6

이미 찜한 이벤트 재요청

  1. 서버 idempotent: 204 동일
  2. 중복 행 없음
S7

무한 스크롤

  1. 20건씩 누적, last=true 시 추가 호출 차단
S8

풀투리프레시

  1. 0페이지부터 재로드 → 다른 디바이스 변경 반영
S9

네트워크 단절 중 토글

  1. 낙관 patch → 서버 실패 → 캐시 재조회로 자연 롤백
  2. 토스트로 안내

자주 빠뜨리는 케이스

레드 플래그
  • 낙관 patch만 하고 실패 시 롤백 안 함 — 실제 상태와 어긋남.
  • 두 화면(상세/목록) 불일치 — invalidate 누락.
  • 서버에서 멱등 보장 안 함 — 중복 행 생성.
  • PRIVATE 가드 누락 — 비멤버가 찜 가능해지면 가시성 정책 위반.
