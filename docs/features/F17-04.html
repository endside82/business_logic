F17-04. 정기모임 생명주기 (발행·종료·취소·재오픈)

모임 본체는 초안 → 모집 중 → 종료(또는 어디서든 취소)로 흘러간다. 종료는 모든 회차의 출석이 확정된 뒤에만 통과하고, 취소는 멤버 전원에게 자동 환불을 발사한다. 정산은 종료가 확정된 다음 별도 단계로 만들어져, "정산 후 새 환불"이 생기는 사고를 막는다.

취소는 종착 상태
머니 안전 — 종료와 정산 사이에 빈틈 없음

종료는 모든 회차의 출석이 확정된 뒤에만 통과한다(미확정 회차가 하나라도 있으면 막힘). 종료가 확정된 직후 정산이 만들어지고, 그 사이 멤버 자가 취소는 차단된다. 자격박탈로 인한 환불은 회차 확정 시점(=종료 이전)에만 발생하므로 정산 금액에 자연 반영된다 — 정산이 만들어진 뒤에 새 환불이 발생할 가능성이 0이다.

각 액션의 사용자 의미

발행

모집을 연다

고정형은 회차가 약속한 수만큼 채워져 있어야 한다(8회 코스라면 8회차 등록 완료). 발행되면 일반 검색에 노출되고 사용자 신청이 들어오기 시작한다.

종료

모든 회차 출석 확정 후에만

고정형은 미래에 잡힌 회차가 없고 모든 회차의 출석이 확정돼야 한다. 종료가 확정된 직후 정산이 만들어진다.

취소

멤버 전원 자동 환불

호스트가 모임을 접는 행동. 활성 멤버 전원에게 "호스트 취소" 사유로 비례 환불을 발사한다(이미 제공된 회차분은 제외). 미래 회차도 자동 취소된다. 한 번 취소된 모임은 되살릴 수 없다.

재오픈

종료를 풀어 다시 모집

주로 변동형의 자동 종료를 풀거나, 호스트가 코스를 더 연장하고 싶을 때 사용. 기존에 취소된 회차는 되살아나지 않는다.

서버 계약

MethodPath사용자 의미
POST/api/v1/regular-meetings/{id}/publish초안 → 모집 중
POST/api/v1/regular-meetings/{id}/close모집 중 → 종료 (모든 회차 확정 필수)
POST/api/v1/regular-meetings/{id}/cancel초안/모집 중 → 취소 + 멤버 전원 환불
POST/api/v1/regular-meetings/{id}/reopen종료 → 모집 중 (취소된 회차는 되살리지 않음)
PATCH/api/v1/regular-meetings/{id}본체 수정 (초안만)
DELETE/api/v1/regular-meetings/{id}본체 삭제 (초안만, 영구 삭제)

모든 액션은 호스트 본인만 호출 가능. 다른 사용자가 시도하면 권한 없음으로 거절된다.

운영자가 알아야 할 것

현금 가드
  • 종료가 막혀 있으면 출석 미확정 회차가 남은 것 — 호스트에게 어떤 회차가 남았는지 안내.
  • 취소 환불은 비동기로 멤버마다 따로 진행 — 일부 실패해도 모임 취소 자체는 완료되고, 실패한 환불은 별도 큐로 들어가 자동 재시도된다.
  • 재오픈은 변동형 위주 — 고정형도 호출은 가능하나, 회차 확정을 다시 갈아엎고 닫아야 하므로 사실상 운영 케이스가 드물다.
  • 본체 수정은 초안 단계만 — 발행 후에는 수정 불가. 노출 정보가 흔들리지 않도록.
  • 본체 삭제도 초안만 — 발행된 모임은 멤버·결제·회차 이력 보존 의무로 삭제 불가.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 호스트가 출석 확정을 미루면 종료 버튼이 막혀 있음 — 다음 회차로 못 넘어가는 멤버 항의 가능. 자동 마감 스케줄러는 기본 비활성.
  • 취소 환불 일부 실패 — 자동 재시도가 돌지만 사용자 입장에서는 환불이 늦어 보일 수 있음. 안내 필요.
  • 재오픈 후 회차 추가 → 다시 종료 — 정산이 멱등이라 한 번만 생성되지만, 호스트 입장에서 "정산이 안 늘었네?" 혼동 가능.
