F17-04. 정기모임 생명주기 (publish / close / cancel / reopen)

모임 본체 4상태 머신. close는 정산 트리거이고 cancel은 일괄 호스트 환불을 일으킨다. 모든 전이는 도메인 메서드 transitionTo(target)로 통과하며, close/cancel은 findByIdForUpdate로 잠궈서 정산/환불 경합을 차단한다.

상태 머신

flowchart LR A[DRAFT] -->|publish| B[OPEN] A -->|cancel| C[CANCELED] B -->|close| D[CLOSED] B -->|cancel| C D -->|reopen| B

CANCELED는 terminal. 허용되지 않은 전이는 REGULAR_MEETING_INVALID_STATUS.

close (OPEN → CLOSED) 가드

  • findByIdForUpdate(meetingId) 락 — pay와 직렬화
  • FIXED: 미래 OPEN 세션 무 + activeSessionCount == totalSessionCount
  • FIXED: 모든 정션 attendanceFinalizationStatus == FINALIZED (F17-09 finalize 완료)
  • 통과 시 afterCommit 동기화 등록 → tryCreateSettlement(REQUIRES_NEW) 호출(F17-10)
  • 실패해도 close 자체는 롤백 안 됨. 1h failsafe 스케줄러가 재시도

cancel (DRAFT|OPEN → CANCELED) 흐름

  • findByIdForUpdate 락 (close 정산 스냅샷과 cancel REQUIRES_NEW 환불 경합 차단)
  • ACTIVE 멤버 전원 HOST_CANCEL pro-rata 환불 (F17-08)
  • 미래 활성 세션 일괄 CANCELED
  • CANCELED는 terminal — 재개 불가
