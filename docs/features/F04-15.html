F04-15. 클럽 기금 인출 요청

OWNER가 클럽 기금에서 본인 지갑으로 인출을 요청한다. 가용 잔액(잔액 − PENDING 인출 − 환불 예비금)을 미리 계산해 보여주고, 월 1회 + 최소 10,000원 정책을 따른다. 실제 송금은 관리자 승인 시점에 일어난다.

하위 동작
2
시나리오
6+
수수료
5% + 원천 3.3%
제한
월 1회 · 최소 1만원

이 기능 안의 하위 항목

동작 1

인출 요청(OWNER)

가용 잔액 검증 + PENDING 등록. 잔액 차감은 승인 시점에.

동작 2

관리자 승인 → 지갑 입금

4중 분개 + 채무 상계 + 지갑 입금 + 푸시.

사용자 시나리오 6개

S1

10만원 인출 요청

  1. 수수료·실수령액 실시간 계산
  2. PENDING INSERT
S2

비OWNER 시도

  1. UI 미노출
  2. 우회 시 CLUB_NOT_OWNER
S3

가용 잔액 부족

  1. 잔액 − PENDING 인출 − 환불 예비금 계산
  2. 부족 시 INSUFFICIENT_BALANCE + 부족액 안내
S4

월 1회 제한

  1. 이미 PENDING/PAID 있음 → DUPLICATE_TRANSACTION
S5

최소 금액 미달

  1. UI 가드, 우회 시 INVALID_PAYMENT_AMOUNT
S6

관리자 승인 후 지갑 입금

  1. 기금 차감 + 채무 상계 + 지갑 적립 + 4중 분개
  2. WITHDRAWAL_PAID 푸시

서버 계약

MethodPath
POST/api/v1/clubs/{id}/fund/withdraw
GET/api/v1/clubs/{id}/fund/withdrawals?page=&size=
