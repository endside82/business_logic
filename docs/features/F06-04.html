F06-04. 결제 수단 관리

PG 결제·자동충전·구독 결제에 쓰일 카드 등 수단을 사용자당 최대 5개 보관. 기본 수단은 항상 1개 존재한다.

이 기능 안의 하위 항목

동작 1

등록 (첫 등록은 기본)

동작 2

기본 전환

동작 3

별칭 수정

동작 4

삭제 (자동충전 연결 시 차단)

사용자 시나리오

S1

첫 등록

  1. count=0이므로 자동 기본
S2

두 번째 등록 + 기본 전환

  1. 새 카드 PATCH default → 이전 기본 해제
S3

5개 초과

  1. 한도 안내
S4

기본 카드 삭제 시도

  1. UI 차단, 우회 시 INVALID
S5

자동충전 연결 카드 삭제

  1. 자동충전 비활성 안내
S6

별칭 수정

서버 계약

MethodPath
GET/api/v1/wallet/payment-methods
POST/api/v1/wallet/payment-methods
PATCH/api/v1/wallet/payment-methods/{id}/default
