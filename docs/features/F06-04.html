F06-04. 결제 수단 관리

충전(F06-02)과 자동 충전(F06-05)에 쓰이는 카드를 등록·삭제·기본 설정하는 자리. 기본으로 정한 카드 한 장은 충전 화면에서 이미 선택된 채로 떠올라, 사용자가 매번 카드를 다시 고르지 않게 한다. 최대 다섯 장까지 등록할 수 있고, 한 장을 기본으로 둔다. 본 화면은 결제를 일으키지 않는다 — 등록한 카드는 다음 충전 시점에 처음으로 쓰인다.

최대 등록 장수
5장
기본 결제수단은 항상 1장 지정
기본 카드 자동 승계
있음
기본 카드를 지우면 남은 카드가 기본이 됨
설계 의도 — 충전 화면에서 카드를 고민하지 않게 한다

매 충전마다 카드를 다시 고르게 하면 충전 빈도가 떨어진다. 기본 카드 한 장을 미리 정해 두면 충전 화면이 그 카드를 이미 선택한 상태로 떠올라, 사용자는 금액만 정하고 결제 창으로 넘어간다. 여러 장을 둘 수 있는 이유는 카드 한 장이 만료되거나 한도 초과로 막혔을 때 다른 카드로 즉시 갈아탈 수 있게 하기 위해서다. 기본 카드를 지우면 남은 카드 중 하나가 자동으로 기본이 되도록 두어, 사용자가 "기본 카드가 없어 충전이 막히는" 경험을 하지 않게 한다.

사용자 시나리오

첫 등록

첫 결제수단을 추가한다

가입 직후 카드가 한 장도 없는 사용자. "결제 수단 추가"를 누르면 카드 입력 시트가 떠오르고, 카드사·뒷 4자리·별칭(예: "급여 카드")을 받는다. 등록을 마치면 첫 카드는 자동으로 기본으로 설정된다 — 기본 카드 지정을 사용자에게 별도로 묻지 않는다.

이 흐름이 남기는 것 — 사용자가 "기본 설정"이라는 추가 결정 없이 충전을 시작할 수 있게 한다.

두 번째 카드

새 카드를 추가하고 그것을 기본으로 바꾼다

기본 카드(A)를 가진 사용자가 새 카드(B)를 추가한다. B는 일단 기본이 아닌 상태로 들어온다. 사용자가 B의 메뉴 → "기본으로 설정"을 누르면 A의 기본 표시가 해제되고 B가 기본이 된다 — 기본은 항상 한 장만 존재.

이 흐름이 남기는 것 — 사용자가 충전에 자주 쓰는 카드를 명시적으로 결정할 수 있게 한다.

기본 카드 정리

만료된 기본 카드를 삭제 — 다른 카드가 자동 기본 승계

기본 카드(A)가 만료되어 정리하려는 사용자. A를 삭제하면 남은 카드(B 또는 C 중 하나)가 자동으로 기본이 된다. "기본 카드가 없는 상태"가 잠시도 생기지 않게 — 그 사이에 충전 시도가 들어와 막히는 사고를 막는다.

이 흐름이 남기는 것 — 사용자가 카드 정리 후 충전 화면에 들어왔을 때 별도로 기본 카드를 다시 정하지 않아도 된다.

한도

다섯 장이 채워진 상태에서 추가하려 함

이미 다섯 장이 등록된 사용자에게는 화면 하단의 "결제 수단 추가" 버튼 자체가 안 보인다 — 사용자가 시도하고 거절되는 헛걸음을 막는 정책. 한 장을 정리한 뒤에야 추가 버튼이 다시 나타난다.

자동 충전 영향

자동 충전이 묶여 있던 카드를 지움

자동 충전(F06-05)이 카드 A에 묶여 있는데 A를 삭제한다. 결제수단 화면 자체는 정상적으로 삭제되지만, 다음 자동 충전 시점에 "카드를 찾을 수 없다"는 이유로 자동 충전이 동작하지 않는다. 사용자가 결제 시점에 잔액 부족을 인지하고 자동 충전 설정으로 돌아가 새 카드를 지정해 줘야 정상화된다.

이 흐름이 남기는 것 — 본 화면이 자동 충전 설정까지 연결되어 변경하지 않는다 — 사용자의 명시적 결정으로 자동 충전을 재설정하게 둔다.

마지막 카드

마지막 한 장을 지운다

한 장만 남은 카드를 사용자가 삭제한다. 결제수단 0장 상태가 되고, 다음 충전 시도에서 "결제수단을 등록해 주세요" 안내로 본 화면이 다시 떠오른다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
등록 개수 표기 "등록된 결제 수단 (N/5)" 카운터가 화면 상단에. 사용자가 한도까지 얼마나 남았는지 인지. 5장이 되면 추가 버튼이 사라진다는 정책을 사용자가 미리 학습.
카드 카드 카드사 + 뒷 4자리 마스킹 + 사용자 별칭 + 기본 배지(기본 카드 한 장). 사용자가 여러 카드를 가진 경우 별칭으로 즉시 구분. 기본 배지는 충전 화면에서 어느 카드가 자동 선택될지를 사용자가 시각적으로 미리 확인.
카드 메뉴 각 카드에 점 세 개 메뉴 → "기본으로 설정" / "삭제". 액션 두 개를 메뉴에 넣어 카드 본체를 깔끔하게 유지. 삭제는 확인 다이얼로그를 거쳐 한 번 더 사용자 의사를 받는다.
빈 상태 카드 일러스트 + "등록된 결제 수단이 없습니다" 안내. 처음 진입한 사용자에게 "여기는 카드를 등록하는 자리"라는 사실을 즉시 인지하게.
추가 버튼 화면 하단 "결제 수단 추가". 5장이 채워지면 사라진다. 사용자가 추가할 수 있을 때만 보여서, 시도-거절의 헛걸음을 막는다.

운영자가 알아야 할 것

결제수단 화면의 정책 결정점
  • 기본 카드는 항상 한 장이 존재해야 한다. 기본 카드가 없는 상태가 잠시라도 생기면, 그 사이에 들어온 충전 시도가 카드 선택 단계에서 막힌다. 자동 승계 동작이 깨지는 회귀가 들어가면 충전 전환이 즉시 떨어진다.
  • 결제수단 화면의 카드 삭제는 자동 충전 설정을 자동으로 변경하지 않는다. 사용자가 자동 충전에 묶인 카드를 지웠을 때, 그 사실을 본 화면에서 경고하지 않는다는 점은 운영팀이 알고 있어야 한다 — 사용자 신고("자동 충전이 안 됐다") 발생 시 카드 변경 이력과 자동 충전 설정의 카드 식별자를 함께 봐야 한다.
  • 최대 다섯 장 한도는 사용자 사고 방지용이다. 여러 장이 필요한 사용자가 거의 없다는 가정 아래의 보수적 한도. 비즈니스 카드 사용이 늘어나면 한도 상향을 검토.
  • 카드 별칭은 사용자가 자유 입력한다. 잘못 입력한 별칭(예: 카드 번호 전체)이 그대로 저장될 수 있다는 점은 운영 안내·검증 메시지에서 다뤄야 할 영역.
