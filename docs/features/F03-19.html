F03-19. 일정 변경 제안·참가자 합의 (RS-002)

호스트가 이벤트 일정을 바꿀 때, 변경 규모에 따라 즉시 반영(AUTO)과 48시간 합의(MAJOR) 중 하나로 처리된다. MAJOR 변경이면 참가자 한 명 한 명에게 제안이 발송되고, 참가자는 동의 또는 거절을 선택한다. 거절하면 무료 취소 + 100% 환불이 즉시 발생한다. 모두 응답하면 호스트가 최종 확정을 누른다.

설계 의도 — 큰 변경은 사용자 동의 없이 강행할 수 없다

일정 변경은 언뜻 호스트 권한처럼 보이지만, 1시간 이상 시간이 밀리거나 장소가 바뀌거나 가격이 오르면 참가자 입장에서는 "다른 이벤트"에 가까워진다. 이 경우 호스트가 일방적으로 바꿀 수 없도록 참가자에게 48시간 응답 기회를 준다. 거절하면 패널티 없이 나갈 수 있고, 48시간 내 무응답은 수락으로 간주한다. 경미한 변경(60분 미만·장소 그대로·가격 동결)은 즉시 반영해 운영 마찰을 줄인다.

기본 흐름 — MAJOR

장소가 바뀌어서 참가 여부를 다시 결정해야 한다

호스트가 집결지를 A에서 B로 바꾼다. 앱은 "장소 변경 포함 — 큰 변경입니다" 배너를 수정 화면에 표시하고, 저장하면 참가자 전원에게 "일정 변경 제안" 알림이 간다. 참가자는 변경 전/후 장소를 나란히 보면서 동의 또는 거절을 선택한다.

이 흐름이 남기는 것 — 호스트가 변경을 강행하는 대신, 참가자에게 "이 변경으로도 참가할 건지" 선택권을 준다. 거절해도 패널티가 없다.

거절 → 환불

변경된 장소가 너무 멀어서 거절하고 나간다

참가자가 "거절"을 누르면 참가가 자동으로 취소되고 결제 금액이 100% 환불된다. 환불 처리에 수수료가 붙지 않는다. "참가가 취소되고 환불이 처리됩니다"라는 확인 화면이 한 번 더 나온다.

이 흐름이 남기는 것 — 변경을 원하지 않는 사람이 손해 없이 빠질 수 있어야 분쟁이 줄어든다.

48시간 무응답

알림을 놓쳐 응답 기한을 넘겼다

48시간이 지나도 응답하지 않으면 시스템이 자동 수락으로 처리한다. 이후 응답을 시도하면 "기한이 지났습니다" 안내가 나온다.

이 흐름이 남기는 것 — 기한이 있어야 호스트가 무기한 대기 없이 변경을 확정할 수 있다.

호스트 — 현황 확인

누가 동의했고 누가 아직 안 답했는지 확인

호스트는 "일정 변경 제안 현황" 화면에서 동의/거절/미응답/자동수락 인원을 실시간으로 확인한다. 전원 응답 완료 후 "확정" 버튼이 활성화되고, 누르면 이벤트가 변경된 일정으로 업데이트된다.

호스트 — 철회

변경 제안을 취소하고 원래 일정을 유지하기로 했다

호스트가 현황 화면에서 "철회"를 누르면 모든 제안이 취소되고 이벤트는 기존 일정을 유지한다. 이미 거절한 참가자가 있어도 환불은 이미 처리된 상태다.

AUTO 변경

30분 일찍 시작하는 변경 — 참가자 동의 없이 즉시 반영

시간 변동이 60분 미만이면 "경미한 변경"으로 분류돼 즉시 반영된다. 참가자에게는 "일정이 변경됐습니다" 알림만 간다. 별도 동의 절차 없음.

참가자가 화면에서 보는 것

항목내용이유
변경 전/후 비교기존 시간·장소·가격과 새 시간·장소·가격이 나란히 표시된다.변경 규모를 직관적으로 파악해야 결정이 빠르다. 텍스트만으로는 "얼마나 바뀌었는지"가 안 보인다.
응답 기한 카운트다운남은 시간(예: "23시간 45분 남음")이 표시된다.기한이 보여야 사용자가 "내일까지 봐도 되는지" 알 수 있다.
호스트 메시지호스트가 직접 쓴 변경 사유가 표시된다.호스트의 말이 없으면 "왜 바꿨지?"라는 의심이 생긴다.
거절 시 안내"거절하면 참가가 취소되고 환불이 처리됩니다"라는 확인 메시지.거절 버튼을 눌러도 실수인지 의도인지 구분이 필요하다.

운영자가 알아야 할 것

운영 주의사항
  • 전원 응답 전 호스트가 확정 불가. 1명이라도 미응답이면 "확정" 버튼이 비활성 상태다. 48시간 이후 시스템이 자동 수락으로 전환하면 그때 버튼이 활성화된다.
  • 가격 인상은 반드시 MAJOR 처리된다. 가격을 올리면서 AUTO(즉시 반영)가 되는 경우는 없다. 단, 현재 앱에서 가격 인상이 포함된 변경을 호스트가 직접 요청하는 UI는 구현되지 않았다.
  • 거절로 인한 환불은 100% 즉시 처리. 위약금이나 수수료가 붙지 않는다. 이미 거절한 참가자에게 호스트가 "철회"를 눌러도 환불을 되돌릴 수 없다.
  • RESCHEDULE_APPLIED 알림은 확정 알림. 변경이 최종 확정됐음을 알리는 알림이다. 현재 이 알림은 일반 이벤트 상세로만 연결되며 전용 화면은 없다.

레드 플래그

자주 발생하는 케이스
  • 가격 인상 변경이 MAJOR로 분류되지 않는 경우 — 현재 앱의 이벤트 편집 화면에 가격 변경 입력 UI가 없어, RescheduleProposalCreateParam.newPrice가 항상 미전송된다. 분류기가 newPrice=null로 수신하면 가격 인상 조건을 체크하지 못해 MAJOR가 아닌 AUTO로 즉시 반영될 수 있다.
  • 확정 후 이벤트 상세에 이전 일정이 표시되는 경우 — 확정 직후 앱 캐시가 갱신되지 않아 기존 시간·장소가 남아 있을 수 있다. 화면을 나갔다 다시 들어오면 해소된다.
  • 대형 이벤트에서 1명 미응답으로 확정 지연 — 참가자 수가 많을수록 1명의 미응답이 48시간 내내 확정을 막는다. 스케줄러가 48시간 후 자동 수락으로 처리하므로 최대 49시간(48h + 스케줄러 1h 주기)까지 지연 가능하다.
