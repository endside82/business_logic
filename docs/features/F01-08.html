F01-08. 소셜 계정 연결 해제

더 이상 사용하지 않는 외부 소셜 계정을 본 서비스 계정에서 분리한다. 단, 모든 로그인 수단이 사라지지 않도록 "최소 한 가지 로그인 수단"은 항상 유지된다 — 마지막 남은 수단을 해제하려고 하면 차단된다.

설계 의도 — 사용자가 본인 계정을 잠그지 못하게 막는다

소셜 연결 해제 자체는 사용자의 선택권이지만, 마지막 남은 로그인 수단까지 해제하면 사용자는 본인 계정에 다시 들어올 방법을 잃는다. 그래서 본 단위는 해제 직전에 "이 사용자가 비밀번호도 가지고 있는가" 그리고 "다른 소셜 계정도 연결되어 있는가"를 한 번에 확인하고, 해제 후에도 최소 한 가지 수단이 남는 경우에만 해제를 허용한다. 사용자가 본인 실수로 계정을 잠그는 사고를 시스템 차원에서 차단한다.

사용자 시나리오

기본 흐름

이메일 가입자가 카카오 연결을 해제

이메일/비밀번호로 가입한 뒤 편의를 위해 카카오를 추가 연동했던 사용자가 카카오 연결을 끊는다. 비밀번호는 여전히 살아있기 때문에 해제가 허용되고, 그 이후엔 이메일 로그인만으로 들어간다.

이 흐름이 남기는 것 — 외부 의존을 줄이고 싶은 사용자의 통제권이 보장된다.

차단

소셜만 쓰는 사용자가 마지막 소셜을 해제 시도

카카오로만 가입해 비밀번호가 없는 사용자가 카카오 연결을 해제하려고 한다. 시스템이 즉시 차단한다 — "마지막 로그인 수단은 해제할 수 없습니다" 안내가 떠오른다. 사용자는 다른 소셜을 먼저 추가하거나 비밀번호를 먼저 설정해야 한다.

이 흐름이 남기는 것 — 사용자가 본인 계정을 잠그는 사고를 미리 차단. CS의 "내 계정에 못 들어간다" 문의가 본 단위 차단으로 사전 예방된다.

분리 후 결과

해제 후 같은 소셜로 다시 로그인

카카오 연결을 끊은 뒤 다시 카카오 로그인을 시도하면 서비스는 그 사람을 처음 보는 사용자로 인지한다 — 새 계정이 만들어진다. 이전 계정의 활동 데이터는 옮겨지지 않는다.

이 흐름이 남기는 것 — 해제는 단순한 일시 분리가 아니라 영구 분리에 가깝다는 사실. 사용자가 알고 결정하도록 운영 측 안내가 필요.

운영자가 알아야 할 것

정책 결정점
  • 마지막 수단 차단 정책이 본 단위의 핵심. "한 사용자가 가진 소셜 연결 개수 + (비밀번호 있음이면 +1)"이 2 이상일 때만 해제 허용. 이 정책이 누락되면 사용자가 본인 계정에 못 들어가는 사고가 발생한다.
  • 외부 소셜 제공자에는 별도 해제 신호를 보내지 않는다. 본 서비스 측 매핑만 끊는다 — 사용자가 본인 카카오 계정에서 "권한 회수"를 직접 해야 외부 측 동의도 끝난다. CS에 "카카오 설정에서 권한이 아직 보인다"는 문의가 들어올 수 있음을 매뉴얼에 포함.
  • 해제 후 같은 소셜로 재로그인 시 새 계정이 만들어진다. 사용자가 "이전 계정 데이터로 복귀하고 싶다"고 요청해도 자동 병합은 안 된다 — 운영 측이 이 사실을 사용자에게 사전 안내해야 한다.
  • 해제 후에도 다른 기기의 진행 중인 세션은 끊기지 않는다. 본 단위는 외부 계정 매핑만 끊고 세션은 건드리지 않는다 — 보안이 더 중요한 경우 사용자가 명시적으로 다른 기기에서 로그아웃해야 한다.
