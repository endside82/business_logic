F02-04. 추천 이벤트 더보기 · 필터 · 무한 스크롤

홈의 추천 섹션을 펼친 전용 화면. 페이지 단위 무한 스크롤과 클라이언트 사이드 필터(날짜·가격·정렬)를 갖는다. 위치 권한이 없으면 “가까운 순” 정렬은 비활성. 빈 상태와 에러 상태를 명확히 구분하고, 유사 이벤트 카드를 통한 깊이 탐색 경로(스택 누적)도 제공한다.

알려진 한계

필터·정렬은 클라이언트 사이드로 수행된다. 한 페이지(20건) 안에서만 의미가 있고, 예를 들어 “무료”만 적용 시 그 페이지에 무료가 없으면 빈 화면이 노출될 수 있다. 서버 필터 지원은 향후 확장 후보.

이 기능 안의 하위 항목

동작 1

페이지 단위 조회 + 무한 스크롤

20건씩 누적 로드. 마지막 페이지에서 “더 이상 없습니다” 안내.

동작 2

날짜 필터

오늘 · 이번 주 · 이번 달 · 전체. 칩으로 전환.

동작 3

가격 필터

전체 · 무료 · 유료.

동작 4

정렬 선택

추천순 · 가까운 순(위치 권한 필요). 권한 없으면 “가까운 순” 비활성.

동작 5

추천 상세 진입

카드 탭 → 추천 상세(SCR-HF-003)로 이동.

필터·정렬 상태 흐름

flowchart TD A["진입"] --> B["1페이지 호출 (전체·추천순)"] B --> C["리스트 렌더"] C --> D{"사용자 액션"} D -->|"스크롤 끝"| E["다음 페이지 호출"] D -->|"필터 변경"| F["리스트 reset → 1페이지 재호출"] D -->|"정렬 변경"| F D -->|"X 탭"| G["전체 필터 초기화 → 1페이지 재호출"] D -->|"카드 탭"| H["추천 상세 진입"] E --> C F --> C G --> C

사용자 시나리오 11개

S1 · Happy Path

더보기 → 무한 스크롤 → 상세

  1. 홈의 “더보기” 탭 → 더보기 화면 진입
  2. 1페이지 20건 로드
  3. 스크롤 끝까지 → 2페이지 20건 추가
  4. 카드 탭 → 추천 상세(SCR-HF-003) → “이벤트 상세 보기”
S2 · 필터

가격 “무료”만

  1. “무료” 칩 탭 → 리스트 reset, 1페이지 재호출
  2. 클라이언트 필터 적용 → 페이지 내 무료만 남음
  3. 페이지에 무료가 적으면 결과가 비어 보일 수 있음(한계)
S3 · 정렬

가까운 순

  1. 위치 권한 허용 상태에서 “가까운 순” 선택
  2. 거리 점수 내림차순으로 정렬
  3. 권한 없으면 정렬 옵션 비활성

한계: 페이지 단위 정렬이라 전체 카탈로그 거리 정렬은 아님.

S4 · 끝

마지막 페이지

  1. 30건만 존재하는 신규 사용자
  2. 1페이지 20, 2페이지 10(last=true)
  3. 리스트 끝에 “더 이상 추천 이벤트가 없습니다”
  4. 추가 스크롤 호출 무시
S5 · 빈 상태

맞춤 추천 0건

  1. 관심사 미설정 신규 사용자
  2. 1페이지 빈 배열
  3. “맞춤 추천 이벤트가 없습니다 · 관심사 태그를 추가해보세요”
S6 · 초기 실패

진입 즉시 5xx

  1. 1페이지 호출 실패
  2. 전체 화면 에러 + “다시 시도”
S7 · 추가 페이지 실패

무한 스크롤 중 일부 실패

  1. 2페이지 호출만 실패
  2. 기존 누적 리스트는 유지
  3. 다음 스크롤 시 다시 시도

UI 보강 여지: 명시적 토스트/푸터 메시지 검토.

S8 · 필터 초기화

X 탭으로 한 번에 reset

  1. 3개 필터 활성(카운터 3) 상태에서 X 탭
  2. 날짜=전체, 가격=전체, 정렬=추천순으로 일괄 변경 → 1페이지 재호출
S9 · 깊이 탐색

유사 이벤트 체인

  1. 추천 상세 A → 유사 카드 B 탭 → 추천 상세 B push
  2. 다시 C, D … 스택 누적
  3. 백버튼으로 거슬러 복귀

UX 관찰: 너무 깊어지면 사용자 미아 위험.

S10 · 직접 진입

딥 링크로 더보기 화면 직진

  1. “더보기” 경유 없이 직접 라우트 push
  2. AppBar “추천 이벤트” + 필터 칩 4종(오늘/이번 주/이번 달/전체, 무료/유료) + 1페이지 카드 동시 노출
  3. 빈 카피는 노출되지 않아야 함(상태 머신 분기)
S11 · 권한 없음 정렬 분기

위치 권한 없이 정렬 시도

  1. 권한 없음 상태에서 정렬 드롭다운 열기
  2. “가까운 순” 항목 disabled
  3. “추천순”만 선택 가능

자주 빠뜨리는 케이스

레드 플래그
  • 클라이언트 필터로 빈 화면이 자주 노출 — 사용자가 “데이터 없음”으로 오해. 서버 필터 도입 검토.
  • 중간 페이지 실패가 침묵 — 사용자가 인지 못함. 푸터 메시지 추가.
  • 위치 권한 없음에서 정렬이 동작하는 척 함 — 비활성 처리 누락.
  • 유사 이벤트 스택 깊이 제한 없음 — UX 관찰 후 정책 수립.
