F08-03. 블록 에디터

DRAFT 플랜의 모든 블록(텍스트·헤딩·이미지·체크리스트·POI 등)을 추가·편집·정렬·삭제한다. 인라인 입력 + 300ms 디바운스로 자동 저장되고, 마크다운 단축키로 블록 타입을 즉석에서 변환하며, 삭제는 soft delete + 실행 취소 토스트로 안전하게 되돌릴 수 있다.

편집 가능 조건

본인 소유 + plan.status=DRAFT 두 가지 모두 통과해야 한다. 비작성자는 화면 진입 자체가 차단되고, 본인이라도 PUBLISHED/HIDDEN 상태에서는 서버가 PLAN_NOT_EDITABLE로 거절한다.

이 기능 안의 하위 항목

동작 1

블록 추가

“+ 블록 추가” 시트에서 타입 선택 → POST. 한도 200 도달 시 PLAN_BLOCK_LIMIT_EXCEEDED.

동작 2

인라인 입력 + 자동 저장

300ms 디바운스로 PATCH(content). 우측 인디케이터 “저장됨/저장 중/재시도 필요” 노출. 네트워크 실패 시 자동 재시도는 없고, 다음 입력이 트리거.

동작 3

마크다운 단축키로 타입 변환

PARAGRAPH 블록에서 “## ” 입력 → HEADING_2로 변환 + content 초기화. 한 블록의 같은 위치에서 진행.

즉석 변환
동작 4

이미지 블록 전용 다이얼로그

이미지·POI·체크리스트 같은 복잡한 블록은 INSERT 후 전용 에디터 다이얼로그에서 properties JSON 편집.

동작 5

Soft Delete + Undo

본인 + 자손 일괄 deleted_at 세팅. 토스트의 “실행 취소”로 복구. 자식 블록도 함께 복구.

사용자 시나리오 9개

S1 · Happy Path

첫 텍스트 블록 + 자동 저장

  1. “블록을 추가해보세요” 빈 상태에서 “+ 블록 추가” → PARAGRAPH
  2. POST 201 + 카드 추가
  3. 본문 입력 → 300ms 디바운스 후 PATCH(content) silent
  4. 인디케이터 “저장됨”
S2 · 단축키

마크다운 “## ” → HEADING_2

  1. PARAGRAPH에서 “## ” 입력
  2. detectMarkdownShortcut → HEADING_2
  3. controller 비움 → PATCH(type=HEADING_2, content=””)
  4. 같은 블록이 헤딩 스타일로 재렌더링
S3 · 이미지

전용 다이얼로그

  1. 시트 → IMAGE 선택 → POST 201
  2. 카드 탭 → showImageBlockEditDialog
  3. 갤러리 선택 + 외부 스토리지 PUT → fileKey 획득
  4. PATCH(properties JSON)로 fileKey 저장
  5. 썸네일 표시
S4 · 안전망

Soft Delete + Undo

  1. 길게 누름 → “삭제”
  2. 본인 + 자손 soft delete
  3. 토스트 + “실행 취소” 액션
  4. 사용자가 취소 탭 → POST /restore → 자손까지 복구
S5 · 한도

200개 초과 추가

  1. POST → 400 PLAN_BLOCK_LIMIT_EXCEEDED
  2. “블록 추가에 실패했습니다” + “다시 시도”
S6 · 상태 차단

PUBLISHED 플랜 편집

  1. 본인 작성이라도 status≠DRAFT
  2. 화면은 진입 가능, 블록 추가/수정 시도 → 400 PLAN_NOT_EDITABLE
  3. 해결: 플랜 hide(UNPUBLISHED) 후 다시 시도
S7 · 권한

비작성자 진입 차단

  1. isCreatorOwned=false → 즉시 “접근 권한이 없습니다”
  2. 서버 호출 없이 화면 차단
S8 · 들여쓰기

자식 블록 생성

  1. 현재 시트는 root에 추가
  2. 자식 추가는 드래그(F08-04) 들여쓰기 또는 TOGGLE/TODO 다이얼로그 내부 항목

UX 노트: 자식 생성을 컨텍스트 시트에 추가하는 것은 후속 후보.

S9 · 네트워크

인라인 편집 중 끊김

  1. 디바운스 후 PATCH → DioException
  2. status=error → 인디케이터 “재시도 필요”
  3. 사용자가 다시 입력하면 자동 재시도

현재 갭: 자동 백오프 재시도 없음. 사용자 타이핑이 트리거.

자주 빠뜨리는 케이스

레드 플래그
  • 디바운스 없이 매 키스트로크 PATCH — 서버 부하 + 동시성 문제.
  • Soft delete 자손 누락 — 부모는 사라졌는데 자식이 남아 트리 깨짐.
  • 복구 시 자손 누락 — 실행 취소 했는데 일부 자식만 살아남는 회귀.
  • 마크다운 단축키 변환 시 controller listener 미제거 — 무한 변환 루프.
  • 네트워크 실패 시 자동 무한 재시도 — 큐가 쌓여 사용자 경험 저하. 사용자 입력이 트리거가 되는 게 안전.
  • PUBLISHED 플랜 편집을 클라이언트만 가드 — 서버에서 PLAN_NOT_EDITABLE로 강제.
  • 이미지 properties JSON 검증 누락 — 잘못된 형식으로 카드 깨짐.
