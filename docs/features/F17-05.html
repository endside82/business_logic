F17-05. 세션 관리 (add / bulk / replace / cancel)

정기모임의 회차(세션)는 모두 event 테이블 위 별개 row이며 eventType=REGULAR_MEETING(3)이다. 정션 regular_meeting_event가 순번·취소대체본·머터리얼라이즈·확정 상태를 잡는다. 호스트는 add / bulk / replace / cancel 4가지 액션을 가진다.

RM 세션은 일반 EventService.publishEvent/cancelEvent, EventCapacitySettingsService, CapacitySettingsService, RecurringEventCreateService 6개 경로에서 직접 mutation이 차단된다. 세션 상태 전이는 모임 본체에 종속되며, RM 서비스는 event.*() 엔티티 메서드를 직접 호출해 도메인이벤트 fanout을 피한다.

정션 정책

  • event_id UNIQUE — 한 이벤트가 두 모임에 속할 수 없음
  • (meeting_id, sequence_no) 중복 허용 — 원본 취소본 + 대체본 두 row 보존
  • 순번당 활성 세션 1개는 서비스가 보장 — event.status != CANCELED 필터
  • origin: ORIGINAL / REPLACEMENT + replacedEventId로 원본 참조
