F17-05. 회차 관리 (추가·일괄·대체·취소)

회차는 일반 이벤트 위에서 동작하지만, 정기모임 컨텍스트로 묶여 일반 이벤트 화면에서의 직접 수정이 막혀 있다. 호스트는 네 가지 액션을 가진다 — 한 회차 추가, 여러 회차 일괄 추가, 취소된 회차의 대체본 추가, 회차 취소. 같은 순번에 대체본이 따로 보관되어 "원본 취소 + 대체본" 이력이 남는다.

설계 의도 — 회차는 정기모임만의 영역

회차도 결국 이벤트의 한 형태라 일반 이벤트 화면을 그대로 재사용할 수 있지만, 일반 이벤트 운영 경로로 회차를 수정하면 모임 본체와 정합성이 깨진다. 그래서 호스트는 정기모임 화면에서만 회차를 손볼 수 있고, 일반 이벤트의 게시/취소/정원 변경 같은 경로는 명시적으로 차단되어 있다.

네 가지 액션

추가

한 회차 추가

다음 순번이 자동 할당된다. 변동형은 정원·가격을 회차마다 따로 줄 수 있고, 고정형은 모임 본체 값을 상속한다.

일괄 추가

여러 회차 한 번에

8회 코스를 단번에 만드는 등 고정형 셋업에 쓴다. 시간 겹침은 호스트가 직접 확인.

대체

취소된 회차의 새 일정

원본을 취소하고 같은 순번에 새 회차를 단다. 원본 행과 대체본이 모두 이력으로 남아 "이 코스의 3회차는 한 번 옮겨졌다"가 보인다.

취소

회차만 닫기

회차 한 건만 닫는다. 고정형 멤버 본체는 영향 없고, 해당 회차의 자동 참석 명단만 빠진다.

일반 이벤트 운영 화면을 통한 회차 손대기는 차단

회차를 일반 이벤트의 게시/취소/정원 변경 화면으로 손대려고 하면 운영 가드가 거절한다. 회차에 대한 모든 변경은 정기모임 전용 화면에서만 가능하다.

운영자가 알아야 할 것

대체본·취소 이력
  • 대체본은 원본과 같은 순번에 붙는다 — 사용자에겐 "3회차가 한 번 옮겨졌다"가 자연스럽게 노출.
  • 취소된 회차는 기본 목록에서 숨김 — 호스트가 "이력 포함" 옵션을 켜면 함께 표시.
  • 고정형 회차 취소 시 멤버 본체에는 영향 없음 — 자동 참석 명단만 갱신.
  • 변동형은 회차마다 정원·가격이 독립적 — 회차별로 다른 가격을 받을 수 있다.

레드 플래그

자주 발생하는 케이스
  • 일괄 추가/대체 호스트 UI 미완성 — API는 준비됐지만 화면 우선순위가 낮아 1차에서 빠짐.
  • 회차 시간 겹침 자동 검출 없음 — 호스트가 같은 시간대에 두 회차를 만들 수 있음. 1차는 호스트 자율.
  • 모집 중 모임에 회차 추가 — 이미 가입한 멤버의 자동 참석 명단이 즉시 갱신되는데, 알림 미배선이라 멤버가 모를 수 있음.
