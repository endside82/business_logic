F03-03. 이벤트 생성 (호스트)

4단계 입력으로 새 이벤트를 만든다. 기본정보 → 일시/장소 → 정원/가격/옵션 → 미리보기. “등록(임시저장)” 또는 “등록 후 발행”의 두 갈래. 유료·사전결제·반복·프라이빗 같은 옵션이 한 화면에서 분기되고, 템플릿 복제로 기존 이벤트를 빠르게 다시 만들 수 있다.

이 기능 안의 하위 항목

동작 1

4단계 입력

단계별 검증을 통과해야 다음으로 진행.

동작 2

발행/임시저장 분기

등록 후 발행은 두 호출(생성 → publish) 연속 실행.

동작 3

유료/사전결제 옵션

가격·환불정책·사전결제 설정. F06과 연동.

동작 4

반복 이벤트 생성

모 이벤트 생성 후 RRULE로 자식 N건 일괄 생성(최대 52).

동작 5

템플릿 복제

기존 이벤트 → 시각만 비우고 prefill.

사용자 시나리오 10개

S1 · Happy Path

무료 모임 생성 후 즉시 발행

  1. FAB → 4단계 입력 → “등록 후 발행”
  2. 이벤트 생성 → publish 호출 → status=OPEN
  3. 상세 화면으로 이동, 목록/마이 이벤트 invalidate
S2 · 임시저장

DRAFT로 저장 후 추후 발행

  1. 4단계까지 진행 후 “등록”만
  2. 마이 이벤트에 “임시저장” 배지
  3. 추후 상세에서 “발행” 탭으로 OPEN 전환
S3 · 검증

시작 시각이 과거

  1. 어제 일자 선택 시 인라인 에러 + 다음 비활성
  2. 우회 시 서버 400
S4 · 유료

유료 + 사전결제

  1. 가격 입력 → 환불정책 자동 표시
  2. “사전결제 필요” 토글 ON
  3. 참가자가 신청 시 결제 차감(F06)
S5 · 프라이빗

프라이빗 모임 생성

  1. 호스팅비 옵션 노출(PERCENTAGE/maxPrice)
  2. 서버: eventType=PRIVATE + PrivateMeetingDetail 생성(phase=WAITING_PAYMENT)
  3. 호스팅비 결제는 별도 흐름(F06)
S6 · 공동호스트

비존재 사용자 ID

  1. 공동호스트로 잘못된 ID 추가
  2. 서버: USER_NOT_FOUND
  3. 토스트 후 사용자 재선택
S7 · 복제

템플릿 복제로 재생성

  1. 상세 더보기 → “이 이벤트로 새로 만들기”
  2. 시각만 비우고 나머지 prefill
  3. 새 시각 입력 후 등록
S8 · 반복

반복 이벤트 일괄 생성

  1. 모 이벤트 생성
  2. 반복 위젯 → RRULE 설정 → 자식 N건 INSERT
  3. 반복 그룹 화면 진입
S9 · 검증

Step 1 필수 미입력 차단

  1. 제목/카테고리 미입력 상태 “다음” 탭
  2. 두 에러 텍스트 동시 노출 + 다음 단계 헤더 미노출
  3. 입력 후 진행
S10 · 반복 다중 요일

BYDAY 다중 요일

  1. 요일 칩에서 월·수·금 선택
  2. 서버: RRULE에 BYDAY=MO,WE,FR + 최대 52 occurrence
  3. 각 occurrence마다 자식 이벤트 생성

서버 계약

MethodPath설명

자주 빠뜨리는 케이스

레드 플래그
  • 등록 후 발행이 두 호출 중 첫 호출만 성공 — DRAFT만 만들어진 채 사용자에게는 “발행됨”으로 보임. 보상 로직 필요.
  • 단계 검증을 클라이언트만 신뢰 — 서버에서도 동일 검증을 강제해야 한다.
  • 반복 occurrence 상한(52) 누락 — 무한 생성 위험.
  • 복제 시 시각까지 복사 — 과거 시각으로 이벤트가 만들어지는 회귀.
