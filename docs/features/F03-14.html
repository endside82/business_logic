F03-14. 이동수단 공통 베이스

이벤트에 이동수단 정책을 붙이는 공통 설정 단위. 호스트는 이벤트 초안에서 이동수단 없음, 카풀, 버스 중 하나를 고르고, 모집 공개 후에는 이동수단 방식 자체를 바꾸지 못한다. 카풀과 버스 기능은 이 설정을 전제로만 열린다.

설계 의도 — 이동수단 기능의 단일 스위치

카풀과 버스는 둘 다 참석자의 이동을 바꾸는 기능이라 동시에 켜질 수 없다. 본 단위는 이벤트마다 하나의 이동수단 방식을 정하고, 하위 기능이 자기 방식이 아닐 때 동작하지 않도록 막는 공통 가드다. 공개 후 방식 변경을 막는 이유는 이미 운전자 지원, 탑승자 선택, 좌석 배정이 시작됐을 수 있기 때문이다.

사용자 시나리오

초안 단계

호스트가 카풀 이벤트로 설정

이벤트 생성 중 이동수단을 카풀로 바꾸면 이동수단 설정이 만들어지고, 이후 카풀 지원·배정 화면이 열릴 수 있다.

전환

카풀에서 버스로 바꾼다

초안 상태에서는 이동수단 방식 전환이 가능하다. 단, 기존 카풀 지원자·탑승자 데이터는 더 이상 유효하지 않으므로 삭제된다.

모집 공개 후

모집 공개 후 방식 변경 차단

이미 참가자에게 이동 정책이 공개된 뒤에는 없음·카풀·버스 사이 전환이 막힌다. 우회 시도도 서버에서 거절된다.

개별 이동

공개 후 개별 이동 허용 여부만 조정

이동수단 방식 자체는 고정되지만, 자차 이동 허용 여부는 운영 상황에 따라 조정할 수 있다. 카풀·버스 배정을 받지 않는 참가자의 선택지를 통제하는 토글이다.

사용자·호스트가 할 수 있는 동작

동작누가무엇이 일어나는가
이동수단 설정 보기이벤트 참여자이 이벤트에 적용된 이동수단 방식과 자차 이동 허용 여부를 확인한다.
이동수단 설정 정하기호스트이동수단 방식(없음·카풀·버스)과 개별 이동 허용 여부를 설정한다.

운영자가 알아야 할 것

이동수단 방식 전환은 데이터 삭제를 동반한다
  • 설정이 없으면 기본값은 이동수단 없음 + 자차 이동 허용. 신규 이벤트가 이동 기능을 쓰지 않아도 별도 설정이 필요 없다.
  • 초안 단계의 방식 전환은 하위 데이터 정리를 포함한다. 카풀에서 버스로 바꾸면 카풀 지원·탑승자 기록이 제거된다.
  • 모집 공개 이후 방식 변경은 정책상 금지다. 이미 공개된 이동 약속을 뒤집으면 참가자 피해가 크다.
