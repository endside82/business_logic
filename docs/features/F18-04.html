F18-04. 분쟁 증빙·공개범위·보존

분쟁 케이스에 첨부된 증빙 파일이 누구에게 보이고, 언제까지 보존되며, 언제 삭제되는지를 결정하는 정책 레이어. 서버 필터는 CS_ONLY 항목만 제거하며, PARTIES·HOST_ONLY·PUBLIC_SUMMARY는 그대로 전달한다 (현재 builder는 HOST_ONLY를 emit하지 않으므로 실질적으로 PARTIES·PUBLIC_SUMMARY만 전달됨). 클라이언트는 받은 목록을 그대로 표시한다. 활성 케이스의 증빙은 법적 보존(evidenceFrozen)으로 삭제가 차단되며, 케이스 종결 후 1년이 지나야 자동 정리된다. 활성 케이스에 연관된 사용자는 계정 삭제도 차단된다.

이 기능이 분쟁 해결 도메인에서 가장 먼저 이해해야 하는 이유

legal hold는 계정 삭제와 데이터 익명화 자체를 차단한다. source별 finder 기준에 해당하는 역할 (OperationalIssue·WarningReport·공개Report·CARPOOL·DateBlock reporter, PurchaseRefundDispute seller, WarningAppeal member, MeetingSettlementAppeal 호스트, ClubMembershipAction 대상 멤버, USER_DISPUTE reporter/target 양측)로 OPEN/IN_REVIEW/ESCALATED 케이스에 연관되어 있으면, 그 케이스가 종결될 때까지 계정을 삭제할 수 없다. 이 판정은 자동으로 이루어지며 사용자에게도 알려져야 한다.

증빙 공개범위 4단계

실제 서버 필터링(filterByActorVisibility)은 CS_ONLY 항목만 제거한다. PARTIES/HOST_ONLY/PUBLIC_SUMMARY는 유지되며, 현재 빌더는 HOST_ONLY를 emit하지 않는다. 클라이언트는 수신한 목록을 추가 처리 없이 그대로 표시한다.

활성 케이스(OPEN/IN_REVIEW/ESCALATED)에 연관된 사용자는 계정 삭제 요청이 차단된다. 단, source별 finder 범위가 제한적이어서 SETTLEMENT_APPEAL appealer(비호스트), PurchaseRefundDispute buyer 측의 경우 hold가 미검출될 수 있다(후속 개선 과제). 케이스가 전부 종결(RESOLVED/CLOSED)되면 차단이 해제된다.

사용자 시나리오

증빙 조회

케이스 상세에서 증빙 확인

참가자가 케이스 상세를 열면 자신이 볼 수 있는 증빙 목록이 나타난다. 현재 서버 필터는 CS_ONLY 항목만 제거한다. PARTIES·HOST_ONLY·PUBLIC_SUMMARY는 응답에 유지된다. 다만 현재 builder가 HOST_ONLY를 생성하지 않아 실질적으로 PARTIES·PUBLIC_SUMMARY만 전달된다. CS_ONLY 증빙은 응답에서 제거되어 노출되지 않는다.

이 흐름이 남기는 것 — 운영팀의 내부 메모나 CS 전용 파일이 실수로 노출되지 않는다.

evidenceFrozen

진행 중인 케이스의 증빙 — 삭제 불가

케이스 상태가 OPEN·IN_REVIEW·ESCALATED이면 "증빙 보존 중" 안내 배너가 표시된다. 증빙 삭제 endpoint 자체가 미구현이므로 frozen 여부와 관계없이 삭제는 불가하며, 증빙은 조회 전용이다.

legal hold

분쟁 중에 계정 삭제 시도

활성 케이스에 연관된 상태에서 계정 삭제를 요청하면 서버가 차단한다. 케이스가 모두 종결될 때까지 계정을 삭제할 수 없다는 안내가 필요하다. 이 상태는 서버가 자동으로 판정한다.

이 흐름이 남기는 것 — 분쟁 중 증거를 은닉하거나 책임을 회피하기 위한 계정 삭제를 막는다.

증빙 제출

이의제기 시 파일 첨부

이의제기(F18-03) 시 최대 5개 파일을 함께 제출할 수 있다. 본인 소유 파일만 첨부 가능하고, 이미 다른 케이스에 사용 중이거나 삭제된 파일은 서버가 차단한다. 분쟁 직접 접수(F18-02) 시에는 v1에서 첨부 UI가 제공되지 않는다.

자동 정리

오래된 종결 케이스 증빙 정리

케이스가 RESOLVED/CLOSED로 종결된 뒤 1년이 지나면 매일 05:00에 증빙 파일이 자동으로 삭제된다. 파일 일부가 삭제 실패하면 해당 파일만 다음 날 재시도한다 — 케이스 row 자체는 삭제되지 않는다.

운영자가 알아야 할 것

운영 주의사항
  • purchase_refund_dispute 증빙이 자동 정리되지 않는다. 파일 구조가 달라 보존 스케줄러 대상에서 제외됐다. 이 source의 오래된 증빙 파일은 별도 정리가 필요하다.
  • legal hold 탈퇴 차단 시 사용자 안내가 필요하다. "왜 계정 삭제가 안 되는지" 앱이 설명하지 못하면 CS로 문의가 집중될 수 있다.
  • 증빙 삭제 endpoint 미구현. DisputeCaseController에 DELETE endpoint가 없으며, EvidenceGallery는 frozen 시 안내 배너만 표시한다. 증빙은 조회 전용이다. 삭제 기능 추가 여부는 제품 정책 결정이 필요하다.
  • 증빙 첨부 분쟁 접수는 v1 미제공. 분쟁 직접 접수(F18-02) 시 파일 업로드 UI가 없다. 서버는 수신할 준비가 되어 있어 v2에서 추가 예정.

레드 플래그

현재 알려진 미구현·주의 사항
  • 증빙 첨부 접수 v1 미제공. 분쟁 접수 시 증빙 없이 텍스트만 제출된다. 운영팀이 처리할 때 맥락이 부족할 수 있다.
  • 플랜 마켓 환불 분쟁 증빙 보존 미구현. purchase_refund_dispute의 오래된 증빙 파일이 영구적으로 남는다. 별도 cleanup 스케줄러 필요.
  • 증빙 삭제 endpoint 미구현 확정. DisputeCaseController에 DELETE endpoint가 없으며 증빙은 조회 전용이다. frozen 상태에서는 "증빙 보존 중" 안내 배너만 표시되고, 삭제 버튼은 현재 제공되지 않는다. 삭제 기능 추가 여부는 제품 정책 결정이 필요하다.
