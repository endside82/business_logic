F18-04. 분쟁 증빙·공개범위·보존

분쟁 케이스에 첨부된 증빙 파일이 누구에게 보이고, 언제까지 보존되며, 언제 삭제되는지를 결정하는 정책 레이어. 공개범위는 운영팀 전용·분쟁 당사자 공개·호스트 공개·요약 공개 네 등급으로 설계되어 있으나, 현재는 운영팀 전용 메모만 가려지고 나머지 등급 구분은 설계만 되어 있을 뿐 화면에는 아직 반영되지 않았다. 진행 중인 케이스의 증빙은 케이스가 끝날 때까지 삭제·수정 없이 보존되며, 케이스가 종결되고 1년이 지나야 자동 정리된다. 진행 중인 케이스에 연관된 사용자는 계정 삭제도 차단된다.

분쟁이 진행 중인 동안에는 계정 삭제와 개인정보 익명화 자체가 차단된다. 운영 이슈·클럽 신고·공개 신고·카풀·데이트 안전신고의 신고자, 플랜 마켓 환불 분쟁의 판매자, 클럽 경고 이의의 당사자, 모임 정산 이의의 호스트, 강퇴·차단 처분의 대상 멤버, 이용자 간 분쟁의 신고자·피신고자 양측 중 하나로 진행 중인 케이스에 연관되어 있으면, 그 케이스가 종결될 때까지 계정을 삭제할 수 없다. 이 판정은 자동으로 이루어지며 사용자에게도 알려져야 한다.

현재 동작은 운영팀 전용 증빙만 화면에서 가린다. 당사자·호스트·요약 등급은 모두 그대로 노출되며, 나머지 등급 구분은 설계만 되어 있을 뿐 아직 화면에 반영되지 않았다. 사용자에게는 받은 목록이 추가 처리 없이 그대로 보인다.

계정 삭제 차단 — 누가 차단되는가

진행 중인 케이스(접수됨·검토 중·상위 이관)에 연관된 사용자는 계정 삭제 요청이 차단된다. 다만 차단 판정이 일부 역할을 놓칠 수 있어, 모임 정산 이의를 낸 사람이 호스트가 아닌 경우와 플랜 마켓 환불 분쟁의 구매자 측은 차단이 잡히지 않을 수 있다(후속 개선 과제). 케이스가 전부 종결(처리 완료·종결)되면 차단이 해제된다.

사용자 시나리오

증빙 조회

케이스 상세에서 증빙 확인

참가자가 케이스 상세를 열면 자신이 볼 수 있는 증빙 목록이 나타난다. 현재는 운영팀 전용 증빙만 가려지고, 당사자·호스트·요약 등급은 그대로 보인다. 나머지 등급 구분은 설계만 되어 있고 아직 화면에 반영되지 않았다. 운영팀 전용 증빙은 화면에서 제거되어 노출되지 않는다.

이 흐름이 남기는 것 — 운영팀의 내부 메모나 운영팀 전용 파일이 실수로 노출되지 않는다.

증빙 보존

진행 중인 케이스의 증빙 — 삭제 불가

케이스가 접수됨·검토 중·상위 이관 상태이면 "증빙 보존 중" 안내 배너가 표시된다. 증빙을 삭제하는 기능 자체가 아직 없으므로 보존 여부와 관계없이 삭제는 불가하며, 증빙은 조회 전용이다.

계정 삭제 차단

분쟁 중에 계정 삭제 시도

진행 중인 케이스에 연관된 상태에서 계정 삭제를 요청하면 차단된다. 케이스가 모두 종결될 때까지 계정을 삭제할 수 없다는 안내가 필요하다. 이 판정은 자동으로 이루어진다.

이 흐름이 남기는 것 — 분쟁 중 증거를 은닉하거나 책임을 회피하기 위한 계정 삭제를 막는다.

증빙 제출

이의제기 시 파일 첨부

이의제기(F18-03) 시 최대 5개 파일을 함께 제출할 수 있다. 본인 소유 파일만 첨부 가능하고, 이미 다른 케이스에 사용 중이거나 삭제된 파일은 서버가 차단한다. 분쟁 직접 접수(F18-02)에서도 2026-06-06(W14 S4) 증빙 첨부가 배선되어 최대 5개 파일을 함께 제출할 수 있다.

자동 정리

오래된 종결 케이스 증빙 정리

케이스가 처리 완료 또는 종결된 뒤 1년이 지나면 매일 05:00에 증빙 파일이 자동으로 삭제된다. 파일 일부가 삭제 실패하면 해당 파일만 다음 날 재시도한다 — 케이스 기록 자체는 삭제되지 않는다.

자동 정리 대상 범위

종결 후 1년이 지난 케이스의 증빙을 매일 자동 정리하는데, 분쟁 종류에 따라 포함 여부가 다르다.

운영자가 알아야 할 것

운영 주의사항
  • 플랜 마켓 환불 분쟁 증빙이 자동 정리되지 않는다. 파일 구조가 달라 자동 정리 대상에서 제외됐다. 이 분쟁의 오래된 증빙 파일은 별도 정리가 필요하다.
  • 계정 삭제가 차단될 때 사용자 안내가 필요하다. "왜 계정 삭제가 안 되는지" 앱이 설명하지 못하면 고객 문의가 집중될 수 있다.
  • 증빙을 삭제하는 기능이 아직 없다. 보존 중일 때는 안내 배너만 표시되고, 증빙은 조회 전용이다. 삭제 기능 추가 여부는 제품 정책 결정이 필요하다.
  • 분쟁 직접 접수 시 증빙 첨부가 배선됐다 (2026-06-06 W14 S4). 분쟁 직접 접수(F18-02) 화면에서 최대 5개 증빙을 함께 제출할 수 있다. 다만 플랜 마켓 환불 분쟁(F08-14) 접수는 서버 계약 한계로 아직 첨부할 수 없어 서버 후속 작업이 필요하다.

레드 플래그

현재 알려진 미구현·주의 사항
  • 통합 분쟁 접수 증빙 첨부 배선 완료 (2026-06-06 W14 S4). 통합 분쟁 접수 화면에서 최대 5개 증빙을 함께 제출할 수 있다. 다만 플랜 마켓 환불 분쟁 접수 화면은 서버 계약 한계로 아직 첨부할 수 없어 서버 후속 작업이 필요하다.
  • 플랜 마켓 환불 분쟁 증빙은 자동 정리되지 않는다. 이 분쟁의 오래된 증빙 파일이 영구적으로 남는다. 별도 정리 방안이 필요하다.
  • 증빙을 삭제하는 기능이 아직 없다. 증빙은 조회 전용이며, 보존 중일 때는 "증빙 보존 중" 안내 배너만 표시되고 삭제 버튼은 현재 제공되지 않는다. 삭제 기능 추가 여부는 제품 정책 결정이 필요하다.
