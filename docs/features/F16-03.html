F16-03. 시즌 (목록·과거 랭킹·내 스냅샷)

마일리지를 기간 단위로 끊어 운영하는 시즌제. 멤버는 시즌 목록, 종료된 시즌의 랭킹(마감 시점 기록을 동결한 스냅샷), 그 시즌 본인 기록을 본다. 시즌이 끝나면 다음 예약 시즌이 자동으로 시작된다.

설계 의도 — 시즌은 "다시 시작할 수 있는 무대"

누적 점수만 있으면 후발 주자가 따라잡기 어렵다. 시즌제는 일정 기간마다 무대를 새로 열어 모두에게 기회를 준다. 종료된 시즌의 랭킹은 마감 순간의 기록을 그대로 동결해 보존하므로 나중에 봐도 그대로다.

시즌 한 살이

flowchart LR S["예정"] --> A["진행"] A -->|"운영진 마감"| C["종료 + 스냅샷 동결"] C -.->|"다음 예정 시즌 자동 시작"| A2["다음 시즌 진행"]

시즌은 예정 → 진행 → 종료로 흐른다. 운영진이 마감하면 그 시점 멤버별 기록이 스냅샷으로 동결되고, 시작 시각이 도래한 다음 예정 시즌이 자동으로 진행 상태가 된다.

화면 구성

목록

시즌 목록 (상태 필터)

전체 또는 예정/진행/종료 상태로 시즌을 본다.

과거 랭킹

종료 시즌 스냅샷 랭킹

마감 시점에 동결된 순위·점수를 본다. 적립 기준 또는 잔액 기준으로 정렬.

내 기록

본인 시즌 스냅샷

해당 시즌의 본인 적립·잔액·순위를 본다. 기록이 없으면 빈 상태로 보인다.

멤버가 할 수 있는 동작

동작누가무엇이 일어나는가
시즌 목록 보기클럽 멤버전체 또는 예정·진행·종료 상태로 시즌을 본다.
과거 시즌 랭킹 보기클럽 멤버종료된 시즌의 마감 시점 동결 순위·점수를 본다.
내 시즌 기록 보기클럽 멤버해당 시즌의 본인 적립·잔액·순위를 본다. 기록이 없으면 빈 상태로 보인다.

운영자가 알아야 할 것 — 비즈니스 리스크

시즌 랭킹의 두 가지 함정
  • 랭킹 정렬 기준 값이 어긋나 있다. 문서·클라이언트는 "적립 기준/잔액 기준"이라는 별도 값을 쓰는데, 서버가 실제로 받는 값은 그와 다르다. 잘못된 값을 보내면 잔액 기준 랭킹에 도달하지 못하거나 오류가 날 수 있다.
  • 마감만 하면 스냅샷이 비어 있을 수 있다. 시즌을 종료해도 멤버별 기록을 채우는 처리가 별도로 보장되지 않으면 과거 랭킹이 비어 보인다. 마감 시 전체 멤버 스냅샷 생성이 함께 일어나는지 확인이 필요하다.
