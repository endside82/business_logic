F03-01. 이벤트 탐색

앱에서 오프라인 모임을 찾는 첫 면. 키워드 검색, 카테고리 좁힘, 시간·거리 정렬, 무한 스크롤을 한 화면에 모아 사용자가 "내가 갈 만한 모임이 있는지"를 빠르게 확인하도록 한다. 로그인 없이도 둘러볼 수 있고, 신청·찜 같은 행동을 누른 순간에만 로그인을 묻는다. 가입 전 사용자가 서비스를 평가하는 핵심 면이다.

이벤트 목록은 사용자가 서비스의 가치를 판단하는 가장 큰 면이다. "이 앱에 내가 갈 만한 모임이 있는지"를 가입 전에 확인하지 못하면 가입 의사 자체가 만들어지지 않는다. 그래서 본 단위는 비로그인에도 카드와 정렬·필터가 모두 동작한다. 로그인 유도는 신청·찜 같은 행동을 누르는 시점에만 들어간다 — 그 전까지는 자유로운 둘러보기다.

사용자 시나리오

기본 흐름

"보드게임"으로 주말 모임 찾기

키워드 입력 후 결과를 카테고리 칩으로 좁히고, "가까운 순"으로 다시 정렬해 마음에 드는 카드의 하트를 누른 뒤 상세로 진입한다. 위치 권한이 있어야 가까운 순이 동작한다.

이 흐름이 남기는 것 — 신청 전에 사용자가 가장 길게 머무르는 면이라, 탐색 행동 데이터가 추천 품질 개선의 핵심 입력값이 된다.

가입 전

비로그인 사용자가 둘러봄

카드와 상세는 모두 보인다. 단, 카드의 본인 상태 배지(찜·신청)는 보이지 않고, 하단 "이벤트 생성" 진입점도 숨겨진다. 카드를 눌러 상세에 가서 "참석 신청"을 누르는 순간에 비로소 로그인 화면이 떠오른다.

이 흐름이 남기는 것 — "가입하고 둘러봐" 대신 "둘러보고 마음 들면 가입" 패턴. 가입 전환의 결정적 진입 경로.

무한 스크롤

20장 보고 더 둘러봄

리스트 끝에 다다르면 다음 20장이 자연스럽게 이어 붙는다. 끝까지 가면 추가 로딩 표시가 사라진다. 위로 당기면 첫 페이지부터 다시 받아 온다.

결과 없음

찾는 단어가 카탈로그에 없을 때

결과가 0건이면 "조건에 맞는 이벤트가 없습니다" 안내. 카테고리 칩을 "전체"로 풀거나 검색어를 비우면 다시 일반 목록으로 복귀한다.

이 흐름이 남기는 것 — 결과 없음을 "오류"로 인지하지 않게 한다. 초기 카탈로그가 비어 보이는 인상은 가입 직후 이탈로 이어진다.

네트워크 단절

지하철에서 다음 페이지가 안 옴

이미 받은 카드는 그대로 보이고 하단 로딩만 사라진다. 화면 전체가 오류로 깜빡이지 않는다. 첫 페이지부터 실패면 "오류가 발생했습니다 + 다시 시도" 버튼이 뜬다.

이 흐름이 남기는 것 — 한 번의 네트워크 실패가 둘러보기 흐름을 깨뜨리지 않게 한다.

유사 이벤트

상세에서 "이런 이벤트는 어떠세요?"

이벤트 상세 하단의 유사 추천을 펼친다. 최대 5장이 보이고, 더 이상 비슷한 카드가 없으면 "유사한 이벤트가 없습니다" 안내만 뜬다 — 빈 상태에서도 헤더는 유지해 "추천이 사라진" 인상은 주지 않는다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
검색바 상단 고정. 입력 후 제출 시점에만 검색 — 타이핑 중 자동 호출 없음. 입력 중 매 글자마다 결과가 흔들리면 사용자가 자기 의도를 잃는다. "제출했을 때"로 검색을 묶어 의사결정 단위와 맞춘다.
카테고리 칩 전체 + 8개 칩. 한 번에 하나만 선택. 다시 누르면 전체로 복귀. 9개 모두를 가로 한 줄에 두어 둘러보기 흐름을 끊지 않는다. 다중 선택은 결정을 늘리지만, 첫 둘러보기에서는 단일 선택이 더 빠르다.
정렬 드롭다운 "총 N개 이벤트" + 정렬 옵션 선택. "가까운 순"은 위치 권한이 없으면 비활성. "몇 개나 있는지"를 먼저 보여 줘 사용자가 검색이 너무 좁혀지지 않았는지 자가 확인.
이벤트 카드 썸네일, 제목, 시작 시각, 주소, 정원, 가격, 본인 상태 배지(로그인 시), 하트. 신청 결정에 필요한 핵심 단서를 카드 한 장 안에 모아, 상세까지 들어가지 않아도 1차 판단이 가능하게.
이벤트 생성 진입점 우하단 + 버튼. 로그인 사용자에게만 노출. 비로그인 사용자가 누르면 어차피 가입을 유도해야 하므로, 가입 후에만 노출해 진입 동선을 깔끔하게 둔다.
마감·취소 카드 흐릿한 오버레이 + 상태 라벨("종료"/"취소"). 이미 끝난 모임을 클릭 가능 상태로 두면 사용자가 잘못 진입한 뒤 답답함이 누적된다. 상태를 카드 단계에서 분명히.

운영자가 알아야 할 것

탐색 화면의 결정점
  • 비로그인 사용자에게도 카탈로그를 보여 주는 게 본 단위의 핵심 정책. "가입해야 둘러볼 수 있다"로 정책이 바뀌면 가입 전환율 자체가 떨어진다.
  • 9개 카테고리 칩은 화면 한 줄에 들어가는 최대치. 늘리면 가로 스크롤이 필요해지고, "전체 카테고리가 있다"는 인상이 약해진다.
  • 위치 권한이 없으면 가까운 순 정렬이 비활성된다. 사용자에게 권한을 다시 묻거나, "위치 권한이 필요해요" 안내를 함께 보여 주는 정책이 필요.
  • 다음 페이지 로딩 실패는 사용자에게 명시적 오류로 보이지 않는다. "더 안 늘어나네" 인상이 누적되지 않도록 페이지 실패율을 운영 모니터링에 포함.
  • 마감·취소 카드는 흐릿하게라도 보여 준다. 카드를 화면에서 빼면 사용자가 "어제 본 모임이 사라졌다"는 혼란을 겪는다 — 상태를 보여 주는 게 안전한 정책.
