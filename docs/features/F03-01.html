F03-01. 이벤트 발견 · 탐색

전체 이벤트 목록을 키워드·카테고리·정렬로 탐색한다. 비로그인 사용자도 같은 화면을 볼 수 있고 액션 시점에 로그인이 유도된다. 무한 스크롤·풀투리프레시·빈 결과·유사 이벤트 추천까지를 단일 단위가 책임진다.

하위 동작
5
시나리오
7

20
게스트
허용

이 기능 안의 하위 항목

동작 1

키워드 검색

검색바 입력 시 0페이지부터 재조회.

동작 2

카테고리 필터

칩 다중 선택. “전체”는 필터 해제.

동작 3

정렬

최신순·가까운순. “가까운순”은 위치 권한 필요.

동작 4

무한 스크롤 + 풀투리프레시

20건씩 누적. 위로 당기면 0페이지 재조회.

동작 5

유사 이벤트 진입

상세 하단 “유사 이벤트 더보기” 진입점.

사용자 시나리오 7개

S1 · Happy Path

키워드 검색 → 필터 → 정렬 → 상세 진입

  1. 이벤트 탭 진입 → 1페이지 20건
  2. “보드게임” 검색 → 12건
  3. SOCIAL 칩 → 7건
  4. “가까운순” 정렬
  5. 하트 탭(낙관적 채워짐) + 카드 탭 → 상세 진입
S2 · 게스트

비로그인 탐색

  1. publicRoute로 진입 허용
  2. 카드의 viewer context(참석 상태·위시리스트)는 모두 null/false
  3. FAB(이벤트 생성) 숨김
  4. 상세 진입은 가능, “참석 신청”은 로그인 유도
S3 · 페이징

무한 스크롤 + 풀투리프레시

  1. 스크롤 끝 200px 도달 → 다음 페이지
  2. 마지막 페이지에서 스피너 사라짐
  3. 위로 당김 → 0페이지부터 재조회
S4 · 빈 결과

검색 0건

  1. 키워드 “도자기수업” 검색 → 0건
  2. 빈 상태 일러스트 + 안내
  3. “전체” 칩으로 복구
S5 · 유사 이벤트

상세 → 더보기

  1. 상세 하단 “유사 이벤트” 가로 스크롤
  2. “더보기” → 유사 이벤트 화면
  3. 카드 탭으로 다른 상세 이동
S6 · 네트워크

지하철에서 추가 페이지 실패

  1. 추가 페이지 호출 실패 → 하단 로더만 사라짐(기존 유지)
  2. 첫 페이지부터 실패면 전체 화면 에러 + “다시 시도”
S7 · 유사 빈 결과

유사 풀이 비어있는 경우

  1. 유사 이벤트 응답이 빈 배열
  2. “이런 이벤트는 어떠세요?” 헤더 유지 + 빈 카피 “유사한 이벤트가 없습니다”

자주 빠뜨리는 케이스

레드 플래그
  • 게스트에게 FAB 노출 — 권한 없는 행동 유도.
  • 위치 권한 없음에서 “가까운순” 활성 — 의도된 disabled 처리 누락.
  • 유사 빈 결과를 에러로 표시 — 헤더 유지 + 빈 카피로 구분.
