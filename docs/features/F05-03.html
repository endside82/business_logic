F05-03. 검색 필터 적용

한 단어로 받은 검색 결과를 카테고리·날짜·가격·거리·지역으로 더 좁혀 본다. 필터 시트 안에서 조건을 바꾸면 하단 적용 버튼이 "지금 조건이면 N개" 라는 미리보기 숫자로 실시간 갱신되어, 사용자는 적용하기 전에 결과 크기를 미리 안다. 적용 후에는 검색바 아래에 활성 필터 칩이 늘어서서, 사용자가 "왜 결과가 줄었는지"를 잊지 않게 한다.

필터는 적용 전에 "이 조건이면 카드가 몇 장 남는지"를 사용자가 알 수 있어야 한다. 그래서 필터 시트의 하단 버튼은 "적용" 같은 정적인 문구가 아니라 "N개 결과 보기"라는 살아 있는 숫자다. 사용자가 카테고리·가격·거리를 조정하면 이 숫자가 잠깐 후 갱신되어, "너무 좁혔다 / 적당하다"는 판단을 적용 전에 내릴 수 있다.

적용 가능한 필터

카테고리

다중 선택 칩

이벤트/클럽 카테고리 15개와 "기타"를 칩으로 노출. 다중 선택 가능. 지역과 같이 쓰면 그 지역에서 인기 있는 카테고리가 별도로 추천 칩으로 떠올라, 사용자가 빈 결과를 만나지 않게 돕는다.

날짜·시간

시작 시각 범위

오늘부터 1년 안의 시작 시각 범위를 선택. "이번 주말", "다음 달 첫째 주" 같은 의도를 받아 낸다.

가격

듀얼 슬라이더 + 무료 토글

최저·최고 가격을 양쪽 핸들로 조정. "무료 이벤트만" 별도 토글로 가격 0원을 빠르게 골라낸다.

거리

위치 권한이 필요한 필터

0~50km 슬라이더. 권한이 없으면 위치 좌표가 비어 거리 조건이 결과에 반영되지 않는다 — 이 사일런트 무효는 운영 이슈로 별도 다룬다.

지역

지역명 부분 일치

"강남", "홍대" 같은 지역명을 선택하면 이벤트 주소에 그 단어가 포함된 결과만 남는다. 지역을 고르면 그 지역에서 인기 있는 카테고리 추천 칩이 함께 떠오른다.

인원수

현재 비활성 — UI에만 존재

"최소 N명 이상" 입력은 화면에 있지만 현재 서버에 해당 조건이 없어 결과에 반영되지 않는다. 사용자에게는 칩이 보이지만 결과는 줄지 않는다 — 이 사일런트 무효는 운영 이슈로 별도 다룬다.

필터 한 번이 만드는 흐름

사용자가 필터 시트를 열어 조건을 바꾸면 미리보기 카운트가 갱신되고, 적용하면 결과 화면이 첫 페이지부터 다시 받아 온다.

사용자 시나리오

기본 흐름

"음식" 검색 결과가 너무 많다

"음식"으로 검색했더니 24건이 나왔다. 필터 시트를 열어 카테고리 "맛집탐방" 칩을 누르니 하단 버튼이 "12개 결과 보기"로 줄어든다. 가격 슬라이더를 0~30,000원으로 좁히니 "8개 결과 보기"로 다시 줄어든다. 사용자는 "이 정도면 적당하다"고 판단하고 적용한다.

이 흐름이 남기는 것 — 미리보기 숫자가 적용 전 결정을 도와주는 핵심 단서. 이 숫자가 없으면 사용자는 매번 적용하고 결과를 보고 다시 풀어야 한다.

지역 + 추천

강남에서 무엇이 인기인지부터 본다

지역을 "강남"으로 고르자 시트 안에 그 지역에서 인기 있는 카테고리 추천 칩이 떠오른다 ("맛집탐방(8) · 스포츠(5) · …"). 사용자가 "맛집탐방(8)" 추천 칩을 누르면 그 카테고리가 자동으로 선택되고 카운트가 갱신된다.

이 흐름이 남기는 것 — "어디서 뭐가 인기인지 모르겠다"는 사용자가 빈 결과를 만나지 않게, 현재 데이터에서 의미 있는 카테고리를 보여 주어 의사결정의 출발점을 만들어 준다.

위치 권한

5km 안의 모임을 찾고 싶다

거리 슬라이더를 5km로 맞추자 시스템이 위치 권한을 묻는다. 사용자가 허용하면 미리보기 카운트가 위치 기반으로 다시 계산된다. 거부하면 거리 조건이 결과에 반영되지 않은 채 "5km 이내" 칩만 화면에 남는다.

이 흐름이 남기는 것 — 권한 거부 후 필터가 적용된 듯한 칩만 보이는 상태는 운영 결함. 사용자가 "왜 5km라 했는데 멀리 있는 게 나오지" 라고 의심하게 둔다.

개별 칩 제거

한 가지 조건만 풀고 싶다

활성 필터 칩 네 개가 보이는 상태에서 "맛집탐방" 칩의 X를 누른다. 그 카테고리만 해제되고 다른 필터는 그대로 유지된 채 결과가 다시 받아 와진다.

전체 초기화

필터를 잘못 걸어 답답하다

여러 조건을 걸다가 결과가 0건이 되었다. 활성 필터 바의 맨 앞 "초기화" 칩을 누르면 모든 조건이 한 번에 풀리고 키워드만 남은 결과로 돌아간다.

이 흐름이 남기는 것 — 한 번에 풀 수 있는 탈출구가 없으면 사용자는 필터 시트를 다시 열어 하나씩 해제하는 답답한 경험을 한다.

잠재 결함

"10명 이상" 으로 좁혔는데 결과가 그대로

사용자가 "최소 인원 10명 이상" 으로 좁히고 적용한다. 검색바 아래에 "10명 이상" 칩은 보이지만 결과 개수는 변하지 않는다 — 이 조건은 현재 서버에 없어서 무시된다.

이 흐름이 남기는 것 — 사용자에게 보이는 칩과 실제 결과 사이의 불일치는 검색의 신뢰를 깎는다. 화면에서 인원수 필터를 빼거나 서버에 해당 조건을 추가해야 한다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
필터 시트 헤더 "필터" 타이틀 / "초기화" 텍스트 버튼 / X 닫기. "초기화"가 닫기와 분리되어 있어야 한 번에 처음부터 다시 만들 수 있다. X는 변경을 버리는 의미라는 점이 분명해진다.
지역 기반 추천 칩 "맛집탐방 (8) · 스포츠 (5) · …" 형태의 추천 카테고리 + 그 지역의 가격/날짜 범위. 지역을 처음 고른 사용자에게 "이 지역에 무엇이 얼마나 있는지"를 보여 줘 빈 결과를 만나지 않게 한다.
미리보기 카운트 버튼 하단 sticky "N개 결과 보기" 버튼. 조건을 바꾸면 잠시 후 숫자가 갱신. "적용해 봐야 결과를 안다"는 답답함을 없앤다 — 이 단위의 가장 중요한 상호작용.
활성 필터 칩 바 검색바 아래 가로 스크롤 칩. 맨 앞에 "초기화" 칩, 이후 적용된 조건별 칩. 결과가 적을 때 "왜 적은지"를 사용자가 한눈에 보고 한 가지씩 풀 수 있게 한다.
거리 슬라이더 + 권한 안내 0~50km 슬라이더. 권한 없음 상태에서 사용 시 한 번 권한 요청 다이얼로그. 거리 필터는 사용자의 위치를 알아야 동작하므로, 거부 시 사용자가 그 한계를 인지하도록 안내가 필요하다 (현재 안내 부족 — 운영 이슈).

운영자가 알아야 할 것

검색 필터의 정책 결정점 + 알면서 둔 결함
  • 인원수 필터는 현재 화면에만 있고 결과에 반영되지 않는다. 사용자에게는 칩이 보이지만 결과 개수는 변하지 않는 사일런트 무효 상태. 화면에서 인원수 필터를 제거하거나 서버에 해당 조건을 추가해야 한다 — 이 불일치가 검색 신뢰를 깎는다.
  • 거리 필터는 위치 권한이 거부되면 조용히 무시된다. "5km 이내" 칩은 그대로 보이지만 멀리 있는 결과가 섞여 나온다. 권한 거부 시 사용자에게 안내 띠가 한 줄 필요하다.
  • 미리보기 카운트는 0.3초 디바운스로 동작한다. 사용자가 슬라이더를 연속해서 움직여도 마지막에 멈춘 값으로만 한 번 호출된다 — 서버 부하 보호.
  • 지역 기반 카테고리 추천은 실시간 집계라 캐시가 없다. 데이터가 많아지면 이 호출이 가장 먼저 느려질 가능성이 있다 — 카운트 결과를 캐시하는 후속 작업이 필요한지 점검 대상.
  • "필터 너무 강함" 빈 결과는 키워드 빈 결과와 안내가 달라야 한다. 현재 두 빈 상태가 같은 메시지를 쓰면 사용자는 "검색 자체가 안 된다"고 오해할 수 있다. "조건을 풀어 보세요" 같은 다른 안내가 필요한지 검토 대상.
