F08-11. 아이템 · 번들 · 플랜 구매

포인트 잔액으로 마켓의 콘텐츠를 사는 한 흐름. 한 번의 "구매하기" 탭이 잔액 차감 · 보유함 적재 · 크리에이터 수익 분배까지를 분리될 수 없는 동작으로 처리한다. 단일 아이템 / 묶음 번들 / 플랜 단건 세 가지 경로가 있고, 각각의 가드(상태·재고·중복·한도·잔액)가 순서대로 검사된다. 잔액이 부족하면 결제가 일어나지 않고 곧장 충전 화면으로 빠질 수 있다. 마켓의 매출이 만들어지는 자리이자 — 크리에이터의 수익이 시작되는 자리.

구매가 일어났는데 보유함에 들어가지 않거나, 보유함에는 있는데 크리에이터 수익이 만들어지지 않거나, 수익은 있는데 잔액이 차감되지 않은 상태는 모두 사용자 신뢰가 무너지는 사고다. 그래서 다섯 단계(상태 → 재고 → 한도 → 잔액 차감 → 보유함·수익 적재)가 한 동작으로 처리된다. 중간에 어디서 실패하면 모든 변경이 함께 되돌려진다.

한 번의 구매가 통과하는 다섯 관문

사용자 입장에서는 한 번의 탭이지만, 서버는 다섯 가드를 순서대로 검사한다. 어느 하나라도 실패하면 결제·적재는 일어나지 않는다.

세 가지 구매 경로

경로 1

아이템 단건 구매

마켓 아이템 상세 하단 "구매하기"가 출발점. 결제 후 보유함에 한 줄이 추가되고, 크리에이터 수익이 가격 그대로 적재된다(수수료·세금 분개). 같은 아이템은 사용자가 여러 번 살 수 있다 — 마켓 아이템은 누적 구매를 막지 않는다.

경로 2

번들 구매 (묶음 할인)

번들 카드 안 "이 번들로 구매하기"가 출발점. 결제는 번들 단위로 한 번 일어나고, 보유함에는 번들이 포함한 아이템들이 각각 한 줄씩 들어간다. 번들에 보너스 포인트가 걸려 있으면 결제 직후 그만큼이 무료 포인트로 적립된다.

경로 3

플랜 직접 구매

플랜 미리보기 화면이나 마켓 플랜 상세에서 "구매하기"가 출발점. 마켓 아이템과 달리 한 사용자가 한 플랜을 한 번만 살 수 있다 — 중복 구매가 차단된다. 가격 0이면 결제 없이 보유 기록만 만들어진다(무료 플랜).

사용자 시나리오

기본 흐름

잔액 충분한 사용자가 한 아이템을 산다

잔액 50,000P인 사용자가 30,000P 플랜 아이템에서 "구매하기"를 누른다. 다섯 가드가 차례로 통과되고 잔액이 30,000P 차감된다. 같은 동작 안에서 보유함에 한 줄이 추가되고, 크리에이터에게는 수익이 적재된다(가격 30,000원에서 수수료 10%·세금 3.3%를 뺀 금액). "구매가 완료되었습니다" 안내 다이얼로그가 뜨고, 확인을 누르면 내 컬렉션 화면으로 이동한다.

이 흐름이 남기는 것 — 한 번의 탭으로 결제부터 보유함 적재, 크리에이터 수익 분배까지 모두 끝난다. 모든 데이터가 동시에 만들어지는 핵심 보장.

잔액 부족

5,000P로 30,000P 아이템을 사려 한다

잔액 부족이 잡혀 결제가 일어나지 않는다. "포인트가 부족합니다. 충전하시겠습니까?" 다이얼로그가 뜨고, 사용자가 "충전하기"를 누르면 충전 화면으로 빠진다. 부족한 금액(25,000P)이 미리 채워진 상태로 충전 화면이 열리는 경우도 있다.

이 흐름이 남기는 것 — 구매가 막혔을 때 사용자가 다음 행동(충전)으로 곧장 이어 갈 수 있게 한다. 결제·충전 도메인을 매끄럽게 잇는 자리.

번들 보너스

묶음을 사면서 보너스 포인트를 받는다

세 개 아이템 + 1,000P 보너스가 걸린 번들에서 "이 번들로 구매하기"를 누른다. 결제 한 번이 일어나고 보유함에 세 줄이 한꺼번에 추가된다. 같은 동작 안에서 1,000P가 무료 포인트로 적립되어 다음 구매에 쓸 수 있다.

이 흐름이 남기는 것 — 단건 구매보다 번들이 유리하다는 가치를 결제 직후 사용자가 체감한다. 마켓 평균 단가를 끌어올리는 자리.

중복 차단

이미 산 플랜을 한 번 더 사려 한다

이미 산 플랜의 직접 구매 버튼을 다시 누른다. 서버가 중복 구매를 거절하고 "이미 구매한 플랜입니다" 안내가 뜬다. 마켓 아이템 단건 구매는 이 가드가 없어 같은 아이템도 누적 구매가 가능하다 — 두 정책 차이는 운영 정렬이 필요한 지점.

재고 소진

마지막 한 개를 두 사용자가 동시에 잡는다

재고 1개 남은 아이템에 두 사용자가 동시에 구매를 시도한다. 한쪽 결제가 먼저 성사되고 재고가 0이 되면, 두 번째 사용자에게는 재고 소진 안내가 뜬다.

한도 차단

이번 달 50번째 구매를 끝낸 사용자가 한 번 더 시도

한 달 동안 50건을 산 사용자가 51번째 구매를 시도한다. 잔액·상태와 무관하게 거절된다. 사용자에게는 "잠시 후 다시 시도해 주세요" 류 안내가 뜨고, 익월이 되어야 다시 구매할 수 있다.

운영 관찰 포인트 — 현재 한도 차단의 안내 카피가 일반 실패와 구분되지 않는다. 정상 운영에서 50건은 충분하지만, 마켓이 커지면 이 한도 조정이 필요할 수 있다.

본인 플랜

크리에이터가 자기 플랜을 사려 한다

작성자 본인이 자기가 만든 플랜의 직접 구매를 시도한다. 거절된다 — "본인 플랜은 구매할 수 없습니다". 자기가 자기 플랜을 사서 매출을 만드는 부정 행위를 막는 가드.

무료 플랜

가격 0인 플랜을 받는다

무료 플랜의 구매 버튼을 누른다. 결제 단계가 건너뛰어지고 보유 기록만 만들어진다. 잔액은 변하지 않고, 사용자는 곧장 본문을 읽을 수 있다.

비로그인

가입 안 한 사용자가 구매를 시도

비로그인 상태로 마켓 아이템 상세에서 "구매하기"를 누른다. "로그인이 필요합니다" 안내가 뜨고 로그인 화면으로 빠진다. 이 화면 자체가 비로그인 진입 차단은 아니지만, 구매 시점에서 인증 가드가 작동한다.

운영자가 알아야 할 것

이 단위에서 무너지면 마켓 매출과 크리에이터 수익 둘 다 무너진다
  • 잔액 차감 · 보유함 적재 · 크리에이터 수익 적재는 한 동작 안에서 처리되어야 한다. 결제가 됐는데 보유함에 안 들어갔거나, 보유함에는 있는데 수익이 안 만들어진 상태는 한 번이라도 발생하면 사용자 신뢰가 즉시 무너진다.
  • 마켓 아이템 단건은 중복 구매가 허용되고, 플랜 단건은 차단된다. UI에서는 같은 "구매하기" 버튼처럼 보이지만 서버 정책이 다르다. 사용자가 같은 마켓 아이템을 여러 번 사는 의도가 정말로 있는지, 아니면 실수 누적인지는 추적해서 판단할 운영 결정점.
  • 번들 구매 시 개별 아이템의 크리에이터 수익은 현재 적재되지 않는다. 번들 전체 단위로만 매출이 잡혀 있어, 한 번들 안의 여러 작성자에게 수익을 나누는 정책이 아직 정렬되지 않았다. 번들 사용이 늘어나면 이 정책 정렬이 시급해진다.
  • 충전 후 자동 재구매는 없다. 사용자가 잔액 부족으로 충전 화면에 갔다가 돌아와도 구매를 다시 눌러야 한다. 충전 후 컨텍스트가 사라지는 경험은 사용자 이탈로 이어질 수 있는 자리.
  • 월 50건 한도는 일반 사용자 보호 + 부정 결제 차단의 합산 정책이다. 풀면 부정 결제 위험이 커지고, 줄이면 단골 구매자가 막힌다.
  • 동시 구매가 재고·월 한도·번들 보너스를 우회하지 못하게 막혀 있다 (2026-06-06). 같은 사용자가 거의 동시에 여러 번 구매를 밀어 넣어도 재고가 마이너스로 떨어지거나 월 50건 한도를 넘기거나 번들 보너스가 중복 지급되지 않는다 — 한 번에 한 처리만 진행되도록 잠금이 걸려 있다.
  • 수수료 10% + 원천세 3.3%는 정산의 시작점이다. 가격 변경이 잦은 운영은 크리에이터 수익 추적이 어려워진다 — 정산 페이지에서 누적 매출과 어긋날 수 있다.
  • 현재 구매 확인 시트는 없다. 사용자가 "구매하기" 한 번에 결제가 일어난다. 실수 결제 비율이 늘어나면 확인 시트 도입을 검토해야 한다.
