F02-03. 섹션 카드 진입 (이벤트·클럽·플랜)

홈의 각 카드가 어디로 보내는지를 결정한다. 본 단위는 “라우팅까지” 책임지고, 잘못된 식별자 처리·인증 가드는 진입한 도메인 단위가 흡수한다. 추천 이벤트는 추천 상세를 한 번 거치는 우회 경로, 나머지는 이벤트/클럽/플랜 상세로 직진하는 단일 경로다.

카드 1

추천 이벤트 카드

추천 상세(SCR-HF-003)를 거쳐 이벤트 상세로 두 단계 이동. 추천 이유·유사 이벤트 노출.

카드 2

예정/인근 이벤트 카드

이벤트 상세로 직진.

클럽 상세로 직진.

카드 4

최신 플랜 카드

마켓 플랜 상세로 직진.

사용자 시나리오 7개

S1

추천 이벤트 → 추천 상세 → 이벤트 상세

  1. 추천 이벤트 카드 탭
  2. SCR-HF-003 진입(추천 이유 + 유사 이벤트)
  3. “이벤트 상세 보기” 탭 → 이벤트 상세
S2

예정/인근 카드 → 이벤트 상세 직진

  1. 카드 탭 → 이벤트 상세로 바로 이동
S3

인기 클럽 카드 → 클럽 상세

  1. 카드 탭 → 클럽 상세로 이동(F04 영역)
S4

최신 플랜 카드 → 마켓 상세

  1. 플랜 카드 탭 → 마켓 플랜 상세(F08 영역)
S5 · 복귀

백버튼으로 홈 복귀 시 캐시 복원

  1. 30초 전 진입 → 카드 탭 → 백버튼
  2. 홈 재 build → 캐시 유효 → 즉시 렌더, 호출 없음
S6 · 잘못된 ID

비정상 식별자

  1. 음수/0 같은 비정상 ID로 라우팅
  2. 본 단위는 라우팅까지만 보장
  3. 상세 화면이 자체적으로 404/400 처리
S7 · 게스트

비로그인 카드 탭

  1. 게스트가 클럽 카드 탭
  2. publicRoute에 포함된 경로면 진입
  3. 인증이 필요한 액션 시점에 로그인 유도

자주 빠뜨리는 케이스

레드 플래그
  • 추천 카드가 추천 상세를 건너뛰고 이벤트 상세로 직진 — 추천 이유·유사 이벤트 노출 누락.
  • 복귀 시 캐시 무시하고 재호출 — 사용자 체감 지연 증가.
  • 게스트 카드 탭이 401 화이트스크린 — publicRoute 정책 확인 필요.
