F02-03. 홈 카드 진입

홈의 네 영역 카드를 탭했을 때 어떤 화면이 열리느냐를 정의한다. 본 단위는 사용자를 다른 화면으로 보내는 데까지만 책임지고, 도착한 화면의 동작은 각 도메인 단위가 맡는다. 카드별로 사용자의 의도가 다르므로 도착 화면도 다르게 둔다.

설계 의도 — "추천을 누른 사람"과 "이벤트를 누른 사람"의 의도는 다르다

같은 이벤트라도 추천 카드에서 들어오는 사용자는 "왜 이게 나에게 추천됐는가"를 먼저 알고 싶어 한다. 반대로 "곧 시작하는 이벤트"에서 들어오는 사용자는 이미 시간·장소 결심이 서 있어 신청 화면으로 바로 가려 한다. 그래서 두 카드의 행선지는 의도적으로 다르다 — 추천 카드는 추천 이유와 유사 카드가 함께 보이는 추천 상세로, 예정/인기 카드는 곧장 이벤트 상세로. 이 분기가 흐려지면 추천의 가치 자체가 사라진다.

카드별 행선지

카드 영역도착 화면그 화면으로 보내는 이유
추천 이벤트 추천 상세 화면 (추천 이유 + 유사 카드를 함께 보여 주는 면) "왜 추천인가"를 사용자에게 설명할 기회를 한 번 끼워 넣는다. 신뢰의 기회.
곧 시작하는 이벤트 이벤트 상세 화면 (신청 가능 상태) 의도가 분명한 카드이므로 한 단계라도 더 끼우면 이탈이 늘어난다.
인기 클럽 클럽 상세 화면 가입 여부를 확인하고 둘러보는 흐름. 카드에서 본인 상태 배지를 이미 본 상태로 도착.
새로 올라온 플랜 마켓 플랜 상세 화면 구매 결정 단서(가격·후기·제작자 정보)가 모인 면으로 직진.

사용자 시나리오

의도 분기

"왜 추천됐는지"가 보고 싶을 때

추천 카드를 누르면 추천 상세로 진입한다. 추천 이유가 한 줄로 먼저 보이고, 유사한 다른 추천이 함께 보인다. 사용자가 "이벤트 상세 보기" 버튼을 누르면 본 이벤트 상세로 들어간다. 한 단계가 더 들어가지만, 이 단계를 거치고 나면 추천 자체에 대한 사용자의 신뢰가 달라진다.

의도 분기

이미 마음이 정해진 카드를 누른다

"곧 시작" 영역 카드는 바로 이벤트 상세로 떨어진다. 추천 설명을 거치지 않고 신청 단계로.

이 흐름이 남기는 것 — 시간 의도가 분명한 사용자에게 불필요한 단계를 끼우지 않는다.

상태 보존

들어갔다가 짧게 돌아옴

상세에서 뒤로가기를 누르면 홈은 직전 데이터·스크롤 위치 그대로다. 사용자는 다시 둘러보기를 이어 갈 수 있다 — 카드 탐색은 보통 여러 장을 비교해 보는 행위.

가입 전

비로그인 사용자가 카드를 누름

인증이 없어도 상세 화면들은 대부분 둘러볼 수 있게 열려 있다. 실제 신청·가입·구매 같은 행동 시점에만 로그인 유도가 떠오른다 — 본 단위는 그 가드를 호출하지 않고, 도착한 화면이 알아서 처리한다.

운영자가 알아야 할 것

행선지 정책 결정점
  • 추천 카드는 항상 추천 상세를 한 번 거치게 둔다. 추천 이유를 보지 않고 바로 상세로 가게 두면 추천 시스템에 대한 신뢰가 측정 불가능해진다.
  • 예정/인기/플랜은 직진으로 두는 것이 사용자 의도와 일치한다. 추후 데이터 분석에서 이탈 지점이 상세에 몰리면 카드 콘텐츠를 손봐야 하지 동선을 손대면 안 된다.
  • 잘못된 식별자(예: 삭제된 이벤트 ID)는 본 단위에서 거르지 않는다. 도착한 화면이 빈 상태로 처리한다 — 진입 가드가 흩어지면 유지보수가 어려워진다.
