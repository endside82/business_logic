F20-01. 1:1 문의

사용자가 카테고리를 골라 관리자에게 스레드를 여는 자유 문의 채널. 계정·결제·이벤트·클럽·신고·기타 6가지 카테고리가 있고, 특정 이벤트·클럽·정산 컨텍스트를 연결하면 그 자리의 호스트도 답변에 참여할 수 있도록 설계됐다. 상태는 접수됨 → 답변 완료 → 종결 3단계이며, 사용자가 추가 메시지를 보내면 다시 접수됨으로 돌아간다.

현재 상태 — 서버 준비됨, 앱 화면 없음

서버 쪽 접수·조회·답장 처리와 본인 참여 확인은 모두 동작한다. 그러나 앱에 1:1 문의 화면과 진입점이 아직 만들어지지 않아 현재 사용자는 앱으로 1:1 문의를 접수할 수 없다 — 서버 준비만 끝난 상태다. 운영 이슈(F20-02)는 앱 화면이 완성돼 있으므로, 결제·환불·정산 건이라면 운영 이슈 채널을 먼저 사용한다.

사용자 시나리오

일반 문의

"비밀번호를 바꾸려는데 이메일이 안 와요"

카테고리 "계정"을 고르고 제목·내용을 작성해 접수한다. 특정 모임이나 거래에 걸린 문의가 아니므로 별도 확인 없이 바로 관리자 대기열에 들어간다. 관리자가 답변하면 상태가 "답변 완료"로 바뀐다. 사용자가 추가 질문을 보내면 "접수됨"으로 되돌아간다.

이 흐름이 남기는 것 — 관리자가 답변까지의 대화 이력 전체를 한 자리에서 볼 수 있다. 사용자도 진행 상태를 문의 목록에서 확인한다.

모임 연결 문의

"이 이벤트 참가비 영수증이 잘못됐어요"

카테고리 "이벤트"를 고르고 해당 이벤트를 지정해 접수한다. 내가 실제로 그 이벤트의 확정 참가자임을 확인한 뒤 문의가 만들어진다 — 참여하지 않은 남의 모임을 끌어와 문의하는 것은 차단된다. 연결된 이벤트가 문의에 기록돼 관리자가 배경을 빠르게 파악할 수 있다.

현재 한계 — 호스트가 직접 답변하는 경로는 아직 동작하지 않는다. 관리자가 중간에서 처리한다.

정산 연결 문의

"정산 금액이 제가 계산한 것과 달라요 (호스트)"

해당 정산을 지정해 접수한다. 그 정산의 호스트가 문의자 본인인지 확인한다. 정산 호스트만 정산에 연결한 문의를 넣을 수 있다.

주요 규칙

종결 뒤 추가 불가

종결된 문의에 메시지 추가 시도

관리자가 문의를 종결 처리하면 사용자는 더 이상 메시지를 추가할 수 없다. 종결된 문의에 추가를 시도하면 막힌다.

내 문의만 볼 수 있다

타인 문의 접근 차단

문의 상세 조회는 작성자 본인만 가능하다. 다른 사람의 문의를 직접 열어 보려 하면 접근이 차단된다.

사용자 추가 메시지 → 재접수

답변 완료 상태에서 사용자가 다시 질문

관리자가 답변을 마친 뒤 사용자가 추가 질문을 보내면 문의 상태가 자동으로 "접수됨"으로 되돌아가 다시 답변 대기열에 들어간다.

알려진 격차

앱 화면 미구현
  • 1:1 문의를 위한 앱 화면과 진입점이 아직 만들어지지 않아, 현재 사용자는 앱으로 이 기능을 쓸 수 없다. 서버 준비만 끝난 상태다.
  • 이벤트·클럽에 연결된 문의에서 호스트가 직접 답변하는 기능이 아직 동작하지 않는다. 답변 이력을 담을 자리는 준비돼 있다.
  • 관리자가 답변할 때 사용자에게 푸시 알림이 실제로 전달되는지 확인이 필요하다.
