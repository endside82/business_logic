F14-03. 위치 공유 만료 연장

모임이 예상보다 길어졌을 때, 위치 공유를 다시 켜는 번거로움 없이 종료 시각을 한 번에 30분 미루는 액션. 위치 공유는 마지막 갱신에서 2시간 뒤 자동으로 멈추므로, 모임이 더 길어지면 사용자가 직접 한 번 더 연장할 자리를 둔다. 만료 30분 전에 푸시 알림이 한 번 가서 사용자가 잊지 않도록 돕는다.

모임이 길어졌을 때 사용자에게 위치 공유를 처음부터 다시 켜라고 요구하면, 권한 흐름·동의 흐름이 다시 한 번 마음에 부담을 만든다. 그래서 본 단위는 "이미 켜져 있는 상태를 유지하면서 종료 시각만 미룬다"로 설계했다. 연장은 한 번에 30분, 너무 길게 미루지 못하게 120분 상한을 두어 사용자가 모임이 끝났는데도 위치를 켜둔 채 잊는 사고를 방지한다.

연장이 일어나는 두 가지 자리

화면 안

위치 화면 시트의 "+30분 연장" 버튼

위치 공유가 켜져 있는 동안 시트에는 "약 N분 후 종료" 카운트다운이 1초 단위로 줄어든다. 그 옆에 작은 "+30분 연장" 버튼이 있어, 사용자가 한 번 누르면 카운트다운이 30분만큼 즉시 늘어난다. 누른 직후 "+ 30분 연장됨" 토스트가 떠 처리 결과를 확인시켜 준다.

화면 밖

만료 30분 전 푸시 알림

위치 공유가 30분 뒤에 끝나는 시점에 사용자에게 푸시 알림이 한 번 간다. 알림을 누르면 곧장 본 이벤트의 위치 화면으로 돌아오게 되어, 사용자가 그 자리에서 연장 버튼을 누르거나 모임이 끝났음을 인지하고 직접 끌 수 있다.

기본 흐름

알림을 보고 연장한다

모임이 길어지고 있는 사용자에게 "위치 공유가 곧 종료됩니다" 푸시가 한 번 간다. 알림을 누르면 위치 화면이 열리고 카운트다운이 보인다. "+30분 연장"을 누르면 즉시 30분만큼 카운트다운이 늘어나고 새 종료 시각의 30분 전에 다시 한 번 알림이 갈 수 있다.

이 흐름이 남기는 것 — 모임이 끝났는데 위치가 계속 보내지는 상황과, 모임 중인데 위치가 갑자기 끊기는 상황 모두를 사용자가 막을 수 있다.

알림 없이 능동적으로

모임이 길어질 것 같아 미리 연장

모임이 한참 진행 중인데 카운트다운이 1시간 5분 남았다. 사용자가 알림을 기다리지 않고 직접 "+30분 연장"을 눌러 종료 시각을 미리 미룬다. 한 번에 30분씩만 늘어나는 단순 구조라 사용자가 지나치게 길게 미루는 사고가 없다.

이 흐름이 남기는 것 — "왜 30분만 늘어나지?"라는 질문은 의도된 설계 — 사용자가 잊었을 때를 대비해 짧은 단위로만 미루도록.

만료 후 시도

이미 끝난 공유를 연장하려고 한다

알림을 늦게 본 사용자가 위치 화면을 열었을 때는 이미 공유가 만료된 상태일 수 있다. "+30분 연장" 버튼이 더 이상 보이지 않거나, 누르면 "연장에 실패했습니다" 토스트가 뜬다. 이 경우 사용자는 위치 공유 토글을 다시 켜서 새로 시작해야 한다(F14-01).

이 흐름이 남기는 것 — 만료된 공유는 연장이 아니라 새로 시작 — 의도된 단계 분리.

한도

너무 길게 미루려 해도 막힌다

본 단위의 연장은 한 번에 최대 120분으로 강제 제한된다. 어떤 값을 보내도 120분이 상한선이며 너무 짧거나 잘못된 값도 1분으로 보정된다.

이 흐름이 남기는 것 — 사용자가 위치 공유를 켜둔 채로 잊는 사고를 정책이 일정 수준에서 끊어준다.

호스트 분기

호스트가 본인 위치를 한 번도 안 보낸 상태에서 연장

호스트는 일행 위치를 보는 데 본인 공유가 필요 없지만, 연장은 본인 좌표가 한 번이라도 기록된 후에만 가능하다. 본인 좌표가 없는 호스트가 연장을 누르면 "연장에 실패했습니다" 토스트가 뜬다.

이 흐름이 남기는 것 — 연장은 자기 위치를 연장하는 동작이지, 일행 위치 노출을 연장하는 동작이 아니다.

알림 차단

이미 끈 사용자에게는 알림이 가지 않는다

이미 위치 공유를 끈 사용자에게는 "곧 종료됩니다" 푸시가 자동으로 보내지지 않는다. 꺼진 사용자에게 알림을 보내는 것은 다시 켜라는 압박이 되므로, 그런 알림은 명시적으로 차단된다.

이 흐름이 남기는 것 — 사용자가 끈 의사 표시가 알림 영역까지 일관되게 적용된다.

화면에서 사용자가 보는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
카운트다운 "N분 M초 후 종료" 텍스트가 1초마다 줄어든다. 만료 시점이 되면 빨간 색으로 "공유 종료됨". "내 위치 공유가 언제 끝나는지"를 사용자가 항상 인지하게 한다 — 위치 공유의 4가지 원칙 중 "공유 상태 시각화"의 실현.
+30분 연장 버튼 카운트다운 오른쪽의 작은 보조 버튼. 본인 위치가 켜져 있고 종료 시각이 설정되어 있을 때만 노출. 꺼진 상태나 만료 후에는 의미 없는 버튼이므로 보이지 않게 한다. 사용자가 "왜 안 늘어나지?"라는 의문을 갖지 않게.
완료 토스트 "+ 30분 연장됨" / 실패 시 "연장에 실패했습니다". 카운트다운이 즉시 늘어나는 시각적 변화에 더해 명확한 결과 통지. 처리 결과 모호함을 없앤다.
만료 30분 전 푸시 "위치 공유가 곧 종료됩니다" 같은 알림 한 줄. 누르면 위치 화면으로 이동. 앱을 안 보고 있는 사용자에게 "지금이 결정 시점"이라는 신호. 사용자가 잊었을 때 자동 멈춤이 시작되기 전에 결정을 미루는 자리.

운영자가 알아야 할 것

연장 정책 결정점
  • 연장 단위는 한 번에 30분으로 고정한다. 사용자가 모임 시간을 직접 입력하게 두면 너무 길게 잡고 끄는 것을 잊는 사고가 늘어난다. 짧게 끊어 가는 것이 자동 만료 정책의 의미를 살린다.
  • 한 번에 최대 120분 상한은 정책 결정. 이 상한은 코드에 강제되어 있다. 늘리는 것은 가능하지만, 그러면 "위치 데이터가 모임 시간대 안에만 흐른다"는 도메인 원칙과 충돌한다.
  • 만료 알림은 새 종료 시각의 30분 전에 다시 보내질 수 있다. 사용자가 연장을 한 번 하면 그 다음 만료 30분 전에 또 한 번 알림을 받는다 — 모임이 길게 이어지는 동안 사용자가 반복해서 결정 시점을 받는 구조.
  • 꺼진 사용자에게는 만료 알림을 보내지 않는다. "꺼져 있는데 알림이 온다"는 사용자가 가장 불쾌하게 받는 신호. 사용자가 끈 의사가 알림 흐름까지 일관되게 적용되어야 한다.
  • 연장 실패 분기(자격 미충족, 본인 좌표 미존재, 이미 만료됨)는 현재 단순 "연장에 실패했습니다" 한 메시지로만 안내된다. 사용자에게는 왜 실패했는지 구분이 잘 안 된다. 분기별 안내 워딩을 다듬으면 사용자가 "다음에 뭘 해야 하는지" 파악이 쉬워진다.
