F09-05. 데이팅 채팅

매칭이 성사된 두 사용자가 처음 대화를 나누는 면. 매칭 직후 곧장 열리는 채팅방에서 1:1 메시지를 주고받고, 상대가 입력 중인지·내가 보낸 메시지를 읽었는지·실시간 연결이 살아 있는지 모든 신호가 한 자리에서 보인다. 모든 메시지는 암호화되어 저장되고, 어느 한쪽이 상대를 차단하면 같은 동작으로 채팅방도 함께 닫힌다. 데이팅에서 사용자가 "관계가 시작된다"는 감각을 처음 받는 화면이다.

매칭만으로는 사용자가 가치를 느끼지 못한다. 두 사람이 실제로 메시지를 한 번이라도 주고받았을 때 비로소 "데이팅이 작동한다"는 인지가 만들어진다. 그래서 본 단위는 매칭 성사 직후 즉시 열리고, 진입과 동시에 미읽음을 0으로 만들고, 상대의 입력 중 표시·내 메시지 읽음 표시를 실시간으로 주고받는다. 네트워크가 불안정해도 같은 메시지가 두 번 저장되지 않는다 — 보낸 기기가 메시지마다 고유 표식을 붙여 중복을 걸러낸다. 안전과 신뢰가 본 화면의 두 축이고, 메시지 암호화·차단 시 채팅방 동시 종료가 그 받침이다.

한 메시지가 송신되는 자리

사용자에게는 한 동작이지만, 그 안에서 암호화·중복 방지·실시간 전달이 순서대로 처리된다.

사용자 시나리오

기본 흐름

매칭 직후 첫 메시지

방금 매칭이 성사된 두 사용자가 채팅방에 처음 들어온다. "대화를 시작해보세요 💬" 라는 빈 상태 안내가 떠 있고, 한쪽이 "안녕하세요"를 입력해 송신한다. 상대에게는 푸시 알림과 함께 메시지가 즉시 도착하고, 상대가 화면을 보고 있다면 곧 "읽음" 표시가 송신자에게 돌아온다.

이 흐름이 남기는 것 — 매칭 성사부터 첫 메시지까지의 시간이 짧을수록 매칭의 가치가 실제 관계로 이어진다. 매칭 → 채팅 → 첫 메시지 동선이 끊기지 않는 게 본 단위의 핵심 목표.

실시간 신호

상대가 입력 중인 게 보인다

상대가 메시지 입력창에 글을 쓰기 시작하면 "상대방이 입력 중..." 표시가 내 화면 하단에 자리한다. 상대가 2초 이상 멈추면 표시도 사라진다. 상대가 송신하면 표시는 즉시 사라지고 메시지가 자리에 들어온다.

이 흐름이 남기는 것 — "내가 보낸 메시지에 대한 응답을 상대가 지금 쓰고 있다"는 즉시성. 데이팅 채팅에서 가장 만족도가 높은 신호 중 하나.

네트워크 끊김

송신 도중 신호가 끊겼다 돌아옴

지하철 진입 직전에 메시지를 보낸다. 신호가 끊겨 화면이 송신 실패 토스트를 보인다. 사용자가 같은 자리에서 다시 송신을 시도한다. 같은 메시지가 두 번 저장되지 않도록 서버가 클라이언트가 부여한 키를 검사해 — 기존 메시지가 있으면 그것을 그대로 반환한다.

이 흐름이 남기는 것 — 사용자가 두 번 보낸 것처럼 보이지 않게 한다. 데이팅에서 같은 메시지가 두 번 도착하는 사고는 사용자 신뢰를 가장 빠르게 깨는 신호 중 하나.

실시간 연결

연결이 잠시 끊겼을 때

실시간 채널이 일시적으로 끊기면 상단에 작은 배지가 "연결 중..." 또는 "재연결 중..."으로 자리한다. 실시간 채널이 죽어 있어도 메시지 송수신 자체는 일반 통신으로 정상 작동한다 — "실시간 연결 실패. 메시지는 정상 전송됩니다." 토스트가 한 번 뜨는 것은 같은 세션에서 한 번뿐.

이 흐름이 남기는 것 — 실시간 신호가 부가 기능임을 사용자가 인지하게 한다. 핵심(메시지 전달)은 그대로 작동한다는 안심.

차단 발동

대화 중에 차단으로 단절

대화를 이어가던 사용자가 부적절한 메시지를 받고 즉시 차단 흐름으로 이동한다. 차단이 확정되면 그 자리에서 채팅방이 닫히고, 다음 송신 시도부터는 메시지가 보내지지 않는다. 채팅방 목록을 새로고침하면 닫힌 방은 활성 목록에서 자연스럽게 사라진다.

이 흐름이 남기는 것 — 데이팅의 가장 중요한 안전 경로 중 하나. 한 동작으로 대화를 완전히 끊을 수 있어야 사용자가 채팅에 안심하고 들어온다.

비활성 프로필

매칭은 살아있지만 본인 프로필이 비활성

본인 프로필을 비활성으로 둔 사용자가 기존 매칭의 채팅방에 진입한다. 메시지는 정상적으로 읽고 스크롤할 수 있지만, 입력바 자리에는 "프로필이 비활성화 상태에서는 메시지를 보낼 수 없습니다" 안내가 자리한다. 본인 프로필을 다시 활성으로 켜면 정상적으로 송신 가능해진다.

이 흐름이 남기는 것 — 비활성 토글의 의미를 채팅에서도 일관되게 유지한다. "매칭은 그대로지만 내가 새로 행동하지는 않는다"는 사용자 의도를 채팅에서도 존중.

푸시 진입

알림 누르고 직접 진입

다른 화면에 있던 사용자가 새 메시지 푸시 알림을 누른다. 앱이 곧장 해당 채팅방을 열고, 진입 즉시 미읽음 0으로 정리된다. 상대 화면에는 "읽음" 표시가 곧 자리한다.

이 흐름이 남기는 것 — 사용자가 어디에 있든 매칭 채팅으로 곧장 들어올 수 있는 경로. 데이팅의 재진입을 가장 자주 만드는 진입점.

만남 제안 진입

채팅에서 만남 제안 화면으로

대화가 무르익은 사용자가 상단 📅 아이콘을 누른다. 만남 제안 화면(F09-06)으로 이동해 일시·장소를 작성하고 제안을 보낸다. 제안을 보낸 직후 다시 채팅으로 돌아오고, "만남을 제안했습니다" 토스트로 결과를 인지한다.

이 흐름이 남기는 것 — 채팅이 그저 메시지 교환이 아니라 오프라인으로 이어지는 다리임을 인지하게 한다. 채팅 → 만남 제안 동선이 데이팅 도메인 가장 가치 있는 전환.

화면에서 사용자가 인지하는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
채팅방 목록 카드 아바타 + 상대 이름 + 마지막 메시지 한 줄 + 시각 + 미읽음 배지. 매칭이 활성화된 모든 대화가 한 자리에 모인다. 닫힌 방은 보이지 않아 사용자가 "지금 살아 있는 대화"에만 집중하게.
채팅 상단 헤더 상대 이름 + 우측 📅 아이콘(만남 제안 진입) + 연결 상태 배지(필요할 때만). 대화의 다음 단계로 가는 경로(📅)와 실시간 연결의 신호를 같은 자리에 둔다. 사용자가 한 화면을 떠나지 않고 다음 행동을 결정.
메시지 버블 내 메시지는 분홍색 우측 정렬, 상대 메시지는 흰 배경 좌측 정렬 + 아바타. 시각은 메시지 옆에 작게. 한 화면에서 누가 누구인지 즉시 식별. 색과 정렬을 좌우로 갈라 시각적 부담을 최소화.
읽음 표시 내가 보낸 메시지의 시각 옆에 "읽음" 두 글자. "상대가 봤다"는 확인은 데이팅에서 송신자에게 가장 의미 있는 신호. 강한 시각 요소 없이 두 글자로만 표현해 과하지 않게.
입력 중 표시 "상대방이 입력 중..." 하단에 한 줄로 자리. "상대가 응답을 쓰고 있다"는 즉시성. 데이팅 채팅에서 만족도가 가장 큰 신호 중 하나라 메시지 영역과 분리된 자리에 둔다.
입력바 회색 배경 둥근 입력창 + 분홍색 원형 송신 버튼. 빈 메시지는 송신되지 않는다. 의도 없는 송신을 막는다. 송신 버튼은 입력창과 명확히 분리되어 한 동작으로 식별.
비활성 안내 본인 프로필 비활성 시 입력바 자리에 "프로필이 비활성화 상태에서는 메시지를 보낼 수 없습니다" 안내. 입력 자체를 막아 사용자가 송신을 시도하다 실패하는 좌절을 미리 차단. 안내문이 다음 행동(활성 토글)을 자연스럽게 가리킨다.
빈 상태 안내 매칭 직후 첫 진입 시 "대화를 시작해보세요 💬". 매칭 직후의 어색함을 가벼운 카피로 받아 준다. 첫 메시지를 누가 보낼지 결정하는 자리를 비워 두지 않는다.

측정해야 하는 것

본 단위가 건강하다고 말하려면 이 다섯 가지 신호가 추적되어야 한다
  • 매칭 성사 → 첫 메시지까지의 시간 — 매칭의 가치가 실제 관계로 이어지는 지표. 너무 길면 매칭이 그저 숫자에 그치고 있다는 신호.
  • 채팅방 진입율 — 매칭이 만들어진 사용자 중 실제로 채팅방을 한 번이라도 연 비율. 낮으면 매칭 → 채팅 동선에 끊김이 있다.
  • 실시간 연결 안정성 — "연결 중·재연결 중" 표시가 사용자에게 보이는 빈도. 잦으면 사용자가 실시간 신호를 신뢰하지 않는다.
  • 차단 발동 빈도 — 채팅 중에 차단이 발동되는 비율과 사유. 특정 사유가 늘어나면 메시지 내용 검수나 후보 카드 단계의 필터 강화가 필요한 신호.
  • 채팅 → 만남 제안 전환율 — 대화가 시작된 매칭 중 만남 제안까지 간 비율. 데이팅 가치 사슬의 가장 중요한 마지막 전환.

운영자가 알아야 할 것

본 단위에서 무너지면 데이팅의 관계 형성이 끊긴다
  • 모든 메시지는 암호화되어 저장된다. 운영팀이 분쟁 조사 등으로 메시지를 봐야 할 때도 별도 권한 절차를 거친다. 평문 노출이 어디에서도 일어나지 않게 카드의 마지막 메시지 미리보기 역시 즉시 풀어 보여주고 다시 저장하지 않는다.
  • 같은 메시지가 두 번 저장되는 사고는 반드시 막아야 한다. 보낸 기기가 붙이는 고유 표식으로 중복 저장이 차단된다. 이 표식이 깨지거나 빠진 채 송신되면 동일 메시지가 중복 저장되는 사고로 이어진다.
  • 차단 발동 시 채팅방은 같은 동작으로 닫힌다. 차단과 채팅방 종료가 분리되면 "차단했는데 채팅방이 살아 있다"는 사고가 즉시 발생한다. 분리 불가능한 동작으로 유지해야 한다.
  • 채팅방 목록은 활성 방만 보여 준다. 닫힌 방은 자연스럽게 사라진다. 사용자가 별도 정리 동작을 하지 않아도 화면이 깨끗하게 유지되는 정책.
  • 실시간 채널이 죽어도 메시지 전달 자체는 살아 있다. 실시간은 부가 신호(입력 중·읽음 표시·즉시 도착)일 뿐 핵심이 아니다. 실시간이 죽었다고 사용자에게 큰 에러로 보이지 않게 — 작은 배지로만 자리한다.
  • 본인 프로필 비활성 시 송신을 막는 건 클라이언트 정책이다. 서버는 이 가드를 두지 않는다 — 만약 클라이언트 가드가 풀리면 비활성 사용자가 메시지를 보낼 수 있는 회귀가 발생한다. 비활성 사용자의 의도를 채팅에서도 존중하기 위한 정책의 자리.
