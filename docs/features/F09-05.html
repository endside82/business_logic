F09-05. 데이팅 채팅

매치된 두 사용자만 들어오는 1:1 채팅. 본문은 AES로 암호화 저장되고, WebSocket으로 실시간 push되며, clientMessageId 멱등키로 같은 메시지가 두 번 들어가지 않게 한다. 차단·비활성 등 사용자 상태 변화가 즉시 채팅에 반영되어 신뢰가 깨지지 않게 한다.

중복·보안 가드 세 가지
  1. clientMessageId 멱등 — 네트워크 재시도가 중복 메시지를 만들지 않음.
  2. 참여자 검증 — 방의 두 user 아니면 GET·POST 모두 차단(NOT_PARTICIPANT).
  3. 방 상태 ACTIVE — 차단·만료된 매치의 방(CLOSED)에는 송신 불가.

이 기능 안의 하위 항목

동작 1

메시지 송수신

TEXT(추가로 IMAGE 가능). 송신 → AES 암호화 → INSERT → 알림 + WebSocket push.

동작 2

읽음 처리

화면 진입·메시지 수신 시 markAsRead → 상대에게 “읽음” 표시 push.

동작 3

타이핑 표시

입력 시작/중지 이벤트를 WebSocket으로. 클라이언트 디바운스로 트래픽 절제.

동작 4

차단·종료 시 채팅 잠금

매치 BLOCKED + 방 CLOSED. 송신 시 NOT_FOUND. 방 목록 폴링에서도 자연 제거.

동작 5

비활성 프로필 송신 차단

본인 isActive=false면 입력바에 안내 + 송신 비활성. 읽기·스크롤은 가능.

사용자 시나리오 10개

S1 · Happy Path

첫 메시지 송수신

  1. A 입장 → GET 빈 페이지 + “대화를 시작해보세요”
  2. A 입력 → typing 이벤트(소켓)
  3. “보내기” → POST /messages → AES 암호화 + INSERT
  4. 알림 + WebSocket push → B에서 새 메시지 노출
  5. B 입장 → markAsRead → A에 읽음 표시
S2 · 멱등

네트워크 끊김 후 재전송

  1. clientMessageId 부여하고 송신 → 응답 대기 중 네트워크 끊김
  2. 재시도 → 같은 clientMessageId로 송신
  3. 서버: 첫 번째 INSERT 또는 unique 충돌 시 기존 메시지 반환
  4. 두 번째 호출은 즉시 기존 메시지 반환

결과: DB에 1건만 존재. 사용자에게 중복 메시지 노출 없음.

S3 · 방 닫힘

CLOSED 방 송신 시도

  1. 한쪽 차단으로 방 CLOSED
  2. 캐시된 화면에서 송신 → DATE_CHAT_ROOM_NOT_FOUND
  3. “메시지 전송에 실패했습니다” 토스트
S4 · 권한

비참여자 조회 시도

  1. 방 ID를 알게 된 외부자가 GET /messages
  2. validateParticipant → 403 DATE_CHAT_NOT_PARTICIPANT
S5 · 즉시 종료

차단 발동 → 채팅 종료

  1. 차단 호출 → 매치 BLOCKED + 방 CLOSED + closedAt
  2. 다음 폴링에서 ACTIVE 필터로 방 목록에서 제거
  3. 양쪽 모두 송신 불가
S6 · 신고

채팅 내 직접 신고 없음

차단으로 단절. 별도 신고는 외부 신고 도메인(F11-04)에서 처리.

S7 · 본인 인증 만료

인증 만료 사용자의 채팅

채팅 컨트롤러에는 isVerified 가드 없음 → 인증 만료라도 기존 채팅은 가능. 새 매치는 F09-03에서 차단됨.

S8 · 푸시 진입

FCM → 채팅 화면

  1. 백그라운드 사용자에게 “새 메시지 + 30자 미리보기” 푸시
  2. 탭 → DeepLink로 /dating/chat/{roomId} push
  3. 진입 즉시 markAsRead → 카운터 0
S9 · 만남 제안

채팅에서 F09-06로 이동

  1. 채팅 헤더 📅 탭
  2. /dating/meeting/{matchId} push
S10 · 비활성

본인 프로필 비활성 상태 송신

  1. 입력바에 “프로필이 비활성 상태에서는 메시지를 보낼 수 없습니다” 안내
  2. 송신 입력 자체 차단, 읽기·스크롤은 가능

자주 빠뜨리는 케이스

레드 플래그
  • clientMessageId UNIQUE 인덱스 누락 — 멱등 가드가 작동 안 함. 트랜잭션 격리수준에서도 race 발생.
  • 방 참여자 검증을 GET만 적용 — POST가 빠지면 외부자가 메시지 송신 가능.
  • 차단 시 방 상태만 바꾸고 사용자 측 화면 폴링은 그대로 — “CLOSED 방”에 계속 송신 시도하는 회귀.
  • 읽음 push 누락 — 보낸 쪽은 상대가 메시지를 봤는지 모름.
  • 비활성 사용자에게 송신 허용 — 매칭 시점의 사용자 상태와 어긋난 대화 발생.
  • 본문을 평문 저장 — AES 암호화 누락 시 데이터 보호 정책 위반.
  • FCM 미리보기에 30자 이상 노출 — 푸시 알림에 민감 정보 노출.
