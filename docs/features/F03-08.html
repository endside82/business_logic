F03-08. QR 체크인

모임 현장에서 사용자가 실제로 도착했는지를 확인하는 면. 호스트는 큐알 코드 또는 6자리 단축 코드를 제시하고, 참석자가 그것을 스캔하거나 입력해 출석을 기록한다. 카메라가 안 되는 사용자나 폰이 방전된 참석자도 호스트의 수동 체크인으로 처리할 수 있도록 세 가지 경로를 모두 두었다.

체크인 경로
3가지
큐알 스캔·단축 코드·호스트 수동
큐알 유효 시간
5분
발급 후 만료, 재발급 시 이전 코드 즉시 무효
시간 윈도우
시작 ±2시간
시간 외에는 체크인 차단
설계 의도 — 세 경로 모두를 두는 이유

현장 체크인은 한 가지 방식만으로는 충분하지 않다. 큐알 스캔이 가장 빠르지만 카메라가 흐리거나 권한이 없는 사용자가 있고, 그때를 위해 6자리 단축 코드가 있다. 단축 코드도 안 되는 폰 방전 참석자를 위해서 호스트가 직접 체크인 가능하게 둔다. 세 경로 모두 같은 정원 가드(시간 윈도우, 참석 확정, 중복 방지)를 통과해야 하며, 사용자 인지는 같은 결과("체크인 완료")로 묶인다.

사용자 시나리오

기본 흐름

큐알 스캔으로 체크인

호스트가 체크인 화면에서 큐알을 제시. 참석자가 자기 폰에서 스캔. 같은 동작 안에서 시간·정원·참석 확정·중복 가드가 모두 통과하면 화면이 체크 완료 상태로 전환되고 "체크인 완료" 토스트가 뜬다.

이 흐름이 남기는 것 — 체크인 가장 빠른 경로. 본 경로의 응답이 느리면 참석자가 줄을 서서 기다리는 운영 비용 증가.

단축 코드

스캔이 안 돼서 6자리 입력

카메라 권한이 없거나 큐알이 흐릿한 사용자가 "코드로 입력하기"를 누른다. 6자리 단축 코드를 호스트 화면에서 받아 입력. 같은 가드를 통과하고 체크인 완료.

호스트 수동

폰이 방전된 참석자를 호스트가 직접 체크인

호스트가 "수동 체크인" 탭에서 참석자 목록을 보고, 해당 사용자에 체크박스를 표시한 뒤 "체크인" 버튼. 여러 명을 한 번에 처리 가능 — 단체로 도착한 그룹을 빠르게 처리.

통계 확인

호스트가 체크인 진행 상황 확인

호스트가 "통계" 탭으로 이동. 원형 프로그레스로 체크인율(예: 8/12, 66.7%)이 표시되고, 미체크인 참석자 목록도 함께 보인다. 모임 직후 호스트가 누가 왔고 안 왔는지를 한 화면에서 확인.

새 큐알 발급

의심스러운 상황 — 호스트가 새 큐알 발급

큐알이 외부에 새 나갔거나 모임이 길어진 경우, 호스트가 "새 큐알 받기"를 누른다. 이전 단축 코드가 즉시 무효화되어 외부 사용자가 우회로 체크인할 수 없다.

이 흐름이 남기는 것 — 안전 운영의 핵심 도구. 자동 재발급은 일부러 안 한다 — 호스트가 의도적으로 발급해야만 새 코드가 만들어진다.

시간 차단

모임 종료 3시간 뒤 체크인 시도

모임 시작 전 2시간 ~ 종료 후 2시간 사이에만 체크인이 가능. 그 시간을 벗어나면 화면 진입 자체가 "체크인 기간이 아닙니다" 안내로 차단.

이 흐름이 남기는 것 — 모임 끝난 뒤 사용자가 다른 사람의 큐알을 외부에서 입력해 체크인을 받는 사고를 막는다.

비참석자 차단

참석 확정이 아닌 사용자가 진입

대기 중이거나 신청만 한 사용자는 체크인 화면에 들어올 수 없다. 우회 호출도 서버에서 차단.

중복 체크인

이미 체크인한 사용자가 다시 스캔

두 번째 시도는 차단되고 "체크인에 실패했습니다" 안내. 같은 사용자가 중복으로 출석 처리되는 사고는 발생 안 함.

큐알 만료

5분이 지난 큐알을 사용

큐알 유효 시간이 지나면 스캔해도 "체크인에 실패했습니다" 안내. 사용자가 호스트에게 다시 발급을 요청해야 한다.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
호스트 — 큐알 탭 큰 큐알 코드 + 6자리 단축 코드(letter-spacing 강조) + 만료 카운트다운 + 새로고침 버튼. 참석자가 멀리서도 보고 스캔할 수 있도록 큐알을 크게. 카메라가 안 되는 사용자를 위한 6자리 코드를 같은 자리에 두어 호스트가 두 안내를 한 화면에서 처리.
호스트 — 수동 체크인 탭 참석자 목록 검색 + 체크박스 + 이미 체크인된 사용자에 표시. 여러 명 선택 후 "체크인" 버튼. 그룹으로 도착한 참석자를 빠르게 처리. 한 명씩 큐알 받기 어려운 현장 상황의 안전장치.
호스트 — 통계 탭 원형 프로그레스(체크인율) + 총 인원/체크인/미체크인 수치 + 미체크인 사용자 목록. 모임 중·후에 호스트가 운영 상황을 한 자리에서 확인. 노쇼 사용자를 신뢰점수 반영 같은 후속 처리로 안내.
참석자 — 스캐너 화면 카메라 뷰포트, "호스트가 제시한 큐알을 스캔하세요" 안내, "스캔이 안 되시나요? 코드로 입력하기" 보조 진입점, "다시 스캔" 버튼. 참석자에게 한 가지 동작만 제시하고, 보조 경로는 명확히 분리. 자동 스캔이 동작하면 별도 버튼 없이 바로 체크인이 진행.
참석자 — 완료 화면 체크 마크 애니메이션 + "체크인 완료" 안내. 현장에서 자기 폰을 보지 않고도 한눈에 완료를 인지할 수 있게.

운영자가 알아야 할 것

체크인의 결정점
  • 큐알 자동 재발급은 일부러 하지 않는다. 만료된 큐알은 호스트가 의도적으로 새로 발급해야만 새 코드가 만들어진다. 자동 재발급은 큐알 외부 노출 위험을 키운다.
  • 새 큐알 발급 시 이전 단축 코드는 즉시 무효화된다. 두 코드가 동시에 살아 있으면 외부 우회 위험이 커진다.
  • 시간 윈도우(시작 -2시간 ~ 종료 +2시간) 밖에서는 체크인 화면 자체가 열리지 않는다. 현장 운영의 안전장치.
  • 수동 체크인의 일괄 처리는 클라이언트 측 순차 호출이다. 중간에 한 사용자가 실패하면 그 시점까지만 반영되고 그 뒤는 멈춘다. 호스트가 결과를 확인하지 않으면 일부 사용자가 빠진 채 지나갈 수 있어, 결과 토스트의 정확성이 중요.
  • 큐알 외부 노출은 사용자의 화면을 거쳐 일어난다. 다른 사용자가 큐알 화면을 사진으로 찍어 멀리서 입력하는 사고가 발생할 수 있다. 5분 만료가 일차 방어, 호스트의 새 큐알 발급이 이차 방어.
  • 중복 체크인은 차단되지만 "이미 체크인했습니다" 같은 구체 메시지는 현재 미지원. 사용자에게는 일반 실패 메시지만 보임. 운영 분석에서 중복 시도 빈도가 크면 메시지 분기 도입 검토.
