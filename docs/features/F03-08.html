F03-08. QR 체크인

참석 확정자가 모임 현장에서 본인을 확인받는 가장 빠른 방법. 호스트가 QR + 6자리 단축코드를 발급하고, 참석자는 카메라 스캔 또는 코드 입력 어느 쪽으로든 통과한다. 시간 윈도우 검증·중복 차단·새 QR 발급 시 이전 코드 즉시 무효화까지 본 단위가 묶어 책임진다.

이 기능 안의 하위 항목

동작 1

QR + 단축코드 발급(호스트)

JWT 5분 + Redis 단축코드. 새 발급 시 이전 코드 즉시 폐기.

동작 2

QR 스캔 체크인(참가자)

카메라로 즉시 통과.

동작 3

단축코드 입력(참가자)

스캔 실패 시 6자리 코드로 fallback.

동작 4

수동 체크인(호스트)

스캔 불가 참석자 직접 체크.

동작 5

통계·노쇼 추적

체크인율 + 미체크인 사용자 목록.

QR 발급 + 체크인 흐름

sequenceDiagram actor H as 호스트 actor U as 참석자 participant S as 체크인 서비스 participant R as Redis H->>S: QR 발급 요청 S->>S: JWT 발급(5분) + 단축코드 생성 S->>R: 단축코드 SET (TTL 5분) S-->>H: QR + "048372" U->>U: QR 스캔 또는 코드 입력 U->>S: 체크인 요청 S->>S: 시간 윈도우·상태·중복 검증 S->>S: event_check_in INSERT S-->>U: 체크인 완료 H->>S: 통계 조회 S-->>H: 12/12 등 + 미체크인 목록

사용자 시나리오 9개

S1 · Happy Path

QR 발급 → 스캔 → 체크인

  1. 호스트가 “체크인” → 3탭(QR/수동/통계) 진입 → QR 자동 발급
  2. 참석자가 카메라로 QR 스캔 → 서버 검증 → 체크인 완료
  3. 호스트 통계 갱신
S2 · fallback

코드로 입력

  1. QR 스캔 실패 → “코드로 입력하기”
  2. 호스트에게 6자리 코드 받아 입력
  3. 서버: Redis 조회 → 통과 → 체크인
S3 · 수동

호스트가 수동 체크인

  1. “수동 체크인” 탭에서 사용자 선택
  2. “체크인” → for-loop로 사용자별 호출
  3. 통계 갱신
S4 · 통계

통계 + 노쇼 목록

  1. “통계” 탭 → 체크인율 원형 프로그레스
  2. 미체크인 사용자 목록 노출
S5 · 시간 외

종료 후 체크인 시도

  1. 이벤트 종료 +2시간 초과
  2. 화면 사전 차단 또는 400 CHECK_IN_NOT_ALLOWED
S6 · 비참석자

WAITING 사용자가 스캔 시도

  1. 라우터 redirect 또는 400 INVALID_INPUT
S7 · 중복

이미 체크인한 사용자

  1. 409 CONFLICT
S8 · QR 재발급

이전 코드 즉시 폐기

  1. 호스트 “새로고침” → Redis에서 이전 코드 DEL
  2. 새 코드 발급
  3. 이전 코드로 시도하던 참석자는 400
S9 · JWT 만료

5분 지난 QR

  1. JWT 만료로 검증 실패 → 400
  2. 호스트가 새로 발급 필요

UX: 자동 재발급은 하지 않고 사용자 액션 요구.

자주 빠뜨리는 케이스

레드 플래그
  • 새 QR 발급 시 이전 코드 폐기 누락 — brute-force 위험.
  • 시간 윈도우 검증을 클라이언트만 — 서버에서도 강제.
  • 중복 체크인 허용 — 통계 왜곡.
  • 참석자에게 호스트 탭 노출 — 권한 분기 누락.
