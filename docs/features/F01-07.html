F01-07. 관심사 태그 관리

사용자가 본인 관심사를 표현하는 태그 묶음을 만들고 유지한다. 온보딩에서 처음 등록한 뒤 사용자의 흥미가 바뀔 때마다 프로필에서 추가·수정·삭제할 수 있다. 이 태그가 추천·검색의 직접 입력값이므로, 사용자가 본 단위를 다루는 빈도와 추천 품질이 정비례한다.

사용자당 최대
20개
초과 등록 차단
태그당 가중치
0.1~5.0
관심도 높을수록 큰 값
정렬
관심도순
가중치 큰 태그가 위에
설계 의도 — 태그는 정적인 라벨이 아니라 사용자 흥미의 시간 표시다

같은 사람도 봄과 가을의 관심사가 다르다. 그래서 본 단위는 태그를 "한 번 정하면 끝"이 아닌 "언제든 바꿀 수 있는" 형태로 두고, 태그마다 가중치를 두어 관심의 깊이까지 표현한다. 가중치는 사용자가 직접 조정할 수도 있고, 향후 추천 알고리즘이 사용자 행동을 보고 자동 조정할 수도 있다. 사용자가 자기 태그를 일년에 몇 번 손보는지가 본 서비스의 추천 품질이 시간에 따라 떨어지지 않는지의 지표가 된다.

사용자 시나리오

최초 등록

온보딩에서 세 개 골라 등록

가입 직후 온보딩 관심사 화면에서 등산·카페탐방·영화 세 개를 고르고 "다음"을 누른다. 세 개가 차례로 등록되고 모두 성공하면 다음 단계로 이동한다. 세 개 모두 가중치는 같은 기본값이다.

이 흐름이 남기는 것 — 추천 시스템에 처음으로 사용자 신호가 들어오는 순간. 본 등록이 없으면 첫 홈 화면이 비어 보인다.

조회

프로필에서 내 태그 목록 확인

설정 화면의 관심사 항목에서 본인이 등록한 태그를 한눈에 본다. 관심도가 높은 태그가 위에 보인다 — 사용자가 본인이 어떤 흥미를 가진 사람인지 한 화면에서 확인할 수 있다.

변경

'러닝'을 '마라톤'으로 변경

관심사가 더 구체화된 사용자가 기존 태그 이름을 손본다. 새 이름이 다른 태그와 안 겹치면 그 자리에서 변경된다. 가중치는 그대로 유지된다 — 이름만 바꿔도 사용자가 쌓아둔 관심 신호는 손실되지 않는다.

중복

이미 가진 태그를 또 추가하려 함

같은 이름의 태그를 다시 등록하면 거절된다. "이미 등록된 태그입니다" 안내가 뜨고, 가중치만 올리고 싶다면 기존 태그를 수정하라고 안내한다.

한도

21번째 태그 등록 시도

이미 20개를 등록한 사용자가 새 태그를 시도하면 "최대 20개까지 등록 가능합니다" 안내가 뜬다. 사용자는 덜 중요한 태그를 삭제하고 새 태그를 추가하거나 그대로 둘 수 있다.

삭제

흥미가 사라진 태그를 정리

더 이상 관심 없는 태그를 삭제한다 — 추천에서 즉시 제외된다. 사용자가 본인 관심사를 한 번씩 정리해 줘야 추천의 노이즈가 줄어든다는 사실이 본 단위의 가치.

운영자가 알아야 할 것

태그 정책과 결정점
  • 태그 카테고리는 클라이언트에 하드코딩되어 있다. 새로운 카테고리(예: "반려동물", "캠핑") 추가는 앱 업데이트 사이클을 따라야 한다 — 시즌성 콘텐츠 반영 속도에 직접 영향. 카테고리 동적 관리가 필요해질 시점을 모니터링.
  • 사용자당 최대 20개 한도는 추천 알고리즘 성능에 맞춘 수치. 한도를 늘리면 알고리즘 입력 차원이 커져 추천 정확도가 떨어질 수 있다 — 한도 변경 시 추천 도메인과 함께 결정.
  • 다른 사용자의 태그를 수정·삭제할 수 없게 본인 소유 검증이 매번 일어난다. 본인이 아닌 태그 ID로 요청하면 "태그를 찾을 수 없습니다"로 응답해서 다른 사용자의 태그 존재 여부 자체를 알 수 없게 한다.
  • 가중치는 현재 사용자가 직접 조정한다. 향후 행동 기반 자동 조정(예: 자주 클릭한 카테고리의 가중치 자동 상승)이 도입되면 본 단위의 사용자 경험에 큰 변화가 생긴다 — 추천 도메인 변경과 함께 추적.
  • 온보딩에서의 일괄 등록은 한 개씩 순차로 처리된다. 중간에 실패하면 일부만 저장되고 다음 시도에서 중복으로 거절될 수 있다 — 사용자가 답답하게 막힐 위험이 있는 알려진 한계점.
