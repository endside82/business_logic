F17-09. 세션 출석 확정·노쇼 (Phase 4)

FIXED 모임 호스트가 매 세션 종료 후 출석 여부를 일괄 확정한다. 결과는 권위 테이블 regular_meeting_session_attendance에 영속((event_id, user_id) UNIQUE). NO_SHOW는 EventAttendance demote + 멤버 카운터 가산. FORFEIT_ON_LIMIT 한도 도달 시 즉시 박탈 + 환불.

finalize 시점 강제 — endTime ≤ now

endTime != null && endTime ≤ now 조건만 통과한다(Codex BLOCKING 1 해소). 미종료 세션 finalize 시도는 REGULAR_MEETING_SESSION_CANNOT_FINALIZE로 거절돼서 NO_SHOW 조기 발생을 차단한다. 정션 attendanceFinalizationStatus == FINALIZED이면 재확정도 거절.

안전망 스케줄러 (opt-in)

@Value("${regular-meeting.auto-finalize.enabled:false}") 기본 OFF (Codex BLOCKING 2 해소). 켜면 endTime+24h grace 경과한 세션을 Redisson lock 하에 정책 기본값(EventCheckIn 기반)으로 일괄 finalize. 이중 finalize는 정션 FINALIZED 가드가 차단.

close 가드 연동

close(F17-04 §3.2)는 모든 정션이 FINALIZED여야 통과한다. 즉 finalize는 반드시 close 전에 완료돼야 하고, FORFEIT 환불도 close 이전(finalize 시점)에 발생해서 settlement gross의 retained paid에 자연 반영된다 → 정산 후 환불 위험 0.

서버 계약

POST /api/v1/regular-meetings/{id}/events/{eventId}/finalize — Body RegularMeetingSessionFinalizeParam? (per-attendee override 옵션, 미지정자는 정책 기본값). 가드: host + FIXED + endTime≤now + 정션 미확정. 응답 204.
