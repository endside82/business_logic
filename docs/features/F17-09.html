F17-09. 회차 출석 확정·노쇼

고정형 호스트가 회차 종료 후 출석 명단을 일괄 확정한다. 결과는 4가지 — 출석·노쇼·공결·세션 취소. 확정은 회차가 종료된 다음에만 가능하고, 한 번 확정된 회차는 다시 확정할 수 없다. 노쇼가 한도에 도달하면 즉시 자격박탈 + 잔여분 환불이 자동 발사된다. 이 단계가 끝나야 코스 종료 버튼이 풀려, 정산 후 환불 사고가 일어나지 않는 구조의 핵심 요소다.

호스트가 매 회차를 직접 확정해야 코스 종료 버튼이 풀린다. 자동 추정만으로 노쇼·박탈을 만들지 않고 호스트의 명시적 판단을 거치게 한 이유는 분쟁 발생 가능성이 높기 때문이다. 안전망으로 종료 시각 + 24시간이 지나도 호스트가 무대응이면 자동 확정하는 옵션이 있지만 기본은 꺼져 있다.

사용자 시나리오

정상 확정

"회차 끝났으니 출석 확정"

호스트가 회차 종료 후 "출석 확정" 화면 진입. 자동 판정된 결과(체크인 기반)를 검토하고 일괄 승인. 체크인 한 멤버는 출석, 안 한 멤버는 노쇼로 들어간다.

이 흐름이 남기는 것 — 호스트가 한 화면에서 한 액션으로 회차를 마무리, 매 회차마다 부담을 줄인다.

공결 표시

"사정 있는 멤버는 공결로"

호스트가 특정 멤버의 결과를 "공결"로 바꾼 뒤 확정. 그 멤버는 노쇼 카운트에 가산되지 않고, 연속 노쇼 카운트는 끊긴다.

이 흐름이 남기는 것 — 호스트가 멤버의 사정을 반영해 부당한 박탈을 막을 수 있는 재량.

세션 취소

"회차 자체를 못 했다"

호스트가 회차 취소(F17-05)를 누르면 그 회차는 "세션 취소"로 마킹된다. 멤버에게 책임 없음 — 노쇼 가산 안 됨.

이 흐름이 남기는 것 — 호스트 사정으로 회차가 못 열렸을 때 멤버가 노쇼 한도에 가까워지지 않게 한다.

자격박탈 트리거

"3회 연속 노쇼 → 박탈"

노쇼 한도가 "자격박탈"로 설정된 모임에서 멤버가 3회 연속(또는 누적) 노쇼에 도달하면 확정 즉시 자격박탈 + 잔여 회차분 자동 환불.

이 흐름이 남기는 것 — 호스트 운영자가 별도 액션 없이 자동으로 노쇼 정책 집행, 모임 분위기 보호.

미종료 확정 시도

"회차 시작 전인데 출석 확정 시도"

회차 종료 시각이 안 지났는데 "출석 확정" 누르면 "회차가 아직 끝나지 않았어요" 안내. 차단.

이 흐름이 남기는 것 — 미래 노쇼를 미리 만들어 멤버를 박탈하는 일을 차단.

재확정 시도

"확정한 회차를 다시 확정"

한 번 확정된 회차에 다시 "출석 확정" 누르면 "이미 확정된 회차예요" 안내. 차단.

이 흐름이 남기는 것 — 결과를 임의로 뒤집어 환불 분쟁이 일어나는 사고를 차단.

호스트 무대응 안전망

"24시간 지나도 호스트가 안 함"

운영팀이 자동 마감을 켠 경우, 종료 +24시간 지나면 시스템이 체크인 기반으로 자동 확정. 기본은 꺼져 있다(운영팀 opt-in).

이 흐름이 남기는 것 — 호스트가 사라진 경우에도 코스가 닫힐 수 있는 비상 출구.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
출석 명단회차 자동 참석 명단이 자동 판정(체크인 기반)으로 미리 채워져 노출.호스트가 빈 명단에서 한 명씩 표시하는 부담을 줄인다.
일괄 확정 버튼화면 하단에 "전체 확정하기" 한 개.한 회차 확정은 한 액션이어야 호스트가 매 회차마다 꾸준히 한다.
공결·세션취소 토글각 멤버별로 결과를 호스트가 바꿀 수 있는 셀렉트.호스트 재량으로 멤버 사정을 반영, 부당 박탈 방지.
노쇼 한도 경고이 회차 확정 시 박탈될 멤버를 미리 노출("3회 노쇼 도달 — 박탈됩니다").호스트가 결정을 내리기 전 영향을 미리 보게 한다.
박탈 환불 안내박탈되는 멤버에 대한 환불 예상액을 같이 노출.호스트가 박탈이 단순 처벌이 아니라 환불 발생을 동반함을 인지하게.

운영자가 알아야 할 것

확정 동작
  • 기본은 체크인 자동 판정 + 호스트 일괄 승인 — 호스트가 명단을 보고 일괄 확정. 1차 출시는 한 명씩 결과 바꾸는 UI 일부 제한.
  • 자동 마감은 기본 꺼짐(opt-in) — 호스트 무대응 시 종료 +24시간 안전망이 있지만, 운영팀이 명시적으로 켜야 한다.
  • 노쇼 → 일반 참석 명단에서 제거 — 회차 정원 카운트가 자연스럽게 갱신.
  • 자격박탈 환불은 즉시 — 박탈 시점에서 잔여 회차분이 자동 환불(코스 종료 이전).
  • 비호스트 확정 시도는 거절 — 권한 가드.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 호스트가 확정을 미루는 경우 — 종료 버튼이 막혀 다음 단계로 못 넘어간다. 자동 마감을 켜지 않으면 영원히 대기.
  • 노쇼·공결 판정 분쟁 — 멤버가 "사정 있어 못 갔다" 주장. 호스트가 공결로 바꾸려면 첫 확정 전에 정해야 한다(재확정 차단).
  • 자격박탈 환불 실패 — 자동 재시도 큐로 가지만 멤버 입장에서는 박탈이 영구. 운영 모니터링 필요.
  • 한 명만 다르게 처리하는 UI 한계 — 1차는 한 화면에서 결과를 일괄로 보고 일부만 바꾸는 동선. 후속에 회차별 명단 화면 강화.
