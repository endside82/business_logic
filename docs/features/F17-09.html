F17-09. 회차 출석 확정·노쇼

고정형 호스트가 회차 종료 후 출석 명단을 일괄 확정한다. 결과는 4가지 — 출석·노쇼·공결·세션 취소. 확정은 회차가 끝난(종료 시각 이후) 다음에만 가능하고, 한 번 확정된 회차는 다시 확정할 수 없다. 노쇼가 한도에 도달하면 즉시 자격박탈 + 잔여분 환불이 자동 발사된다.

호스트가 매 회차를 직접 확정해야 코스 종료 버튼이 풀린다. 자동 추정만으로 노쇼·박탈을 만들지 않고 호스트의 명시적 판단을 거치게 한 이유는 분쟁 발생 가능성이 높기 때문이다. 안전망으로 종료 시각 + 24시간이 지나도 호스트가 무대응이면 자동 확정하는 옵션이 있지만 기본은 꺼져 있다.

네 가지 출석 결과

출석

실제로 참석한 멤버

체크인이 있거나 호스트가 직접 출석 표기. 누적 출석 +1, 연속 노쇼 카운트는 0으로 리셋.

노쇼

오기로 한 사람이 안 옴

일반 참석 명단에서 제거(체크인 정상 무효화), 누적 노쇼 +1, 연속 노쇼 +1. 한도 도달 시 자격박탈 트리거.

공결

사유 있는 결석

노쇼 카운트에 가산되지 않고, 연속 노쇼는 끊긴다(멤버 귀책 아님). 환불 정책은 1차에서 멤버 취소 산식과 동일.

세션 취소

호스트가 회차 자체를 취소한 경우

멤버에게 책임 없음. 노쇼 가산 안 됨, 연속 노쇼 끊김.

노쇼 한도 정책 — 세 가지 모드

정책의미한도 도달 시

자격박탈한도 도달 시 자동 박탈멤버 박탈 + 잔여 회차분 자동 환불 (F17-08)

카운트 방식 두 가지 — 누적(코스 전체 누적 노쇼) 또는 연속(연속 노쇼만). 호스트가 모임 만들 때 한 번 선택.

미종료·재확정 차단

회차 종료 시각이 안 지났으면 확정 호출이 거절된다(미래 노쇼를 미리 만들 수 없게). 또 한 번 확정된 회차는 다시 확정할 수 없다 — 결과를 임의로 뒤집어 환불 분쟁이 생기는 사고를 막기 위함.

운영자가 알아야 할 것

확정 동작
  • 기본은 체크인 자동 판정 + 호스트 일괄 승인 — 호스트가 명단을 보고 일괄 확정. 1차 출시는 한 명씩 결과 바꾸는 UI 미신설.
  • 자동 마감은 기본 꺼짐(opt-in) — 호스트 무대응 시 종료 +24시간 안전망이 있지만, 운영팀이 명시적으로 켜야 한다.
  • 노쇼 → 일반 참석 명단에서 제거 — 회차 정원 카운트가 자연스럽게 갱신된다.
  • 자격박탈 환불은 즉시 — 박탈 시점에서 잔여 회차분이 자동 환불(코스 종료 이전).
  • 비호스트 확정 시도는 거절 — 권한 가드.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 호스트가 확정을 미루는 경우 — 종료 버튼이 막혀 다음 단계로 못 넘어간다. 자동 마감을 켜지 않으면 영원히 대기.
  • 노쇼·공결 판정 분쟁 — 멤버가 "사정 있어 못 갔다" 주장. 호스트가 공결로 바꾸려면 첫 확정 전에 정해야 한다(재확정 차단).
  • 자격박탈 환불 실패 — 자동 재시도 큐로 가지만 멤버 입장에서는 박탈이 영구. 운영 모니터링 필요.
  • 한 명 override UI 부재 — 1차는 일괄 확정만. 일부 멤버만 공결 처리하려면 사후 운영자 개입 필요.
