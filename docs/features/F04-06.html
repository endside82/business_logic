F04-06. 클럽 차단 관리

문제 멤버를 영구 차단해 가입 시도와 초대 자체를 막는다. 차단 처리는 추방 + 가입비 환불 + 기존 신청/초대 정리 + 알림을 한 트랜잭션으로 묶는다.

하위 동작
2
차단·차단 해제
시나리오
8

이 기능 안의 하위 항목

동작 1

차단

사유 입력. 추방+환불+기존 wait 정리+BAN wait INSERT.

동작 2

차단 해제

BAN wait DELETE. 재가입 시도 가능 상태로 복귀.

사용자 시나리오 8개

S1

MEMBER 차단

  1. 사유 입력 → BAN 등록 + 추방 + 환불
  2. 대상에게 CLUB_MEMBER_BANNED 알림
S2

ADMIN→ADMIN 차단 시도

  1. UI 항목 없음, 우회 시 CLUB_NO_PERMISSION
S3

OWNER→ADMIN 차단

  1. OWNER는 ADMIN 차단 가능
S4

이미 차단된 사용자 재차단

  1. INVALID_INPUT(중복)
S5

차단된 사용자 재가입 시도

  1. CTA UI 비활성
  2. 우회 시 CLUB_BANNED
S6

차단된 사용자 초대 시도

  1. CLUB_BANNED
S7

차단 해제

  1. BAN wait DELETE → 재가입 시도 가능
S8

환불 잔액 부족(BUSINESS)

  1. 기금 부족 시 환불 실패 처리
  2. 운영 정리 필요

서버 계약

MethodPath

GET/api/v1/clubs/{id}/bans
