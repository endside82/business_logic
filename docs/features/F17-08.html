F17-08. 환불 (비례 산식 · 세 가지 트리거)

고정형 환불은 이미 진행/제공된 회차만큼만 빼고 나머지를 돌려준다(비례 환불). 세 가지 트리거 — 멤버 본인 취소·호스트 모임 취소·노쇼 자격박탈 — 가 모두 같은 골격을 쓰되 "차감 기준"이 조금씩 다르다. 100원 단위로 절삭되고 음수가 되면 0원으로 클램프된다. 지갑 결제는 자동 환불, 계좌이체는 호스트가 직접 송금하고 "환불 완료"를 눌러야 마감된다.

설계 의도 — "내가 못 받은 만큼만 돌려받는다"

코스 단위로 결제했더라도 환불은 회차 단위 공정성이 필요하다. 시작된 회차는 회사가 이미 자원을 썼다고 보고 환불 대상에서 빼고, 미래 미제공 회차만 비례로 돌려준다. 호스트가 모임을 닫는 경우만 예외 — 호스트가 미제공한 회차는 100% 돌려준다(노쇼한 멤버 책임 아님).

사용자 시나리오

멤버 본인 취소

"3회차까지 듣고 사정이 생겼다"

8회 코스 80,000원 결제, 3회차 시작 후 취소. 산식: 5/8 × 80,000 = 50,000원 환불. 지갑이면 즉시 자동 환불, 계좌이체면 호스트 송금 대기.

이 흐름이 남기는 것 — 사용자가 시점에 따라 환불액이 다르다는 걸 인지하고 결정할 수 있게.

호스트 모임 취소

"호스트가 5회차 진행 후 모임 접음"

8회 중 5회 제공, 3회 남음. 산식: 3/8 × 80,000 = 30,000원 환불. 활성 멤버 전원에게 자동 환불 발사.

이 흐름이 남기는 것 — 호스트 책임 발생 시 미제공분 100% 환불로 멤버 보호.

자격박탈

"노쇼 3회 연속으로 박탈됨"

4회까지 시작된 시점에 박탈. 산식: 4/8 × 80,000 = 40,000원 환불. 멤버 상태 자격박탈 + 자동 환불.

이 흐름이 남기는 것 — 멤버가 자기 책임으로 박탈됐어도 미래 미제공 회차분은 받아갈 수 있게, 호스트 부당이득 방지.

시작 전 취소

"코스 시작 전에 취소"

0회 시작, 8회 남음. 산식: 8/8 × 80,000 = 80,000원 전액 환불.

이 흐름이 남기는 것 — 코스 시작 전 취소는 사용자 부담 없이 빠질 수 있게.

코스 끝나가는 시점

"8회 중 8회 다 시작 후 취소"

모든 회차가 시작됐다면 환불 0원. 코스 가치를 다 받았다는 판단.

이 흐름이 남기는 것 — 코스 마지막에 잠깐 빠지면서 환불받는 악용을 차단.

계좌이체 환불 절차

"호스트가 사용자에게 직접 송금"

계좌이체 결제는 외부 송금이라 환불도 외부. 멤버 상태는 "환불 진행중"으로 들어가고, 호스트가 사용자 계좌로 송금한 뒤 "환불 완료" 버튼을 누르면 마감.

이 흐름이 남기는 것 — 외부 거래의 환불도 시스템이 상태를 추적해 사용자가 인지할 수 있게.

환불 실패

"지갑 환불이 일시적으로 실패"

네트워크·일시 오류 등으로 환불 실패. 자동 재시도 큐로 들어가 추후 자동 재시도. 호스트 취소처럼 다수 환불이 한꺼번에 발사될 때 일부 실패해도 모임 취소 자체는 완료.

이 흐름이 남기는 것 — 일시 오류가 모임 운영 전체를 멈추지 않게.

화면에서 사용자가 인지하는 것

영역사용자가 보는 것그 자리에 둔 이유
취소 직전 dialog"지금 취소하면 X원 환불됩니다" 안내(취소 전).사용자가 환불액을 결정 전에 알고 의사결정하게 한다.
환불 진행중 라벨계좌이체 환불 진행 중일 때 멤버 상태에 라벨이 노란색으로 노출.호스트 송금 대기 상태를 사용자가 명확히 인지하게.
환불 완료 안내지갑 환불은 즉시 거래 이력에 표기, 계좌이체는 호스트 마감 후 라벨 변경.두 결제 방식의 환불 흐름 차이를 사용자에게 명확히.
호스트 환불 알림호스트가 모임 취소 시 "전 멤버에게 환불됩니다" 확인 dialog.호스트가 취소의 환불 책임 규모를 직시하게 해 즉흥 취소를 줄인다.
박탈 후 환불 안내자격박탈 알림에 "잔여 회차분 X원이 환불됩니다" 같이 노출.박탈이 일방적 처벌이 아니라 금전적 정산이 함께임을 알리는 균형.

운영자가 알아야 할 것

금액 산정 규칙
  • 100원 단위 절삭 — 원 단위 금액은 자동으로 100원 미만이 깎인다(일반 결제와 동일).
  • 0 클램프 — 차감이 결제 금액을 넘어가면 0원으로 클램프(음수 환불 금지).
  • 회차 시작 시각 기준 — 멤버 취소·박탈 산식에서 "시작된 회차"는 시작 시각이 지난 회차(노쇼 포함).
  • 호스트 미제공 회차는 무조건 환불 — 호스트가 닫은 회차는 멤버 귀책과 무관하게 100% 환불.
  • 자동 재시도 큐로 환불 실패 추적 — 모든 실패는 별도 큐로 들어가 운영팀이 모니터링.

레드 플래그

분쟁이 일어날 수 있는 케이스
  • 계좌이체 환불 마감 누락 — 호스트가 송금만 하고 마감 안 누르면 멤버 입장에서 영구 "환불 진행중". 호스트 알림 미배선이라 잊기 쉬움.
  • 지갑 환불로 음수 잔액 발생 — 사용자가 환불액보다 적은 잔액을 가진 경우 마이너스 잔액이 일시 발생할 수 있음(다음 적립 시 우선 차감).
  • "24시간 전 100% 환불" 같은 시간 비례 정책 미적용 — 1차 출시는 회차 단위 차감만. 시간 비례 환불은 후속.
  • 공결·세션 취소의 환불 정책 — 1차는 멤버 취소 산식과 동일 처리. 약관 검토 후 별도 정책 가능.
