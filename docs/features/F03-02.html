F03-02. 이벤트 상세 조회

한 이벤트에 대한 정보·CTA·진입점을 한 화면에 모은다. 사용자 역할(호스트·참가자·게스트·클럽 비멤버)과 이벤트 상태(DRAFT/OPEN/CLOSED/CANCELED)의 조합이 모든 액션 노출을 결정한다. 잘못된 CTA가 노출되면 전환율과 신뢰가 동시에 깨지는 단위.

가로축은 이벤트 상태(DRAFT/OPEN/CLOSED/CANCELED), 세로축은 사용자 역할(호스트/참가자/일반 사용자/게스트/클럽 비멤버). 이 매트릭스가 모든 액션바·메뉴·배너의 노출을 결정한다.

사용자 시나리오 8개

S1 · Happy Path

일반 사용자가 OPEN 상세 진입

  1. 카드 탭 → 상세 응답 200
  2. Hero·정보·호스트·설명·유사 이벤트 렌더
  3. 최근 본 이벤트 기록 + 조회 카운트 INCR
  4. “참석 신청” 활성 → F03-05 진입
S2 · 호스트

자기 이벤트 진입

  1. DRAFT도 호스트는 통과
  2. 액션바가 “발행/수정/삭제”로 교체
  3. 운영 메뉴 카드 + “주최자 화면입니다” 배너 노출
S3 · 공유 링크

비로그인 사용자의 딥 링크 진입

  1. OPEN 이벤트라면 비로그인도 조회 가능
  2. viewer context는 null/false
  3. 액션 탭 시 로그인 유도
S4 · DRAFT 비호스트

일반 사용자가 DRAFT 접근

  1. 서버가 EVENT_NOT_FOUND로 마스킹
  2. “이벤트를 찾을 수 없습니다” + 돌아가기
S5 · CLOSED

종료된 이벤트

  1. 흐릿한 오버레이 + “종료된 이벤트” 라벨
  2. “리뷰 작성”(F11) · 사진첩(F03-09) 진입 가능
S6 · CANCELED

취소된 이벤트

  1. 빨간 배지 + 흐림
  2. 액션 모두 비활성
  3. 환불 정보는 알림에서 안내(F06)
S7 · .ics

캘린더 내보내기

  1. 더보기 → “.ics 다운로드”
  2. 파일 저장 후 시스템 공유 시트
  3. 외부 캘린더 앱에 등록
S8 · 클럽 비멤버

비공개 클럽 이벤트 진입

  1. 설명·플랜·사진첩이 가려짐
  2. “클럽 멤버만 볼 수 있습니다” + 클럽 가입 CTA

자주 빠뜨리는 케이스

레드 플래그
  • DRAFT를 비호스트에게 노출 — 미공개 정보 유출. 404 마스킹 필수.
  • 이미 신청한 사용자에게 “신청하기”가 남음 — viewer context 갱신 누락.
  • CANCELED에 “참석 신청” 활성 — 상태별 CTA 매트릭스 깨짐.
  • 클럽 비멤버에게 설명 노출 — 가림 처리 누락은 클럽 정책 위반.
