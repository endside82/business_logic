F20-03. 지원 FAQ 제안

운영 이슈 작성 화면에 진입하면 문의 대상에 맞는 자주 묻는 질문 카드를 최대 5개 보여준다. 문의를 올리기 전에 스스로 해결할 수 있는지 확인하게 하는 가벼운 안내 기능이다. 별도 저장소 없이 서버에 직접 적어 둔 제안 목록을 사용한다.

운영 이슈 작성 화면에 진입하면 먼저 관련 도움말이 카드로 표시된다. "환불 상태 확인"이나 "결제 실패 확인" 같은 짧은 안내로 사용자가 문의 없이 해결할 수 있는지 먼저 확인하게 한다. 도움이 안 되면 아래 폼을 채워 문의한다.

화면 열리면 도움말이 먼저 나온다

운영 이슈 작성 화면이 열릴 때 문의 대상에 맞는 제안을 자동으로 불러온다. 결과가 있으면 문의 폼 위에 "추천 도움말" 섹션이 나타나고, 없거나 오류이면 조용히 숨겨진다.

검색어 좁히기

검색어로 좁힌다

검색어를 함께 보내면 제목과 답변 글에 그 단어가 들어간 항목만 보여 준다. 현재 앱은 검색어 없이 불러오므로 전체 제안이 그대로 표시된다. 실시간 검색 입력은 아직 만들어지지 않았다.

알려진 한계

문구를 운영팀이 직접 바꿀 수 없어 변경마다 개발 작업이 필요

FAQ 제안 문구는 시스템에 내장되어 있고, 운영팀이 직접 추가·수정할 수 있는 관리 화면이 없다. 문구를 바꾸려면 매번 개발 작업과 배포가 필요하다. 향후 운영팀이 직접 편집할 필요가 생기면, 운영팀용 편집 화면과 저장소를 새로 만들어야 한다.
