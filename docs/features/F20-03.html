F20-03. 지원 FAQ 제안

운영 이슈 작성 화면에 진입하면 소스 타입에 맞는 자주 묻는 질문 카드를 최대 5개 보여준다. 문의를 올리기 전에 스스로 해결할 수 있는지 확인하는 경량 안내 기능이다. DB가 없고 서버 코드에 하드코딩된 제안 목록을 사용한다.

운영 이슈 작성 화면에 진입하면 먼저 관련 도움말이 카드로 표시된다. "환불 상태 확인"이나 "결제 실패 확인" 같은 짧은 안내로 사용자가 문의 없이 해결할 수 있는지 먼저 확인하게 한다. 도움이 안 되면 아래 폼을 채워 문의한다.

동작 방식

진입 시 자동 표시

화면 열리면 도움말이 먼저 나온다

운영 이슈 작성 화면(SupportIssueCreateScreen)이 빌드될 때 소스 타입에 맞는 제안을 자동으로 불러온다. 결과가 있으면 문의 폼 위에 "추천 도움말" 섹션이 나타나고, 없거나 오류이면 조용히 숨겨진다.

keyword 필터

검색어로 좁힌다

서버에 query 파라미터를 보내면 제목과 답변 텍스트에 그 단어가 포함된 항목만 반환된다. 현재 Flutter 앱은 항상 query=null을 보내므로 전체 제안이 표시된다. 실시간 검색 입력은 미구현이다.

알려진 한계

DB가 없어 문구 변경 시 코드 수정 필요

FAQ 제안은 SupportFaqSuggestionService의 코드에 직접 작성돼 있다. 운영팀이 문구를 추가하거나 수정하려면 개발자가 코드를 수정하고 서버를 재배포해야 한다. 향후 운영팀이 직접 편집하는 요구가 생기면 support_faq 테이블과 admin CRUD endpoint를 추가해야 한다.
