F08-05. 플랜 발행

작성한 플랜을 마켓에 풀기 직전에 한 번 더 확인하는 화면이다. 제목 길이·블록 수·설명 길이·커버 이미지·가격 다섯 항목의 체크리스트를 띄우고, 모두 통과한 뒤에만 발행을 허용한다. 가격과 설명을 여기서 한 번 더 손볼 수 있고, 커버 이미지를 안 올린 채로 발행하면 마켓에서 빈 썸네일로 나간다는 신호로 미통과를 표시한다. 같은 플랜을 잠시 내렸다가 다시 풀 때도 이 화면을 한 번 거친다.

설계 의도 — 체크리스트는 사용자가 보는 마지막 거울이다

마켓에 한 번 풀린 플랜은 다른 사용자에게 즉시 노출된다. 빈 설명, 빠진 커버, 0개 블록 같은 상태로 노출되면 구매 결정 자체가 일어나지 않는다. 그래서 발행 직전에 다섯 항목을 한눈에 보여 주고, 사용자가 자기 손으로 채워서 통과하게 한다. 서버는 상태 전이만 검사하고 체크리스트 자체는 화면이 책임진다 — 향후 서버 강제가 들어오면 단축 경로(F08-01 인라인 발행)도 이 가드를 거치게 정렬해야 한다.

발행 직전 다섯 가지 확인

확인 1

제목 2~50자

너무 짧거나(2자 미만) 너무 길면(50자 초과) 미통과. 마켓 카드에서 잘리지 않고 보일 수 있는 범위.

확인 2

블록 1개 이상

본문이 빈 플랜은 발행할 수 없다. 사용자가 본문을 한 줄도 안 쓴 채 마켓에 풀리는 사고를 막는다.

확인 3

설명 10~500자

구매자가 카드 단계에서 "이 플랜이 무엇인지" 알 수 있는 최소 길이를 강제한다. 본 화면에서 직접 입력 가능.

확인 4

커버 이미지 등록

마켓 그리드와 인기 가로 스크롤에서 가장 먼저 보이는 시각 신호. 미등록 상태로 발행되지 않게 막는다.

확인 5

가격 설정

항상 통과 처리. 0이면 "무료", 양수면 콤마 포맷으로 보여 준다. 사용자가 가격을 한 번 더 확인하라는 신호.

이 단계에서 작성자는 무료 포인트로 살 수 있게 허용할지와, 허용한다면 무료 결제 가격을 따로 정할 수 있다. 무료로 팔린 분은 작성자에게도 무료 형태로만 들어와 플랫폼 안에서만 쓰이고 현금으로 인출되지 않는다(유료/무료 분리정산, 상세는 결제·정산 정책 문서).

발행 한 번이 일어나는 흐름

사용자가 본 화면에서 가격·설명·커버를 손본 뒤 발행을 누르면, 그 변경분을 먼저 저장하고 그다음 상태를 발행 중으로 바꾸는 두 단계로 진행된다.

사용자 시나리오

기본 흐름

모두 채워서 마켓에 푼다

제목과 블록은 이미 잘 채운 상태로 발행 화면에 들어왔지만 설명이 0자다. 체크리스트에 빨간색 미통과 표시가 두 곳에 떠 있다(설명·커버). 본 화면에서 설명을 12자 채워 넣고, 커버 영역을 눌러 갤러리에서 사진을 고른다. 업로드가 끝나면 모든 체크가 초록색이 된다. "발행하기"를 누르면 확인 다이얼로그 → 발행 호출 → 상세 화면으로 돌아간다.

이 흐름이 남기는 것 — 마켓에 풀리기 전에 자기 플랜이 완성됐는지 작성자가 한 번 더 본다. 미완성 콘텐츠가 카탈로그를 채우는 일이 줄어든다.

재발행

잠시 내렸던 플랜을 다시 풀어 둔다

예전에 발행했다가 판매 중지로 내려놓은 플랜에 들어왔다. 체크리스트는 모두 통과 상태이고 "발행하기"를 한 번에 누를 수 있다. 같은 식별자가 다시 마켓에 노출된다 — 새 플랜이 만들어지는 게 아니다.

미통과 차단

체크리스트가 빨간데 발행을 시도한다

설명 미통과 상태에서 발행 버튼을 누른다. 버튼 자체는 흐리게 표시되지만 누를 수 있다 — 누르면 "발행 조건을 모두 충족해야 합니다" 안내만 뜨고 호출은 일어나지 않는다.

부분 성공

가격·설명은 저장됐는데 발행이 실패했다

가격과 설명을 손본 뒤 발행을 눌렀는데 마지막 단계에서 네트워크가 끊겼다. 가격·설명 변경분은 별도 저장 단계에서 이미 반영됐고, 발행 호출만 실패했다. 사용자에게는 "발행에 실패했습니다" 안내가 뜨고 화면이 그대로 남는다. 다시 발행을 누르면 그 변경분은 다시 보내지 않고 발행 호출만 재시도된다.

운영 관찰 포인트 — 사용자에게는 "가격은 바뀌었지만 발행은 안 됐다"는 부분 성공 상태가 명시되지 않는다. 발행 실패율이 높아지면 안내 카피 보완이 필요하다.

커버 업로드 실패

이미지 업로드가 중간에 실패한다

커버 영역을 눌러 사진을 골랐는데 업로드가 도중에 실패한다. "이미지 업로드에 실패했습니다" 안내만 뜨고 체크리스트는 여전히 커버 미통과 상태로 남는다. 사용자는 같은 사진을 다시 시도하거나 다른 사진을 고른다.

잘못된 상태 전이

이미 발행 중인 플랜에서 발행을 다시 시도한다

화면 캐시가 어긋난 상태로 이미 발행 중인 플랜에 발행 호출이 닿는다. 서버는 작성 중·숨김에서만 발행 중으로 갈 수 있다는 규칙으로 거절하고, 사용자에게는 일반 실패 안내만 뜬다.

권한 가드

다른 사람의 발행 시도가 도달한다

비정상 경로로 다른 사용자가 본 플랜의 발행을 호출한다. 서버는 작성자 본인이 아니라는 이유로 거절한다. 일반 사용자에게는 노출되지 않는 안전 가드.

운영자가 알아야 할 것

체크리스트는 카탈로그 품질의 사실상 1차 필터다
  • 현재 체크리스트 검사는 화면에서만 한다. 서버는 상태 전이만 본다. 인라인 단축 발행(F08-01의 "판매 시작")이나 외부 호출은 이 가드를 우회할 수 있어, 마켓 품질을 책임지는 가드가 한 곳에 모이지 않은 상태다. 향후 서버 검증이 도입되면 단축 경로도 같은 규칙을 통과하게 정렬해야 한다.
  • 가격·설명 저장과 발행은 분리된 두 호출이다. 한쪽만 성공하는 부분 실패가 일어날 수 있다. 사용자에게 명확히 안내되지 않으니, 발행 실패율이 운영 지표로 추적되어야 한다.
  • 확인 다이얼로그는 "발행 후 수정할 수 없습니다"라고 말한다. 실제로는 판매 중지 → 편집 → 재발행이 가능하지만, 사용자에게는 신중을 유도하는 카피로 둔다. 이 카피가 너무 강해서 발행 자체가 위축되는 신호가 있다면 톤 조정 필요.
  • 커버 이미지의 권장 비율은 16:9, 권장 크기는 1200×675px. 다른 비율로 올리면 마켓 카드에서 잘려 보일 수 있다. 안내 문구가 화면에 한 줄로 들어가 있지만, 사용자가 이를 무시한 채 올리는 빈도는 운영 모니터링이 필요하다.
  • 제목 길이 50자는 마켓 카드 두 줄에서 안 잘리는 한계다. 한도를 늘리면 카드 안에서 잘리고, 줄이면 흥미를 끌 만한 제목을 쓸 자유가 줄어든다.
