F04-09. 게시글 댓글 · 대댓글

게시글 하단 1단 들여쓰기 답글까지 지원하는 댓글 시스템. 본인 수정/삭제, ADMIN의 부적절 댓글 삭제, 답글 보존을 위한 soft delete를 분기한다.

하위 동작
4
시나리오
8
깊이
2단
댓글 + 답글

이 기능 안의 하위 항목

동작 1

댓글 작성

입력창 → POST → 카운트 갱신.

동작 2

답글 작성

“@사용자” 자동 prefix, 1단 들여쓰기.

동작 3

수정(작성자)

“(수정됨)” 표시.

동작 4

삭제

자식이 있으면 placeholder로 유지, 없으면 제거.

사용자 시나리오 8개

S1

첫 댓글 작성

  1. 입력 → POST → 카운트 갱신 + 토스트
S2

답글 작성

  1. “@사용자” prefix + 1단 들여쓰기 노출
S3

본인 수정/삭제

  1. 수정: PUT + “(수정됨)”
  2. 삭제: 자식 유무로 placeholder/제거 분기
S4

ADMIN의 댓글 삭제

  1. requireStaff 통과 → soft delete
S5

일반 멤버의 타인 수정/삭제

  1. UI 미노출
  2. 우회 시 COMMENT_NO_EDIT_PERMISSION
S6

비멤버 진입

  1. 게시글 가시성 정책 적용
S7

긴 본문 분기

  1. “더보기” 토글
S8

삭제된 댓글 placeholder

  1. “삭제된 댓글입니다”로 자리 유지

서버 계약

MethodPath
POST/api/v1/clubs/{cid}/posts/{pid}/comments
POST/api/v1/clubs/{cid}/posts/{pid}/comments/{commentId}/replies
PUT/api/v1/clubs/{cid}/comments/{commentId}
DELETE/api/v1/clubs/{cid}/comments/{commentId}
