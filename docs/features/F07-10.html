F07-10. 정산 계좌 · 내 정산 이력 · 호스트 신뢰도

정산을 한 건씩 보지 않고 한 사람을 기준으로 묶어서 보는 횡단 영역. 호스트는 정산을 받을 본인 계좌를 등록·관리하고, 사용자는 본인이 어떤 모임에서 얼마를 주고받았는지를 월별 요약으로 회고한다. 그리고 새 모임에 신청하기 전 그 호스트의 정산 완료율·취소율·이의율 같은 신뢰도 지표를 미리 본다. 모임 한 건 안에서는 보이지 않는 사용자의 패턴과 호스트의 평판이 이 단위에서 한 자리에 모인다.

월별 요약 범위
최대 24개월
기본은 6개월
첫 계좌 등록
자동 기본 지정
한 번에 시작 가능
호스트 신뢰도 공개
전체 사용자
모임 신청 전 미리 확인
설계 의도 — 한 사람의 정산 패턴이 다음 결정에 쓰이게 한다

정산은 한 모임 한 건으로 끝나지 않는다. 사용자가 매달 여러 모임에 참가하고, 호스트가 매달 여러 정산을 만든다. 그런 누적이 한 사람을 기준으로 보이지 않으면 "내가 이 앱에서 얼마를 쓰고 있는지", "이 호스트는 정산을 잘 마무리하는 사람인지" 같은 질문에 답할 수가 없다.

본 단위는 정산이라는 한 건 단위가 아니라 한 사람의 영역으로 데이터를 묶어, 다음 의사결정에 쓰일 수 있게 한다. 특히 호스트 신뢰도는 신청 전 사용자가 미리 보고 결정하는 평판 지표가 되어, 정산을 잘 못 마무리하는 호스트의 모임에 사용자가 휘말리지 않게 한다.

이 단위 안의 세 가지 영역

영역 1 · 정산 수령용 계좌

호스트가 돈을 받을 계좌 등록·관리

은행·계좌번호·예금주를 등록하고, 여러 개일 때 기본 계좌를 지정한다. 첫 계좌는 자동으로 기본 계좌로 잡혀서 호스트가 추가 동작 없이 곧장 정산을 시작할 수 있다. 기본 계좌를 삭제하면 자동 승격은 일어나지 않는다 — 호스트가 다시 명시적으로 지정해야 한다.

영역 2 · 내 정산 이력

참여한 정산을 사람 기준으로 회고

본인이 보낸 돈, 받은 돈, 아직 결제하지 않은 분담을 정산 단위로 묶어서 보여 준다. 월별 요약으로는 "이번 달 3건 / 보낸 48,000원 / 받은 12,000원" 같은 짧은 카드가 위쪽에 떠오른다.

영역 3 · 호스트 신뢰도

다른 사람의 모임에 들어가기 전

호스트가 만든 모든 정산의 완료율·취소율·이의율·평균 완료 시간을 한 카드로 정리. 모임 상세 화면의 호스트 프로필 자리에 함께 노출돼, 사용자가 신청 전 미리 평판을 확인할 수 있다.

사용자 시나리오

기본 흐름 · 첫 계좌 등록

호스트가 정산 수령을 위해 계좌를 등록한다

프로필 메뉴에서 "정산 계좌"로 진입하니 빈 상태다. "계좌 추가" 다이얼로그에서 은행·계좌번호·예금주를 입력하고 추가하면 첫 계좌이므로 자동으로 기본 계좌로 잡힌다. 카드에 "기본" 배지가 함께 표시된다.

이 흐름이 남기는 것 — 호스트가 추가 동작 없이 곧장 정산을 시작할 수 있게 첫 등록을 부드럽게 한다.

계좌 추가 + 기본 변경

두 번째 계좌를 등록하고 기본을 옮긴다

이미 신한은행 계좌가 기본으로 등록된 호스트가 카카오뱅크 계좌를 추가한다. 두 번째 계좌는 자동으로 기본이 아닌 일반 상태로 추가된다. 호스트가 카카오 카드의 "기본 설정"을 누르면 기존 신한의 기본이 풀리고 카카오로 옮겨진다.

기본 계좌 삭제

기본 계좌를 삭제했더니 기본이 없는 상태가 된다

호스트가 신한(기본) 계좌를 삭제한다. 시스템은 카카오 계좌를 자동으로 기본으로 승격시키지 않는다 — 호스트가 카드 화면에서 "기본 계좌가 설정되지 않았습니다" 안내를 받고 명시적으로 다시 지정해야 한다.

이 흐름이 남기는 것 — 기본 계좌는 호스트의 명시적 의지로만 바뀌어야 한다. 자동 승격이 들어가면 호스트가 의도하지 않은 계좌로 정산이 모이는 사고가 날 수 있다.

기본 흐름 · 내 정산 이력

지난 6개월의 정산 참여를 회고한다

사용자가 프로필에서 "정산 히스토리"로 진입한다. 상단에는 월별 요약 카드가 띠로 흐른다 — "2026.04 / 3건 / 보낸 48,000 / 받은 12,000" 같은 형식. 그 아래에 정산 한 건씩 카드가 나열되어 모임 제목, 상태, 본인이 낸 금액·받은 금액이 표시된다.

결제 대기 강조

아직 결제 안 한 정산이 두드러진다

이력 카드 중 본인의 결제가 대기 상태인 건은 "결제 대기" 배지가 붙고 "지금 결제하기" 링크가 함께 노출된다. 사용자가 누르면 해당 정산 현황 화면으로 바로 이동.

이 흐름이 남기는 것 — 미처리 청구가 본인 영역에 모여 있어야 사용자가 잊은 결제를 찾아낸다. 한 모임 안에서만 알림이 가면 그 알림을 놓친 사람은 영영 안 보는 청구가 생긴다.

기간 변경

12개월 또는 24개월로 펼쳐 본다

사용자가 기간 셀렉터를 12개월 또는 24개월로 바꾸면 월별 요약 카드가 그만큼 늘어난다. 24개월을 넘는 값은 서버에서 24개월로 잘려, 일정 범위 안에서 그래프가 일관된다.

기본 흐름 · 호스트 신뢰도

새 모임 신청 전 호스트의 평판을 확인한다

사용자가 새 모임 상세 화면에서 호스트 프로필 카드에 있는 신뢰도 배지를 본다. "완료율 83% (우수)" 같은 라벨과 함께 색상으로 등급이 시각화되어 있다. 펼쳐 보면 누적 정산 12건, 완료 10건, 취소 1건, 이의 1건, 평균 완료 48시간 같은 세부 수치가 함께 노출된다.

이 흐름이 남기는 것 — 정산을 잘 못 마무리하는 호스트의 모임에 사용자가 휘말리지 않게 평판 지표가 신청 전에 보여야 한다.

정산 0건 호스트

신규 호스트는 신뢰도가 표시되지 않는다

호스트가 한 번도 정산을 만든 적이 없으면 신뢰도 배지가 표시되지 않고 "정산 이력 없음" 안내만 보인다. 데이터가 없는 호스트에게 임의의 점수를 부여하는 사고를 막는다.

차단 · 다른 사람 계좌

다른 사용자의 계좌를 수정하려 한다

사용자 A가 우회 호출로 사용자 B의 계좌 식별자에 수정 요청을 보낸다. "계좌를 찾을 수 없습니다" 안내로 거절되어 본인 계좌가 아니라는 사실 자체가 외부로 노출되지 않는다.

이 흐름이 남기는 것 — 계좌 정보는 본인만 접근 가능해야 한다. 권한 거부 메시지로 "다른 사람의 계좌가 존재한다"는 사실까지 노출되지 않게 한다.

화면에서 사용자가 인지하는 것

화면 요소사용자가 보는 것그 자리에 둔 이유
정산 계좌 카드 은행 로고·이름·계좌번호·예금주. 기본 계좌는 "기본" 배지. 호스트가 한눈에 본인의 수령 계좌가 어느 것인지 알 수 있어야 정산 수령 사고가 없다.
월별 요약 카드 띠 월별로 "건수 / 보낸 / 받은 / 대기" 한 줄. 가로 스크롤 또는 세로 스택. 한 모임 단위로는 보이지 않는 패턴이 월 단위로 보여야 사용자가 본인 사용을 회고 가능.
이력 카드 리스트 정산 한 건당 모임 제목, 상태 배지, 본인 결제·수령 금액. 결제 대기 건은 "지금 결제하기" 강조. 이력은 단순 회고가 아니라 미처리 결제를 찾는 도구도 되어야 한다 — 강조가 행동으로 이어진다.
호스트 신뢰도 배지 모임 상세 호스트 프로필 자리에 작은 카드. 완료율과 등급 라벨이 한 줄. 사용자가 모임 신청 전 짧게 평판을 확인할 수 있어야 정산 분쟁이 많은 호스트를 피할 수 있다.
신뢰도 세부 펼침 누적 / 완료 / 취소 / 이의 / 평균 완료 시간. 한 줄 라벨로는 부족할 때 펼쳐서 근거 수치를 볼 수 있게 — 사용자가 직접 판단할 여지를 둔다.

운영자가 알아야 할 것

한 사람의 영역은 사고 발견의 마지막 자리이기도 하다
  • 첫 계좌는 자동으로 기본이 된다. 호스트가 추가 동작 없이 시작할 수 있게 하는 의도. 두 번째부터는 호스트의 명시적 지정 없이는 자동 승격되지 않는다.
  • 기본 계좌 삭제 시 자동 승격은 일어나지 않는다. 자동 승격이 들어가면 호스트가 의도하지 않은 계좌로 정산이 모이는 사고가 날 수 있다 — 명시적 재지정이 안전.
  • 내 정산 이력은 모임 단위 알림이 누락된 사용자의 마지막 발견 지점이다. 한 모임 화면에서 결제 알림을 놓친 사용자도 본인 영역의 "결제 대기" 강조로 찾아낼 수 있다 — 이 강조가 빠지면 미결제 누적이 늘어난다.
  • 월별 요약의 기간 상한은 24개월이다. 그 이상을 요청해도 24개월로 잘린다 — 데이터 누적이 늘어날수록 무거운 조회를 방지하는 안전 장치.
  • 호스트 신뢰도는 다른 사용자에게 공개되는 정보다. 모임 신청 전 평판 확인이 목적 — 정산 0건 호스트에게는 점수가 부여되지 않고 "이력 없음"으로 표시된다.
  • 신뢰도의 "진행 중 건수"는 진행 중인 정산만 센다 (2026-06-05 정정). 그 전까지는 호스트가 아직 작성 중인(신청 전) 정산까지 진행 중으로 섞여 집계됐다 — 비확정 작성분이 평판 숫자를 오염시키던 문제를 바로잡았다. 완료율·취소율 계산은 변동 없다.
  • "내가 참여한 정산 목록"은 준비 중 정산도 담을 수 있게 준비됐지만, 아직 그 목록을 보여 주는 화면이 없다 (2026-06-05). 내 분담금이 걸린 준비 중 정산이 목록에 들어오고, 호스트가 아닌 사람에게는 호스트 메모·독촉 설정 같은 운영 정보가 가려진 채 내려온다 — 지갑 영역에 "내 모임정산 목록" 화면이 생기면 자동으로 적용되는 선구현 상태다. 현재 참가자가 준비 중 정산을 발견하는 공식 경로는 이벤트 상세의 "모임 정산" 입구 하나다.
  • 평균 완료 시간 계산은 현재 정확한 척도가 아니다. 완료 후에도 추가 변경(재요청·상각)이 있으면 마지막 변경 시각으로 계산되어 실제 완료 시간보다 길게 나올 수 있다 — 별도 "완료 시각" 필드 도입이 다음 개선 항목.
  • 계좌번호는 현재 평문으로 저장된다. 본인만 조회 가능하지만 향후 마스킹 표시·암호화가 강화 항목.
  • 다른 사람의 계좌 접근 시도는 "찾을 수 없음"으로 응답된다. "권한 없음"으로 응답하면 다른 사용자의 계좌가 존재한다는 사실이 노출된다 — 의도된 보안 처리.
