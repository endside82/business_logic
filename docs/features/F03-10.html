F03-10. 이벤트 ↔ 플랜 연결

호스트가 이벤트에 자신의 플랜(F08)을 묶어 “이 모임은 이 플랜대로 진행됩니다”를 보여준다. 순서 변경·활성/비활성 토글·삭제까지 본 단위가 책임지고, 일반 참가자에게는 활성 항목만 가로 스크롤로 노출된다.

하위 동작
4
시나리오
8

이 기능 안의 하위 항목

동작 1

플랜 연결

본인 플랜 검색 후 추가.

동작 2

순서 변경(드래그)

sortOrder swap.

동작 3

활성/비활성 토글

플랜은 유지하되 이번 회차 노출 제외.

동작 4

매핑 삭제

해제. 플랜 자체는 삭제 안 함.

사용자 시나리오 8개

S1

첫 플랜 연결

  1. 플랜 검색 시트 → 본인 플랜 선택
  2. sortOrder=0으로 INSERT
S2

두 번째 플랜 + 순서 변경

  1. 추가 → 드래그로 순서 변경 → sortOrder swap
S3

비활성 토글

  1. 활성 스위치 OFF → 카드 흐릿 + 참가자 미노출 후보
S4

매핑 삭제

  1. 휴지통 → 삭제 → 카드 사라짐(남은 sortOrder는 hole 가능)
S5

중복 매핑 시도

  1. 같은 플랜 재추가 → 409 EVENT_PLAN_MAP_ALREADY_EXISTS
S6

플랜 삭제된 후 조회

  1. planTitle이 null로 응답 → UI는 “삭제된 플랜”으로 표기 권장
S7

권한 없음 직접 호출

  1. 403 EVENT_NOT_OWNER, 클라이언트는 라우터로 사전 차단
S8

일반 참가자 미리보기

  1. 이벤트 상세 응답에 매핑 목록 포함
  2. 활성만 가로 스크롤로 노출

자주 빠뜨리는 케이스

레드 플래그
  • 플랜 삭제된 후 null 처리 누락 — 카드 깨짐.
  • 참가자 화면에 비활성 항목 노출 — UI 필터 누락.
  • sortOrder hole 후 정렬 깨짐 — 삭제 후 reorder 권장.
