# 전체 수용 기준 매트릭스

<!-- supporting-doc-status: 2026-05-18 -->

> 문서 상태: **보조 문서**. 기능별 현재 계약, source trace, Gap/Risk 판단은 [PRD_MIGRATION_STATUS.md](../PRD_MIGRATION_STATUS.md)와 각 기능 PRD를 우선한다. 이 문서는 인벤토리, 정책, QA, 기획 운영 기준을 보조하며, 기능 세부 판단은 [FEATURE_PRD_STANDARD.md](../FEATURE_PRD_STANDARD.md) 기준으로 재확인한다.

## 1. 공통 수용 기준

| 범주 | 수용 기준 |
|---|---|
| 기능 커버리지 | 168개 기능이 모두 PRD와 QA 대상에 포함되어야 한다. |
| 시나리오 커버리지 | 정상, 빈 상태, 권한, 상태 변화, 입력 검증, 외부 실패, 중복/동시성, 부수효과를 검토해야 한다. |
| 역할 커버리지 | 게스트, 로그인 사용자, 관계자, 소유자/관리자 역할별 CTA가 정의되어야 한다. |
| 상태 커버리지 | 생성, 진행, 완료, 취소, 만료, 삭제, 차단 상태를 구분해야 한다. |
| 회귀 방지 | 홈/검색 진입, 알림 딥링크, 결제/환불, 위치 권한 흐름이 깨지지 않아야 한다. |

## 2. 기능별 커버리지 인덱스

이 문서는 168개 기능 각각에 대해 원문 시나리오와 도식이 몇 개 있는지 확인하는 검산표다. 기능 카드 상세화를 진행할 때 이 표를 체크박스로 복사해서 쓰면, 특정 기능만 설명하고 예외 시나리오를 빼먹는 일을 줄일 수 있다.

## 카드화 우선순위 제안

기능 카드 상세화는 모든 기능을 한 번에 하지 말고, 아래 기준으로 우선순위를 둔다.

| 우선순위 | 기준 | 해당 예시 |
|---|---|---|
| P0 | 돈, 안전, 계정 삭제, 위치 공유처럼 사용자 피해가 큰 기능 | 결제/환불, 모임 정산, 데이팅 차단, 위치 opt-in/out, 계정 삭제 |
| P1 | 주요 전환율과 직접 연결되는 기능 | 이벤트 신청, 클럽 가입, 플랜 구매, 온보딩 |
| P2 | 운영/CS 문의가 많이 생길 기능 | 알림 설정, 데이터 내보내기, 리뷰/신고, 대기열 |
| P3 | 조회 중심이거나 실패 영향이 작은 기능 | 목록 조회, 통계, 단순 상세 |
