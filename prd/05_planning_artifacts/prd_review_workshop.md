# PRD 리뷰 워크숍 가이드

<!-- supporting-doc-status: 2026-05-18 -->

> 문서 상태: **보조 문서**. 기능별 현재 계약, source trace, Gap/Risk 판단은 [PRD_MIGRATION_STATUS.md](../PRD_MIGRATION_STATUS.md)와 각 기능 PRD를 우선한다. 이 문서는 인벤토리, 정책, QA, 기획 운영 기준을 보조하며, 기능 세부 판단은 [FEATURE_PRD_STANDARD.md](../FEATURE_PRD_STANDARD.md) 기준으로 재확인한다.

## 1. 목표

기획, 디자인, 개발, 운영이 같은 기능을 같은 의미로 이해하는지 확인하고 출시 전 빈 시나리오를 줄인다.

## 2. 90분 진행안

| 시간 | 내용 |
|---:|---|
| 0-10분 | 제품 PRD와 이번 리뷰 범위 확인 |
| 10-25분 | 영역 PRD에서 사용자와 기능 경계 확인 |
| 25-45분 | 정책 PRD로 상태, 권한, 돈, 알림 영향 확인 |
| 45-70분 | 기능 PRD의 시나리오와 수용 기준 검토 |
| 70-85분 | 미결정 사항을 결정 등록부에 기록 |
| 85-90분 | 다음 액션 담당자와 마감일 확정 |

## 3. 리뷰 질문

- 이 기능의 주 사용자는 누구이며, 사용자가 얻는 결과는 무엇인가?
- 사용자가 권한이 없거나 상태가 바뀌었을 때 무엇을 보게 되는가?
- 돈, 알림, 위치, 리뷰/신뢰 중 같이 바뀌는 영역이 있는가?
- 실패 후 재시도, 복구, 다른 화면 이동 중 어떤 동선이 필요한가?
- 운영자가 처리해야 하는 일이 생기는가?
- 법무 또는 개인정보 고지가 필요한가?

## 4. 산출물

- 확정된 기능 PRD
- 추가/수정된 수용 기준
- 결정 등록부 업데이트
- 다음 릴리즈 범위 후보
